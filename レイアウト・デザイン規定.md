# レイアウト・デザイン規定（読み物向け論文要約スライド）

この規定は、論文要約を「発表スライド」ではなく、単体で読める画像資料として生成するための上位規定である。参考画像は雰囲気の補助に留め、最終判断は本 Markdown を優先する。

## 1. 基本方針

- 出力は 16:9、基準解像度は 1920×1080 px とする。
- 外周は上下左右に最低 120px の安全余白を確保する。
- 各ページは「要点が分かるタイトル」「読み物として成立する説明量」「理解を助ける図解」を必ず持つ。
- 発表用の大きな余白だけの構成にしない。読み物として、本文・表・図解を中密度で配置する。
- ただし情報を詰め込みすぎない。1ページの主要情報は 3〜5 ブロックに整理する。
- 章構成は対象 Markdown を優先し、「総評」「査読結果」「総合評価」など、明示的に除外された章は作成しない。

## 2. 配色

全ページで同じ色味を使い、ページごとに色調を変えない。

- 背景: オフホワイト `#F7F6F1`
- 主文字: ディープネイビー `#2B3A4A`
- アクセント: くすんだ青 `#5E7E96`
- 補助文字・ページ番号: 淡いグレー `#9AA4AD`
- 補助図形: 上記色を薄くした淡いブルーグレーのみ

禁止:

- 強い原色、紫・ピンク・オレンジなどの追加アクセント
- 濃いグラデーション、強い影、写真風背景
- ページごとに異なるカラーテーマ

## 3. タイポグラフィ

- 日本語は Noto Sans JP 相当の読みやすいサンセリフを使う。
- タイトル、章ラベル、本文、注記の階層を明確にする。
- タイトルは全ページで同じサイズ感・太さ・色に揃える。
- 本文は読み物用途に合わせ、発表スライドより小さめでもよいが、1920×1080 を縮小表示しても読める余裕を残す。
- 英数字や数式は本文と同じトーンで扱い、過度に装飾しない。

## 4. ページ番号

- 全ページ必須。
- 位置: 右上、x=1800 付近で右揃え、y=84 付近。
- 書式: `03 / 07` のように「現在ページ / 総ページ」。
- 色: `#9AA4AD`
- サイズ: 約 27px。
- 表紙を含め、位置と書式を全ページで揃える。

## 5. タイトル設計

タイトルは章名の言い換えではなく、そのページの結論・要点を明記する。

良い例:

- `BM25は強いが、言い換えには取りこぼしがある`
- `DPRは質問と通路を同じベクトル空間で比べる`
- `DPRはSQuAD以外でBM25を大きく上回った`

避ける例:

- `背景`
- `提案手法`
- `結果`
- `まとめ`

中身スライドの標準位置:

- 章ラベル: x=120, y=126 付近、約24px、アクセント色
- タイトル: x=120, y=180〜232 付近、約52〜60px、太字、ネイビー
- タイトル下罫線: x=120 から幅 180〜270px、y=279 付近、3px、アクセント色
- 本文・図版エリア: y=330〜990、左右 120px の内側

表紙:

- 表紙も要点タイトルにする。
- 論文名はタイトルではなくサブタイトルとして扱う。
- 著者・会議名などは小さくまとめる。
- 表紙タイトルは x=144, y=450 前後、約78〜90px を目安にし、読み物としての要点が最初に見えるようにする。

## 6. 情報量

読み物として使うため、1ページに次の程度の情報量を持たせる。

- 要点タイトル: 原則1行、長い場合のみ自然な2行
- 本文ブロック: 3〜5個
- 各ブロック: 見出し + 1〜2文、または短い箇条書き
- 数値・比較: 表、棒グラフ、KPIカードで表現
- 図解: 本文を置き換えるのではなく、理解を補助する

目安:

- 表紙: 要点タイトル + 論文名 + 3行要約 + 小図解
- 背景: 問題設定 + 既存手法の限界 + 例
- 手法: アーキテクチャ図 + 学習の要点
- 実験: コーパス、データセット、指標
- 結果: 主要数値、比較表、例外条件
- 考察: 成功要因、制約、実用上の示唆
- まとめ: 3つの結論 + 後続技術への接続

## 7. レイアウト

推奨パターン:

- 2カラム: 左に説明、右に図解
- 3カード: 成功要因・制約・示唆などの整理
- 表 + KPI: 結果や実験条件
- フロー図: 処理手順やアーキテクチャ

カード:

- カードは情報単位を区切るために使う。
- 枠線は細く淡くし、角丸や影を強調しない。
- カードの中にカードを入れない。

図解:

- 線は細く、淡く、直線またはゆるい曲線にする。
- 図解は本文と同じ色体系で統一する。
- アイコンは意味を補助する場合のみ使う。
- 装飾目的だけの泡・ドット・幾何学模様は避ける。ただし、背景の空気感を作る淡い線・小さなベクトル風モチーフは控えめに使用できる。

## 8. 画像生成時の注意

- `image_gen` でページ単位に生成する。
- 日本語の正確性が重要な箇所は、短く明確な文にする。
- 生成プロンプトには、ページ番号、章ラベル、タイトル、本文、図解ラベルを明示する。
- 各ページに一意の識別子を入れて生成し、ログから該当画像を確実に取り出せるようにする。
- 出力後、必要に応じて 1920×1080 にリサイズする。
- 最終確認では、ファイル名、寸法、ページ数、除外章の不在、色味、タイトルの要点化を確認する。

## 9. ファイル命名

対象 ID が `P001` の場合:

- 出力ディレクトリ: `pdf/P001/`
- ファイル名: `P001_01.png`, `P001_02.png`, ...
- 総ページ数は作成対象章数に合わせる。
- `総評` を除外する場合は、総評を数に含めない。
