# 9.2 RA-LLM / retrieval-aware training

## 9.2.1. RA-LLMの範囲

RA-LLMは「検索結果をpromptへ貼る」方式だけを指さない。in-context RAG、FiDのようなfusion型、cross-attentionでretrievalを組み込む方式、REALM・RETROのようなpretraining統合型を区別する。通常RAGはindex更新で知識を替えられるが、学習型はmodel weightとdata lifecycleも管理対象になる。検索時の補助と、retrievalを前提に振る舞いを学習させる設計の境界を示し、運用負荷を比較する。

**参照論文**
- [P006] [Leveraging Passage Retrieval with Generative Models for Open Domain Question Answering](https://arxiv.org/abs/2007.01282)
- [P008] [REALM: Retrieval-Augmented Language Model Pre-Training](https://arxiv.org/abs/2002.08909)
- [P003] [Improving language models by retrieving from trillions of tokens](https://arxiv.org/abs/2112.04426)

## 9.2.2. architectureの比較

FiDは各passageを独立encodeしてdecoderで融合し、REALMはpretrainingからretrieverを学び、RETROは大規模外部databaseをchunk単位で参照する。三者は取得位置、頻度、学習可能部分、index規模、推論costが異なる。同じ「retrieval-augmented」と呼んでも、application側の差し替えやすさとmodel内部の結合度は逆方向に動く。図ではretrieverからgeneratorへ情報が入る位置を並べ、知識更新・rollback・監査の違いへつなげる。

**参照論文**
- [P006] [Leveraging Passage Retrieval with Generative Models for Open Domain Question Answering](https://arxiv.org/abs/2007.01282)
- [P008] [REALM: Retrieval-Augmented Language Model Pre-Training](https://arxiv.org/abs/2002.08909)
- [P003] [Improving language models by retrieving from trillions of tokens](https://arxiv.org/abs/2112.04426)

## 9.2.3. training target

学習対象は一つではない。retrieverを正解evidenceへ寄せる、generatorへ根拠利用を教える、両者をjointに最適化する、RA-DITのようにretrieverとLMへdual instruction tuningを行う選択がある。どのfailureを直すかでtargetを決め、検索漏れをgenerator tuningで隠さない。引用形式の不安定、distractor追従、retrieval不要時の過剰検索を別課題とし、学習後もzero-shot baselineとcomponent単位の指標を残す。

**参照論文**
- [P070] [Retrieval-Augmented Dual Instruction Tuning](https://arxiv.org/abs/2310.01352)
- [P066] [RAFT: Adapting Language Model to Domain Specific RAG](https://arxiv.org/abs/2403.10131)

## 9.2.4. 学習データ設計

Retrieval-aware trainingの基本例は、question、gold evidence、distractor、cited answer、unanswerable caseで構成する。RAFTは正解文書とdistractorを同時に与え、根拠を参照しながら答える能力をdomain固有設定で訓練する。文書版、ACL、quote spanをtraining dataにも保持し、未来文書や権限外sourceの混入を防ぐ。模範回答だけでなく、何を無視すべきかと拒否条件をlabel化し、test corpusとの文書重複を検査する。

**参照論文**
- [P066] [RAFT: Adapting Language Model to Domain Specific RAG](https://arxiv.org/abs/2403.10131)
- [P076] [Teaching language models to support answers with verified quotes](https://arxiv.org/abs/2203.11147)

## 9.2.5. negative samplingとdistractor耐性

Negativeは難しさより種類が重要である。random、semantic hard negative、near duplicate、旧版、scope違い、矛盾文書を区別し、false negativeを避ける。RAFTはdistractorの中からoracle documentを使う学習を通じ、根拠選択を強化する。検索recallだけでなく、正しいevidenceを採用し誤evidenceを無視するreader能力を測る。古い版を常に誤りとせず、質問時点に応じてpositiveが入れ替わるtemporal caseも作る。

**参照論文**
- [P066] [RAFT: Adapting Language Model to Domain Specific RAG](https://arxiv.org/abs/2403.10131)
- [P001] [Dense Passage Retrieval for Open-Domain Question Answering](https://arxiv.org/abs/2004.04906)

## 9.2.6. fine-tuningを選ぶ条件

Fine-tuningは、検索結果が正しいのにmodelが根拠を無視する、引用形式が安定しない、特定domainの回答規約がpromptだけで定着しない場合に検討する。先にdata整備、hybrid、rerank、packing、promptを改善し、上流failureをweight更新で覆わない。RAFTの利点はdomain RAGの利用行動を学ぶ点であり、頻繁に変わる事実の保存先をmodelへ戻すことではない。十分なlabel、回帰suite、rollbackが用意できるかも導入条件にする。

**参照論文**
- [P066] [RAFT: Adapting Language Model to Domain Specific RAG](https://arxiv.org/abs/2403.10131)
- [P070] [Retrieval-Augmented Dual Instruction Tuning](https://arxiv.org/abs/2310.01352)

## 9.2.7. 実装・version管理

学習型RAGではbase model、adapter、retriever、embedding、index、training data、promptを別versionとして記録する。modelとindexを同時更新すると原因が追えないため、互換matrixを持ち独立rollbackできるようにする。training dataのsource snapshotとlicense、PII処理、random seed、checkpointをmanifest化し、model cardへ用途と限界を記す。新adapterを既存indexで評価し、その後index更新を別実験にする段階的releaseを採る。

**参照論文**
- [P154] [Model Cards for Model Reporting](https://arxiv.org/abs/1810.03993)
- [P094] [Hidden Technical Debt in Machine Learning Systems](https://papers.nips.cc/paper/5656-hidden-technical-debt-in-machine-learning-systems)

## 9.2.9. applicationとrisk tier

RA-LLMの採用はdomainとrisk tierで変わる。社内QAでは引用習慣、契約・法務では条項と例外、医療・金融では拒否と専門家review、codeではrepository context、推薦では説明可能性が重要になる。高risk領域でfine-tuned modelの流暢さを最終判断へ使わず、原文citationとhuman approvalを残す。学習dataが特定部門・言語へ偏る場合のcoverageも評価し、通常RAGへ戻るfallbackを用意する。

**参照論文**
- [P066] [RAFT: Adapting Language Model to Domain Specific RAG](https://arxiv.org/abs/2403.10131)
- [P103] [Towards Trustworthy Retrieval Augmented Generation for Large Language Models: A Survey](https://arxiv.org/abs/2502.06872)
