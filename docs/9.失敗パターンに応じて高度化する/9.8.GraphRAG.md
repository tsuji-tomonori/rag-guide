# 9.8 GraphRAG

**この節の目的:** entity・relation・community・pathを使い、flat vector retrievalが苦手な関係探索、multi-hop、corpus全体のsensemakingを扱う。

## 9.8.1. GraphRAGが解く問題
  - global sensemaking、relation query、multi-hop、依存関係、影響分析を例示する。
  - 単純な類似検索で十分なqueryをGraphRAGへ送らない。
## 9.8.2. graphの種類
  - knowledge graph、document graph、citation graph、entity-chunk bipartite graph、community hierarchyを区別する。
  - 既存graph利用と文書からの自動構築を分ける。
## 9.8.3. Graph-Based Indexing
  - entity / relation抽出、entity resolution、chunk provenance、community detection、summary生成を扱う。
  - 抽出modelとontologyをversion管理する。
## 9.8.4. query classification
  - local entity query、path / relation query、global questionを分類する。
  - flat vector、graph local、graph globalのroutingを設計する。
## 9.8.5. Graph-Guided Retrieval
  - entity linking、neighbor expansion、path search、Personalized PageRank、subgraph optimizationを比較する。
  - hop数、node数、token budgetを制約する。
## 9.8.6. Graph-Enhanced Generation
  - node、edge、path、community summaryをevidence setへ変換する。
  - 回答claimからgraph elementと元chunkへcitationを張る。
## 9.8.7. hybrid vector + graph
  - vectorでseedを探しgraphで展開する方式と、graphで絞りtextを読む方式を比較する。
  - RRFやsource priorityでflat結果とgraph結果を統合する。
## 9.8.8. update・quality・ACL
  - entity merge / split、relation誤抽出、stale edge、incremental updateを扱う。
  - node / edge / community summaryへACLを伝播する。
