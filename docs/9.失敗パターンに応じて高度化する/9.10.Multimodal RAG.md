# 9.10 Multimodal RAG

## 9.10.1. 対象モダリティとユースケース

Multimodal RAGでは、文書page、figure、chart、table、screenshot、audio、videoを同じ「画像」と呼ばず、質問と回答のmodalityも整理する。たとえばUI状態の確認、グラフ傾向、会議録の発言箇所では必要な証拠単位が異なる。ColPaliは文書page画像をmulti-vector表現で検索し、OCR textだけに依存しない方向を示した。ユースケースごとにtext-to-image、image-to-text、時間区間検索の必要性を定義する。

**参照論文**
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)
- [P067] [RAG-Anything: All-in-One RAG Framework](https://arxiv.org/abs/2510.12323)

## 9.10.2. ingestion

IngestionではOCR、layout analysis、table extraction、figure–caption binding、ASR、keyframe・segment生成を別stageにする。page、bbox、cell、timestampをstable IDとして保持し、text化した内容から元regionへ戻れるようにする。LayoutLMv3はtext・image・layoutを統合し、PubTables-1Mは表の検出と構造認識を扱った。OCR confidenceやASR誤りをmetadataへ残し、低品質artifactを確定事実としてindexへ公開しない。

**参照論文**
- [P083] [LayoutLMv3: Pre-training for Document AI with Unified Text and Image Masking](https://arxiv.org/abs/2204.08387)
- [P100] [PubTables-1M: Towards comprehensive table extraction from unstructured documents](https://arxiv.org/abs/2110.00061)
- [P090] [DocLayNet: A Large Human-Annotated Dataset for Document-Layout Analysis](https://arxiv.org/abs/2206.01062)

## 9.10.3. text-first / vision-native / hybrid

Text-firstはOCR・captionを通常RAGへ載せ、debugしやすい。Vision-nativeはpage画像を直接encodeし、layoutや図を保てるがindexと推論costが高い。Hybridは両候補を融合する。ColPaliはpage screenshotをlate-interactionで検索し、従来の複雑なdocument processingを減らす性能を示した。一方、文字列exact matchや細かな数値はtext経路が強い。文書種別とquery sliceで三方式を比較し、万能な一経路に統一しない。

**参照論文**
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)
- [P083] [LayoutLMv3: Pre-training for Document AI with Unified Text and Image Masking](https://arxiv.org/abs/2204.08387)

## 9.10.4. index設計

IndexはText、Table、Image／Page、Video Segmentへ分ける案と、共通embedding空間を比較する。caption、OCR text、bbox、cell header、timestamp、modality、ACLをmetadataにし、同一pageの複数表現をgroup IDで結ぶ。RAG-Anythingはtext、image、table、equationを横断する統合frameworkを志向するが、各modalityの誤りと更新契約は残る。別indexなら専門retrieverを選べ、共通空間ならcross-modal検索が容易になるtrade-offを示す。

**参照論文**
- [P067] [RAG-Anything: All-in-One RAG Framework](https://arxiv.org/abs/2510.12323)
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)

## 9.10.5. retrieval

Retrievalにはtext-to-image、image-to-text、page、region、table row、video segment、cross-modalがある。queryが「赤い警告が出る画面」のような視覚表現ならpage画像を、エラーコードならOCR textを優先する。modalityごとにtop-kとrerankerを変え、hitしたregionをdocument候補へ集約する。ColPali型multi-vector検索ではどのpatchが反応したかを保持し、最終回答へ渡す前にpage・regionのACLとconfidenceを再確認する。

**参照論文**
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)
- [P096] [M3-Embedding: Multi-Linguality, Multi-Functionality, Multi-Granularity Text Embeddings Through Self-Knowledge Distillation](https://arxiv.org/abs/2402.03216)

## 9.10.6. fusion

同一pageのOCR、figure、table、captionが別候補として返ったら、重複ではなく相互補完の可能性がある。page／figure／table／timestamp IDでbundleし、modality内順位を作ってからRRF、router、evidence graphで統合する。raw cosineを異なるencoder間で直接加算せず、source trustとparse confidenceを別特徴量にする。あるmodalityが候補を独占しないquotaを置き、融合後も各evidenceの原locationを保持する。

**参照論文**
- [P032] [Reciprocal Rank Fusion Outperforms Condorcet and Individual Rank Learning Methods](https://dl.acm.org/doi/10.1145/1571941.1572114)
- [P067] [RAG-Anything: All-in-One RAG Framework](https://arxiv.org/abs/2510.12323)

## 9.10.7. generation

回答生成ではtext LLMとVLMを証拠内容で使い分ける。OCRで十分な説明文はtext LLM、図の位置関係、UI状態、chartの傾向は元画像をVLMへ渡す。画像を一度captionへ変換してから答えると、細部や否定が失われるため、重要なvisual claimはregion画像と周辺textを併記する。VLMにも根拠外推測禁止とcitation IDを課し、数値読取や計算はtable cell・専用toolで検証する。

**参照論文**
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)
- [P100] [PubTables-1M: Towards comprehensive table extraction from unstructured documents](https://arxiv.org/abs/2110.00061)

## 9.10.8. citation

Multimodal citationはdocument IDとpageだけでなく、figure／table ID、bbox、cell、audio・video timestampを単位にする。利用者が元regionや該当区間を再表示でき、回答claimからvisual evidenceへ直接辿れるUIを作る。OCR textを引用しても原画像と違う場合があるため、抽出textとimage snapshotを両方保存する。動画では回答時点のsegment hashとtranscript versionを残し、後日の編集でcitationがずれないようにする。

**参照論文**
- [P128] [Measuring Attribution in Natural Language Generation Models](https://arxiv.org/abs/2112.12870)
- [P061] [ColPali: Efficient Document Retrieval with Vision Language Models](https://arxiv.org/abs/2407.01449)

## 9.10.9. 品質・安全性・コスト

品質はOCR、caption、layout、ASR、vision embedding、rerank、VLM generationをstage別に測る。文字recallが高くてもtable列が崩れればQAは失敗するため、region retrieval、cell accuracy、timestamp overlap、visual claim supportを加える。顔、PII、画面上のcredential、画像内prompt injectionをsecurity testへ含め、不要なpage画像を外部modelへ送らない。storageとVLM tokenの費用をtext-first baselineと比較し、視覚情報が必要なqueryだけrouteする。

- [P100] [PubTables-1M: Towards comprehensive table extraction from unstructured documents](https://arxiv.org/abs/2110.00061)
- [P047] [Not what you've signed up for: Compromising Real-World LLM-Integrated Applications with Indirect Prompt Injection](https://arxiv.org/abs/2302.12173)
