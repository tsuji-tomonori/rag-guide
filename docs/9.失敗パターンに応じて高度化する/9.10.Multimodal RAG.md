# 9.10 Multimodal RAG

**この節の目的:** PDFページ、図、表、画像、スクリーンショット、音声、動画を検索可能な証拠として扱い、モダリティ固有の引用と評価を設計する。

## 9.10.1. 対象モダリティとユースケース
  - 文書画像、figure、chart、table、screenshot、audio、videoを分ける。
  - 検索queryがtextかimageか、回答がtextかvisual explanationかを整理する。
## 9.10.2. ingestion
  - OCR、layout analysis、table extraction、figure / caption binding、ASR、keyframe / segment作成を扱う。
  - page、region、cell、timestampなどの位置情報を保持する。
## 9.10.3. text-first / vision-native / hybrid
  - 抽出textで通常RAG、ページ画像embedding、両者のhybridを比較する。
  - OCR誤り、layout情報損失、index cost、debug容易性を評価軸にする。
## 9.10.4. index設計
  - Text Index、Table Index、Image / Page Index、Video Segment Indexを分ける案と共通空間を比較する。
  - caption、OCR text、bounding box、table cell、timestampをmetadataにする。
## 9.10.5. retrieval
  - text-to-image、image-to-text、page retrieval、region retrieval、video segment retrieval、cross-modal retrievalを扱う。
  - モダリティごとにtopKとrerankerを変える。
## 9.10.6. fusion
  - 同一page / figure / table / timestampのevidenceを束ねる。
  - score単純加算ではなく、rank fusion、modality router、evidence graphを比較する。
## 9.10.7. generation
  - text LLMとVLMの使い分けを示す。
  - 図の位置関係、UI状態、chart傾向は画像入力を保つ。
## 9.10.8. citation
  - document_id、page、figure / table id、bounding box、cell、timestampを引用単位にする。
  - UIで元regionや動画区間を再表示できるようにする。
## 9.10.9. 品質・安全性・コスト
  - OCR、caption、layout、ASR、vision embedding、VLM generationの誤りを段階別に測る。
  - 顔・個人情報・画面上の秘密情報、画像prompt injectionを扱う。
