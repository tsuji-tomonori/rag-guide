RAG（Retrieval-Augmented Generation、検索拡張生成）とは、AIが回答を作る前に必要な資料を探し、その資料を根拠として回答する考え方です。LLMの内部にある知識だけに頼るのではなく、社内規程、製品資料、手順書、問い合わせ履歴、論文、コード、データベースなど、確認可能な情報を参照します。

RAGという名称は、Lewisらが2020年に発表した論文 *Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks* で広く知られるようになりました。原論文では、生成モデルのパラメータに保持された知識を **parametric memory**、外部の検索可能な文書インデックスを **non-parametric memory** と呼び、両者を組み合わせて回答を生成します。[^lewis-rag]

| 種類 | どこに知識があるか | 長所 | 主な制約 |
|---|---|---|---|
| Parametric memory | LLMの学習済みパラメータ | 応答が速く、文章生成や一般的な推論に強い | 更新しにくい、出典を直接示しにくい、組織固有の知識を保証できない |
| Non-parametric memory | 文書、検索インデックス、DB、APIなどの外部知識源 | 更新しやすい、出典・版・権限を管理しやすい | 検索漏れ、古い資料、権限設定、検索品質の影響を受ける |

現在の実務で使われる「RAG」は、原論文の特定モデルだけを指す言葉ではありません。キーワード検索、ベクトル検索、ハイブリッド検索、SQL、ナレッジグラフ、APIなどを使って外部情報を取得し、その結果をLLMの回答に反映するアプリケーション設計を広く指します。
