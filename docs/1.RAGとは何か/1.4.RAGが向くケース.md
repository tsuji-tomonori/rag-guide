# 1.4 RAGが向くケース

| 必読 | [KILT: a Benchmark for Knowledge Intensive Language Tasks](https://arxiv.org/abs/2009.02252) — Petroni et al., 2020 | open-domain QA、fact checking、dialogue、slot filling等を「外部知識が必要なタスク」として整理する枠組み | Abstract、task taxonomy、provenance評価 |
| 必読 | [Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks](https://arxiv.org/abs/2005.11401) — Lewis et al., 2020 | knowledge-intensive QA、fact verification、generationにおけるRAGの基本的な適用範囲 | Abstract、実験タスク |
| 推奨 | [Leveraging Passage Retrieval with Generative Models for Open Domain Question Answering](https://arxiv.org/abs/2007.01282) — Izacard & Grave, 2020/2021 | 複数文書を統合する質問でretrievalとgenerationを組み合わせる価値 | Abstract、multi-passage結果 |

| 補助 | [RepoCoder: Repository-Level Code Completion Through Iterative Retrieval and Generation](https://arxiv.org/abs/2303.12570) — Zhang et al., 2023 | コードリポジトリ内の分散した文脈を検索し、生成へ利用する代表例 | Abstract、iterative retrieval-generation概要 |
