# 1. RAGとは何か

検索拡張生成（Retrieval-Augmented Generation：RAG）は、質問に関係する外部資料を検索し、取得した内容を根拠として大規模言語モデル（Large Language Model：LLM）が回答を生成する方式です。
LLMが学習時に得た知識だけに頼らず、社内規程、製品資料、手順書、問い合わせ履歴など、組織が更新できる情報を回答時に利用します。

RAGには検索と生成の両方が含まれます。
検索システムが質問に関係する文書や箇所を選び、LLMが質問に合わせて内容を統合します。
回答には参照元を示し、根拠が不足する場合には回答を保留するよう設計できます。
この仕組みによって、利用者は回答だけでなく、判断材料となった原文まで確認できます。

ただし、RAGはLLMの誤りを自動的に取り除く技術ではありません。
正しい資料を取得できなければ、回答に必要な情報はLLMへ届きません。
取得した資料が古い場合や、LLMが否定・例外・適用条件を読み違えた場合にも誤答が生じます。
RAGの価値は、誤りをゼロにすることではなく、資料、検索、根拠の選別、生成という工程ごとに失敗を調べられることにあります。

本章では、RAGの定義、LLM単体や検索システムとの違い、適する用途、適さない用途を順に説明します。
最後に研究の流れをたどり、現在のRAGが検索器とLLMを接続するだけの仕組みではなく、更新、権限、引用、評価を含むシステムとして扱われる理由を整理します。
