# 1. RAGとは何か

RAGとは、AIが答える前に必要な資料を探し、その資料を根拠にして回答する考え方です。AIの中にある知識だけに頼るのではなく、社内規程、製品資料、手順書、問い合わせ履歴など、確認できる情報を使って答えを作ります。そのため、答えの理由をたどりやすくなり、古い知識や思い込みだけで答えてしまう問題を減らせます。

この章では、RAGを「検索できるチャット」としてではなく、業務で信頼して使うための仕組みとして説明します。どのような場面に向いているのか、逆に任せるべきではない場面はどこか、根拠が足りないときにどう振る舞うべきかを整理します。

RAGの目的は、AIの間違いを完全になくすことではありません。大切なのは、答えに使った情報を確認できるようにし、分からないときには無理に答えず、失敗したときには原因を見直せるようにすることです。本章では、RAGを理解するための出発点を示します。
