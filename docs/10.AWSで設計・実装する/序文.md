# 10. AWSで最小RAGを設計・実装する

本章は、Amazon Web Services（AWS）でRAGを初めて動かす人のための、独立したハンズオンです。
第1章から第9章を先に読んでいなくても、サンプル文書の配置から検索結果の確認、根拠付き回答のAPI取得までを実行できます。

扱うのは、RAGを構成する次の四工程だけです。

1. **検索前処理（Pre-retrieval）**：文書をAmazon S3へ置き、Knowledge Baseへ取り込みます。
2. **検索（Retrieval）**：`Retrieve` APIで質問に近いチャンクを取得します。
3. **検索後処理（Post-retrieval）**：上位5件の順位、スコア、本文、出典を確認します。
4. **生成（Generation）**：`RetrieveAndGenerate` APIで回答と引用を取得します。

この四工程の意味は、[第2章 アーキテクチャ](../2.アーキテクチャ/序文.md)で説明しています。
本章では理論を繰り返さず、一つの最小構成へ対応付けます。

AWSサービスの画面、対応リージョン、モデルIDは変わることがあります。
AWS固有情報は2026年7月11日に確認した内容です。
実行時にも、本文からリンクしたAWS公式ドキュメントを確認してください。

### 10.0.1 完了時にできること

ハンズオンを終えると、次の状態になります。

- 三つの架空のMarkdown文書がS3に配置されています。
- 文書が分割、埋め込み、索引化され、Knowledge Baseの同期が完了しています。
- API応答で検索結果の1位から5位、関連度スコア、本文、S3 URIを確認できます。
- API応答で、文書に基づく回答と引用元をJSONとして取得できます。
- どのAWSリソースに費用が発生し得るかを理解し、作成したリソースを削除できます。

想定所要時間は60分から90分です。
AWSアカウントの権限やリソース作成の待ち時間によって前後します。

### 10.0.2 この章の範囲

この章に含めるものは次のとおりです。

- 一つのAWSアカウントと東京リージョン
- 検証用の公開情報だけを想定した三つのテキスト文書
- S3の原文バケット
- Amazon Bedrock Knowledge Bases
- Quick createで作るAmazon S3 Vectorsのベクトルバケットとインデックス
- Amazon Titan Text Embeddings V2
- `Retrieve` API
- `RetrieveAndGenerate` APIとAnthropic Claude Haiku 4.5
- ローカル端末またはAWS CloudShellから実行するPythonクライアント

次のものは含めません。

- Web画面、API Gateway、AWS Lambda
- 利用者認証、文書ACL、テナント分離
- VPC、PrivateLink、顧客管理KMS鍵
- Guardrails、WAF、監視、負荷試験
- Infrastructure as Code、CI/CD、複数環境
- 本番データ移行、高可用性、災害復旧
- Agentic RAG、GraphRAG、画像・音声・動画

これらは本番設計では重要ですが、RAGの四工程を理解して動かす最初のハンズオンへ混ぜません。
この章で作る構成を、そのまま本番公開しないでください。

### 10.0.3 なぜこのKnowledge Baseを選ぶか

Amazon Bedrockには、Managed Knowledge Baseと、ベクトルストアを選ぶ標準Knowledge Baseがあります。
AWSはManaged Knowledge Baseを推奨していますが、現行の`RetrieveAndGenerate` APIはManaged Knowledge Baseでは利用できません。
本章では回答をこのAPIで受け取るため、次の構成を選びます。

- **Knowledge Base**：ベクトルストアを選ぶ標準Knowledge Base
- **ベクトルストア**：コンソールのQuick createで作るS3 Vectors
- **データソース**：S3

S3 VectorsのバケットとインデックスはAWSが作成しますが、ハンズオン終了時には自分で削除対象を確認します。
この制約は、[RetrieveAndGenerate API](https://docs.aws.amazon.com/bedrock/latest/APIReference/API_agent-runtime_RetrieveAndGenerate.html)と[S3 Vectorsを使うKnowledge Base](https://docs.aws.amazon.com/AmazonS3/latest/userguide/s3-vectors-bedrock-kb.html)で確認できます。

### 10.0.5 初期値の考え方

本章では、選択肢を増やさないために初期値を一つに固定します。

| 項目 | 初期値 |
|---|---|
| リージョン | `ap-northeast-1` |
| 文書 | UTF-8のMarkdown 3件 |
| parser | Default parser |
| chunking | Default chunking、約300トークン |
| embedding | `amazon.titan-embed-text-v2:0`、1024次元、float |
| vector store | S3 Vectors、Quick create |
| 検索件数 | 5件 |
| 検索方式 | Semantic search |
| 生成 | `anthropic.claude-haiku-4-5-20251001-v1:0` |

これらはハンズオンを一度動かすための値です。
品質を改善するための比較や、本番要件に合わせた変更は本章の対象外です。
