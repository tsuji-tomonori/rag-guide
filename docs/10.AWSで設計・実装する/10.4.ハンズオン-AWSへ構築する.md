# 10.4 ハンズオン：データを配置してKnowledge Baseを作る

ここでは、サンプル文書をS3へ配置し、Amazon Bedrock Knowledge Baseへ取り込みます。
質問APIはまだ実行しません。

## 10.4.1 Step 0：開始位置を確認する

コマンドはリポジトリのルートで実行します。

```bash
pwd
test -f 'docs/10.AWSで設計・実装する/samples/data/support-policy.md'
aws sts get-caller-identity
```

`test`が終了コード0になり、AWS identityが検証アカウントのものなら進みます。

リージョンを固定します。

```bash
export AWS_REGION=ap-northeast-1
export AWS_DEFAULT_REGION="$AWS_REGION"
aws configure set region "$AWS_REGION"
```

以降のコマンドを別のterminalで実行する場合は、環境変数をもう一度設定してください。

## 10.4.2 Step 1：サンプル文書を読む

三つの文書を確認します。

```bash
sed -n '1,160p' \
  'docs/10.AWSで設計・実装する/samples/data/product-overview.md'

sed -n '1,160p' \
  'docs/10.AWSで設計・実装する/samples/data/support-policy.md'

sed -n '1,160p' \
  'docs/10.AWSで設計・実装する/samples/data/security-guide.md'
```

`support-policy.md`に次の二点があることを確認します。

- スタンダードプランの受付時間は、平日09:00から17:00までの日本標準時
- 問い合わせ方法は、Webフォームまたはメール

この原文が、後で検索と回答を判定する正解です。

## 10.4.3 Step 2：S3原文バケットを作る

一意なbucket名を作ります。

```bash
export ACCOUNT_ID="$(aws sts get-caller-identity --query Account --output text)"
export LAB_SUFFIX="$(date +%Y%m%d%H%M%S)"
export DOC_BUCKET="rag-four-elements-${ACCOUNT_ID}-${LAB_SUFFIX}"
echo "$DOC_BUCKET"
```

bucketを東京リージョンへ作成します。

```bash
aws s3api create-bucket \
  --bucket "$DOC_BUCKET" \
  --region "$AWS_REGION" \
  --create-bucket-configuration LocationConstraint="$AWS_REGION"
```

Public Accessをすべて遮断します。

```bash
aws s3api put-public-access-block \
  --bucket "$DOC_BUCKET" \
  --public-access-block-configuration \
'BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true'
```

設定を確認します。

```bash
aws s3api get-bucket-location --bucket "$DOC_BUCKET"
aws s3api get-public-access-block --bucket "$DOC_BUCKET"
```

locationが`ap-northeast-1`で、四つのPublic Access設定がすべて`true`なら成功です。

## 10.4.4 Step 3：文書をS3へ配置する

三つのMarkdownを`source/` prefixへアップロードします。

```bash
aws s3 cp \
  'docs/10.AWSで設計・実装する/samples/data/' \
  "s3://${DOC_BUCKET}/source/" \
  --recursive \
  --exclude '*' \
  --include '*.md'
```

配置結果を確認します。

```bash
aws s3 ls "s3://${DOC_BUCKET}/source/" --recursive
```

次の三ファイルだけが表示されることを確認します。

```text
source/product-overview.md
source/security-guide.md
source/support-policy.md
```

ファイルが0件なら、リポジトリのルートでコマンドを実行したか確認します。

## 10.4.5 Step 4：モデルを確認する

embedding modelとgeneration modelが東京リージョンで参照できることを確認します。

```bash
aws bedrock get-foundation-model \
  --model-identifier amazon.titan-embed-text-v2:0 \
  --region "$AWS_REGION" \
  --query 'modelDetails.[modelId,modelLifecycle.status]' \
  --output table

aws bedrock get-foundation-model \
  --model-identifier anthropic.claude-haiku-4-5-20251001-v1:0 \
  --region "$AWS_REGION" \
  --query 'modelDetails.[modelId,modelLifecycle.status]' \
  --output table
```

両方が`ACTIVE`であることを確認します。
Anthropic modelを初めて使うAWSアカウントでは、Bedrock consoleのModel catalogから利用目的の登録を求められる場合があります。
`AccessDeniedException`の場合は、モデルの存在ではなく、利用登録と実行identityの権限を管理者へ確認します。

## 10.4.6 Step 5：標準Knowledge Baseを作る

Knowledge BaseはAWS Management Consoleから作ります。
S3 Vectorsのvector bucket、index、service roleをQuick createすることで、初学者がIAM policyやindex schemaを手作業で組み立てずに済みます。

1. [Amazon Bedrock console](https://console.aws.amazon.com/bedrock/)を開きます。
2. 右上のリージョンが**Asia Pacific (Tokyo)**であることを確認します。
3. 左側の**Knowledge bases**を選びます。
4. **Create knowledge base**を選びます。
5. Knowledge Baseの種類では、ベクトルストアへ接続する標準Knowledge Baseを選びます。
6. **Managed Knowledge Baseは選びません**。
7. Knowledge Base名へ`rag-four-elements-kb`を入力します。
8. IAM service roleは**Create and use a new service role**を選びます。
9. data source typeは**Amazon S3**を選びます。
10. data source名へ`rag-four-elements-source`を入力します。
11. S3 URIへ、Step 3で作った`source/`を指定します。

S3 URIは、terminalで次のコマンドを実行すると表示できます。

```bash
echo "s3://${DOC_BUCKET}/source/"
```

次に、取り込みを設定します。

1. parsing strategyは**Default parser**を選びます。
2. chunking strategyは**Default chunking**を選びます。
3. custom transformation、multimodal data、supplemental storageは設定しません。
4. embedding modelは**Titan Text Embeddings V2**を選びます。
5. dimensionsは`1024`を選びます。
6. vector typeは`Float`を選びます。

最後に、vector storeを設定します。

1. **Quick create a new vector store**を選びます。
2. vector storeは**Amazon S3 Vectors**または**S3 vector bucket**を選びます。
3. encryptionはハンズオンの既定値を使います。
4. 設定を確認し、Knowledge Baseを作成します。

画面のラベルは変更されることがあります。
重要なのは、次の組み合わせになっていることです。

```text
Standard Knowledge Base
  + S3 data source
  + Titan Text Embeddings V2 / 1024 / float
  + S3 Vectors / Quick create
```

作成手順の最新情報は、[S3 data sourceへ接続するKnowledge Baseの作成](https://docs.aws.amazon.com/bedrock/latest/userguide/knowledge-base-create.html)を参照してください。

## 10.4.7 Step 6：作成されたリソースを記録する

作成完了後、Knowledge Baseの詳細画面で次を記録します。

```yaml
knowledge_base_name: rag-four-elements-kb
knowledge_base_id: YOUR_KB_ID
data_source_name: rag-four-elements-source
data_source_id: YOUR_DATA_SOURCE_ID
service_role: YOUR_SERVICE_ROLE_NAME
source_bucket: YOUR_DOC_BUCKET
vector_bucket: YOUR_VECTOR_BUCKET_NAME
vector_index: YOUR_VECTOR_INDEX_NAME
```

Knowledge Base IDは、AWS CLIでも確認できます。

```bash
aws bedrock-agent list-knowledge-bases \
  --region "$AWS_REGION" \
  --query \
"knowledgeBaseSummaries[?name=='rag-four-elements-kb'].[name,knowledgeBaseId,status]" \
  --output table
```

表示されたIDを環境変数へ設定します。

```bash
export KB_ID=YOUR_KB_ID
test -n "$KB_ID"
```

`YOUR_KB_ID`を実際のIDへ置き換えてください。

## 10.4.8 Step 7：data sourceを同期する

Knowledge Baseの詳細画面でdata sourceを選び、**Sync**を実行します。
同期中は、S3文書がparser、chunking、embedding、indexingの順に処理されます。

statusが**Complete**になるまで待ちます。
完了後、同期履歴またはstatisticsで次を確認します。

- scanned documents：3
- newまたはindexed documents：3
- failed documents：0

項目名はコンソールの版によって異なる場合があります。
少なくとも、三つの文書が処理され、失敗が0件であることを確認します。

CLIからdata source IDを確認する場合は、次を実行します。

```bash
aws bedrock-agent list-data-sources \
  --knowledge-base-id "$KB_ID" \
  --region "$AWS_REGION" \
  --query 'dataSourceSummaries[].[name,dataSourceId,status]' \
  --output table
```
