# 10.5 ハンズオン：APIで順位と回答を確認する

ここでは、付属のPythonクライアントから二つのAmazon Bedrock APIを実行します。

1. `Retrieve`で、検索されたchunkの順位、score、本文、S3 URIを確認します。
2. `RetrieveAndGenerate`で、RAGされた回答と引用をJSONとして受け取ります。

コンソールのテスト画面ではなく、最後までAPI応答を使います。

## 10.5.1 Step 1：Python環境を用意する

Python 3.11以降を確認します。

```bash
python3 --version
```

仮想環境を作り、最新のAWS SDK for Pythonをインストールします。

```bash
python3 -m venv .venv-rag-handson
source .venv-rag-handson/bin/activate
python -m pip install --upgrade pip boto3
python -c 'import boto3; print(boto3.__version__)'
```

既存の古いboto3を使うと、最新のBedrock API形状を認識できない場合があります。
このハンズオン専用の仮想環境を使います。

`jq`があるかも確認します。

```bash
jq --version
```

`jq`がない場合でもAPI実行には影響しません。
後続の`jq`コマンドは省略し、保存したJSON全体をテキストエディターで確認してください。

## 10.5.2 Step 2：APIの変数を設定する

10.4で記録したKnowledge Base IDを設定します。

```bash
export AWS_REGION=ap-northeast-1
export AWS_DEFAULT_REGION="$AWS_REGION"
export KB_ID=YOUR_KB_ID
export GENERATION_MODEL_ARN=\
"arn:aws:bedrock:${AWS_REGION}::foundation-model/anthropic.claude-haiku-4-5-20251001-v1:0"
```

`YOUR_KB_ID`は、実際のIDへ置き換えます。

値を確認します。

```bash
printf 'region=%s\nkb=%s\nmodel=%s\n' \
  "$AWS_REGION" "$KB_ID" "$GENERATION_MODEL_ARN"
```

Knowledge Base IDが空または`YOUR_KB_ID`のままなら、APIを実行しません。

## 10.5.3 Step 3：代表質問をAPIへ送る

次のコマンドは、`Retrieve`と`RetrieveAndGenerate`を順に一回ずつ呼び出します。
どちらの呼び出しにも費用が発生し得ます。

```bash
python \
  'docs/10.AWSで設計・実装する/samples/rag_api.py' \
  'スタンダードプランのサポート受付時間と問い合わせ方法を教えてください。' \
  | tee /tmp/rag-four-elements-result.json
```

成功すると、一つのJSON documentが標準出力へ表示され、同じ内容が`/tmp/rag-four-elements-result.json`へ保存されます。

## 10.5.4 Step 4：検索順位を確認する

検索結果だけを表示します。

```bash
jq '.retrieval' /tmp/rag-four-elements-result.json
```

応答は次の形です。scoreの数値と本文の区切りは実行ごとに異なるため、以下は短縮例です。

```json
{
  "requested_results": 5,
  "returned_results": 3,
  "results": [
    {
      "rank": 1,
      "score": 0.72,
      "text": "# Example Support Cloud サポートポリシー ...",
      "source_uri": "s3://.../source/support-policy.md"
    },
    {
      "rank": 2,
      "score": 0.41,
      "text": "# Example Support Cloud 製品概要 ...",
      "source_uri": "s3://.../source/product-overview.md"
    }
  ]
}
```

次を確認します。

- `rank`は1から順に並んでいます。
- `support-policy.md`が1位です。
- 1位の`text`に、平日、09:00、17:00、Webフォーム、メールが含まれます。
- `source_uri`からS3の原文へ戻れます。

`requested_results`は上限です。
登録されたchunkが少ない場合、`returned_results`は5未満になります。

scoreはモデルの確信度や正答確率ではありません。
この質問で期待文書が上位に来たかを確認するために使います。

これでRAGの二つ目と三つ目の工程を確認できました。

- **検索**：質問に関連するchunkを最大5件取得しました。
- **検索後処理**：順位付きの候補を上位5件へ絞り、本文と出典を確認しました。

## 10.5.5 Step 5：RAG回答を確認する

生成結果だけを表示します。

```bash
jq '.generation' /tmp/rag-four-elements-result.json
```

応答は次の形です。回答文は完全一致する必要はありません。

```json
{
  "answer": "スタンダードプランのサポート受付時間は、祝日を除く平日の09:00から17:00までです。時刻は日本標準時です。問い合わせはWebフォームまたはメールで行えます。",
  "citations": [
    {
      "generated_text": "...",
      "source_uri": "s3://.../source/support-policy.md",
      "source_text": "..."
    }
  ],
  "session_id": "..."
}
```

次を確認します。

- `answer`に、平日の09:00から17:00、日本標準時が含まれます。
- `answer`に、Webフォームまたはメールが含まれます。
- `citations`が1件以上あります。
- 引用の`source_uri`に`support-policy.md`が含まれます。
- `source_text`が回答内容を実際に支えています。
- `session_id`が返っています。

これでRAGの四つ目の工程である生成を確認できました。

## 10.5.6 Step 6：質問を変えて順位の変化を見る

セキュリティに関する質問を送ります。

```bash
python \
  'docs/10.AWSで設計・実装する/samples/rag_api.py' \
  '保存データと通信はどの方式で暗号化されますか。' \
  | tee /tmp/rag-security-result.json
```

検索結果を確認します。

```bash
jq '.retrieval.results[] | {rank, score, source_uri}' \
  /tmp/rag-security-result.json
```

今度は`security-guide.md`が1位になることを確認します。
生成回答にはAES-256とTLS 1.3が含まれ、引用も`security-guide.md`を指すはずです。

質問によって1位の文書が変われば、ファイル名を固定的に返しているのではなく、質問とchunkの意味的な近さで検索していることを確認できます。

## 10.5.7 付属クライアントが呼ぶAPI

検索では、次の設定を使います。

```python
response = client.retrieve(
    knowledgeBaseId=KNOWLEDGE_BASE_ID,
    retrievalQuery={"text": question},
    retrievalConfiguration={
        "vectorSearchConfiguration": {
            "numberOfResults": 5,
            "overrideSearchType": "SEMANTIC",
        }
    },
)
```

回答生成では、次の設定を使います。

```python
response = client.retrieve_and_generate(
    input={"text": question},
    retrieveAndGenerateConfiguration={
        "type": "KNOWLEDGE_BASE",
        "knowledgeBaseConfiguration": {
            "knowledgeBaseId": KNOWLEDGE_BASE_ID,
            "modelArn": GENERATION_MODEL_ARN,
            "retrievalConfiguration": {
                "vectorSearchConfiguration": {
                    "numberOfResults": 5,
                    "overrideSearchType": "SEMANTIC",
                }
            },
        },
    },
)
```

完全な実装は、[rag_api.py](samples/rag_api.py)にあります。
