# 2. アーキテクチャ

RAGを業務で使うには、質問に答える画面だけを作っても十分ではありません。あらかじめ資料を集めて整える作業、利用者の質問に合わせて資料を探す作業、利用状況や変更内容を管理する作業を分けて考える必要があります。役割を分けることで、問題が起きたときにどこを直すべきかを判断しやすくなります。

この章では、RAGの仕組み全体を大きく三つに分けて整理します。一つ目は、資料を準備して検索しやすくする部分です。二つ目は、利用者が質問したときに資料を探し、回答を作る部分です。三つ目は、利用するAI、資料の版、利用記録、権限、費用などを管理する部分です。

どの製品やサービスを使うかを決める前に、まず「どの作業をどこで行うか」を決めることが重要です。本章では、RAGを小さな試作から業務利用へ広げるための全体像を示します。
