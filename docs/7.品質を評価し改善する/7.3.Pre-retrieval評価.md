## 7.3.1. 評価対象のsnapshotを固定する
- source corpus、document version、parser、chunker、embedding、index設定を記録する。
- 同じ文書集合を異なるpipelineへ流し、差分比較できるようにする。

## 7.3.2. Ingestionを評価する
- 取り込み成功、部分失敗、retry、重複投入、欠落を測る。
- 作成、更新、削除、復元の各eventが規定時間内に反映されるか確認する。
- source countとindex countを定期照合する。

## 7.3.3. Parser／OCRを評価する
- text extractionだけでなく、reading order、heading、page、table、listを確認する。
- OCR文書では文字誤り、数字、記号、固有名詞、列対応を重点評価する。
- layout benchmarkのscoreとdownstream QAの差を併記する。

## 7.3.4. Chunkingを評価する
- gold evidenceが一つのchunkへ収まる割合を測る。
- chunk長、overlap、隣接重複、section跨ぎ、表分断を確認する。
- fixed、semantic、hierarchical、parent-childを同じgold setで比較する。
- document、sentence、propositionなど検索粒度を比較する。

## 7.3.5. Metadataを評価する
- document_id、chunk_id、title、page、section、version、effective dateを検査する。
- 欠損、型不一致、誤値、重複ID、source URL不整合を測る。
- citationとfilterに必要なmetadataがchunk単位で残るか確認する。

## 7.3.6. ACLを評価する
- tenant、role、group、document permissionをfixture化する。
- 権限外chunk混入と、過剰制限による正解根拠欠落を別々に測る。
- 権限変更と削除がindexへ反映される時間を測る。
- ACL漏えいはzero-tolerance gateにする。

## 7.3.7. Embeddingを評価する
- model、version、dimension、normalizationの混在を検査する。
- 日本語、英語、日英混在、cross-languageでretrieval性能を比較する。
- domain queryとhard negativeを使い、embedding候補を比較する。

## 7.3.8. Index構築を評価する
- vector count、document count、metadata countの整合性を確認する。
- stale vector、削除残存、重複vector、index build failureを検出する。
- ANN設定ではrecall、latency、memory／storageのtrade-offを測る。

## 7.3.9. Downstream影響を測る
- parser／chunker／embedding変更ごとにretrieval recall@kとQA品質を比較する。
- 上流scoreが改善しても、最終citationやanswer completenessが悪化していないか確認する。

## 7.3.10. 回帰suiteを作る
- PDF、表、画像、OCR、長文、multi-column、多言語の代表文書を固定する。
- 新しい失敗文書をfixtureへ追加し、同じfailureを再発させない。
