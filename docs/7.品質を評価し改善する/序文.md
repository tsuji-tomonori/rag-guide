# 7. 品質を評価し改善する

RAGの良し悪しは、最終的な答えだけを見ても正しく判断できません。答えが間違っている原因は、資料の準備、資料の探し方、候補の選び方、回答の作り方、権限の扱いなど、さまざまな場所にあり得ます。そのため、どこで失敗したのかを分けて確認することが重要です。

この章では、あらかじめ用意した質問と正しい根拠を使って、RAGの仕組みを試す方法を扱います。正しい資料を見つけられたか、答えが資料に沿っているか、出どころを正しく示せているか、答えられない質問に無理に答えていないかを確認します。

設定や資料を変更したときには、前より良くなったのか、別の質問で悪くなっていないかを比べる必要があります。本章は、勘ではなく確認結果に基づいてRAGを改善していくための章です。
