# 7.6 Generation評価

## この節の役割

- answer correctnessとfaithfulness／groundednessを区別する。
- 回答をclaimへ分解し、各claimとcitation sourceの対応を評価する。
- completeness、relevance、矛盾処理、answerability、formatを測る。
- rule、claim checker、LLM judge、人手評価を組み合わせる。

## 7.6.1. Generation条件を固定する
  - 同じqueryとfinal contextでmodel／promptだけを比較する。
  - model、prompt version、temperature、max tokens、output schemaを保存する。
## 7.6.2. Answer correctnessを評価する
  - exact matchだけでなく、key fact、数値、日付、条件、手順をrubric化する。
  - 許容表現、許容差、必須claim、禁止claimを定義する。
  - reference answer自体の誤りを定期的にレビューする。
## 7.6.3. Claimへ分解する
  - 回答をatomic claimへ分ける。
  - claimごとにsupported、contradicted、not-enough-informationを判定する。
  - 重大claimと補助claimをseverityで区別する。
## 7.6.4. Faithfulnessを評価する
  - contextにない主張、根拠の過剰一般化、条件の取り落としを検出する。
  - 外部常識として正しくてもcontextにない主張はgroundedness上で区別する。
  - unsupported claim rateと重大unsupported claim件数を分ける。
## 7.6.5. Context utilizationを評価する
  - gold evidenceがcontextに存在する条件で、回答へ利用されたかを見る。
  - 必要根拠の見落としと、不要根拠への依存を分ける。
## 7.6.6. Citation precisionを評価する
  - citation先が対応claimを支持するか確認する。
  - documentは正しいがspanが違う、旧版、別claimを支えるcaseを分ける。
  - quoteを出す場合は原文との一致をrule-basedに確認する。
## 7.6.7. Citation recall／completenessを評価する
  - 根拠が必要なclaimへcitationがある割合を測る。
  - 複数根拠が必要なclaimではcitation setの完全性を見る。
  - citation不要の一般文へ過剰引用していないかも確認する。
## 7.6.8. Answer completeness／relevanceを評価する
  - 必須論点、例外条件、前提、手順の抜けをrubric化する。
  - 冗長さとcompletenessを別軸にし、短い回答を自動的に高評価しない。
## 7.6.9. 矛盾と鮮度を評価する
  - 新旧版、複数source、反対証拠がある場合の断定、併記、保留を評価する。
  - effective dateやsource priorityに沿っているか確認する。
## 7.6.10. Answerability／Refusalを評価する
  - answerable、unanswerable、ambiguous、permission-denied、out-of-scopeを分ける。
  - correct refusal、false refusal、unsafe answer、correct clarificationを区別する。
  - 権限外情報の存在を示唆しないrefusalになっているか確認する。
## 7.6.11. Noise／Counterfactual robustnessを評価する
  - 無関係根拠が混ざっても正解できるか確認する。
  - model内部知識とretrieved evidenceが異なるとき、指定根拠を正しく扱えるかを見る。
## 7.6.12. Format adherenceを評価する
  - JSON schema、Markdown、table、citation IDをdeterministicに検査する。
  - parse failureと内容品質を分ける。
## 7.6.13. 評価器を校正する
  - rule、NLI／claim checker、LLM judge、人手評価を使い分ける。
  - judgeを人手gold setで校正し、言語・domain・回答長別の誤差を見る。
## 7.6.14. Failure sampleを保存する
  - unsupported、contradiction、citation mismatch、omission、over-refusal、format failureを分類する。
  - query、context、answer、判定理由をcase_idで結ぶ。
