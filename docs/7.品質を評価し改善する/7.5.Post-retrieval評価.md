# 7.5 Post-retrieval評価

## この節の役割

- reranking、filtering、deduplication、compression、context packingを段階別に評価する。
- first-stageで回収したgold evidenceが最終contextまで残ったかを測る。
- 品質改善とlatency／token／cost増加を同時に比較する。
- 矛盾、旧版、重複、根拠位置によるfailureを検出する。

## 7.5.1. Stageごとの入出力を保存する
  - first-stage candidates、reranked candidates、filtered set、compressed set、final contextを記録する。
  - 各根拠にID、rank、score、除外理由、source versionを付ける。
## 7.5.2. Rerankingを評価する
  - rerank前後のMRR、nDCG、precision@k、gold rankを比較する。
  - first-stage `20/50/100`からfinal `5/10`へ絞る組合せを評価する。
  - exact-match、少数言語、短い根拠が不当に落ちるcaseを確認する。
  - pairwise、pointwise、listwiseの再現性とcostを比較する。
## 7.5.3. Filteringを評価する
  - relevance、source trust、freshness、effective date、ACLを目的別に分ける。
  - 不適切根拠除去率とgold evidence誤除去率を同時に測る。
  - filtering ruleとmodel-based filteringの差を比較する。
## 7.5.4. Deduplicationを評価する
  - exact duplicate、near duplicate、隣接chunk、旧版／新版重複を区別する。
  - context内重複率、document diversity、gold coverageを測る。
  - MMR等で多様性を上げたときに重要根拠を落としていないか確認する。
## 7.5.5. Compressionを評価する
  - token削減率とgold evidence retentionを対で測る。
  - 抽出型と要約型を分け、数値、例外条件、否定表現の欠落を確認する。
  - 圧縮後もcitationが元source spanへ戻れるか確認する。
## 7.5.6. Context packingを評価する
  - token budget、根拠順序、source grouping、parent-child展開を比較する。
  - gold evidenceを先頭、中央、末尾へ移すposition testを行う。
  - 必要根拠がcontextにあるのに未使用となるcaseを測る。
## 7.5.7. Evidence setの完全性を評価する
  - answerに必要な根拠がすべて残っているか測る。
  - primary evidence、supporting evidence、conflicting evidenceを区別する。
  - multi-hopで一部根拠だけが残るpartial coverageを検出する。
## 7.5.8. 矛盾と版を評価する
  - 新旧版、effective date、source trustに従って優先できるか確認する。
  - 解消不能な矛盾を隠さず、generationへ明示的に渡せるか評価する。
## 7.5.9. End-to-end影響を測る
  - post-retrieval scoreだけでなく、answer correctness、faithfulness、citationへの影響を測る。
  - stage単体改善が最終回答を悪化させるcaseを確認する。
## 7.5.10. 性能とcostを評価する
  - stage別latency、LLM token、reranker call、cost/queryを記録する。
  - 高cost構成を全queryへ適用するか、routingするかを比較する。
## 7.5.11. Failure reasonを固定する
  - rerank落ち、filter誤除去、dedup誤除去、compression欠落、packing位置を分類する。
  - 除外理由とstage差分をtraceから再現できるようにする。
