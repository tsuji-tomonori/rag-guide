# P089：DiskANN: Fast Accurate Billion-point Nearest Neighbor Search on a Single Node

## 1. はじめに

本論文は、**10億件規模の高次元ベクトルに対する近似最近傍探索（ANNS）を、64GBのDRAMと市販SSDを備えた単一マシンで実現する**システム「DiskANN」を提案したNeurIPS 2019採択論文である。中核となる新しいグラフ索引アルゴリズムとして **Vamana** を導入し、高再現率、低レイテンシ、高い収容密度を同時に達成することを目的としている。([NeurIPS Proceedings][1])

SIFT1Bでは、16コアのマシン上で、論文要旨の動作点として**毎秒5,000件超、平均3ms未満、1-recall@1が95%以上**と報告されている。1-recall@1は、各クエリについて真の最近傍1件を正しく返せた割合を表す。

## 2. 背景

当時の高性能ANNSは、大きく二系統に分かれていた。

HNSWやNSGなどのグラフ型手法は高い再現率を実現する一方、グラフと非圧縮ベクトルを主記憶に保持するため、10億件規模では多数のサーバーや大容量DRAMが必要になる。対してFAISSやIVF-PQ系手法は、転置インデックスとProduct Quantizationによってメモリ使用量を抑えられるが、圧縮誤差により、真の最近傍を1位で返す1-recall@1が低下しやすい。

グラフ索引を単純にSSDへ移すだけでは、1クエリ当たり数百回のランダム読み出しが発生する。SSD上で数msの応答時間を実現するには、ランダムアクセス数を数十回程度、逐次的なI/Oラウンド数を10回未満、できれば5回程度まで削減する必要がある。

## 3. 提案手法

### 3.1 Vamanaグラフ

Vamanaは、各データ点を頂点とする有向グラフを構築する。主なパラメータは、最大出次数 (R)、探索候補数 (L)、枝刈り強度 (\alpha) である。

まず各頂点がランダムな (R) 個の頂点へ接続するグラフを初期化し、データ集合のメドイドを探索開始点とする。各点 (p) に対して現在のグラフ上でGreedySearchを行い、最終候補だけでなく**探索中に訪問した全頂点**を隣接候補として利用する。その後、RobustPruneによって最大 (R) 本まで枝を選択し、逆向きの枝も追加する。処理は、最初に (\alpha=1)、次に指定された (\alpha\geq1) で、データ全体を2周する。

RobustPruneでは、点 (p) に最も近い候補 (p^*) を選んだ後、

[
\alpha d(p^*,p') \leq d(p,p')
]

を満たす候補 (p') を削除する。(\alpha>1) では候補削除の条件が厳しくなり、より多様な長距離エッジが残る。この結果、グラフ次数を増やしたときに直径や探索ホップ数を削減しやすくなる。HNSWやNSGが実質的に (\alpha=1) の枝刈りを用いるのに対し、Vamanaはこの値を明示的に調整できる点が重要である。

### 3.2 低メモリでの10億点索引構築

10億点を一括してVamanaへ入力すると構築時のメモリが不足するため、DiskANNはデータを (k) 個のクラスタに分割し、各点を最近接する複数のクラスタへ重複配置する。論文では主に (k=40)、各点を最近接する (\ell=2) クラスタへ割り当てている。

各クラスタのVamana索引を個別に構築した後、全クラスタの辺集合を単純に合併する。クラスタ間にデータ点の重複があるため、検索時に複数シャードへ問い合わせなくても、統合後の単一グラフに必要な接続性が形成される。

### 3.3 DRAMとSSDの役割分担

DiskANNでは、次のようにデータを配置する。

* **DRAM**：各ベクトルの短いPQコードと、頻繁に訪問される一部の頂点
* **SSD**：グラフの隣接リストとフル精度ベクトル

SSD上では、各頂点のフル精度ベクトルと最大 (R) 個の隣接頂点IDを固定長領域に格納する。次数が小さい場合はゼロで埋めるため、頂点IDから格納位置を直接計算でき、オフセット表をDRAMに保持する必要がない。

### 3.4 BeamSearchと暗黙的再ランキング

通常のGreedySearchでは、1頂点ずつ隣接リストを読むため、SSDへの逐次的な往復回数が多くなる。DiskANNのBeamSearchは、候補のうち上位 (W) 頂点、典型的には2、4、8頂点の隣接リストを同時に読み出す。これによりI/Oラウンド数を削減する。ただし (W) が大きすぎると、不要な距離計算とSSD帯域消費が増える。

PQ距離は探索方向の決定に利用するが、最終結果はフル精度距離で再評価する。隣接リストとフル精度ベクトルを同じSSDセクタに配置するため、頂点を展開した際に追加I/Oなしでフル精度ベクトルも取得できる。これにより、PQの圧縮誤差を抑えながら高い1-recall@1を実現する。

## 4. 実験方法

実験には主に次の2台が用いられた。

| マシン      | 構成                                                         | 用途          |
| -------- | ---------------------------------------------------------- | ----------- |
| z840     | Xeon E5-2620v4、16コア、64GB DDR4、Samsung 960 EVO 1TB×2 RAID-0 | 低メモリ索引構築・検索 |
| M64-32ms | 32 vCPU、1,792GB DDR3                                       | 10億点の一括索引構築 |

インメモリ比較にはSIFT1M、GIST1M、DEEP1Mを使用し、VamanaをHNSWおよびNSGと比較した。10億点実験には128次元のSIFT1Bと96次元のDEEP1Bを使用し、DiskANNをFAISSおよびIVFOADC+G+P系手法と比較した。評価指標は主に再現率対平均レイテンシ、構築時間、探索ホップ数である。すべての実験はユークリッド距離を対象としている。

## 5. 結果

### インメモリ性能

SIFT1M、GIST1M、DEEP1Mでは、VamanaとNSGが概してHNSWを上回り、960次元のGIST1MではVamanaが両手法を上回った。DEEP1Mの構築時間は、Vamanaが129秒、HNSWが219秒、NSGが480秒であった。

SSD検索で重要となるI/Oホップ数は、98%の5-recall@5を達成する条件で、VamanaがHNSWやNSGより**2～3倍少ない**と報告されている。HNSWとNSGでは次数を増やしてもホップ数が頭打ちになる一方、Vamanaは (R) と (\alpha) を増やすことでホップ数をさらに削減できた。

### 10億点索引

SIFT1Bの一括索引は、約1.1TBのピークメモリを使って約2日で構築され、平均次数は113.9であった。一方、重複シャードを合併する方式は、64GB未満のメモリで約5日を要し、348GB、平均次数92.1の索引となった。合併索引のレイテンシ増加は、同一再現率に対して一括索引比で最大約20%に抑えられた。

一括索引はSIFT1Bで**1-recall@1＝98.68%を5ms未満**で達成した。またDiskANNは、95%以上の1-recall@1を3.5ms未満で実現している。論文要旨では、別の運用点として、5,000 QPS超、平均3ms未満、95%以上の1-recall@1が報告されている。

同程度のDRAM使用量で比較したIVFOADC+G+Pでは、16バイトコードが37.04%、32バイトコードが62.74%で1-recall@1が頭打ちになったのに対し、DiskANNは最終的に100%まで到達した。これは、PQを探索候補の絞り込みに使いつつ、SSD上のフル精度ベクトルで再ランキングする設計の効果である。

## 6. 考察

本論文の本質的な貢献は、既存のグラフ索引を単にSSDへ移したことではなく、**グラフ構造、枝刈り、並列I/O、データ配置、圧縮、再ランキングをSSDの特性に合わせて協調設計したこと**にある。

特に、Vamanaの (\alpha) は「近傍探索に適した局所的な辺」と「I/Oラウンドを減らす長距離辺」のバランスを調整する。BeamSearchはランダムI/Oを少数の並列バッチにまとめ、フル精度ベクトルの同一セクタ配置は再ランキングの追加I/Oをほぼ排除する。個々の技術は既知要素を含むが、システム全体として高再現率とSSD検索を両立させた点が強い。

一方、次の制約がある。

第一に、「64GBで10億点を扱う」という主張は、主として**合併索引の構築と検索**に対応する。最高性能の一括索引構築には約1.1TBのメモリが使われており、合併方式では構築時間と検索レイテンシが増える。

第二に、本論文の実験はユークリッド距離を対象とし、オンライン挿入・削除や継続的更新は評価していない。また、平均レイテンシが中心で、p95やp99などのテールレイテンシ、SSDの耐久性、消費電力、障害時の挙動は示されていない。

第三に、10億点での評価対象はSIFT1BとDEEP1Bが中心であり、ハードウェア構成も限定されている。キャッシュ、PQコード長、ビーム幅、重複シャード数など、システム構成要素ごとの包括的なアブレーションがあれば、設計判断の一般性はさらに明確になったと考えられる。

## 7. まとめ

DiskANNは、SSD上にグラフとフル精度ベクトルを置き、DRAMにはPQ圧縮ベクトルとホットな頂点だけを保持することで、10億点規模の高再現率ANNSを単一ノードへ収容した。

技術的な要点は、Vamanaの (\alpha)-RobustPruneによる低直径グラフ、重複シャードからの索引統合、並列SSD読み出しを行うBeamSearch、追加I/Oなしのフル精度再ランキングである。これらにより、従来はDRAM常駐が必要と考えられていた高性能グラフ探索を、市販SSD上で実用的なレイテンシに落とし込んだ。

## 8. 総評

**査読結果：採択（NeurIPS 2019）**

査読では、個々の構成要素が既存技術の拡張であり、方法論上の新規性が弱いという指摘があった。また、QPS対再現率の比較、検索処理全体のオーバーヘッド、引用や記述の改善も求められた。一方で、実験は広範かつ説得力があり、実務家にとって高い価値を持つとの評価も受けている。([NeurIPS Proceedings][2])

公式メタレビューは、新規性への懸念を認めながらも、**堅牢な最先端実装、非常に強い実験結果、実務への大きな影響**を理由に、公開に値すると判断した。([NeurIPS Proceedings][3])

総合的には、理論上まったく新しい探索原理を提示した論文というより、アルゴリズムとストレージシステムを精密に統合し、従来困難だった10億点・高再現率・単一ノード検索を実証したシステム研究である。新規性は漸進的だが、問題設定の重要性、性能改善の大きさ、後続のベクトル検索システムへの影響を考えると、**採択は妥当**と評価できる。
