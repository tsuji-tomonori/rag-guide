# P132「RankT5: Fine-Tuning T5 for Text Ranking with Ranking Losses」

**書誌情報**：Honglei Zhuangら、SIGIR 2023 Short Paper。T5を文書再ランキングへ適用し、分類・テキスト生成ではなく、ランキング損失によって直接学習する手法を提案した研究である。([Google Research][1])

## 1. はじめに

本論文は、T5を検索結果の再ランキングに利用する際、従来のように「関連あり／なし」を表すトークンを生成させるのではなく、**各クエリ・文書ペアに対する実数のランキングスコアを直接出力させる**RankT5を提案している。

さらに、モデルを通常の二値分類損失だけでなく、文書間の順序を考慮するpairwise損失や、候補文書集合全体を考慮するlistwise損失でファインチューニングする。主な結論は、**モデル規模を大きくするだけでなく、ランキング問題に適した目的関数を用いることが重要であり、特にlistwise損失が有効である**というものである。

## 2. 背景

検索システムでは、第一段階のretrieverが大量の文書から候補を抽出し、第二段階のrerankerが候補文書を高精度に並べ替える構成が一般的である。本論文は後者の再ランキングを対象とし、各クエリ (q_i) と候補文書 (d_{ij}) に対してスコア (\hat y_{ij}\in\mathbb{R}) を計算する。

従来のmonoT5は、入力を次のようなテキストとしてT5に与える。

[
\text{Query: }q_i\ \text{Document: }d_{ij}\ \text{Relevant:}
]

そして、関連文書なら`true`、非関連文書なら`false`を生成するよう学習し、推論時には両トークンのlogitからランキングスコアを計算する。この方法には、以下の不整合がある。

* 学習目標はトークン生成・二値分類である。
* 実際に最適化したい対象は、候補文書間の順位である。
* あるクエリに対する複数文書の相対関係を、pointwiseな学習では直接扱わない。

そこで著者らは、T5の出力をランキングスコアへ変更し、Learning to Rankで使われるランキング損失を直接適用する。

## 3. 提案手法

### 3.1 Encoder–Decoder型RankT5

入力は次の形式で構成される。

[
s_{ij}=\text{Query: }q_i\ \text{Document: }d_{ij}
]

T5のencoderとdecoderを通して語彙全体の非正規化logit (z) を得るが、語彙方向のsoftmaxは適用しない。

[
z=\mathrm{Dense}(\mathrm{Dec}(\mathrm{Enc}(s_{ij})))
]

語彙中の未使用トークン`<extra_id_10>`に対応するlogitを、そのままランキングスコアとする。

[
\hat y_{ij}=z_{\texttt{<extra_id_10>}}
]

重要なのは、特定トークンのlogitを使うこと自体ではなく、**そのlogitにランキング損失を直接適用すること**である。トークンを生成する必要がないため、通常の自己回帰的な文章生成処理も不要となる。

### 3.2 Encoder-only型RankT5

第二の構造では、T5のdecoderを完全に除去する。encoderが出力するトークン表現をpoolingし、全結合層で1次元のスコアへ変換する。

[
\hat y_{ij}
===========

\mathrm{Dense}\bigl(\mathrm{Pool}(\mathrm{Enc}(s_{ij}))\bigr)
]

Poolingには先頭トークン表現または平均poolingを使用できる。付録実験では両者の性能差はほとんどなかった。

この構造はBERT型cross-encoderに近いが、初期値としてT5のencoderを利用する点が異なる。decoderがランキング性能に本当に必要かを検証するための構造でもある。

### 3.3 ランキング損失

RankT5では、以下の4種類を比較する。

| 損失      | 学習単位    | 特徴                     |
| ------- | ------- | ---------------------- |
| PointCE | 文書単体    | 各文書を関連／非関連に二値分類する      |
| Pair    | 文書対     | 正例のスコアが負例より高くなるよう学習する  |
| Softmax | 文書リスト全体 | 同一クエリの全候補にsoftmaxを適用する |
| Poly1   | 文書リスト全体 | Softmax損失へ多項式補正項を追加する  |

Softmaxのlistwise損失は概略的には次式である。

[
\mathcal{L}_{\mathrm{Softmax}}
==============================

-\sum_j y_{ij}
\log
\frac{\exp(\hat y_{ij})}
{\sum_{j'}\exp(\hat y_{ij'})}
]

PointCEが各文書を独立に扱うのに対し、SoftmaxとPoly1は、同一クエリに対する候補文書間の競争を明示的にモデル化する。

## 4. 実験方法

### 4.1 データセット

**MS MARCO Passage Ranking**では、約53万件の学習クエリ、約6,800件の開発クエリ、880万件以上のpassageを使用する。各クエリについてdual-encoderで上位1,000件を取得し、RankT5で再ランキングする。

**Natural Questions（NQ）**では、5万件以上の学習クエリと8,000件の開発クエリを使用し、同様にdual-encoderによる上位1,000件を対象とする。両データセットとも関連度ラベルは二値である。

### 4.2 学習データ

各クエリについて、1件の正例と、retriever上位1,000件から一様ランダムに選んだ (m-1) 件の負例でリストを構成する。T5-Largeではハードウェア制約から最大リスト長を36とし、主実験では1正例＋35負例を使用する。

PointCEの場合は、正例数と負例数のバランスを取るため正例をupsamplingする。評価時は学習時の36件だけでなく、retrieverの上位1,000件すべてを再ランキングする。

### 4.3 比較手法と設定

主な比較対象はBERT-Large reranker、monoT5、RankT5 Encoder–Decoder、RankT5 Encoder-onlyである。RankT5の既定モデルはT5-Largeで、追加実験ではT5-Base、T5-Large、T5-3Bを比較する。

評価指標はMRR@10、NDCG@5、NDCG@10である。学習率は (1\times10^{-4})、バッチサイズは32リストで、リスト長36の場合は1バッチ当たり1,152クエリ・文書ペアとなる。実装にはT5Xと、JAXベースのLearning to RankライブラリRaxを用いる。

加えて、以下を分析している。

* モデルサイズの影響
* 学習時のリスト長の影響
* MS MARCOで学習したモデルによるBEIR 15データセットへのzero-shot転移

## 5. 結果

### 5.1 主結果

主要な結果は次の通りである。NQでは指標によって最良モデルが異なるため、各指標の最高値を示している。

| データセット   | モデル                   |     MRR@10 |     NDCG@5 |    NDCG@10 |
| -------- | --------------------- | ---------: | ---------: | ---------: |
| MS MARCO | monoT5                |     0.4156 |     0.4448 |     0.4843 |
| MS MARCO | RankT5 EncDec + Poly1 | **0.4343** | **0.4640** | **0.5025** |
| NQ       | monoT5                |     0.5406 |     0.5861 |     0.6079 |
| NQ       | RankT5の最高値            | **0.5689** | **0.6068** | **0.6291** |

MS MARCOでは、Encoder–Decoder＋Poly1が全指標で最高性能となった。monoT5との差はMRR@10で約0.019、NDCG@10で約0.018である。

NQではEncoder-only＋Poly1がMRR@10で0.5689を達成し、Encoder–Decoder＋SoftmaxがNDCG@10で0.6291を達成した。monoT5に対する改善はMRR@10で約0.028、NDCG@10で約0.021である。

### 5.2 損失関数の比較

SoftmaxとPoly1によるlistwise学習は、ほぼ一貫してPointCEとPairを上回った。特に、モデルサイズをBase、Large、3Bへ変更してもSoftmaxのPointCEに対する優位性は維持され、すべて統計的に有意と報告されている。

この結果は、事前学習モデルを単純に大型化しても、ランキング損失が提供する文書間の相対的な学習信号を完全には代替できないことを示している。

### 5.3 モデル構造の比較

Encoder–DecoderとEncoder-onlyの間には、一貫した勝者が存在しなかった。同一損失を使った場合の性能は概ね近く、十分な教師データがある状況では、T5 decoderの事前学習知識やpooling能力が必須ではない可能性が示された。

### 5.4 学習リスト長

リスト長を5、10、20、30、36と増やすと、両データセットで性能が向上した。NQでは30付近で飽和する傾向がある。

特に重要なのは、SoftmaxがPointCEを上回るためには、概ね20～30件以上の文書を含むリストが必要だった点である。listwise損失は、少数の負例だけでは十分な文書間比較を学習できないと考えられる。

### 5.5 Zero-shot性能

MS MARCOで学習したEncoder-onlyモデルを、BEIRの15データセットへ直接適用した結果は次の通りである。

| 学習損失    | BEIR平均NDCG@10 |
| ------- | ------------: |
| PointCE |        0.5024 |
| Softmax |    **0.5241** |

Softmaxは15データセット中11件でPointCEを上回った。TREC-COVID、BioASQ、NFCorpusなど、MS MARCOとは語彙や分野が大きく異なる医療・生物医学系データでは、平均約0.046の改善が報告されている。著者らは、listwise学習が個別の分類パターンの記憶よりも、より一般的な「関連性」の概念を学習させた可能性を指摘している。

## 6. 考察

本研究の最も重要な知見は、**RankT5の性能向上が特殊なモデル構造だけでなく、目的関数とランキング問題の整合性から生じている**点である。Encoder–DecoderとEncoder-onlyの差は小さい一方、PointCEとlistwise損失の差はモデルサイズを変えても残っている。したがって、「より大きいT5を使うこと」より、「候補文書間の競争を明示的に学習すること」の方が本質的な寄与と解釈できる。

また、listwise損失がzero-shot性能も改善したことは興味深い。二値分類では各文書の絶対的な関連確率を学習しやすいのに対し、listwise学習では、クエリごとにどの文書を相対的に上位へ置くべきかを学習する。この差が分布外データへの頑健性につながった可能性がある。

一方で、次の制約がある。

第一に、主実験はMS MARCOとNQという、二値関連度を持つpassage rankingに限定されている。多段階の関連度ラベル、複数正例が豊富なデータ、長文書ランキングに同じ結論が成立するかは未検証である。

第二に、負例はretriever上位1,000件から一様ランダムに選ばれており、hard negative miningや動的な負例選択との組み合わせは十分に検討されていない。

第三に、listwiseなのは学習損失であり、推論時には各クエリ・文書ペアを個別に処理する。そのため、上位1,000件をcross-encoderで処理する計算量は依然として大きい。Encoder-only化による速度、メモリ量、パラメータ数の削減効果も定量評価されていない。

第四に、「listwise損失が抽象的な関連性を学習させる」という説明は妥当な仮説ではあるが、表現解析や誤り分析によって直接検証されたものではない。Zero-shot改善の原因については追加実験が必要である。

## 7. まとめ

RankT5は、T5をトークン生成器として扱う従来のmonoT5から離れ、ランキングスコアを直接出力するモデルへ変更した手法である。

主な成果は次の三点に集約できる。

1. Encoder–Decoder型とEncoder-only型の双方で、T5を実数スコア出力型rerankerとして構成できる。
2. SoftmaxやPoly1などのlistwise損失が、分類損失やpairwise損失を上回る。
3. Listwise学習の効果はモデルサイズを大きくしても残り、分布外のBEIRデータでも平均性能を改善する。

特に、「ランキングタスクでは、事前学習モデルの強さだけでなく、学習目的を評価対象である順位へ合わせる必要がある」という、明快で再利用性の高い結論を示した研究である。
