# P141：Analyzing Leakage of Personally Identifiable Information in Language Models

## 1. はじめに

本論文は、言語モデルが学習データ中の個人識別情報（Personally Identifiable Information: PII）をどの程度漏えいするかを、攻撃者の知識量に応じた三つの脅威モデルで体系化した研究である。著者らは、PII漏えいを「抽出・再構成・推論」に分類し、それぞれをゲームベースで定式化したうえで、GPT-2を法律文書、企業メール、医療施設レビューにファインチューニングして評価している。

中心的な結論は、差分プライバシー（DP）はPII漏えいを大幅に低減するものの、単独では完全に防止できず、PIIスクラビングとの併用が必要だという点である。本論文はIEEE Symposium on Security and Privacy 2023に掲載されている。([microsoft.com][1])

## 2. 背景

従来のプライバシー攻撃では、あるデータが学習に使われたかを判定するメンバーシップ推論、または学習文そのものの抽出が主に研究されてきた。しかし、これらは必ずしも「どの情報が個人に関する機微情報なのか」を区別しない。

本論文は、次の二つを明確に区別している。

* **by whom型の漏えい**：誰のデータが学習に使われたかを明らかにするメンバーシップ推論
* **about whom型の漏えい**：学習データが誰について何を述べていたかを明らかにするPII漏えい

PIIには、氏名、住所、電話番号など単独で個人を識別できる直接識別子だけでなく、年齢、性別、職業など、組み合わせによって個人を識別できる準識別子も含まれる。

既存防御には主に以下がある。

**PIIスクラビング**では、NERにより氏名や住所を検出して`[MASK]`へ置換する。ただし、NERには未検出があり、複雑なPIIや文脈依存の情報を完全には除去できない。

**差分プライバシー**は、単一レコードまたは単一ユーザーの追加・削除が学習結果へ与える影響を制限する。一方、同じ氏名や住所が複数のメール、レコード、ユーザーにまたがって繰り返し現れる場合、その情報自体の保護は保証されない。すなわち、DPは主として「誰が提供したか」を保護するが、「誰について書かれていたか」を必ずしも保護しない。

## 3. 提案手法

著者らは、攻撃者がブラックボックスAPIを通じて任意のプロンプトに対する次トークン確率分布を取得できると仮定し、PII漏えいを三つに分類する。

| 攻撃                 | 攻撃者が持つ情報      | 目的                    |
| ------------------ | ------------- | --------------------- |
| PII Extraction     | モデルAPIのみ      | 学習データ中に存在したPIIを広く発見する |
| PII Reconstruction | PIIをマスクした学習文脈 | 文脈に対応するPIIを復元する       |
| PII Inference      | マスク文脈とPII候補集合 | 候補集合から正しいPIIを選ぶ       |

### PII抽出

モデルに空プロンプトなどを入力して大量の文章を生成し、NERでPII候補を抽出する。抽出結果について、学習データ中のPIIに対する再現率と適合率を測定する。

さらに著者らは、モデルを無制限にサンプリングしなくても、特定のPIIが抽出されやすいかを推定する「lazy estimation」を提案している。生成文中で同種のエンティティが現れる位置を検出し、そのエンティティを対象PIIに置き換えた際の条件付き確率を平均することで、PIIの抽出可能性を推定する。

### PII再構成

従来のTAB攻撃は、マスク位置より前の接頭辞だけを使い、モデルから続きのトークンを生成していた。提案攻撃は、接尾辞も含む文全体の情報を利用する。

処理は以下のようになる。

1. 対象以外のマスクを公開RoBERTaで補完する。
2. 対象GPT-2から複数の文をサンプリングする。
3. 生成文からNERでPII候補を収集する。
4. 各候補をマスク位置に挿入する。
5. 接頭辞・候補・接尾辞からなる全文のパープレキシティを計算する。
6. パープレキシティが最小の候補を返す。

これにより、自己回帰モデルであるGPT-2でも、接尾辞を間接的に利用した双方向的な推定が可能になる。

### PII推論

再構成と同じスコアリングを用いるが、攻撃者には正解を含むPII候補集合があらかじめ与えられる。各候補をマスク位置に挿入し、全文のパープレキシティが最小となる候補を正解として選択する。

また、ファインチューニング前の公開GPT-2も知っていたPIIは「ベースライン漏えい」として除外する。これにより、ファインチューニング用データから新たに学習されたPIIだけを評価しようとしている。

## 4. 実験方法

評価対象は以下の三領域である。

* **ECHR**：欧州人権裁判所の判例記述
* **Enron**：Enron社の実メール
* **Yelp-Health**：医療・健康関連施設に対するYelpレビュー

ECHRは11万件以上のレコード、Yelp-Healthは約7.9万件のレビューを含む。Enronについては、約60万通の原コーパスから一部を抽出して利用している。データを学習、シャドウモデル用検証、テストへ分割した。

モデルにはGPT-2 Small、Medium、Large、XLを使用し、それぞれ約1.24億、3.55億、7.74億、15.57億パラメータを持つ。次の四条件でファインチューニングした。

1. 防御なし
2. DP学習
3. PIIスクラビング
4. DP＋スクラビング

PII検出にはFlairとPresidio／spaCyを組み合わせ、氏名、組織、場所、電話番号、メールアドレスなど21種類のエンティティを扱った。

DP学習では、バッチサイズ64、勾配クリッピングノルム1.0、4エポックとし、プライバシー予算を

[
(\varepsilon,\delta)=(8,1/N)
]

に設定した。

抽出実験では、空プロンプトからtop-kサンプリング（(k=40)）により1クエリ当たり256トークンを生成し、15,000クエリ、合計約400万トークンを調査した。公開GPT-2による既知PIIを除外するため、別途50,000クエリ、約1,300万トークンをベースラインとして生成している。

再構成では64個の候補を生成し、推論では候補数100および500を評価した。指標は、テストパープレキシティ、メンバーシップ推論のROC-AUC、抽出の適合率・再現率、再構成・推論のTop-1正解率である。

## 5. 結果

主要結果は次の通りである。

| 評価                           |   防御なし | DP（(\varepsilon=8)） |
| ---------------------------- | -----: | ------------------: |
| ECHR・GPT-2 Large抽出適合率        | 29.56% |               2.92% |
| ECHR・GPT-2 Large抽出再現率        | 22.96% |               2.98% |
| Enron・GPT-2 Large抽出再現率       |  7.23% |               2.31% |
| Yelp-Health・GPT-2 Large抽出再現率 | 13.63% |               6.51% |
| ECHR・再構成正解率                  | 18.27% |               0.55% |
| ECHR・推論正解率、候補100             | 70.11% |               8.32% |
| Enron・推論正解率、候補100            | 50.50% |               3.78% |
| Yelp-Health・推論正解率、候補100      | 28.31% |               4.29% |

### 抽出

ECHRのGPT-2 Largeでは、学習データ中のPIIの約23%が抽出され、生成されたPIIの約30%が実際の学習PIIだった。モデルが大きいほど、おおむね漏えいが増加した。

DPはECHRにおける抽出再現率を約23%から約3%へ低下させた。しかし、漏えいを完全にはなくしていない。EnronやYelp-Healthでは、DP後も適合率・再現率がECHRより高いケースがある。

PIIの出現回数と漏えい頻度には強い関係があり、学習データ中で約2倍多く出現するPIIは、おおむね約2倍の頻度で漏えいした。また、DPは特に複数トークンからなる長いPIIの抽出を抑制したが、短い氏名やメールアドレスなどは残りやすかった。

### 再構成

提案攻撃は、接頭辞だけを使うTAB攻撃を大きく上回った。

* ECHR・GPT-2 Large：5.81%から18.27%
* Enron・GPT-2 Medium：0.67%から7.26%
* Yelp-Health・GPT-2 XL：1.99%から6.40%

EnronのGPT-2 Mediumでは約10.8倍の改善であり、論文が主張する「最大10倍」の根拠となっている。正解PIIが候補集合に含まれている場合には約70%の確率で正しく選択できたため、最大のボトルネックは正解候補を生成できるかどうかであった。DPモデルでは再構成率はほぼ1%以下に抑えられた。

### 推論

候補数100のランダム正解率は1%であるが、DP適用後でもECHRで8.32%、Enronで3.78%、Yelp-Healthで4.29%となった。したがってDPは絶対的な漏えい率を大幅に削減する一方、情報を持つ攻撃者に対してはランダム水準を明確に上回る信号を残している。

### 防御方法の比較

ECHR・GPT-2 Largeでは、防御なしモデルのメンバーシップ推論AUCが0.96だった。スクラビングのみではAUCが0.82であり、PIIを消しても学習レコードの所属情報は残る。一方、DPまたはDP＋スクラビングではAUCが約0.5となった。

スクラビングは測定上のPII抽出・再構成をほぼ防止したが、メンバーシップ推論には十分でない。DPとスクラビングを併用すると両方の攻撃を抑制できるものの、パープレキシティが上昇し、モデル効用が低下した。

## 6. 考察

本論文の重要な貢献は、プライバシー保護を単に「学習参加の秘匿」として扱わず、「モデルが誰についての情報を再現するか」という別の軸を導入した点にある。レコード単位DPでは、複数レコードに繰り返し現れる同一人物の情報を十分に保護できないという指摘は、メール、医療、組織内文書などの現実的なデータに対して重要である。

また、GPT-2のような自己回帰モデルであっても、候補を全文へ挿入してパープレキシティを比較すれば、接尾辞を利用した再構成が可能であることを示した点も有用である。これは、モデルAPIが単純な次トークン生成しか提供していなくても、スコアリング方法によってより強い攻撃が構成できることを意味する。

一方、以下の制約がある。

第一に、スクラビングと漏えい測定の双方に同じNER系ツールを使用している。NERが見逃したPIIは、スクラビングされないだけでなく評価時にもPIIとして数えられない可能性がある。このため、測定値は実際の漏えいの下限と考えるべきである。著者らもこの点を主要な限界として認めている。

第二に、基本評価は文字列の完全一致であり、略称、表記揺れ、意味的に同一な人物表現を十分に扱っていない。例えばフルネームとイニシャルが同一人物である場合や、複数の準識別子を組み合わせるリンク可能性攻撃は評価外である。したがって、実際の攻撃者が外部情報やエンティティ解決を利用した場合、漏えいリスクはさらに高くなる可能性がある。

第三に、モデルはGPT-2系列に限定され、英語の三データセット、単一の主要DP設定で評価されている。現代の指示調整済みLLM、チャットAPI、検索拡張モデル、より大規模なモデルへ結果を直接一般化できるとは限らない。

第四に、攻撃者が次トークンの確率ベクトルを取得できるという仮定は、生成文だけを返すAPIより強い。さらに、再構成・推論攻撃にはマスク済み文脈や候補集合が必要であり、攻撃者の事前知識が成功率を大きく左右する。

今後の方向性としては、PIIの重複回数やDPによる保護度を考慮し、危険なPIIだけを選択的に除去する「DP-informed scrubbing」、データ重複除去、意味的なPII照合、人手アノテーションを含む独立した漏えいベンチマークが有望である。

## 7. まとめ

本論文は、言語モデルからのPII漏えいを、抽出・再構成・推論という三つの脅威に分類し、形式的定義と具体的攻撃を提示した。

実験から、次の点が明らかになった。

* 防御なしモデルでは、学習PIIの相当部分が抽出・再構成可能である。
* モデル規模とPIIの重複回数が漏えいを増加させる。
* 接尾辞を利用する提案再構成攻撃は、既存攻撃より最大約10倍強い。
* DPは漏えいを大幅に抑制するが、特に候補付き推論では信号が残る。
* スクラビングとDPは代替関係ではなく、異なる脅威を防ぐ補完的な防御である。

結論として、PII保護にはDPだけでもスクラビングだけでも不十分であり、両者の保護効果を考慮した選択的・効用保存型の防御設計が必要である。

## 8. 総評

**長所**は、PII漏えいを明確な脅威モデルとして定式化したこと、形式的定義と実証実験が対応していること、三つの実データ領域と複数モデルサイズで防御効果を比較したこと、既存再構成攻撃を大幅に改善したことである。再現用コードも公開されている。([GitHub][2])

**弱点**は、NER依存による評価の循環性、GPT-2中心の限定的なモデル選択、確率ベクトルを取得できるという比較的強いAPI仮定、完全一致中心の漏えい判定、DP予算のアブレーション不足である。特に「スクラビング条件で測定漏えいが0%」という結果は、同じNERを防御と評価に使用している以上、絶対的安全性を意味しない。

**査読結果（私見）：採択／Accept。**

主要な限界はあるものの、問題設定の新規性、攻撃手法の改善幅、DPとスクラビングの役割を分離した分析、セキュリティ・プライバシー分野への実務的示唆は十分に強い。実際の公刊結果としても、本論文は査読を経てIEEE Symposium on Security and Privacy 2023に採択・掲載されている。([microsoft.com][1])
