# P151：Generating Is Believing: Membership Inference Attacks against Retrieval-Augmented Generation

## 1. はじめに

本論文は、RAG（Retrieval-Augmented Generation）の**外部知識データベースに、特定の文書・サンプルが登録されているか**を推定するメンバーシップ推論攻撃を扱う。著者らは、RAGの生成結果と対象サンプルの類似性を利用する **S²MIA（Semantic Similarity Membership Inference Attack）** を提案している。論文はYuying Liらによるもので、ICASSP 2025に採択・掲載された。([arXiv][1])

一言でまとめると、**「RAGに対象文書が格納されていれば、その文書を手掛かりに質問した際、生成結果が元文書に似やすくなる」という現象を利用した攻撃**である。

## 2. 背景

一般的なRAGは、外部データベース、検索器、LLMの三要素から構成される。ユーザーのクエリに対して検索器が関連文書を取得し、取得文書とクエリをLLMへ入力して回答を生成する。

従来のメンバーシップ推論攻撃は、学習データに対するモデルの過学習を利用することが多い。しかし、RAGの外部データベースは通常、LLMの学習には使用されない。そのため、モデル損失やトークン確率だけを用いる従来型攻撃を、そのままRAGの検索データへ適用することは難しい。([arXiv][2])

一方、対象文書が検索データベースに存在すれば、その文書自身が検索結果に含まれ、生成内容へ強く反映される可能性がある。本論文はこの**検索によって生じる直接的な情報反映**を、メンバーシップ信号として利用する。

想定攻撃者は、LLMのパラメータ、検索器の設定、外部データベースの内容にはアクセスできないが、対象RAGへ問い合わせ、生成テキストと生成確率を取得できる。また、データベースのおおよそのデータ分布を知っていると仮定される。

## 3. 提案手法

S²MIAでは、対象サンプル (x_t) を次の二つに分割する。

[
x_t=x_t^q\oplus x_t^r
]

ここで、(x_t^q) はRAGへ入力するクエリ部分、(x_t^r) は推定時に比較対象とする残りの部分である。実験では、質問回答データの「質問」をクエリ、「回答」を残りの部分としている。

クエリをRAGへ入力して生成テキスト (\bar{x}^g) を得た後、次の二種類の特徴量を計算する。

第一は、対象サンプルと生成テキストの類似度である。

[
S_{\mathrm{sem}}=\mathrm{BLEU}(x_t,\bar{x}^g)
]

第二は、生成テキストのパープレキシティ (PPL_{\mathrm{gen}}) である。対象サンプルがデータベース内に存在する場合、関連文書が検索されるため、生成結果は対象サンプルに近くなり、同時に生成時の不確実性も低くなる、すなわちBLEUは高く、パープレキシティは低くなると仮定する。

推論方法には二つの方式がある。

**S²MIA-T** は閾値方式であり、

[
S_{\mathrm{sem}}\geq\theta_{\mathrm{sem}}
\quad\land\quad
PPL_{\mathrm{gen}}\leq\theta_{\mathrm{gen}}
]

を満たしたサンプルをメンバーと判定する。

**S²MIA-M** はモデル方式であり、BLEUとパープレキシティを入力特徴量とする教師あり二値分類器を学習する。論文ではニューラルネットワークやXGBoostなどを利用可能としている。

## 4. 実験方法

データセットにはNatural QuestionsとTriviaQAを使用している。各データセットの80％をRAGの外部データベースに入れたメンバー、残り20％を非メンバーとした。

RAGの生成モデルには、Llama-2-7B、Llama-2-13B、Vicuna、Alpaca、GPT-3.5-turboの5種類を使用した。検索器にはDPRとContrieverを使用し、上位5件を取得する設定である。

攻撃用参照データとして、メンバー1,000件と非メンバー1,000件を使用し、さらに別の2,000件で攻撃性能を評価した。評価指標はROC AUCとPR AUCである。

比較手法は、Loss Attack、Zlib Entropy Attack、Min-k% Probability Attack、Neighborhood Attack、およびRAGへ直接メンバーか否かを質問するRAG-MIAの5種類である。さらに、クエリのパラフレーズ、生成プロンプトの変更、検索結果のリランキングという三つの防御策も検証した。([arXiv][2])

## 5. 結果

従来のLLM向け攻撃であるLoss、Zlib、Min-k%、Neighborhoodは、多くの条件でROC AUCが0.5前後となり、ほぼランダム推定にとどまった。これに対してRAG専用のRAG-MIAは約0.76～0.85、S²MIAは最大でROC AUC 0.893、PR AUC 0.903を達成した。

表1の10条件を単純平均すると、概ね以下となる。

| 手法      | 平均ROC AUC |  平均PR AUC |
| ------- | --------: | --------: |
| RAG-MIA |     0.807 |     0.818 |
| S²MIA-T | **0.855** | **0.856** |
| S²MIA-M |     0.848 |     0.854 |

ただし、S²MIA-TはTriviaQAのLlama-2-13BおよびVicunaではRAG-MIAを下回っており、すべての設定で両方の提案方式が優れているわけではない。S²MIA-Mはこれらの条件でもRAG-MIAを上回っている。

類似度指標の比較では、BLEUとパープレキシティの組み合わせが最も高性能だった。Natural QuestionsにおけるS²MIA-TのROC AUCは、BLEUのみでは0.751だったが、パープレキシティを追加すると0.892まで上昇した。コサイン類似度、ユークリッド距離、内積よりもBLEUの方が高い性能を示した。

DPRとContrieverの比較では、ROC AUCの差は比較的小さく、検索器が変わっても攻撃信号は残った。

一方、防御策を適用すると結果は大きく低下した。パラフレーズおよびプロンプト変更ではROC AUCが約0.49～0.61となり、ほぼランダムに近い。リランキングでは約0.59～0.70であり、攻撃信号が一定程度残った。

## 6. 考察

本研究の重要な点は、RAGに対するプライバシー漏洩を、LLMの「学習データ記憶」ではなく、**検索された文書が生成結果へ反映される現象**として捉えたことである。手法もBLEUとパープレキシティという比較的単純な特徴量だけで構成されており、複数のLLMと検索器で有効性を確認している。

ただし、以下の問題がある。

第一に、攻撃者の知識に関する仮定が十分に整理されていない。脅威モデルでは、攻撃者はデータベース内のサンプルを知らないとされるが、S²MIA-TとS²MIA-Mの学習には、メンバーと非メンバーのラベルが付いた参照データを使用している。実環境で既知メンバーをどのように取得するのか、シャドーRAGを構築するのか、攻撃者自身が挿入した文書を利用するのかが明確でない。

第二に、パープレキシティの計算には生成トークンの確率が必要である。したがって、最も高性能な構成は、生成テキストだけが返される通常のブラックボックスAPIよりも強いアクセス権を仮定している。テキストのみで動作するBLEU版は存在するものの、性能は明確に低下している。

第三に、「semantic similarity」と呼んでいる中心指標がBLEUである点には注意が必要である。BLEUは主としてn-gramの語句一致を測るため、この攻撃は意味的類似性というより、RAGが取得文書の表現をどの程度再利用・コピーするかを検出している可能性が高い。パラフレーズ防御によって性能が大幅に低下する結果も、この解釈と整合する。([arXiv][2])

第四に、評価は質問回答データに限定され、攻撃クエリとして元の質問をそのまま使用している。長文文書、チャンク分割、重複文書、曖昧なユーザークエリ、検索失敗、複数回問い合わせなど、実運用のRAG条件に対する一般化は未検証である。また、分散や信頼区間、低FPR領域でのTPRも報告されていない。

第五に、防御策がRAG本来の回答精度をどれだけ低下させるかが評価されていない。特に検索を妨害するパラフレーズは、攻撃だけでなく正常な検索性能も下げる可能性があるため、プライバシーと回答性能のトレードオフを測定する必要がある。

さらに、要旨と結論では「三つの防御を回避できる」と述べているが、実験本文ではパラフレーズとプロンプト変更が攻撃を効果的に緩和すると記述されている。表4の数値も後者を支持しており、論文の主張には明確な不整合がある。([arXiv][1])

## 7. まとめ

本論文は、RAGの外部データベースに対するメンバーシップ推論攻撃S²MIAを提案した。対象サンプルの一部をクエリとしてRAGへ入力し、生成結果とのBLEUおよび生成パープレキシティから、対象サンプルが検索データベースに存在するかを推定する。

従来型MIAがほぼランダム推定にとどまる一方、S²MIAは多くの条件でROC AUC 0.8台後半を達成した。したがって、RAGの外部データベースは、LLMを再学習していなくても、生成インターフェースを通じてメンバーシップ情報を漏洩し得る。

ただし、攻撃用の既知メンバーを必要とする点、生成確率へのアクセス、QA形式への依存、BLEUの表層的一致への依存などから、現実のブラックボックスRAGにそのまま適用できるとは限らない。

## 8. 総評

**長所**は、RAG特有のプライバシー問題を早期に明示した点、攻撃原理が簡潔で理解しやすい点、複数の生成モデル・検索器・防御条件を検証した点である。特に「検索結果に含まれたこと自体が生成文へ痕跡を残す」という観察は重要である。

**弱点**は、脅威モデルと参照データの入手方法の不整合、テキスト確率への依存、実験対象の狭さ、再現に必要な分類器設定等の不足、および防御回避に関する主張と実験結果の矛盾である。

**実際の掲載結果：ICASSP 2025採択・掲載。** ([IEEE Xplore][3])

**独自の査読判定：Weak Accept（弱採択、6/10）。**

研究課題の重要性と攻撃アイデアの明快さは採択に値する。一方で、攻撃者が既知メンバーを取得する方法、純粋なテキストのみのブラックボックス条件、実運用RAGでの評価、防御と回答性能のトレードオフを追加し、過大な防御回避の主張を修正する必要がある。
