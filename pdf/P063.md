# 1. はじめに

本論文「**Few-Shot Conversational Dense Retrieval**」は、複数ターンの会話履歴を考慮して関連文書を検索するモデル **ConvDR** を提案した研究である。著者は Shi Yu、Zhenghao Liu、Chenyan Xiong、Tao Feng、Zhiyuan Liuで、SIGIR 2021に採択・掲載された。([arXiv][1])

要点は、従来のように会話中の短い質問を完全な検索クエリへ書き換えるのではなく、**会話履歴全体を直接1つの密ベクトルへ変換して検索する**ことである。さらに、通常の検索用Dense Retrieverを教師として利用する知識蒸留により、会話検索用の関連度ラベルが少ない状況でも学習できるようにしている。

# 2. 背景

会話検索では、現在の質問だけでは検索意図が明確でないことが多い。例えば、

> 「ダーウィンの理論とは？」
> 「それは種分化とどう関係する？」

という会話では、「それ」がダーウィンの理論を指すことを過去の発話から補う必要がある。

従来手法では、現在の質問と会話履歴から、

> 「ダーウィンの理論は種分化とどう関係するか」

のような独立したクエリを生成し、その後にBM25やDense Retrieverで検索していた。しかし、クエリ書き換えモデルが誤った語を生成したり、重要な文脈を脱落させたりすると、その誤りが後段の検索へ伝播する。また、生成された語と文書中の語との語彙不一致も生じる。([arXiv][2])

Dense Retrievalは、クエリと文書を意味ベクトルに変換するため、この語彙不一致を緩和できる。一方でDense Retrieverは大量の関連度ラベルを必要とする傾向があり、少数の会話しか利用できない会話検索とは相性が悪い。この「**Dense Retrievalのデータ依存性**」と「**会話検索データの少なさ**」の矛盾が、本研究の中心的な課題である。([arXiv][2])

# 3. 提案手法

## ConvDR

ConvDRは、BERTベースのDual Encoderで構成される。

会話の第 (k) ターンでは、現在の質問だけでなく、それ以前の質問を連結してクエリエンコーダへ入力する。

[
\mathbf{q}_k
============

g_{\mathrm{ConvDR}}(q_1,\ldots,q_k)
]

文書も別のエンコーダによってベクトル化する。

[
\mathbf{d}=g_{\mathrm{doc}}(d)
]

検索スコアは単純な内積である。

[
s(q_k,d)=\mathbf{q}_k^\top\mathbf{d}
]

したがって、GPT-2などで文章としてクエリを書き換える処理は不要になる。

## 教師・生徒型の知識蒸留

少量データでConvDRを学習するため、通常の単発検索用Dense Retrieverである **ANCE** を教師モデルとして利用する。

教師には、人手で文脈を補完した「オラクル書き換えクエリ」 (q_k^*) を入力する。

[
\mathbf{q}_k^*
==============

g_{\mathrm{ANCE}}(q_k^*)
]

一方、生徒であるConvDRには、生の会話履歴 (q_1,\ldots,q_k) を入力する。生徒の出力を教師の出力へ近づけるため、平均二乗誤差を最小化する。

[
L_{\mathrm{KD}}
===============

\operatorname{MSE}
(\mathbf{q}_k,\mathbf{q}_k^*)
]

つまり、ConvDRは「正しい検索意図がDense空間のどこに配置されるべきか」を教師から学ぶ。文書ベクトルは、会話検索と通常検索で意味が変わらないという仮定の下、ANCEの文書表現をそのまま継承・固定する。([arXiv][2])

## ランキング損失

関連文書ラベルが利用できる場合は、正例文書のスコアを負例文書より高くするランキング損失も用いる。

[
L=L_{\mathrm{KD}}+L_{\mathrm{Rank}}
]

ただし、後述するように、ラベルが極端に少ないCAsTではランキング損失が逆効果となり、知識蒸留のみを使う方が高性能だった。

# 4. 実験方法

実験には以下の3データセットを使用している。

| データセット         | 設定       | 特徴                                 |
| -------------- | -------- | ---------------------------------- |
| TREC CAsT 2019 | Few-shot | 50会話・479質問。ただし評価済み部分は20会話・173質問    |
| TREC CAsT 2020 | Few-shot | 25会話・216質問。過去のシステム応答を参照する質問も含む     |
| OR-QuAC        | 十分な教師データ | 学習31,526質問、評価5,571質問の大規模な合成会話検索データ |

CAsTではMRR、NDCG@3、Hole@10を、OR-QuACではMAP@10、Recall@5、MRR@5を評価指標とした。Hole@10は上位10件のうち関連度判定が付与されていない文書の割合であり、CAsTの評価の不確実性を表す。([arXiv][2])

比較対象には、BM25、ANCE、GPT-2による自動クエリ書き換え、手動書き換えクエリ、BERTリランカーなどが含まれる。CAsTでは5分割交差検証を行い、実装手法間の差は置換検定 (p<0.05) で検定している。文書ベクトルは固定し、会話クエリエンコーダのみを主として学習した。([arXiv][2])

# 5. 結果

## 初段検索

主要な初段検索結果は次の通りである。

| データセット                       |                    ConvDR |           ANCE＋自動書き換え |           ANCE＋手動書き換え |
| ---------------------------- | ------------------------: | --------------------: | --------------------: |
| CAsT-19 MRR / NDCG@3         |         **0.740 / 0.466** |         0.665 / 0.409 |         0.740 / 0.461 |
| CAsT-20 MRR / NDCG@3         |         **0.501 / 0.340** |         0.375 / 0.255 |         0.591 / 0.422 |
| OR-QuAC MAP@10 / R@5 / MRR@5 | **0.607 / 0.750 / 0.616** | 0.451 / 0.584 / 0.457 | 0.507 / 0.642 / 0.515 |

CAsT-19では、ConvDRは自動書き換え方式を明確に上回り、手動書き換えを使ったANCEとMRRで同等、NDCG@3ではわずかに上回った。CAsT-20では手動書き換えには届かないものの、自動書き換えとの差を大きく縮めている。OR-QuACでは、手動書き換えを利用したモデルも上回った。([arXiv][2])

## リランキング

BERTリランカーを追加すると、OR-QuACではMAP@10が0.758、Recall@5が0.850、MRR@5が0.773となった。CAsT-19でもConvDRとBERTの組合せは高性能だった。

一方、CAsT-20ではConvDRとRRFを組み合わせた結果がMRR 0.545、NDCG@3 0.392であり、公開ベースラインのh2oloo_RUN2が示した0.621、0.493には届いていない。したがって「すべての設定で最良」ではなく、特に強い成果は**初段Dense Retrieval**にある。([arXiv][2])

## 学習方法の比較

CAsT-19のNDCG@3は以下のようになった。

* Zero-shot：0.202
* 知識蒸留のみ：**0.466**
* ランキング損失のみ：0.084
* 知識蒸留＋ランキング損失：0.157

少量の関連度ラベルを直接使うと、過学習や不安定化によって性能が低下した。対して、大量データがあるOR-QuACでは、ランキング損失のみでMRR@5が0.588、両損失を併用すると最高の0.616となった。これは、**少量データでは教師の表現を模倣し、大量データではランキング学習も併用する**という使い分けが必要であることを示す。([arXiv][2])

## 推論速度

CAsT-19における1クエリ当たりの総処理時間は、自動書き換え＋ANCEが805ms、ConvDRが560msだった。OR-QuACでは727msから327msへ短縮された。バッチ検索時のConvDRは、それぞれ17.4msと16.1msである。

速度向上の主因は、トークンを逐次生成するクエリ書き換え処理を削除したことである。([arXiv][2])

# 6. 考察

ConvDRが有効な最大の理由は、自然言語のクエリを書き換えるのではなく、**検索に必要な意味表現そのものを直接学習したこと**にある。自動書き換えでは、固有名詞の取り違え、代名詞解決の失敗、重要語の欠落が発生する。ConvDRでは、これらを明示的な文章として生成する必要がない。

分析実験では、ConvDRのクエリベクトルが手動書き換えクエリのベクトルに近づき、関連文書群の中央付近に配置されることが示された。また、会話が長くなるにつれて自動書き換え方式の性能が低下するのに対し、ConvDRは後半ターンでも比較的安定していた。([arXiv][2])

文脈ターンを1つずつ除去する実験では、重要な語を含むターンを除去した場合ほどConvDRのベクトルが大きく変化した。一方、無関係なターンを除去しても変化は小さかった。これは、すべての履歴を同じ重みで平均しているのではなく、検索意図に関係する履歴を選択的に利用していることを示唆する。([arXiv][2])

具体例として、ConvDRは「ACL」を会話履歴からスポーツ外傷のAnterior Cruciate Ligamentと判断できたが、手動書き換えを含む他方式はAccess Control Listに関連する文書を検索した。また、「Stanford Experiment」と「Milgram Experiment」の混同や、「speciation」に対するダーウィンの文脈の欠落も回避している。([arXiv][2])

ただし、次の制約がある。

第一に、Few-shotとはいっても完全な教師なしではない。知識蒸留には、人手で文脈を補完したオラクル書き換えクエリが必要である。削減されるのは主として文書関連度ラベルであり、クエリ書き換えのアノテーションコスト自体は残る。([arXiv][2])

第二に、CAsTの評価対象は小さく、ConvDRのHole@10もCAsT-19で40.3%、CAsT-20で42.2%と高い。検索上位に未判定文書が多いため、指標には相応の不確実性がある。([arXiv][2])

第三に、OR-QuACでは同一会話の関連文書がすべて同じWikipediaエンティティのページに存在する。この合成データ特有の構造によって、現実の横断的な会話検索より問題が単純化されている可能性がある。著者自身もOR-QuAC上の大幅な改善は慎重に解釈すべきだとしている。([arXiv][2])

さらに、入力が256トークンを超えると古いターンを破棄するため、非常に長い会話への適用性は十分検証されていない。文書表現を教師モデルから固定的に継承する設計も、会話特有の情報要求に文書側を適応させる余地を制限している。([arXiv][2])

# 7. まとめ

本論文は、会話検索を「クエリ書き換え問題」としてではなく、**文脈依存クエリを直接Dense表現へ写像する問題**として再定式化した。

ConvDRは、

1. 会話履歴全体をBERTで直接符号化する
2. 通常検索用ANCEの文書表現を再利用する
3. 手動書き換えクエリのDense表現を知識蒸留する
4. 内積だけで高速に初段検索する

という比較的単純な構成で、自動クエリ書き換え方式を大きく上回った。特に、少量データでは関連度ラベルによるランキング学習よりも、教師の表現を模倣する知識蒸留の方が有効であるという結果が重要である。

# 8. 総評

**実際の査読結果：採択。** 本論文は査読付き国際会議 **SIGIR 2021** に採択され、会議録の829–838ページに掲載されている。今回確認できた公開情報には、個別査読者のスコアや査読コメントは含まれていない。([arXiv][1])

**私見の査読判定：Accept。**

提案自体は複雑ではないが、「文章として正しいクエリを書き換える必要はなく、検索空間上で正しい位置を模倣すればよい」という問題設定が明快である。初段検索での一貫した改善、Few-shotと十分な教師データがある場合の詳細なアブレーション、推論速度、埋め込み可視化、文脈除去実験、事例分析まで揃っており、主張を多面的に裏付けている。

一方で、オラクル書き換えへの依存、小規模かつ不完全なCAsT評価、OR-QuACの合成的性質、CAsT-20のリランキングでは最良でない点は重要な弱点である。それでも、会話Dense Retrievalの基本的な設計指針を示した研究として、採択は妥当と評価できる。
