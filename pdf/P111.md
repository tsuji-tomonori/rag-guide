# P111：Learning Passage Impacts for Inverted Indexes

**著者**：Antonio Mallia、Omar Khattab、Torsten Suel、Nicola Tonellotto
**提案モデル**：DeepImpact
**採録先**：SIGIR 2021 Short Paper、pp. 1723–1727

## 1. はじめに

本論文は、BERTなどの文脈化言語モデルによる高精度な検索と、従来の転置インデックスによる高速な検索を両立させることを目的としている。

提案手法のDeepImpactは、文書中の各単語について「その単語が検索スコアへどれだけ寄与すべきか」を表す単一のインパクト値を学習する。学習した値は通常の転置インデックスに格納できるため、検索時にBERTを実行せず、クエリ語に対応するインパクト値を加算するだけでランキングできる。論文は、DeepImpactが従来の転置インデックス型検索を大きく上回り、ColBERTと組み合わせた場合には高い精度と低遅延を両立できると報告している。([arXiv][1])

## 2. 背景

一般的なニューラル検索システムは、BM25などで候補文書を取得し、その候補をBERT系モデルで再ランキングするカスケード構成を採る。しかし、BERTによるクエリ・文書間の詳細な相互作用は計算量が大きく、候補が数百から数千件ある場合には検索遅延が問題となる。([arXiv][1])

本論文が主に改善対象とする既存手法は、次の2つである。

**DeepCT**は、BERTから得られる文脈表現を使い、各文書語の擬似的な語頻度を推定する。ただし、各単語を独立した回帰問題として学習するため、複数のクエリ語のインパクトが協調して文書スコアを形成するようには最適化されない。また、もともと文書に存在しない語を追加できないため、語彙不一致には弱い。

**DocT5Query**は、文書から関連しそうなクエリを生成して文書へ追加する。これにより、文書中の既存語を反復して頻度を変える「Rewrite」と、文書に存在しなかった語を追加する「Inject」が生じる。ただし、追加後のスコアリング自体はBM25に依存しており、各語の適切な重要度は直接学習していない。([arXiv][1])

MS MARCOにおける分析では、Rewrite単独はMRR@10が0.215、Inject単独は0.194である一方、両方を含む通常のDocT5Queryは0.278となった。RecallについてはInjectが0.912、Rewriteが0.878であり、新語の追加は候補の網羅性、既存語の書き換えは上位順位の改善に寄与していることが示されている。

## 3. 提案手法

DeepImpactは、DocT5Queryによる語彙拡張と、文書全体のランキング目的に基づくインパクト学習を組み合わせる。

### 文書拡張

まず、DocT5Queryが生成した語のうち、元の文書に存在しない語を文書表現へ追加する。これにより、クエリと文書が異なる表現を用いる語彙不一致を緩和する。

その後、元の文書語と追加語の両方について、DeepImpactが新たなインパクト値を推定する。したがって、DocT5Queryが生成した語の出現回数をそのままBM25に渡すのではなく、各語が検索にどの程度重要かをニューラルモデルが再評価する。([arXiv][1])

### ニューラルネットワーク

元の文書語と拡張語を`[SEP]`で区切り、BERT-baseへ入力する。BERTは文書内の文脈を考慮した各トークンの表現を生成する。

各ユニーク語の最初の出現位置の表現を、ReLUを備えた2層MLPへ入力し、語・文書ペアごとに1つのスカラー値 (I(t,d)) を出力する。これがインパクト値である。

クエリ (q) と文書 (d) のスコアは、両者に共通して出現する語のインパクト値の和として計算される。

[
S(q,d)=\sum_{t\in q\cap d} I(t,d)
]

クエリ側のニューラルエンコーダや、クエリ・文書間のTransformer計算は不要である。ただし、検索時のマッチング自体は基本的に完全一致であり、意味的な対応は事前の文書拡張に依存する。([arXiv][1])

### 学習方法

学習にはMS MARCO公式の三つ組データを用いる。

* クエリ
* 関連文書
* 非関連と推定された文書

各文書について、クエリ語と一致するインパクト値を加算し、関連文書と非関連文書のスコアを求める。関連文書のスコアが高くなるよう、pairwise softmax cross-entropyでモデル全体を最適化する。

BERT-base、最大入力長160トークン、バッチサイズ32、学習率 (3\times10^{-6})、Adam、10万イテレーションという設定で学習している。([arXiv][1])

### インデックスへの格納

学習後は、クエリを与えずに全文書のインパクト値を事前計算する。浮動小数点値をそのまま格納するとインデックスが大きくなるため、線形量子化によって8ビットの整数へ変換する。

著者らの実験では、8ビット量子化による検索精度の低下は観測されなかった。検索時には、通常の転置リストからクエリ語のインパクト値を取得し、その整数値を加算する。([arXiv][1])

## 4. 実験方法

実験にはMS MARCO Passage Rankingコーパスを使用し、以下のクエリ集合で評価している。

* MS MARCO Dev Queries
* TREC Deep Learning Track 2019
* TREC Deep Learning Track 2020

第一段検索では、BM25、DeepCT、DocT5Query、DeepImpactを比較した。再ランキング実験では、DeepImpactが取得した上位1,000件をEPICまたはColBERTで再ランキングし、ANN検索を候補生成に用いるColBERT End-to-Endと比較した。([arXiv][1])

評価指標はMRR@10、NDCG@10、MAP、第一段のRecall、平均応答時間MRTである。有意差検定にはBonferroni補正を適用した対応ありt検定を使用し、有意水準を (p<0.05) としている。

転置インデックスはAnseriniで構築し、CIFF形式を介してPISAへ変換した。検索にはMaxScoreを使用し、上位1,000件を取得している。DocT5Queryでは各パッセージに40個の生成クエリを連結した。

遅延評価はIntel Xeon Platinum 8268のCPU 1コアで行い、ColBERTにはNVIDIA RTX8000を使用した。CPU手法とGPU手法でハードウェアが異なるため、実時間の比較はシステム全体の実装条件を含むものとして解釈する必要がある。([arXiv][1])

## 5. 結果

### 第一段検索

MS MARCO Dev Queriesにおける結果は次のとおりである。

| 手法             |   NDCG@10 |    MRR@10 |       MAP |      MRT |
| -------------- | --------: | --------: | --------: | -------: |
| BM25           |     0.235 |     0.188 |     0.196 | 13.24 ms |
| DeepCT         |     0.298 |     0.244 |     0.252 | 10.91 ms |
| DocT5Query     |     0.338 |     0.278 |     0.286 | 12.62 ms |
| **DeepImpact** | **0.385** | **0.326** | **0.332** | 58.64 ms |

DeepImpactはDocT5Queryに対して、相対値でNDCG@10を約13.9%、MRR@10を約17.3%、MAPを約16.1%改善した。MS MARCO Devでは、全有効性指標について他手法との差が統計的に有意である。

TREC 2019でも、DeepImpactはNDCG@10で0.695、MRR@10で0.863、MAPで0.456となり、DocT5Queryの0.648、0.799、0.405を上回った。TREC 2020でもDeepImpactは0.651、0.820、0.426であり、全指標で最良だった。

一方、第一段検索単独の応答時間は58.64ミリ秒であり、DocT5Queryの12.62ミリ秒より約4.6倍遅い。著者らは、DeepImpactが生成するスコア分布がMaxScoreの枝刈りに適していないことを原因として挙げている。したがって、「転置インデックスを使うため常にBM25並みに高速」という結果ではない。([arXiv][1])

### 再ランキング

MS MARCO Devでは、DeepImpactで候補を取得してColBERTで再ランキングする構成が、NDCG@10で0.425、MRR@10で0.362、MRTで81.00ミリ秒となった。

ColBERT End-to-Endは、NDCG@10が0.424、MRR@10が0.361、MRTが380.97ミリ秒である。すなわち、精度はほぼ同等またはわずかに高く、処理時間は約4.7分の1である。TREC 2019と2020を含めると、DeepImpact＋ColBERTはColBERT End-to-Endに対して約4.4～5.1倍の高速化を達成した。

### 候補数を減らした場合

候補数 (k=10) の第一段Recallは、BM25が0.394、DeepCTが0.484、DocT5Queryが0.542、DeepImpactが0.584だった。ColBERTで再ランキングした後のMRR@10も、DeepImpactが0.350で最も高い。

一方、(k=200) ではDeepImpactとDocT5Queryの再ランキング後MRR@10がともに0.361、(k=1000) ではともに0.362となる。このことから、DeepImpactの利点は、特に小さな候補集合から関連文書を取りこぼさずに取得する場面で大きい。

## 6. 考察

DeepImpactの有効性は、DocT5QueryとDeepCTの長所を補完的に組み合わせた点にある。DocT5Queryによる語の追加で候補のRecallを改善し、その後、各語のインパクト値をランキング損失から直接学習することで上位順位の精度を改善している。

特に重要なのは、単語ごとの擬似語頻度を独立して学習するのではなく、クエリに含まれる複数語のインパクト値の合計を学習目的にしている点である。この設計により、ある単語の適切な重みを他のクエリ語との関係を含めて調整できる。

ただし、以下の制約がある。

第一に、第一段検索単独の遅延は既存の疎検索手法より大きい。論文の「高速」という主張は、主としてColBERT End-to-Endとの比較であり、BM25やDocT5Queryとの比較では成立していない。

第二に、評価対象がMS MARCOコーパスと、同じコーパスを用いるTREC DLクエリに限られる。異なる分野、長文文書、多言語検索、分布外クエリに対する一般化は検証されていない。

第三に、全パッセージに対するDocT5Query生成とBERTによるインパクト推定が必要だが、文書拡張時間、インパクト推論時間、最終的なインデックス容量などのオフラインコストは十分に報告されていない。

第四に、検索時には依然として語の完全一致を使う。拡張時に生成されなかった同義語や言い換えには対応できない。著者らも、より緩いクエリ・文書語マッチング、より高度な語彙拡張、MaxScoreに適したスコア分布の学習を今後の課題としている。([arXiv][1])

## 7. まとめ

本論文は、文書拡張と文脈依存の語重み学習を組み合わせ、通常の転置インデックスで利用できるDeepImpactを提案した。

DeepImpactは、MS MARCOの第一段検索においてBM25、DeepCT、DocT5Queryを大きく上回った。また、DeepImpactで候補を生成しColBERTで再ランキングすることで、ColBERT End-to-Endと同等以上の精度を維持しながら、最大約5.1倍の高速化を実現した。

一方、DeepImpact単独の第一段検索は既存の疎検索より遅く、オフライン計算量や異種データへの一般化についても追加検証が必要である。

## 8. 総評

**査読結果：採択。** 本論文はSIGIR 2021のShort Paperとして正式に採録され、プログラム上でも「Short; sp1392」として発表されている。今回確認した公開情報では、個別の査読コメントや査読スコアは公開されていない。([sigir.org][2])

総合的には、採択は妥当と評価できる。特に、ニューラルモデルの出力を既存の転置インデックスへ落とし込み、検索エンジンの実装資産を利用できる形にした点は実用性が高い。また、候補生成のRecall改善が再ランキングコストの削減につながることを、ColBERTとの比較で定量的に示している。

一方で、効率性の主張には注意が必要である。第一段検索そのものはDocT5Queryより大幅に遅く、真の貢献は「最速の疎検索」ではなく、**高品質な候補集合を小さなカットオフで生成し、高価な再ランキング処理を削減できること**にある。短報としては明確で強い結果を提示しているが、一般化性能、インデックス容量、オフラインコスト、構成要素ごとのアブレーションは後続研究で補う必要がある。
