# P122：Deduplicating Training Data Makes Language Models Better

## 1. はじめに

本論文は、大規模言語モデルの学習コーパスに含まれる**重複・近似重複テキスト**が、モデルの性能、評価の信頼性、学習効率、訓練データの記憶に与える影響を調査した研究である。

著者らは、C4などの一般的な言語モデル用データセットには、同一文書だけでなく、テンプレートの一部だけを変更した文書や、長い共通部分列を持つ文書が大量に含まれていることを示した。例えばC4には、同一の61単語の文章が訓練集合に61,036回、検証集合に61回出現していた。さらに、通常のC4で学習したモデルが無条件生成したトークンの1%以上が、訓練データから50トークン以上逐語的にコピーされた部分であった。([ACL Anthology][1])

論文の中心的な主張は、**大規模化の前に学習データを適切に重複除去するだけで、モデルをより効率的かつ安全に学習でき、評価も正確になる**というものである。

## 2. 背景

Webから収集した大規模コーパスには、次のような重複が混入する。

* 同じ記事が複数のサイトに転載される
* 商品ページや航空券ページのように、一部の地名・価格だけが異なる
* 広告、利用規約、ライセンス、定型文が多数のページに付加される
* Webクローラが同じページの別URLや別バージョンを収集する

C4とRealNewsでは、作成時点ですでにハッシュ等による重複除去が行われていた。しかし、段落全体の一致や先頭100文字の一致を使う単純な処理では、少数の可変フィールドを含むテンプレート文書や、書式だけが異なる転載記事を検出できなかった。([ACL Anthology][2])

重複データには主に三つの問題がある。第一に、同じテキストを何度も学習するため、モデルがそのテキストを過剰に記憶する。第二に、訓練集合と検証集合の間に重複があると、汎化性能ではなく記憶能力によってPerplexityが改善し、評価が過大になる。第三に、情報量を増やさないデータの反復に計算資源が消費される。

## 3. 提案手法

著者らは、性質の異なる二つの重複除去法を提案する。

### EXACTSUBSTR：完全一致部分列の除去

EXACTSUBSTRは、文書全体ではなく、複数文書間で完全一致する**長い部分列**を検出する。

全データをBPEトークン列として連結し、その列に対してSuffix Arrayを構築する。辞書順で隣接するSuffix間の共通接頭辞を調べることで、総当たり比較をせずに、重複部分列をほぼ線形時間で検出できる。

本実験では、50 BPEトークン以上完全一致する部分列を重複とみなし、一方から削除する。50という閾値は、25トークン一致を人手確認して誤検出が見つからなかったうえで、さらに保守的に倍増して設定された。([ACL Anthology][2])

### NEARDUP：近似重複文書の除去

NEARDUPは、文書全体がほぼ同じだが、一部だけ異なるケースを検出する。

各文書を5-gram集合に変換し、MinHashによって文書間のJaccard類似度を効率的に近似する。MinHashの署名長は9,000、LSHのパラメータは (b=20)、(r=450) としている。候補対についてさらに編集距離を計算し、正規化編集類似度が0.8を超える文書を重複と判定する。その後、類似文書のグラフから連結成分を構成し、重複文書群を削減する。([ACL Anthology][2])

EXACTSUBSTRは文書の一部分だけがコピーされたケースに強く、NEARDUPは地名や日付などが少し変更されたテンプレート文書に強い。両者は代替関係ではなく、相補的な手法である。

## 4. 実験方法

対象データセットは、C4、RealNews、LM1B、英語版Wiki-40Bの4種類である。C4は約3.6億Web文書、RealNewsは約3,100万のニュース文書、LM1Bは約3,000万文、Wiki-40B英語版は約290万Wikipedia記事から構成される。([ACL Anthology][2])

C4について、以下の3条件でdecoder-only Transformerを学習した。

1. C4-ORIGINAL：元のC4
2. C4-NEARDUP：NEARDUPによる重複除去後
3. C4-EXACTSUBSTR：EXACTSUBSTRによる重複除去後

モデルは1.5BパラメータのXLモデルと110MパラメータのBaseモデルである。Baseモデルは各条件について3乱数シード、計9モデルを学習した。最大系列長は512、語彙数は約50,000、学習期間は各データセット約2エポックである。XLモデルは24層・32 attention heads、Baseモデルは12層・12 attention headsで、最適化にはAdafactorを使用した。([ACL Anthology][2])

評価項目は以下である。

* C4、LM1B、Wiki-40BにおけるPerplexity
* 訓練データと重複する検証例／重複しない検証例の性能差
* 無条件生成における訓練データの逐語的再生成率
* 32トークンのプロンプトから正解の続きを再生成する割合
* 公開済みTransformer-XL、GROVER、GPT-Neoにおける重複の影響

生成実験ではtop-(k) samplingの (k=50) を用い、各モデルから最大512トークンの系列を10万件生成した。生成トークンが訓練データ中の50トークン以上の完全一致部分列に属する場合を「記憶されたトークン」と判定した。([ACL Anthology][2])

## 5. 結果

### データセット中の重複量

NEARDUPが訓練集合中の近似重複例として検出した割合は、C4が3.04%、RealNewsが13.63%、LM1Bが4.86%、Wiki-40Bが0.39%であった。

EXACTSUBSTRが重複部分列として検出した訓練トークンの割合は、C4が7.18%、RealNewsが19.4%、LM1Bが0.76%、Wiki-40Bが2.76%であった。RealNewsでは、約5分の1のトークンが50トークン以上の重複部分列に含まれていた。([ACL Anthology][2])

### 訓練・検証集合間の漏洩

検証例のうち、訓練集合内に近似重複が存在した割合は、C4で4.60%、RealNewsで14.35%、LM1Bで4.92%、Wiki-40Bで0.72%だった。

したがって、通常の検証集合で測定されるPerplexityには、未知データへの汎化だけでなく、訓練データの記憶による改善が含まれている。([ACL Anthology][2])

### Perplexity

重複除去モデルは、C4全体および重複のないC4検証例では、元データモデルとほぼ同等のPerplexityを示した。一方、訓練集合と重複する検証例では、元データモデルだけが著しく低いPerplexityを示した。これは性能向上ではなく、訓練例の記憶による評価値の押し上げと解釈できる。

LM1BとWiki-40Bによる外部評価では、NEARDUPモデルが一貫して良好であり、XLのEXACTSUBSTRモデルはWiki-40B上でPerplexityを約3ポイント改善した。つまり、データ量を削減しても性能は悪化せず、一部では約10%改善した。([ACL Anthology][2])

### 訓練データの記憶

2エポック学習後の無条件生成で、訓練データからコピーされた50トークン以上の部分列に属するトークンの割合は次の通りだった。

| 学習データ          | 記憶トークン率 |
| -------------- | ------: |
| C4-ORIGINAL    |  1.571% |
| C4-NEARDUP     |  0.264% |
| C4-EXACTSUBSTR |  0.168% |

重複除去によって、逐語的な訓練データ再生成は約6～10分の1に減少した。([ACL Anthology][2])

重複訓練例の先頭32トークンをプロンプトとして与えた場合、元データモデルは40%以上の確率で元文書の続きをほぼそのまま生成した。重複除去モデルではこの割合が大幅に低下したものの、完全にはゼロにならず、より厳しい重複除去が必要であることも示された。([ACL Anthology][2])

### 既存モデルへの影響

LM1Bで学習されたTransformer-XLのPerplexityは、重複検証例で10.11、重複のない検証例で23.58だった。評価データが訓練データと重複しているだけで、Perplexityが半分以下になっている。

また、公開済みGROVER-Mega生成文の1.38%、GPT-Neo 1.3B生成文の5%以上のトークンが、それぞれの訓練集合中の50トークン以上の完全一致部分列に属していた。([ACL Anthology][2])

### 計算効率

C4に対するEXACTSUBSTRでは、トークン数が177.3Bから165.4Bに減少した。NEARDUPモデルの学習時間は約131時間から126時間に短縮された。

重複検出自体についても、96コア、768GBメモリの計算機上で、約350GBのC4のSuffix Array構築が12時間未満、その後の重複除去が1時間未満で完了した。モデル学習に比べれば、前処理コストは小さい。([ACL Anthology][2])

## 6. 考察

本論文の重要な点は、重複除去を単なるストレージ節約ではなく、**モデル性能、プライバシー、評価の妥当性、計算効率を同時に改善するデータ品質管理**として位置付けたことである。

特に説得力があるのは、データセット中の重複率を測定するだけでなく、実際に1.5Bパラメータモデルを再学習し、重複除去がPerplexityを悪化させず、記憶を約10分の1に減らすことを因果的に示している点である。

一方、以下の限界がある。

第一に、分析対象が英語データセットに限定されている。第二に、モデルの再学習実験は主にC4で行われており、コード、会話、書籍、多言語コーパスへの一般化は直接検証されていない。第三に、評価は主にPerplexityと逐語的記憶であり、下流タスクや事実知識の保持への影響は十分に調べられていない。

また、すべての重複が有害とは限らない。引用、帰属表現、重要な定義、検索やclosed-book QAで必要な知識まで削除される可能性がある。著者ら自身も、重複除去によって帰属部分だけが除去され、内容と出典の結び付きが弱まる危険や、重複除去だけではパスワード・医療情報等の機密データを排除できないことを指摘している。([ACL Anthology][2])

## 7. まとめ

本論文は、言語モデル用コーパスに含まれる重複を、EXACTSUBSTRとNEARDUPという二つのスケーラブルな方法で検出・除去した。

主要な結論は以下である。

* 一般的な言語モデル用データセットには大量の完全重複・近似重複が存在する
* 訓練・検証集合間の重複により、Perplexityが過大評価される
* 重複除去してもモデル性能は低下せず、外部データでは改善する場合がある
* 訓練データの逐語的再生成を約10分の1に削減できる
* データ量と学習時間を削減できる
* 重複除去はプライバシー対策の一部としても有効である

したがって、重複除去は大規模言語モデルの学習パイプラインにおける、事実上の標準前処理とすべきであるというのが本論文の結論である。([ACL Anthology][2])
