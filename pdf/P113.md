# P113：Open-Domain Question Answering Goes Conversational via Question Rewriting

**著者**：Raviteja Ananthaほか
**掲載先**：NAACL-HLT 2021 Main Conference、pp. 520–534 ([ACL Anthology][1])

## 1. はじめに

本論文は、会話中の省略や代名詞を含む質問を、単独でも意味が通じる質問へ書き換えることで、既存のオープンドメイン質問応答システムを会話型質問応答へ拡張する研究である。

例えば、直前にヨガについて話している状況での「Does it help in reducing stress?」を、「Does Yoga help in reducing stress?」へ書き換える。これにより、検索器や読解モデルは会話履歴を直接処理せずとも、通常の単発質問として回答を探索できる。論文の中心的な成果は、約1.36万会話・約8万質問を含むデータセット **QReCC** の構築と、その上での質問書き換え・文書検索・回答抽出を組み合わせたベースラインの提示である。([ACL Anthology][2])

## 2. 背景

従来のQuACやCoQAなどの会話型QAデータセットでは、回答対象が単一文書内に限定されることが多かった。しかし実際の情報探索では、会話の途中で話題が変化したり、異なるウェブページから回答を探したりする必要がある。

一方、TREC CAsTは大規模な文書集合からの会話型検索を扱っていたが、会話数が80件と小さく、学習データや回答スパンを十分に提供していなかった。OR-QuACなども存在したが、Wikipediaを中心とした限定的な文書集合であり、実ウェブに近い会話型QAを包括的に学習・評価する環境は不足していた。QReCCは、次の3タスクを一つのデータセットで扱えるように設計されている。

* 質問書き換え：Question Rewriting、QR
* パッセージ検索：Passage Retrieval、PR
* 読解・回答抽出：Reading Comprehension、RC

そのため、本研究は単なる質問書き換えモデルの提案というより、**オープンドメイン会話型QAを学習・分析するための基盤整備**に重点を置いている。([ACL Anthology][2])

## 3. 提案手法

### QReCCデータセット

QReCCは、QuAC、TREC CAsT、Natural Questionsの質問を会話の種として構築された。最終的な規模は13,598会話、79,952質問であり、1会話当たりの平均質問数は6である。各質問には、回答、会話履歴を補完した質問書き換え、人手で回答を発見したウェブページの出典が付与されている。([ACL Anthology][2])

質問書き換えは次の4種類に分類される。

| 種類          | 内容           | 例                                              |
| ----------- | ------------ | ---------------------------------------------- |
| Insertion   | 不足する語を追加     | “What are the main types?” → “…types of Yoga?” |
| Removal     | 不要な語を削除      | 会話依存の補足表現を除去                                   |
| Replacement | 語を削除し、別の語を追加 | “Does it help?” → “Does Yoga help?”            |
| Copy        | 書き換え不要       | 元の質問が既に自己完結                                    |

全体の約52%がReplacement、約38%がInsertionであり、単純な代名詞の共参照解決だけでなく、省略された情報を生成する能力が必要となる。([ACL Anthology][2])

### 質問応答パイプライン

提案ベースラインは、BERTseriniに質問書き換えを追加した3段階構成である。

1. **質問書き換え**
   現在の質問と過去の質問・回答列から、自己完結した質問を生成する。

2. **パッセージ検索**
   書き換えた質問をBM25へ入力し、約5,400万パッセージから上位100件を検索する。

3. **回答抽出**
   BERT-Largeが各パッセージから回答スパンを抽出する。最終スコアはBM25の検索スコアとBERTの読解スコアを線形補間して計算する。([ACL Anthology][2])

質問書き換えモデルとして、共参照解決器、PointerGenerator、GECOR、CopyTransformerなどを比較し、最終的に **Transformer++** を採用している。Transformer++は、質問側と会話文脈側に対応する2種類の語彙分布を生成し、トークンごとに重み付けして統合するGPT-2ベースのモデルである。([ACL Anthology][2])

## 4. 実験方法

### データ分割

| 分割          |   会話数 |    質問数 |
| ----------- | ----: | -----: |
| Train       | 約8.7K | 約50.8K |
| Development | 約2.2K | 約12.7K |
| Test        | 約2.8K | 約16.4K |

回答出典ページはWayback Machineから取得され、Common Crawlのランダムなウェブページを妨害文書として追加している。最終的な検索対象は約1,000万ページ、54,241,550パッセージである。([ACL Anthology][2])

### 質問書き換えの評価

人間の判定との相関を調べるため、Exact Match、BLEU、METEOR、ROUGE、Universal Sentence Encoder、検索結果の重複度などを比較した。質問書き換えは参照文と表面的に一致しなくても意味的に正しい場合があるため、単純な文字列一致だけでは十分でないという問題意識に基づいている。

モデル比較には、人間との相関が高かった以下の3指標を使用した。

* ROUGE-1 Recall
* Universal Sentence Encoder類似度
* ウェブ検索結果のRecall@10

### 質問応答の評価条件

QA性能は次の3条件で評価された。

* **End-to-End**：質問書き換え、検索、回答抽出をすべて実行
* **Known Context**：人間が示した正解パッセージを直接入力し、検索誤差を除外
* **Extractive Upper Bound**：検索上位100件から、正解とのF1が最大となるスパンを選択

これにより、質問書き換え、検索、読解のどの段階がボトルネックになっているかを切り分けている。([ACL Anthology][2])

## 5. 結果

### 質問書き換え

人間の評価とのPearson相関は、従来よく使われていたBLEUが0.58であったのに対し、ROUGE-1 Recallが0.63、USEが0.67、検索結果のRecall@10が0.80となった。したがって、下流の検索結果を直接比較する指標が、人間による意味的な正しさの判断に最も近かった。

Transformer++は、全自動モデル中で最高性能を示した。

| 書き換え方法            | ROUGE-1 R |      USE |   検索R@10 |
| ----------------- | --------: | -------: | -------: |
| 共参照解決器            |      67.1 |     82.3 |     56.1 |
| CopyTransformer   |      86.1 |     92.8 |     79.4 |
| **Transformer++** |  **89.5** | **95.2** | **83.2** |
| 人手書き換え            |      94.6 |     97.3 |     87.2 |

単純な共参照解決より、文脈中の語をコピーしながら不足情報を生成できるモデルの方が大幅に優れていた。([ACL Anthology][2])

### パッセージ検索

| 質問形式              |        MRR | Recall@10 | Recall@100 |
| ----------------- | ---------: | --------: | ---------: |
| 元の質問              |     0.0343 |      6.12 |      11.71 |
| **Transformer++** | **0.1586** | **26.52** |  **41.51** |
| 人手書き換え            |     0.1994 |     32.78 |      49.36 |

Transformer++による書き換えで、Recall@10は6.12から26.52へ約4.3倍になった。会話依存の質問をそのまま検索器へ渡すことが、検索性能を大きく損なっていたことが分かる。([ACL Anthology][2])

### End-to-End QA

| 設定                     | 質問形式              |        F1 |       EM |
| ---------------------- | ----------------- | --------: | -------: |
| End-to-End             | 元の質問              |      9.07 |     0.32 |
| End-to-End             | **Transformer++** | **19.10** | **1.01** |
| End-to-End             | 人手書き換え            |     21.82 |     1.23 |
| Known Context          | Transformer++     |     32.34 |     4.04 |
| Known Context          | 人手書き換え            |     36.42 |     4.70 |
| Extractive Upper Bound | —                 |     75.45 |    25.07 |

Transformer++によってEnd-to-End F1は9.07から19.10へ倍増し、人手書き換えの21.82に近づいた。一方、正解パッセージを与えてもF1は32.34であり、検索だけでなく回答抽出にも大きな改善余地がある。

## 6. 考察

本研究の最も重要な知見は、**質問書き換えが単なる前処理ではなく、会話型検索の成否を左右する主要構成要素である**ことを、下流の検索・QA性能によって定量的に示した点である。特に、元質問とTransformer++の間の検索性能差は大きく、会話文脈を検索クエリへ明示的に反映する必要性が確認できる。

また、End-to-EndとKnown Contextの差から検索誤差の影響が、Known ContextとExtractive Upper Boundの差から読解モデルの限界が分かる。QReCCの回答には、原文の単一スパンではなく、複数箇所を要約・言い換えた回答も含まれる。このため、単純な抽出型QAだけでなく、生成型・要約型の回答モデルが必要である。([ACL Anthology][2])

主な限界は次のとおりである。

* 検索対象文書の追加正解判定に、回答とのトークンF1が0.8以上というヒューリスティックを用いており、人手による十分な検証は行われていない。
* 検索結果に基づく書き換え評価は人間との相関が高い一方、検索エンジンや時点によって結果が変化し、計算コストと再現性に問題がある。
* 同一質問に複数の正しい回答表現が存在するため、Exact Matchや単一参照F1は、実際には正しい回答を過小評価する。
* 会話は既存QAデータを種にした自己対話形式で構築されており、実ユーザーとシステム間の自然な失敗、訂正、曖昧性を完全には再現していない可能性がある。([ACL Anthology][2])

## 7. まとめ

本論文は、質問書き換え・検索・読解を統合して評価できる大規模データセットQReCCを構築し、質問書き換えによって既存のオープンドメインQAを会話型QAへ転用できることを示した。

主要な結論は以下である。

* 質問書き換えにより検索性能とEnd-to-End QA性能が大幅に向上する。
* 単純な共参照解決では、省略された情報を補えない。
* Transformer++は人手書き換えに近い性能を示すが、依然として差が残る。
* 検索と回答抽出の両方が未解決のボトルネックである。
* 抽出型QAだけでは扱えない要約・言い換え回答が多く、生成型QAが必要である。([ACL Anthology][2])

## 8. 総評

**査読結果：採択（Accept）**

実際に本論文は、**NAACL-HLT 2021のMain Conference論文として採択・掲載**されている。個別査読者の点数や査読コメントは、確認できる公開記録には掲載されていない。([ACL Anthology][1])

内容に基づく査読評価としても **Accept** が妥当である。新規モデル単体の技術的革新性よりも、実ウェブ規模の会話型QAデータセット、タスク分解可能なアノテーション、評価指標の検証、強力な初期ベースラインを同時に提供した点に高い価値がある。特にQReCCは、後続の会話型検索・Retrieval-Augmented QA研究を促進する基盤として影響力が大きい。

一方で、検索コーパスの再構築性、正解パッセージ判定のヒューリスティック、単一参照による回答評価には改善の余地がある。したがって評価は、**強いデータセット論文として採択。ただし評価設計と自然会話性には留保あり**、という位置付けになる。
