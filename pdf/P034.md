**対象論文**
Stephen Robertson and Hugo Zaragoza, “The Probabilistic Relevance Framework: BM25 and Beyond,” *Foundations and Trends in Information Retrieval*, Vol. 3, No. 4, pp. 333–389, 2009. 本稿は新規アルゴリズムの実験論文ではなく、確率的関連性枠組みからBM25・BM25Fまでを導出・整理するサーベイ／チュートリアル論文である。([nowpublishers.com][1])

## 1. はじめに

本論文の目的は、情報検索における代表的なランキング関数BM25を、単なる経験的な式としてではなく、**文書がクエリに関連する確率を推定する枠組み**から説明することである。

中心となる考え方は、各クエリ–文書対に観測できない「関連／非関連」の変数を置き、関連確率が高い順に文書を並べることである。そこから二値独立モデル、関連フィードバック、BM25、構造化文書向けのBM25F、非テキスト特徴の統合、パラメータ最適化までを一つの流れとして整理している。([staff.city.ac.uk][2])

重要なのは、本稿がBM25の新しい性能値を報告する論文ではない点である。著者ら自身が「主として実験的なサーベイではなく、理論展開、論理、仮定を重視する」と明記している。([staff.city.ac.uk][2])

## 2. 背景

確率的関連性枠組み、Probabilistic Relevance Framework: PRFでは、検索クエリをユーザーの情報要求の不完全な表現とみなし、関連性を「その情報要求に対して、ユーザーがその文書を提示してほしいと思うか」と捉える。

基本モデルは、次の仮定に立つ。

第一に、関連性は他の文書とは独立に、対象文書と情報要求の組から判定できる。第二に、関連性は関連／非関連の二値で表される。著者らは、段階的関連性や複数検索意図を考えれば、これらの仮定には議論の余地があることも認めている。([staff.city.ac.uk][2])

その上で、**確率ランキング原理**、Probability Ranking Principle: PRPを採用する。すなわち、利用可能な情報の下で文書を関連確率の降順に並べれば、その情報から得られる最良の検索性能が期待できる、という原理である。([staff.city.ac.uk][2])

実際に扱えるスコア関数へ変換するため、以下の近似・仮定も導入される。

* 文書は基本的にbag-of-wordsとして表現する。
* 関連／非関連という条件の下で、各語の出現は条件付き独立である。
* 初期検索ではクエリに含まれる語だけを関連性の証拠として扱う。
* 確率そのものではなく、文書の順位だけが保存されればよいとする。

条件付き語独立性は現実には厳密に成立しないが、単純で計算可能な加算型スコアを得られ、転置インデックス上で効率的に実装できるという実務上の利点がある。([staff.city.ac.uk][2])

## 3. 提案手法

### 3.1 PRFから加算型ランキング関数への変換

出発点は関連確率 (P(\mathrm{rel}\mid d,q)) である。これをオッズ比に変換し、ベイズの定理を適用し、文書間の順位に影響しない項を除去する。さらに語の条件付き独立性を仮定し、対数を取ることで、積を語ごとの和に変換する。

最終的には、次の形式のスコアが得られる。

[
\mathrm{score}(d,q)
===================

\sum_{t\in q\cap d}W_t(tf_{t,d})
]

すなわち、クエリと文書の双方に現れる語について、語ごとの証拠を加算する。存在しない語を明示的に処理しなくてよいため、転置インデックスとの相性がよい。([staff.city.ac.uk][2])

### 3.2 二値独立モデルとRSJ重み

語の頻度を無視して、語が文書に存在するか否かだけを扱うと、Binary Independence Modelになる。

関連文書数を (R)、語 (t) を含む関連文書数を (r_t)、全対象文書数を (N)、語を含む文書数を (n_t) とすると、Robertson–Spärck Jones重みは次のようになる。

[
w_t^{RSJ}
=========

\log
\frac{
(r_t+0.5)(N-R-n_t+r_t+0.5)
}{
(n_t-r_t+0.5)(R-r_t+0.5)
}
]

0.5は、確率推定が0または1となって対数重みが無限大になることを避けるための擬似カウントである。

関連性判定が一件もない場合、(R=r_t=0) とすることで、次のIDFに近い式が得られる。

[
w_t^{IDF}
=========

\log
\frac{N-n_t+0.5}{n_t+0.5}
]

この意味で、IDFは「関連文書に現れやすく、非関連文書には現れにくい語ほど重くする」という確率的関連性重みの特殊ケースとして解釈できる。([staff.city.ac.uk][2])

### 3.3 関連フィードバックとクエリ拡張

ユーザーから関連性判定が得られた場合、IDFをRSJ重みに更新できる。ただし著者らは、既存クエリ語の重みを変更するだけでは一般に十分な改善が得られず、関連文書から新しい語を追加するクエリ拡張が必要だと説明する。

追加候補語は、関連文書と非関連文書の平均スコア差をどれだけ広げるかを表す「offer weight」によって順位付けされる。擬似関連フィードバックでは、初回検索の上位5～10文書程度を関連文書と仮定して同じ処理を行う。平均的には改善が期待できる一方、初回検索が失敗した難しいクエリでは、誤った方向へクエリが拡張される危険がある。([staff.city.ac.uk][2])

### 3.4 ElitenessモデルとBM25

BM25の核心は、同じ語が文書内に何度も出現しても、その追加的な証拠は次第に小さくなるという**語頻度の飽和**である。

著者らは、各語–文書対に「eliteness」という潜在二値変数を導入する。これは、その文書が当該語の表す概念を本質的に扱っているかどうか、すなわち一種のaboutnessを表す。語頻度はelitenessに依存し、elitenessが関連性と関係すると仮定する。語頻度分布を2つのポアソン分布の混合として扱う2-Poissonモデルから、語頻度が増えると重みは増加するものの、一定の上限へ漸近する性質が導かれる。([staff.city.ac.uk][2])

ただし、厳密な2-Poisson式はパラメータ推定が困難である。そのため、著者らは主要な形状を次の単純な関数で近似する。

[
\frac{tf}{k+tf}
]

ここは重要であり、BM25の飽和関数は生成モデルから完全に閉形式で導出されたものではなく、理論から得られる定性的な形状に対する実用的なパラメトリック近似である。([staff.city.ac.uk][2])

### 3.5 文書長正規化

長い文書で語の出現回数が多くなる理由として、論文は二つの極端なケースを考える。

* **Verbosity**：同じ内容を冗長に表現している。
* **Scope**：より多くの話題や内容を扱っている。

前者なら文書長で完全に正規化すべきだが、後者なら長い文書に含まれる語の証拠を過度に減らすべきではない。そこで、パラメータ (b) によるソフトな正規化を用いる。

[
B=(1-b)+b\frac{|d|}{\mathrm{avgdl}}
]

(b=0) なら正規化なし、(b=1) なら完全な文書長正規化となる。([staff.city.ac.uk][2])

### 3.6 BM25

以上を組み合わせると、論文中のBM25は概略次式となる。

[
\mathrm{score}_{BM25}(d,q)
==========================

\sum_{t\in q\cap d}
w_t^{RSJ}
\frac{
tf_{t,d}
}{
k_1\left((1-b)+b\frac{|d|}{\mathrm{avgdl}}\right)+tf_{t,d}
}
]

関連性判定がない通常の検索では、(w_t^{RSJ}) をIDFに置き換える。

* (k_1)：語頻度がどの程度速く飽和するかを制御する。
* (b)：文書長正規化の強さを制御する。
* IDF／RSJ重み：語の識別力を表す。

実装でよく見られる分子の ((k_1+1)) は、全語に共通する係数であり、本稿の順位付けでは省略しても文書順位は変わらない。([staff.city.ac.uk][2])

### 3.7 BM25F

BM25Fは、タイトル、本文、アンカーテキストなど、文書を複数のフィールドまたはstreamに分ける拡張である。

重要な設計思想は、各フィールドでBM25スコアを計算して後から足すのではなく、**同一語について各フィールドから得られた証拠を先に統合し、その後で飽和関数を適用する**ことにある。

[
\widetilde{tf}_t
================

\sum_s
v_s
\frac{tf_{s,t}}
{(1-b_s)+b_s|d_s|/\mathrm{avglen}_s}
]

[
w_t^{BM25F}
===========

w_t^{RSJ}
\frac{\widetilde{tf}_t}{k_1+\widetilde{tf}_t}
]

ここで (v_s) はフィールドの重要度、(b_s) はフィールド別の長さ正規化係数である。タイトルやアンカーテキストの一致に、本文一致より強い証拠を与えられる。([staff.city.ac.uk][2])

### 3.8 非テキスト特徴と最適化

PRFは語だけに限定された枠組みではない。文書の新しさ、ファイル形式、リンク次数、PageRankなども、関連／非関連での特徴値の尤度比としてスコアに加算できる。

[
\mathrm{score}(d,q)
===================

\mathrm{BM25F}(d,q)
+
\sum_i\lambda_iV_i(f_i)
]

著者らは、対数、飽和型有理関数、シグモイドなどを特徴変換として挙げている。特徴が増えると自由パラメータも増えるため、キャッシュ、グリッド探索、Robust Line Search、多次元探索、勾配ベース学習を用いて最適化する。([staff.city.ac.uk][2])

## 4. 実験方法

本論文には、通常の実験論文に見られる新規データセット、比較ベースライン、統一評価指標、学習・検証・テスト分割、統計的有意差検定は設定されていない。

代わりに、以下を証拠として用いている。

1. PRFから各ランキング式を導く理論的展開。
2. 過去の多数の情報検索実験から得られた経験則。
3. TREC Web Searchなど、既発表研究のパラメータ例。
4. BM25／BM25Fの既存実装に対するソースコード調査。
5. 過去に著者らが使用したパラメータ最適化手法の解説。

TREC 2003のtopic distillationとname page検索についてBM25Fのフィールド重みを例示しているが、これは本論文が新たに実施した統一比較実験ではない。([staff.city.ac.uk][2])

また、既存検索エンジンのBM25実装についてはコードを目視確認したものの、著者ら自身はテストを実行していないと明記している。したがって、これは実装検証実験ではなく、コードレビューに近い。([staff.city.ac.uk][2])

パラメータ最適化については、関連性判定付きクエリ集合を固定し、Average Precision、Precision@k、MRR、NDCGなどの評価値 (M(\theta)) を最大化する問題として定式化している。ただし、これらの順位評価関数は非連続で、局所最大や平坦領域を持ち、直接的な勾配が得られにくいことが指摘されている。([staff.city.ac.uk][2])

## 5. 結果

本稿の主要な「結果」は性能値ではなく、理論的・方法論的な整理である。

第一に、関連確率によるランキングを、転置インデックスで効率的に計算可能な**語ごとの加算スコア**へ変換できることを示した。

第二に、二値独立モデルからRSJ重みが得られ、関連性情報がない場合にはIDF型の重みに帰着することを示した。

第三に、潜在的なelitenessモデルから語頻度の飽和という性質を抽出し、文書長のソフト正規化と組み合わせることでBM25を説明した。

第四に、フィールド別の証拠を語単位で統合するBM25Fと、リンク情報など非テキスト特徴を加算する一般化を示した。([nowpublishers.com][1])

実務的な経験則としては、多くの状況で

[
0.5<b<0.8,\qquad 1.2<k_1<2
]

が妥当な範囲とされる。ただし最適値は文書種別、クエリ、コレクションによって変化し、BM25Fではフィールド重みに応じて (k_1) のスケールも変化する。([staff.city.ac.uk][2])

さらに、BM25とBM25Fは小さなパラメータ変化に対して比較的頑健だが、新しいコレクションでは適切な最適化によって改善が得られる。一方、最適化には人手による関連性判定と大きな計算費用が必要で、得られた値が別コレクションへ一般化しない可能性がある。([staff.city.ac.uk][2])

## 6. 考察

### 評価できる点

最大の貢献は、BM25を「IDFと語頻度を適当に組み合わせた式」ではなく、**関連性確率、語の識別力、語頻度飽和、文書長正規化を接続したモデル**として説明したことである。

特に優れているのは、理論的仮定と実装上の工夫を明確に対応付けている点である。条件付き独立性から語ごとの加算が生まれ、「出現しない語の項」を除去する操作から転置インデックス上の効率的な計算が可能になる。この理論と検索エンジン実装の接続は非常に明瞭である。

また、著者らはモデルの弱点を隠していない。二値関連性、語独立性、位置情報の欠如、パラメータの自由度、関連確率を直接校正できないことなどを本文中で明示している。

### 注意すべき点

BM25は完全に第一原理から一意に導出された式ではない。2-Poissonモデルから飽和という定性的性質は得られるが、実際の

[
tf/(k+tf)
]

は、その形状を再現するために選ばれた実用的近似である。文書長係数 (b) と飽和係数 (k_1) も、モデル自体から値が決まるわけではない。したがって、BM25は「純粋な確率モデル」というより、**確率論的な根拠と経験的パラメータ化を組み合わせたモデル**と理解するのが正確である。([staff.city.ac.uk][2])

また、順位を保存するために文書に依存しない項を捨てているため、BM25スコアをそのまま「関連確率80%」のような校正済み確率として解釈することはできない。閾値判定や確率校正が必要な用途では追加モデルが必要になる。([staff.city.ac.uk][2])

語独立性は、同義語、複合語、フレーズ、意味的に相関する語を多く含むクエリ拡張によってさらに破れやすくなる。位置情報や語順を取り込む方法も紹介されているが、著者らはPRFの自然な拡張とは言い難いと評価している。([staff.city.ac.uk][2])

BM25Fについても、各フィールドをまたいだIDFの適切な定義は未解決問題として残されている。極端に長いフィールドがほぼ全語を含む場合、通常の文書単位IDFが不自然になる可能性がある。([staff.city.ac.uk][2])

サーベイとしての弱点は、文献を系統的に収集・選別した手順がなく、「多くの実験で有効だった」という主張の一部に個別の引用が付されていないことである。著者ら自身も、長期間に多数の研究者が行った実験を背景とし、すべてを完全には参照しないと説明している。したがって、定量的なメタ分析ではなく、専門家による理論的・歴史的統合として読むべきである。([staff.city.ac.uk][2])

## 7. まとめ

本論文は、BM25を次の三要素に分解して理解するための基礎文献である。

[
\boxed{
\text{語の希少性}
\times
\text{語頻度の飽和}
\times
\text{ソフトな文書長正規化}
}
]

これを確率的関連性枠組みの中に位置づけ、関連フィードバック、クエリ拡張、BM25F、非テキスト特徴、パラメータ最適化へ拡張している。

一方で、本稿は新しい検索ベンチマークを提示する実験論文ではない。BM25の成立過程、数式の意味、モデル上の仮定、実装上の設計判断を理解するためのチュートリアル兼理論的総説である。
