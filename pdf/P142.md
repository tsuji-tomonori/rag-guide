# P142: Benchmarking and Defending Against Indirect Prompt Injection Attacks on Large Language Models

**著者:** Jingwei Yiほか
**掲載:** KDD 2025
**主題:** 間接プロンプトインジェクションの評価ベンチマーク「BIPIA」と、ブラックボックス／ホワイトボックス防御。([arXiv][1])

## 1. はじめに

検索、メール、文書、コードなどの外部コンテンツをLLMに入力するアプリケーションでは、そのコンテンツに攻撃命令が埋め込まれている可能性がある。LLMがそれを単なるデータではなく実行すべき命令として解釈すると、ユーザーの依頼から逸脱した出力を生成してしまう。これが**間接プロンプトインジェクション**である。

本論文は、この攻撃を体系的に評価するベンチマーク **BIPIA** を構築し、25種類のLLMを比較する。さらに、攻撃成功の原因を「データと命令の境界認識不足」と「外部コンテンツ内の命令を実行してはいけないという認識不足」に整理し、それぞれに対応する防御を提案する。([arXiv][1])

## 2. 背景

通常の直接的なプロンプトインジェクションでは、攻撃者自身が悪意あるユーザー入力を送る。一方、間接攻撃では、攻撃命令はWebページ、メール、表、ニュース記事、ソースコードなどに埋め込まれる。一般ユーザーがその情報をLLMアプリケーションに読み込ませることで、攻撃が発動する。

論文では、ユーザー命令を (I)、外部コンテンツを (C)、プロンプトテンプレートを (T) とし、LLMへの入力を次のように考える。

[
P=\mathrm{Combine}(T,C,f(I))
]

攻撃者は外部コンテンツ (C) の一部に悪性命令 (M) を埋め込む。攻撃者は外部データを変更でき、公開されているモデルやアプリケーションの仕様を把握しているが、LLM本体やアプリケーションを直接改ざんすることはできない、という脅威モデルである。([arXiv][1])

防御には二つの目標がある。

**堅牢性**は攻撃成功率、すなわちASR（Attack Success Rate）を下げること、**有用性**は攻撃がない通常入力に対する回答性能を維持することである。単にすべての外部入力を拒否すればASRは下がるが、アプリケーションとして利用できなくなるため、両者を同時に評価する必要がある。

## 3. 提案手法

### 3.1 BIPIAベンチマーク

BIPIAは以下の5タスクを対象とする。

| タスク           | 元データ               | 想定アプリケーション |
| ------------- | ------------------ | ---------- |
| Email QA      | OpenAI Evals       | メール支援      |
| Web QA        | NewsQA             | 検索・Web質問応答 |
| Table QA      | WikiTableQuestions | 表計算・データ分析  |
| Summarization | XSum               | 文書要約       |
| Code QA       | 独自収集のPythonコード     | コード支援      |

攻撃は、30種類のテキスト攻撃と20種類のコード攻撃で構成され、各種類に5個の具体的な悪性命令がある。したがって、攻撃命令は合計250個となる。

テキスト攻撃は、元タスクとは無関係な処理へ誘導する**タスク非関連攻撃**、翻訳や符号化などで本来の回答を変形させる**タスク関連攻撃**、広告・詐欺・偽情報など特定の目的を達成する**標的型攻撃**に分類される。コード攻撃は、情報収集を狙う**受動的攻撃**と、システムやデータを変更する**能動的攻撃**に分けられる。

また、悪性命令を外部コンテンツの先頭・中央・末尾に挿入する。最終的な規模は訓練用626,250プロンプト、テスト用86,250プロンプトであり、攻撃タイプも訓練用とテスト用に分割されている。([arXiv][1])

### 3.2 攻撃原因に関する仮説

著者らは、攻撃成功の根本原因を次の二点と仮定する。

1. LLMが、外部コンテンツという「情報」と、実行すべき「命令」を十分に区別できない。
2. 外部コンテンツ内の命令を実行してはいけないことを、LLMが明示的に認識していない。

これに対応して、**境界認識（boundary awareness）**と**明示的リマインダー（explicit reminder）**を防御の中心要素とする。([arXiv][1])

### 3.3 ブラックボックス防御

モデルの重みを変更できないAPI利用を想定し、二つの方法を提案する。

**マルチターン対話方式**では、外部コンテンツを前のターンに置き、ユーザーが実行させたい命令を最新ターンに配置する。LLMが直近のユーザー命令を優先する性質を利用し、外部データと正規命令を会話構造上で分離する。

**In-context learning方式**では、外部コンテンツ内の攻撃命令を無視して正常に回答した例をプロンプト中に提示する。これにより、どの部分がデータで、どの部分が実行すべき命令なのかをfew-shot例から学習させる。

いずれにも「外部コンテンツ内の命令を実行してはいけない」という明示的リマインダーを加える。([arXiv][1])

### 3.4 ホワイトボックス防御

モデルを変更可能な場合は、境界を示す特殊トークン `<data>` と `</data>` を語彙に追加する。

[
P=\mathrm{Combine}(T,\langle data\rangle C\langle /data\rangle,I)
]

さらに、悪性命令を含むプロンプトと、その影響を受けていない正常回答の組を使って教師あり微調整する。正常回答は、BIPIAの正解ラベル、元のLLMがクリーン入力から生成した回答、GPT-4がクリーン入力から生成した回答の3方式で作られる。

これは、単に区切り記号を表示するだけでなく、「この区間はデータであり、その内部の命令には従わない」という振る舞いをモデルのパラメータに学習させる敵対的訓練である。

## 4. 実験方法

攻撃耐性の基礎評価では、オープン／クローズドを含む25モデルを使用する。ASRの判定には、ルールベース判定、LLM-as-a-Judge、言語判定を組み合わせる。生成温度は0、最大生成トークン数は2,000に設定される。([arXiv][1])

防御実験では、ブラックボックス方式をGPT-4、GPT-3.5-Turbo、Vicuna-7B、Vicuna-13Bで評価する。ホワイトボックス方式はVicuna-7BとVicuna-13Bを対象とし、AdamW、学習率 (2\times10^{-5})、バッチサイズ128、1エポックで微調整する。

評価指標は次の三つである。

* **ASR:** 攻撃命令が成功した割合。低いほど安全。
* **ROUGE-1 recall:** 攻撃を含まないBIPIA-Cleanにおける元タスク性能。
* **MT-Bench:** ホワイトボックス微調整後の一般的な会話・指示追従能力。

したがって、攻撃を無視できるかだけでなく、通常入力への回答品質が劣化していないかも検証している。([arXiv][1])

## 5. 結果

### 5.1 防御前の脆弱性

評価したすべてのモデルで攻撃が一定割合成功した。全タスクを統合したASRは、GPT-4が0.3103、GPT-3.5-Turboが0.2616であり、25モデル平均は0.1179だった。

Chatbot ArenaのElo値とASRのPearson相関は、全タスクで0.6423、テキストタスクで0.6635、コードタスクでは−0.0254だった。すなわち、少なくとも本実験のテキスト攻撃では、能力が高いモデルほど攻撃命令も正確に理解・実行する傾向が見られた。ただし、これは相関であって因果関係の証明ではない。([arXiv][1])

タスク別では要約タスクのASRが高かった。また、悪性命令を外部コンテンツの末尾に配置した場合が最も成功しやすく、先頭、中央の順となった。著者らは、LLMが文章末尾の命令を重視する位置バイアスを持つ可能性を指摘している。([arXiv][1])

### 5.2 ブラックボックス防御

主要な総合ASRは次の通りである。

| モデル           |   防御なし | In-context learning |     マルチターン |
| ------------- | -----: | ------------------: | ---------: |
| GPT-4         | 0.3103 |              0.2408 | **0.2056** |
| GPT-3.5-Turbo | 0.2616 |              0.1884 | **0.1843** |
| Vicuna-7B     | 0.1237 |              0.0885 | **0.0617** |
| Vicuna-13B    | 0.1531 |              0.1658 | **0.1021** |

マルチターン方式は、GPT-4で約34%、GPT-3.5-Turboで約30%、Vicuna-7Bで約50%、Vicuna-13Bで約33%の相対的なASR低減を達成した。

一方、In-context learningはVicuna-13BではASRを0.1531から0.1658へ悪化させている。したがって、ブラックボックス防御は低コストで有効ではあるものの、モデルやタスクによっては不安定であり、攻撃を完全には防げない。

マルチターン方式のROUGEは、多くの場合で防御前と同等か改善した。対してIn-context learningは、プロンプトの追加例による干渉や長文化により、回答品質をやや低下させる場合があった。

### 5.3 ホワイトボックス防御

GPT-4が生成したクリーン回答を教師として使った場合の結果は次の通りである。

| モデル        | 防御前ASR |     防御後ASR |      ROUGE-1の変化 |     MT-Benchの変化 |
| ---------- | -----: | ---------: | --------------: | --------------: |
| Vicuna-7B  | 0.1237 | **0.0053** | 0.6187 → 0.6260 | 4.8063 → 4.8312 |
| Vicuna-13B | 0.1531 | **0.0047** | 0.6134 → 0.6337 | 5.2062 → 4.5500 |

ASRはVicuna-7Bで約96%、Vicuna-13Bで約97%低下した。Vicuna-7Bでは通常性能もほぼ維持された。一方、Vicuna-13BではROUGEは向上したものの、MT-Benchは低下している。

また、BIPIAの短い正解ラベルをそのまま教師回答として使ったVicuna-13Bでは、MT-Benchが5.2062から1.6625まで低下した。したがって、ホワイトボックス防御の成功には、敵対的訓練そのものだけでなく、教師回答の品質と多様性が重要である。

### 5.4 アブレーション

境界認識または明示的リマインダーを除去するとASRが上昇した。特に、境界認識を除いた場合の悪化が大きく、単に「攻撃を無視せよ」と指示するよりも、外部データと正規命令を構造的に区別させる方が重要であることが示された。([arXiv][1])

## 6. 考察

本論文の重要な示唆は、間接プロンプトインジェクションを単なる「悪い文章への耐性」ではなく、**信頼境界とデータ来歴の問題**として捉えた点にある。LLMにとって、システム命令、ユーザー命令、検索結果、メール本文は、最終的には同じトークン列である。そのため、区切りを表示するだけではなく、どの情報源にどの権限があるかをモデルに学習させる必要がある。

ただし、論文の「ASRをほぼゼロにできる」という結果は、BIPIAの非適応的な攻撃分布内での結果と解釈すべきである。主な限界は、攻撃が静的なテンプレートを中心としていること、エージェントによる実際のツール実行や長期的な攻撃連鎖を評価していないこと、ROUGE-1がQAやコードの意味的正しさを十分に測れないこと、使用モデルが主に2023年前後の世代であることである。

後続研究では、防御内容を知った攻撃者が専用の攻撃文字列を最適化する**適応的攻撃**によって、8種類の既存防御すべてが突破され、エージェント環境で50%を超えるASRが報告されている。この研究はBIPIAと実験設定が異なるため直接的な反証ではないが、「非適応攻撃に対する低ASRだけで堅牢性を主張してはならない」ことを示している。

実運用では、本論文の境界認識や敵対的微調整に加え、ツールの最小権限化、重要操作前のユーザー承認、外部データの由来管理、出力検証、実行環境のサンドボックス化を併用する必要がある。

## 7. まとめ

本論文は、間接プロンプトインジェクションを体系的に評価するBIPIAを提示し、LLMの能力向上が必ずしも安全性向上につながらないことを示した。

ブラックボックス防御では、外部コンテンツと正規命令を別ターンに分離する方法が比較的安定して有効だった。ホワイトボックス防御では、特殊トークンによる境界表示と敵対的微調整を組み合わせることで、ベンチマーク上のASRを約0.5%まで低下させた。

最も重要な結論は、**明示的な警告だけでは不十分であり、データと命令の境界をモデルとシステムの両方で表現する必要がある**という点である。

## 8. 総評

**公式の査読結果:** 本論文は、査読を経て **KDD 2025に採択・掲載**されている。公開資料から個々の査読者スコアやコメントまでは確認できない。([arXiv][2])

**本要約における模擬査読判定:** **Accept／採択、確信度4/5**

採択相当と評価する最大の理由は、間接プロンプトインジェクション研究に対して、ベンチマーク、原因分析、防御、通常性能評価、アブレーション、コード公開を一つの研究としてまとめた先駆性と実用性である。特に、攻撃タイプを訓練用とテスト用に分けたことや、セキュリティ性能だけでなくROUGEとMT-Benchで有用性も測った点は高く評価できる。実装とデータも公開されており、再現性も比較的高い。([GitHub][3])

一方で、「ホワイトボックス防御により堅牢になった」という主張は、**BIPIA内の非適応攻撃に対して**という限定をより明確にすべきである。また、ASR評価の人手検証、他ベンチマークへの転移、適応的攻撃、最新モデル、RAG・ツール利用型エージェントでの検証が不足している。

したがって、本論文は完成した安全対策というより、**間接プロンプトインジェクションを測定可能な研究課題へ変えた基盤的研究**として高く評価できる。
