# 1. はじめに

**RAG-Anything: All-in-One RAG Framework** は、テキストだけでなく、画像・表・数式を含む長大な文書を一つのRAG基盤で扱うためのフレームワークである。中心的な発想は、非テキスト情報を単なる説明文に変換して検索するのではなく、文書内の構造や他の要素との関係を知識グラフとして保持することにある。論文は香港大学の研究グループによるもので、2025年10月14日にarXivへ投稿された。([arXiv][1])

# 2. 背景

従来のRAGは、文書をテキストチャンクへ分割し、埋め込み類似度などで関連部分を検索する方式が中心である。しかし、実際の研究論文、財務報告書、技術文書には、グラフ、複数パネルからなる図、表、数式などが含まれる。これらを平坦なテキストへ変換すると、表の行・列対応、図のパネルとキャプションの関係、数式と変数定義の対応といった構造が失われる。

著者らは、マルチモーダルRAGには、異種データを統一的に表現すること、文書レイアウトや階層構造を保持して分解すること、テキスト質問から画像・表・数式へ横断的にアクセスすること、という三つの課題があると整理している。特に長文書では、根拠が複数ページ・複数モダリティに分散するため、通常のチャンク検索では対応が難しい。([arXiv][2])

# 3. 提案手法

RAG-Anythingは、次の処理を行う。

**マルチモーダル知識の統一化**では、入力文書をテキスト、画像、表、数式などの原子的なコンテンツ単位へ分解する。この際、図とキャプション、表と説明文、数式と周辺の定義などの位置的・階層的関係を保持する。

**デュアルグラフ構築**では、二種類の知識グラフを別々に作る。非テキスト要素についてはVLMを用いて詳細説明とエンティティ要約を生成し、画像・表・数式をアンカーノードとするクロスモーダル知識グラフを構築する。一方、本文テキストからはLightRAGやGraphRAGに近いエンティティ・関係抽出を行い、テキスト知識グラフを作る。その後、エンティティ名を主なキーとして両グラフを統合する。

**検索インデックス**は、統合知識グラフと、エンティティ・関係・各コンテンツ単位の埋め込みから構成される。

**クロスモーダル・ハイブリッド検索**では、質問中の「図」「表」「数式」などの語から期待されるモダリティを推定し、次の二経路で候補を取得する。

* グラフ上のエンティティ照合と近傍展開による構造的ナビゲーション
* 埋め込み類似度による意味検索

両者を構造的重要度、意味類似度、モダリティ適合度によって統合・再ランキングする。最後に、検索用のテキスト説明だけでなく元の画像も復元し、質問、構造化されたテキスト文脈、視覚情報をVLMへ同時に入力して回答を生成する。([ar5iv][3])

# 4. 実験方法

評価には二つの長文書マルチモーダルQAベンチマークが使われた。

**DocBench**は229文書・1,102問からなり、平均66ページ、約46,377トークンで、学術、金融、政府、法律、ニュースを含む。**MMLongBench**は135文書・1,082問、平均47.5ページ、約21,214トークンで、研究報告書、論文、ガイドブック、財務報告書など7種類の文書を含む。

比較対象は、GPT-4o-miniによる直接回答、テキスト中心のLightRAG、マルチモーダル知識グラフ型のMMGraphRAGである。すべての方式でGPT-4o-miniを基盤モデルとして使用し、文書解析にはMinerU、埋め込みにはtext-embedding-3-large、再ランキングにはbge-reranker-v2-m3を用いている。回答は一文に制限され、正解判定もGPT-4o-miniが担当した。直接入力方式では、文書を144 dpiの画像として最大50ページまで入力している。([arXiv][2])

# 5. 結果

DocBenchの総合正解率は、GPT-4o-miniが51.2%、LightRAGが58.4%、MMGraphRAGが61.0%、RAG-Anythingが**63.4%**だった。特にマルチモーダル問題では、MMGraphRAGの66.0%に対してRAG-Anythingは**76.3%**と、10.3ポイント上回った。一方、回答不能問題では46.0%であり、MMGraphRAGの60.5%を大きく下回っている。

MMLongBenchでは、GPT-4o-miniが33.5%、LightRAGが38.9%、MMGraphRAGが37.7%、RAG-Anythingが**42.8%**だった。研究報告書、学術論文、ガイドブック、財務報告書などで最良の結果を示したが、チュートリアルや行政文書では必ずしも最高ではない。

長さ別の分析では、DocBenchの101～200ページで68.2%対54.6%、200ページ超で68.8%対55.0%と、MMGraphRAGとの差が13ポイント以上に広がった。MMLongBenchでも、11～50ページで3.4ポイント、51～100ページで9.3ポイント、101～200ページで7.9ポイントの改善が報告されている。

アブレーションでは、通常のチャンク検索のみが60.0%、再ランキングを除いたモデルが62.4%、完全版が63.4%だった。したがって、グラフ構築による改善が約3.4ポイント、再ランキングによる追加改善が約1ポイントであり、主要な効果はグラフ表現から得られている。

# 6. 考察

本研究の強みは、画像・表・数式を同じ埋め込み空間へ押し込むだけでなく、表の行―列―セル―単位、図のパネル―軸―凡例―キャプションといった関係を明示的にモデル化した点にある。ケーススタディでも、複数パネル図の対象パネルの選択や、財務表の特定の行・年・数値の対応付けで効果が示されている。長文書ほど改善幅が大きい点も、構造検索と意味検索を併用する設計仮説と整合する。([arXiv][2])

一方、実験上の懸念は大きい。比較手法が三つに限られ、関連研究で言及しているVisRAGなどの強力な文書RAG方式が比較に含まれていないため、「最先端を上回る」という主張の一般性は十分ではない。また、直接入力のGPT-4o-miniだけが最大50ページに制限されており、平均66ページ、金融文書では平均192ページに達するDocBenchでは、長文書比較がRAG方式に有利になっている可能性がある。([arXiv][2])

正解判定に回答生成と同系列のGPT-4o-miniを使用している点にも、自己評価バイアスの可能性がある。人手評価、複数評価器、信頼区間、統計的有意差は報告されていない。さらに、検索Recall、根拠の正確性、引用位置の正しさなど、検索部分を直接測る指標もない。

実用面では、各画像・表・数式に対するVLM処理、エンティティ抽出、グラフ構築が必要になるため、索引作成のAPIコスト、処理時間、メモリ使用量、グラフ規模の増加が重要になるが、効率評価は示されていない。エンティティ名を主要キーとするグラフ統合も、同名異義語や表記揺れに弱い可能性がある。

また、RAG-Anythingは回答不能問題で弱く、関連しそうなグラフ経路を積極的に探索する結果、根拠が存在しない場合にも回答を生成してしまう可能性が示唆される。著者自身も、テキストを優先して誤った根拠を取得する問題や、結合セル・不規則レイアウトを持つ表で全方式が失敗する問題を報告している。([arXiv][2])

# 7. まとめ

本論文は、マルチモーダル文書を「テキスト・画像・表・数式という別々のデータ」ではなく、「相互に接続された知識エンティティ」として扱うRAGフレームワークを提案した。デュアルグラフとハイブリッド検索により、特に長文書と構造化された表・図を必要とする質問で改善を示している。

ただし、現時点の成果は、普遍的なマルチモーダルRAGの完成形というより、**文書構造を知識グラフとして保持することの有効性を示した有望なシステム研究**と捉えるのが適切である。

# 8. 総評

**公開されている査読結果**については、arXivではv1のプレプリントとして掲載され、OpenReview上でも「CoRR 2025」として登録されているだけで、会議・論文誌の採択判定や査読コメントは公開されていない。したがって、公開情報からは正式な査読済み論文とは確認できない。公式リポジトリでも本論文は「technical report」として案内されている。([arXiv][1])

**模擬査読判定：Weak Reject／大幅修正後に再評価**

着想、実装価値、長文書での改善は高く評価できる。一方で、トップ会議のフルペーパーとしては、比較手法の不足、50ページ制限による比較条件の非対称性、LLM自己評価、効率分析と統計検定の欠如が主要な弱点となる。強力な文書RAGとの公平な比較、人手を含む評価、検索性能の分解評価、索引作成コストの測定、回答不能判定の改善が追加されれば、採択に近づく研究である。
