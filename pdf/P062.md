# 1. はじめに

**CONQRR: Conversational Query Rewriting for Retrieval with Reinforcement Learning** は、対話履歴に依存する質問を、検索器が扱いやすい独立した検索クエリへ書き換える手法を提案した論文である。Zeqiu Wuらによって発表され、EMNLP 2022のメイン会議論文集に掲載された。([ACL Anthology][1])

例えば、対話中の「Who won?」という質問だけでは検索できないが、履歴を参照して「Who won MasterChef season 10?」と書き換えれば、通常の検索エンジンでも適切な文書を取得できる。

本論文の中心的な主張は、**人間が自然だと考える書き換えと、検索性能を最大化する書き換えは必ずしも一致しない**という点にある。そこで、正解の書き換え文を模倣する教師あり学習だけでなく、検索結果を報酬として用いる強化学習によって書き換えモデルを訓練する。([ar5iv][2])

---

# 2. 背景

対話型質問応答では、現在の質問が過去の発話に依存することが多い。代名詞、省略、話題の継続などが含まれるため、現在の質問だけを一般的な検索器に入力しても、検索意図を正しく解釈できない。

従来研究には、大きく二つの方向性がある。

一つは、対話履歴を直接扱えるように検索器自体を再学習する方法である。ただし、特定のDual Encoderなどにアーキテクチャが限定され、既存の検索エンジンや外部APIのようなブラックボックス検索器には適用しにくい。

もう一つは、対話質問を単独で理解可能な質問へ変換するクエリ書き換えである。しかし、従来のモデルは人手で作られた書き換えを正解として学習していた。人間のアノテータは「文脈を知らない人にも意味が通じる文」を作るが、それが検索器にとって最良の語句を含むとは限らない。

CONQRRはこの目的関数の不一致を問題視し、書き換え文の自然さではなく、**実際に関連パッセージを上位取得できたか**を直接最適化する。

# 3. 提案手法

## 3.1 基本モデル

書き換えモデルにはT5-baseを用いる。入力は現在の質問と過去の対話履歴を連結した文字列で、直近の発話を先頭に置く。通常の教師ありモデルであるT5QRは、人手書き換え文を出力するよう交差エントロピー損失で学習する。

CONQRRは、T5または学習済みT5QRを初期値として、さらに検索報酬による強化学習を行う。([ar5iv][2])

## 3.2 弱教師となる正例パッセージ

QReCCでは各質問の直後に回答文がある。この回答文字列とのトークン重複F1が最も高いパッセージを、弱い正例パッセージとして採用する。

標準設定では、まず人手書き換えをBM25に入力して上位100件を取得し、その中から回答文との重複が最大のパッセージを選ぶ。したがって、提供された正解パッセージラベル自体は使わないが、標準設定の正例作成には人手書き換えが間接的に使われている。

0%書き換え教師の実験では、人手書き換えの代わりに対話履歴全体をBM25に入力して正例候補を作る。

## 3.3 強化学習の報酬

各訓練例について、モデルは次の二種類の書き換えを生成する。

* サンプリングによる書き換え (q_s)
* 推論時にも用いる貪欲デコードの書き換え (q_g)

各バッチには、訓練例ごとの弱正例と困難負例をまとめた候補パッセージ集合を作る。生成したクエリを固定検索器に入力し、対応する弱正例がバッチ内候補の1位になった場合を1、それ以外を0とする。

[
\mathrm{score}(q)
=================

\mathbf{1}
[\text{正例パッセージが1位}]
]

サンプリング出力の相対報酬は、

[
r(q_s,q_g)
==========

\mathrm{score}(q_s)-\mathrm{score}(q_g)
]

である。貪欲出力より検索結果が良ければ生成確率を上げ、悪ければ下げる。これはSelf-Critical Sequence Trainingに相当する。全コーパスで毎回MRRを計算する代わりに、バッチ内の二値的な1位判定を使うことで計算量を抑えている。

学習には、強化学習損失だけを使うCONQRR（RL）と、教師あり損失を組み合わせるCONQRR（mix）がある。

[
\mathcal{L}_{mix}
=================

\alpha \mathcal{L}*{RL}
+
(1-\alpha)\mathcal{L}*{CE}
]

混合設定では (\alpha=0.99) であり、検索報酬を大きく重視している。推論時は、貪欲デコードで一つの書き換えを生成し、それを固定検索器に入力する。

# 4. 実験方法

評価にはQReCCを使用する。QReCCは約1.4万の英語対話、約8万の質問回答ペア、検索対象となる約5,400万パッセージからなる。データはQuAC由来のQuAC-Conv、Natural Questions由来のNQ-Conv、TREC CAsT由来のTREC-Convに分かれる。TREC-Convは訓練データに存在せず、ドメイン外評価として使われる。([ar5iv][2])

検索器には次の二種類を用いる。

* **BM25**：Pyserini、(k_1=0.82)、(b=0.68)
* **Dual Encoder（DE）**：MS MARCOで学習されたT5-base Dual Encoderを固定して使用

比較対象は、GPT2＋弱教師、Transformer++、教師ありT5QR、人手書き換えである。評価指標はMRR、Recall@10、Recall@100である。正解パッセージラベルがないテスト例を除外した更新版評価では、合計8,209例が対象となる。

CONQRRはT5-base、約2.2億パラメータで、バッチサイズ64、10,000ステップ、各例につき5個の書き換えをサンプリングする。訓練には8 TPUノードを使用し、教師あり学習は約2時間、強化学習は約9時間かかる。BM25を報酬計算に使用する場合、訓練中は計算を簡略化したBM25-lightを代用し、最終評価では通常のPyserini BM25を用いる。

# 5. 結果

## 5.1 全体的な検索性能

更新版評価による主要結果は以下の通りである。

| 検索器  | 書き換えモデル    |       MRR | Recall@10 | Recall@100 |
| ---- | ---------- | --------: | --------: | ---------: |
| BM25 | T5QR       |     0.328 |      52.5 |       84.7 |
| BM25 | CONQRR mix |     0.373 |      58.5 |   **90.2** |
| BM25 | CONQRR RL  | **0.383** |  **60.1** |       88.9 |
| BM25 | 人手書き換え     |     0.398 |      62.6 |       98.5 |
| DE   | T5QR       |     0.361 |      56.2 |       75.9 |
| DE   | CONQRR mix |     0.395 |      61.9 |       81.8 |
| DE   | CONQRR RL  | **0.418** |  **65.1** |   **84.7** |
| DE   | 人手書き換え     |     0.422 |      64.8 |       84.0 |

CONQRRは両検索器の全指標でT5QRを上回った。DEではCONQRR（RL）がRecall@10とRecall@100で人手書き換えも上回り、MRRも0.418対0.422とほぼ同等である。BM25では人手書き換えが依然として最良だが、CONQRRは教師ありT5QRとの差を大きく縮めている。論文発表時点ではQReCCの固定BM25設定における最高性能とされている。

## 5.2 ドメイン外性能

CONQRR（mix）は、QuAC-Conv、NQ-Conv、TREC-ConvのすべてでT5QRを概ね上回った。一方、純粋なCONQRR（RL）は、訓練時に現れないTREC-Convで性能が低下した。

したがって、強化学習だけでは検索器や訓練分布に過適合しやすく、少量でも教師あり損失を残した混合学習の方がドメイン外で安定している。

## 5.3 書き換え教師の削減

人手書き換えラベルを0%、1%、10%、100%と変化させた実験では、CONQRRは0%または1%の書き換え教師でもT5QRを大きく上回った。

ただし、これは完全な教師なし学習ではない。回答文字列と検索器から作った弱教師信号を利用しており、正確には**人手書き換えラベルを必要としない弱教師付き学習**である。

## 5.4 話題継続と話題転換

論文では、現在の正解パッセージが過去ターンと同じ文書に由来する場合を「話題集中」、異なる文書に由来する場合を「話題転換」として分析している。

話題集中では、対話履歴全体をそのまま検索器に入力する方法が非常に強く、BM25ではMRR 0.620、DEでは0.551となり、人手書き換えやCONQRRを上回った。

一方、話題転換では、長い対話履歴に過去の話題が混入するため、履歴全体の検索性能が大きく低下する。CONQRRは不要な履歴を圧縮することで履歴入力を上回るが、人手書き換えにはまだ及ばない。これは、クエリ書き換えの主要な価値が、単なる省略補完よりも**話題転換時のノイズ除去**にあることを示している。

## 5.5 定性分析

CONQRRが人手書き換えより良い50例を調べたところ、70%では、人手書き換えが省略した有用な固有名詞や文脈情報をCONQRRが追加していた。

逆にCONQRRが人手書き換えより悪い50例では、56%が文脈不足、34%が質問意図の誤解であった。例えば「ケトダイエットとパレオダイエットの共通点」を聞いているのに、CONQRRが「パレオダイエットと石器時代ダイエットの共通点」と誤って解釈する例が示されている。

# 6. 考察

本研究の最大の強みは、**生成文の表面的な正しさではなく、下流の検索性能に目的関数を合わせたこと**である。人手書き換えを唯一の正解とせず、検索器ごとに有効な語句を学習できる点は、現在の検索拡張生成やRAGに通じる考え方である。

また、固定された検索器を変更せず、入力クエリ側を適応させるという設計には実用上の利点がある。検索器の再学習では文書インデックスの再構築が必要になる場合があるが、クエリ書き換えモデルなら既存インデックスを保持できる。

一方、以下の限界がある。

第一に、報酬は全5,400万パッセージ上のMRRではなく、バッチ内候補に対する二値的な1位判定である。この近似報酬が実際の全コーパス検索と一致しない可能性がある。

第二に、標準実験の弱正例作成では、人手書き換えを使ってBM25上位100件を取得している。「正解書き換えを使わない」という主張は、0%教師実験には当てはまるが、標準設定には完全には当てはまらない。

第三に、「任意のブラックボックス検索器に適用可能」という主張は設計上のもので、実験はBM25と一種類のDual Encoderに限られる。さらにBM25実験では、訓練中の報酬計算に実際のPyseriniではなくBM25-lightを使っている。検索APIしか提供されず、任意のバッチ内候補を順位付けできない環境では、訓練コストや実装可能性が問題になる。

第四に、検索器を再学習できる条件では、対話履歴全体で学習した検索器がCONQRRを上回る。特に話題転換条件でも、微調整済みDEのMRRは履歴入力0.312、CONQRR 0.259である。したがってCONQRRの優位性は、主として**検索器を固定しなければならない条件**に限定される。

第五に、最終的な回答生成精度は測定しておらず、改善がCQAシステム全体の回答品質につながるかは未検証である。英語のQReCCと、質問・回答が交互に続く標準的な対話形式だけを対象としている点も一般化上の制約である。

さらに、著者らは2～3回の実行で類似傾向を確認したとしているが、表にはランダムに選んだ1回の結果だけを掲載しており、標準偏差や統計的有意差は報告していない。([ar5iv][2])

---

# 7. まとめ

CONQRRは、対話中の質問を固定検索器向けに書き換えるT5ベースのモデルであり、検索結果を報酬とする強化学習によって訓練される。

主な成果は次の通りである。

* 教師ありT5QRをBM25・Dual Encoderの両方で上回った。
* 人手書き換えラベルがなくても、回答と検索結果から得られる弱教師信号で高い性能を示した。
* 人間に自然な書き換えが、必ずしも検索に最適ではないことを実証した。
* 話題転換ではクエリ書き換えが重要だが、話題が継続する場合は対話履歴全体を直接使う方が強い場合がある。
* 純粋な強化学習はドメイン外で不安定で、教師あり損失との混合がより堅牢である。

# 8. 総評

**公開上の査読結果は「採択」**である。本論文は **EMNLP 2022のメイン会議論文**として、10000～10014ページに掲載されている。少なくともarXivとACL Anthologyの公開ページには、個別査読者の点数や査読コメントは掲載されていない。([ACL Anthology][1])

査読的に評価すると、**採択は妥当**である。特に、クエリ書き換えを検索目的に直接整合させる問題設定、固定検索器を対象とした明確な実用性、教師量・話題転換・文脈長・生成例まで含む分析の充実度は高く評価できる。

一方で、「任意のブラックボックス検索器への適用可能性」は実証範囲より広い主張であり、近似報酬、弱教師のノイズ、統計的検定の欠如、微調整可能な検索器では優位性が失われる点には注意が必要である。

**総合判定：Accept。**
固定検索器を利用する対話検索という条件下では、新規性と実験的根拠の双方が十分にあり、その後のRAGにおける検索器適応やretriever-aware query rewritingにもつながる意義のある研究である。
