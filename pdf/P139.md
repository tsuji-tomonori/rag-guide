# P139：Know What You Don’t Know: Unanswerable Questions for SQuAD

**著者**：Pranav Rajpurkar, Robin Jia, Percy Liang
**発表**：ACL 2018 Short Papers, pp. 784–789

## 1. はじめに

本論文は、機械読解モデルに「本文中に答えがある場合は回答し、答えがない場合は回答を控える」能力を要求するデータセット **SQuAD 2.0** を提案した研究である。

従来のSQuADでは、すべての質問に対して本文中に正解スパンが存在した。そのためモデルは、質問との関連性が最も高そうな文字列を選ぶだけでも高得点を得られ、本文が本当にその回答を支持しているかを十分に確認する必要がなかった。

SQuAD 2.0では、既存の回答可能質問に、クラウドワーカーが敵対的に作成した **53,775件の回答不能質問**を追加する。論文中ではこのデータセットをSQuADRUNとも呼んでいる。重要なのは、単に無関係な質問を追加するのではなく、本文と強く関連し、誤答として選びたくなる「もっともらしい回答候補」を含む質問を作成した点である。([ACLアントロジー][1])

## 2. 背景

SQuAD 1.1を含む従来の抽出型質問応答では、モデルは本文から開始位置と終了位置を選び、回答スパンを抽出する。しかし正解の存在が保証されている場合、モデルは次のような浅いヒューリスティクスに依存できる。

* 質問と本文の単語重複
* 質問が要求する回答型と、本文中の固有表現型との対応
* 質問中のキーワードに最も近いスパンの選択

既存研究にも回答不能な負例は存在したが、TF-IDFで別段落を割り当てる方法などでは、質問と本文の関連性が低くなりやすかった。また、規則による自動生成では、固有表現、数値、反義語の置換などにパターンが限定され、モデルが生成規則を学習しやすかった。

そこで著者らは、良質な回答不能質問が満たすべき条件として、次の二点を設定した。

1. **関連性**：質問が本文の話題と関連していること
2. **もっともらしい回答の存在**：質問が要求する型と一致する誤答候補が本文中に存在すること

これにより、モデルは表面的な関連性だけでなく、本文がその回答を本当に含意しているかを判断する必要がある。([ACLアントロジー][1])

## 3. 提案手法

### 3.1 回答不能質問の作成

著者らはDaemo上のクラウドワーカーに、SQuADのWikipedia記事を提示した。各段落について、ワーカーは最大5件の回答不能質問を作成した。

質問作成時には、次の条件が課された。

* その段落だけでは回答できないこと
* 段落中の事物や固有表現を参照すること
* 本文中に、正解と誤認し得る「もっともらしい回答スパン」が存在すること
* そのもっともらしい回答スパンも併せて選択すること
* 既存のSQuAD質問を参考にし、回答可能質問に似た形式にすること

ワーカーには段落当たり約7分が割り当てられ、報酬は時給10.50ドルに設定された。1記事について25問以下しか作成しなかったワーカーの質問は、タスクを十分理解していない可能性があるとして除外された。([ACLアントロジー][1])

### 3.2 データセット構成

| 分割     |        総質問数 |    回答不能質問数 |
| ------ | ----------: | ---------: |
| 訓練     |     130,319 |     43,498 |
| 開発     |      11,873 |      5,945 |
| テスト    |       8,862 |      4,332 |
| **合計** | **151,054** | **53,775** |

開発・テスト集合では、回答可能質問と回答不能質問がおおむね1対1になるよう構成されている。一方、訓練集合では回答可能質問が回答不能質問の約2倍存在する。記事単位の分割は元のSQuADを踏襲し、開発・テスト集合からは回答不能質問を収集できなかった記事を除外している。([ACLアントロジー][1])

### 3.3 モデルの回答方式

評価対象モデルは、通常の回答スパン分布に加えて、質問が回答不能である確率を予測する。

回答不能確率が閾値を超えた場合、モデルは回答を出力せず棄却する。閾値は各モデルについて開発集合上のF1が最大になるよう調整され、テスト時にはその閾値が固定される。

したがってSQuAD 2.0は、単なるスパン抽出ではなく、次の二つを同時に評価する課題となる。

* 本文中に回答が存在するかの判定
* 存在する場合の正しいスパン抽出

([ACLアントロジー][1])

## 4. 実験方法

### 評価モデル

以下の三つの既存モデルを使用した。

| モデル        | 概要                        |
| ---------- | ------------------------- |
| BNA        | BiDAFに回答不能判定を追加したモデル      |
| DocQA      | DocumentQAの回答不能対応版        |
| DocQA＋ELMo | DocQAに文脈化単語表現ELMoを追加したモデル |

本論文の中心的貢献は新しいモデルではなく、データセットと評価設定の提案である。

### 評価指標

評価にはSQuADと同様に以下を用いる。

* **Exact Match（EM）**：予測回答と正解が完全一致した割合
* **F1**：予測回答と正解回答のトークン重複に基づくF1

回答不能質問では、モデルが回答を棄却した場合にEMとF1を1、何らかの回答を出した場合を0とする。

### 人間性能の測定

開発・テスト集合の全質問について、追加のクラウドワーカーが回答を付与した。回答可能質問と回答不能質問は混ぜて提示され、ワーカーは回答スパンを選択するか、回答不能と判定した。

質問当たり平均4.8件の回答を収集し、多数決で最終回答を決定した。同数の場合には回答不能より回答可能を優先し、長いスパンより短いスパンを優先した。([ACLアントロジー][1])

### 比較実験

著者らは、SQuAD 2.0以外に二種類の自動生成負例を用意した。

* **TFIDF**：質問を同じ記事中の別段落に割り当てる
* **RULEBASED**：固有表現、数値、反義語などを規則的に置換する

また、モデルが回答不能質問に誤って回答した場合、その回答がクラウドワーカーの指定した「もっともらしい回答」と一致するかも調査した。([ACLアントロジー][1])

## 5. 結果

### 5.1 SQuAD 2.0の主要結果

| システム       |    テストEM |    テストF1 |
| ---------- | -------: | -------: |
| BNA        |     59.2 |     62.1 |
| DocQA      |     59.3 |     62.3 |
| DocQA＋ELMo | **63.4** | **66.3** |
| 人間         |     86.9 |     89.5 |

最良モデルであるDocQA＋ELMoのF1は66.3であり、人間の89.5を **23.2ポイント**下回った。

同じモデル構成をSQuAD 1.1で評価した場合、DocQA＋ELMoはF1 85.8、人間は91.2であり、差は5.4ポイントである。したがって、人間とモデルの差はSQuAD 2.0で大幅に拡大した。

なお、常に「回答不能」と予測するベースラインでもテストF1 48.9を得る。これはテスト集合の約半数が回答不能質問であるためであり、66.3というモデル性能は人間性能よりもこの単純ベースラインに近い。([ACLアントロジー][1])

### 5.2 自動生成負例との比較

DocQA＋ELMoの開発集合F1は次の通りである。

| 負例の種類           |       F1 |
| --------------- | -------: |
| SQuAD＋TFIDF     |     83.0 |
| SQuAD＋RULEBASED |     89.6 |
| SQuAD 2.0       | **67.6** |

SQuAD 2.0のF1は、TF-IDF負例の場合より15.4ポイント低い。規則生成負例との差は22.0ポイントである。

この結果は、単純な自動生成負例ではモデルが生成パターンや質問・本文間の不一致を容易に検出できるのに対し、人間が敵対的に作成した質問では、そのような表面的手掛かりが弱いことを示している。([ACLアントロジー][1])

### 5.3 回答不能質問の内訳

開発集合からランダムに選んだ100件を人手分析した結果は次の通りである。

| 種類             |  割合 |
| -------------- | --: |
| 否定の追加・削除       |  9% |
| 反義語            | 20% |
| 固有表現・数値・日付の交換  | 21% |
| 相互排他的な条件       | 15% |
| 成立しない条件        |  4% |
| その他の非含意        | 24% |
| 実際には回答可能だったノイズ |  7% |

93%は実際に回答不能であり、単純な固有表現置換だけでなく、否定、条件、相互排他性、本文から導けない前提など、多様な現象を含んでいた。([ACLアントロジー][1])

### 5.4 もっともらしい回答の効果

回答不能質問に対して誤って回答したケースだけを見ると、その予測は高い頻度でクラウドワーカーが指定したもっともらしい回答と一致した。

例えばDocQA＋ELMoでは、誤った回答ともっともらしい回答との一致度がEM 54.9、F1 69.2だった。人間についてもEM 46.4、F1 60.6であり、人間も同じ誤答候補に誘導されている。

したがって、もっともらしい回答は、モデルだけを狙った不自然な妨害ではなく、人間にも一定の混乱を与える有効なディストラクタとして機能した。

## 6. 考察

本研究の重要性は、質問応答を「最も関連しそうな文字列を探す課題」から、「本文による根拠の有無を確認する課題」へ変更した点にある。

SQuAD 1.1では回答の存在が保証されているため、モデルの高いEM・F1が、必ずしも本文理解や含意判定能力を意味しなかった。SQuAD 2.0は、回答候補が本文中に存在していても、その候補が質問への答えとして支持されていなければ棄却することを要求する。この構造は自然言語推論における含意・矛盾・中立の判定に近い。

一方、次の限界もある。

第一に、回答不能質問は「自然に発生したユーザー質問」ではなく、本文を見た作業者が意図的に作成したものである。そのため、語彙選択や質問構造にクラウドソーシング由来のアノテーション・アーティファクトが含まれる可能性がある。

第二に、人手分析した回答不能質問は100件に限られ、そのうち7%は実際には回答可能だった。大規模データセットとしては比較的良好な品質であるものの、回答不能性には意味的な曖昧さが残る。

第三に、開発・テスト集合では回答可能と回答不能がほぼ均等である。実運用では回答不能質問の出現率が異なるため、開発集合で最適化した棄却閾値が、そのまま実環境で適切とは限らない。

第四に、評価がEMとF1を中心としており、回答不能確率の較正、誤回答のリスク、棄却率と精度のトレードオフは十分に扱われていない。実利用を考えると、誤回答を避ける選択的予測としての評価も必要である。

第五に、対象は英語Wikipedia上の抽出型質問応答であるため、他言語、専門分野、自由記述回答、検索を伴うオープンドメインQAへの一般化は別途検証が必要である。

## 7. まとめ

本論文は、SQuADに53,775件の敵対的な回答不能質問を追加し、SQuAD 2.0を構築した。

質問は本文と関連し、本文中にもっともらしい誤答候補を含むよう、人間によって作成された。その結果、既存モデルはSQuAD 1.1ではF1 85.8を達成したにもかかわらず、SQuAD 2.0では66.3まで低下した。人間とのF1差も5.4ポイントから23.2ポイントへ拡大した。

したがってSQuAD 2.0は、モデルが「答えられる質問」と「本文だけでは答えられない質問」を区別できていないことを明確に示し、その後の回答不能判定付き質問応答の標準的な評価設定を確立した研究である。([ACLアントロジー][1])

## 8. 総評

### 査読結果

**採択**。ACL 2018の査読を通過し、**Volume 2: Short Papers**に掲載された。ACL Anthologyの公開情報では、個別の査読スコアや査読コメントは掲載されていない。([ACLアントロジー][2])

### 総合評価

本論文の価値は、新しいニューラルネットワークを提案したことではなく、既存モデルの性能を過大評価していたベンチマーク上の前提を修正した点にある。

特に優れている点は、単純な無関係質問を負例とせず、「質問との関連性」と「もっともらしい誤答候補」を明示的に設計したことである。自動生成負例との比較、人間性能の測定、誤答候補への誘導分析も行われており、短い論文ながら提案の有効性を複数の角度から検証している。

一方、自然なユーザー質問との分布差、回答不能ラベルの曖昧さ、閾値較正の問題は残る。それでも、「常に何かを答えるモデル」から「根拠がないときは答えないモデル」へ研究目標を転換した影響は大きい。

**査読者視点での判定はStrong Accept相当**である。モデル手法としての新規性は限定的だが、課題設定、データ構築、実証的インパクトが非常に強く、質問応答研究の評価基準を更新した代表的なデータセット論文と評価できる。
