# P152：It's About Time: Incorporating Temporality in Retrieval Augmented Language Models

## 1. はじめに

本論文は、検索拡張言語モデルが文書の**意味的関連性**だけでなく、質問時点に対する**時間的関連性**も考慮して文書を取得する手法「TempRALM」を提案する。

通常のRAG／RALMでは、「ウィンブルドン男子優勝者は誰か」という質問に対して、異なる年の大会結果が意味的にはほぼ同じ文書として検索される。そのため、質問が行われた時点で最新の結果ではなく、古い結果を根拠として回答する可能性がある。TempRALMは、質問と文書のタイムスタンプを検索スコアに組み込み、この問題を軽量な再ランキングによって改善する。公開されているarXiv版では、ベースラインのATLASに対して最大74%の相対性能向上が報告されている。([arXiv][1])

以下では公開本文であるarXiv v2を中心に説明する。後に著者を追加した最終版が、2025 IEEE Conference on Artificial Intelligence（IEEE CAI 2025）の会議録に収録されている。([SJSU ScholarWorks][2])

## 2. 背景

大規模言語モデルが保持する知識は、基本的に学習データを取得した時点の静的なスナップショットである。新しい知識をモデル内部に反映するには再学習や追加学習が必要となり、計算・金銭・環境面のコストが大きい。

RAGはこの問題に対し、外部文書集合から関連文書を検索し、その内容を言語モデルに与える。しかし、標準的な密検索器は主としてクエリと文書の意味的類似度を用いるため、次のような状況に弱い。

* 同じ出来事について、異なる時点の文書が複数存在する。
* 質問に「最新の」などの明示的な時間表現がない。
* 質問日時と文書内の年が一致しない。
* 質問時点より未来の文書が検索対象に含まれる。

例えば2019年の大会結果を尋ねる質問に2019年のタイムスタンプを付けた場合、検索器は「2019」という文字列の一致によって正しい文書を取得できることがある。一方、同じ質問を2020年初頭に行うと、質問側の「2020」と正解文書側の「2019」が一致しないため、検索精度が急落する。既存RALMは、この「2020年時点で最も新しい過去の結果は2019年」という関係を明示的には扱わない。([arXiv][3])

## 3. 提案手法

### 3.1 意味的検索スコア

クエリ (q) と文書 (d) の意味的関連度は、ATLASの密検索器と同様に、クエリ埋め込みと文書埋め込みの内積で求める。

[
S_{\mathrm{sem}}(q,d)
=====================

\langle f_\theta(q),f_\theta(d)\rangle
]

ここまでは通常の密検索と同じである。

### 3.2 時間的検索スコア

クエリのタイムスタンプを (t_q)、文書のタイムスタンプを (t_d) とすると、時間的スコアは概ね次式で定義される。

[
S_{\mathrm{temp}}(q,d)
======================

\frac{\alpha}{t_q-t_d}
]

(\alpha) は時間スコアの強さを調整する係数である。この設計により、質問時点より過去にあり、かつ質問時点に近い文書ほど大きなスコアを得る。

時間スコアと意味スコアは数値範囲が異なるため、時間スコアを意味スコアの平均・標準偏差に合わせて正規化し、両者を加算する。

[
S_{\mathrm{TempRALM}}(q,d)
==========================

S_{\mathrm{sem}}(q,d)
+
\widehat{S}_{\mathrm{temp}}(q,d)
]

したがって、意味的に無関係な最新文書を単純に優先するのではなく、**意味的に関連する候補の中から、質問時点に近い文書を優先する**設計となっている。

### 3.3 未来文書のマスキング

文書が質問時点より未来にある場合、すなわち (t_d>t_q) なら、その文書のスコアを (-\infty) に設定して検索結果から除外する。

[
S_{\mathrm{TempRALM}}(q,d)=-\infty
\qquad
\text{if }t_d>t_q
]

これにより、過去時点の質問に対して、その時点ではまだ存在していなかった情報が使われる「未来情報リーク」を防ぐ。

### 3.4 既存RAGへの導入方法

TempRALMは検索器を最初から学習し直すのではなく、次の手順で導入される。

1. 既存検索器によって通常より多くの候補文書を取得する。
2. 各候補に時間スコアを付与する。
3. 意味スコアと時間スコアの合計で再ランキングする。
4. 上位文書のみを言語モデルのreaderに渡す。

したがって、モデルの再事前学習や文書インデックスの再構築を必要とせず、既存のRAGシステムに後処理として追加できる点が大きな特徴である。([arXiv][1])

## 4. 実験方法

### 4.1 ベースライン

ベースラインには、few-shot学習に対応した検索拡張モデルATLASを使用している。ATLASは、Contriever系の密検索器、T5ベースの言語モデル、Fusion-in-Decoder構成からなる。TempRALMは同じモデルと事前学習済みパラメータを用い、検索ランキング部分にのみ時間処理を追加している。主な実験はNVIDIA A100 40GB GPU上で実施された。([arXiv][3])

### 4.2 文書コーパス

実験では、テニスの4大大会に関するATP／WTAの表データを自然言語文書へ変換している。

* 対象期間：1978～2019年
* 対象：4大大会の各年の大会結果
* 文書数：約50,000件
* 各文書に大会・年・選手などの情報とタイムスタンプを付与

同じ大会について年ごとに類似した文書が多数存在するため、意味検索だけでは年を区別しにくく、時間的検索を評価するための制御されたデータセットとなっている。

### 4.3 学習データ

few-shot学習の規模として、以下の3条件を設定している。

* 32-shot
* 64-shot
* 128-shot

学習例は1978～2018年の範囲から構成され、特定の質問形式に偏らないよう調整されている。

### 4.4 テストデータ

各128問からなる2つのテストセットを用いる。

* **TPQ-2019**：2019年の出来事について、2019年時点のタイムスタンプで質問する。
* **TPQ-2020**：同じ2019年の出来事について、質問タイムスタンプだけを2020年に変更する。

質問内容と正解は基本的に同じであり、質問日時だけが年境界を越える。この設計により、モデルが単なる年表記の文字列一致に依存しているのか、質問日時と文書日時の相対関係を扱えているのかを検証できる。評価指標は回答のExact Matchと、検索器のRecall@1／Recall@5である。各回答実験は5回実行され、その平均が報告されている。

## 5. 結果

### 5.1 回答のExact Match

| few-shot数 | TPQ-2019 ATLAS | TPQ-2019 TempRALM | TPQ-2020 ATLAS | TPQ-2020 TempRALM |
| --------: | -------------: | ----------------: | -------------: | ----------------: |
|        32 |          64.84 |         **71.72** |          37.65 |         **55.93** |
|        64 |          74.68 |         **75.78** |          40.93 |         **68.44** |
|       128 |          76.88 |         **77.80** |          41.72 |         **72.65** |

TPQ-2019では両モデルの差は比較的小さい。一方、質問タイムスタンプだけを2020年に移したTPQ-2020では、ATLASの性能が大幅に低下するのに対し、TempRALMは高い性能を維持している。

TPQ-2020におけるATLASからの相対改善率は、32-shotで約49%、64-shotで約67%、128-shotで約74%である。学習例が多くなるほど、時間的に適切な文書を与えることによるreader側の利益も大きくなっている。

### 5.2 検索性能

64-shot条件における検索結果は次の通りである。

| テストセット   | モデル      | Recall@1 | Recall@5 |
| -------- | -------- | -------: | -------: |
| TPQ-2019 | ATLAS    |     0.54 |     0.63 |
| TPQ-2019 | TempRALM | **0.63** | **0.75** |
| TPQ-2020 | ATLAS    |     0.24 |     0.63 |
| TPQ-2020 | TempRALM | **0.64** | **0.75** |

特にTPQ-2020のRecall@1は0.24から0.64へ上昇している。これは、正しい文書が上位5件に含まれるだけでなく、最上位へ配置されやすくなったことを示す。ATLASのRecall@5がTPQ-2019とTPQ-2020で同じ0.63である一方、Recall@1だけが大きく低下している点からも、問題の一部が「候補文書を全く見つけられないこと」ではなく、「時間的に正しい文書を最上位に順位付けできないこと」にあると分かる。

### 5.3 モデルサイズによる違い

64-shot条件では、BaseモデルとLargeモデルの双方でTempRALMがATLASを上回った。

* TPQ-2020、Base：28.13 → 35.94
* TPQ-2020、Large：40.93 → 68.44

したがって、時間的再ランキングの効果は単一のモデルサイズだけに限定されない。ただしLargeモデルの改善幅が特に大きく、正しい文書を取得した後に、その内容から正解を生成する能力も重要であることが示唆される。

## 6. 考察

### 有効性

本論文の重要な知見は、時間依存質問の失敗が、必ずしもLLMの時間推論能力だけに由来するわけではなく、**検索ランキングの段階ですでに発生している**ことを示した点にある。

ATLASは、質問と文書に同じ年が含まれるTPQ-2019では比較的高性能だが、質問日時を1年進めただけで性能が低下する。これに対しTempRALMは、絶対的な年の一致ではなく、「質問時点以前で最も近い文書」という相対的関係をスコア化するため、年境界を越えた質問に対して頑健である。([arXiv][3])

### reader側の限界

著者らは、正解文書が検索されているにもかかわらず、言語モデルが誤答するケースも確認している。したがって、時間的再ランキングによって検索問題が改善しても、readerが文書から適切な答えを抽出・生成できなければ、最終的な回答精度は完全には改善しない。Largeモデルで利益が大きい結果は、この検索器とreaderの相互作用を反映している可能性がある。

### 限界

第一に、実験領域がテニス大会に限定されており、文書も表データから生成された定型的な文章である。ニュース、論文、法律、医療記録など、時間表現が曖昧で複雑な実文書への一般化は未検証である。

第二に、本手法はすべての質問について「質問時点以前で最も新しい文書ほど有用」という帰納バイアスを置いている。しかし、「2010年当時の優勝者」「最初に達成した人物」「今後予定されている大会」といった質問では、単純な近接度や未来文書の排除が不適切になる。

第三に、発行時刻、出来事の発生時刻、文書の更新時刻、有効期間など、現実の情報検索で必要となる複数種類の時間を区別していない。明示的なタイムスタンプが正確に付与されていることも前提となる。

第四に、時間スコアの式をそのまま解釈すると (t_q=t_d) で特異点が発生するため、実装上の微小定数や境界処理を明確にする必要がある。また、未来マスキング、時間距離スコア、正規化、候補文書の過剰取得を個別に評価するアブレーションも十分ではない。

第五に、5回平均は報告されているものの、標準偏差、信頼区間、統計的有意差、再ランキングによるレイテンシ増加などは明確に示されていない。最大74%という値も相対改善率であり、128-shot TPQ-2020での絶対改善は41.72から72.65への30.93ポイントである点には注意が必要である。

## 7. まとめ

TempRALMは、RAGの検索スコアに質問時刻と文書時刻の関係を組み込むことで、時間依存質問に適した文書を取得する手法である。

中心となる工夫は次の3点である。

1. 意味スコアに時間近接度スコアを加える。
2. 質問時点より未来の文書を除外する。
3. 既存検索器の候補を再ランキングし、再学習やインデックス再構築を不要にする。

制御実験では、通常のATLASが質問日時の年を変えただけで大幅に性能を落とすのに対し、TempRALMは高い検索・回答性能を維持した。特に128-shotのTPQ-2020では、Exact Matchが41.72から72.65へ改善した。([arXiv][1])

なお、IEEE CAI 2025に収録された最終版の抄録では、ATLASに対する最大74%の改善に加え、検索付き商用LLMに対する32%の改善、および時間質問応答用の正解データを自動生成する「TablePedia」の導入も報告されている。ただし、これらの詳細は公開arXiv v2の本文には含まれていない。([SJSU ScholarWorks][2])

## 8. 総評

**査読結果：採択。**
本論文の最終版はIEEE CAI 2025の会議論文として採択され、75～82ページに収録されている。DOIは `10.1109/CAI64502.2025.00019` である。公開されている情報からは、個別査読者のコメントや評点までは確認できない。([SJSU ScholarWorks][2])

**独自の査読評価：Weak Accept相当。**

採択を支持する最大の理由は、複雑な時間推論モデルを導入するのではなく、既存RAGに容易に組み込める再ランキングとして問題を解決している点である。質問タイムスタンプだけを変更する対照実験も分かりやすく、意味検索が年表記の一致に依存する失敗を明確に可視化している。最大74%の相対改善とRecall@1の大幅な向上も、提案部分が実際に検索順位を改善していることを裏付ける。

一方、評価データが単一の定型的領域に限られ、時間スコアの各構成要素に対するアブレーションや実行コスト分析が不足している。そのため、「一般的な時間認識RAGを確立した」というより、**RAG検索に時間メタデータを加えることの有効性を、単純かつ説得力のある形で実証した研究**と評価するのが妥当である。
