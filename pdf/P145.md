# P145「Deep Learning with Differential Privacy」

**著者**：Martín Abadi, Andy Chu, Ian Goodfellow, H. Brendan McMahan, Ilya Mironov, Kunal Talwar, Li Zhang
**発表先**：ACM CCS 2016, pp. 308–318
**主題**：深層ニューラルネットワークを、形式的な差分プライバシー保証付きで学習する方法の提案。([Google Research][1])

## 1. はじめに

深層学習では、大規模かつ代表性の高い学習データが必要となる。一方、医療記録、行動履歴、入力文、写真などの機微情報を含むデータを使った場合、学習済みモデルが個々の訓練事例を記憶し、その情報を漏えいする危険がある。

本論文は、モデル構造やパラメータを攻撃者に完全に公開したとしても、個々の訓練レコードの影響を数学的に制限できる学習法を提案する。中心的な成果は、現在一般に **DP-SGD（Differentially Private Stochastic Gradient Descent）** と呼ばれる学習アルゴリズムと、反復学習によるプライバシー損失を高精度に計算する **Moments Accountant** である。

---

## 2. 背景

### 差分プライバシー

データセット (D) と、そこから一つのレコードを追加または削除した (D') を隣接データセットとする。ランダム化アルゴリズム (M) が次を満たすとき、(M) は ((\varepsilon,\delta))-差分プライバシーを満たす。

[
\Pr[M(D)\in S]
\leq e^\varepsilon\Pr[M(D')\in S]+\delta
]

直感的には、特定の一人のデータが学習に含まれている場合と含まれていない場合で、モデル出力の分布が大きく変化しないことを意味する。一般に、(\varepsilon) と (\delta) が小さいほど強いプライバシー保証となる。

### 深層学習への適用上の問題

通常のSGDでは、訓練事例から勾配を計算し、それを何千回、何万回とモデルに反映する。そのため、次の問題が生じる。

* 個別事例が勾配に与える影響には上限がない。
* 各更新にノイズを加えても、反復によりプライバシー損失が累積する。
* 従来の合成定理では、この累積損失を過大評価し、実用的な精度が得られない。
* 深層学習は非凸最適化であり、凸モデル向けの既存の差分プライバシー手法をそのまま適用しにくい。

本論文は、学習後のパラメータに一度だけノイズを加えるのではなく、**SGDの各更新時点で個々の事例の影響を制御する**方針を採用する。

---

## 3. 提案手法

### 3.1 DP-SGD

各学習ステップで、次の処理を行う。

まず、訓練データからランダムに事例集合 (L_t) を抽出し、各事例 (x_i) について個別勾配を計算する。

[
g_t(x_i)=\nabla_{\theta_t}\mathcal{L}(\theta_t,x_i)
]

次に、各勾配の (L_2) ノルムが上限 (C) 以下になるようクリッピングする。

[
\bar g_t(x_i)
=============

\frac{g_t(x_i)}
{\max\left(1,\frac{\lVert g_t(x_i)\rVert_2}{C}\right)}
]

これにより、一つの事例が更新に及ぼせる影響が最大 (C) に制限される。

クリッピング後の勾配和にガウスノイズを加え、平均勾配を作る。

[
\tilde g_t
==========

\frac{1}{L}
\left(
\sum_{i\in L_t}\bar g_t(x_i)
+
\mathcal{N}(0,\sigma^2C^2I)
\right)
]

最後に、通常のSGDと同様にモデルを更新する。

[
\theta_{t+1}=\theta_t-\eta_t\tilde g_t
]

ここで (C) はクリッピング閾値、(\sigma) はノイズ倍率、(L) は一度のノイズ付加単位である lot の大きさである。論文では、計算上の小単位を batch、ノイズを一度加える論理的な単位を lot と区別している。

### 3.2 Moments Accountant

単純な合成定理では、各ステップの ((\varepsilon,\delta)) を逐次加算するため、長時間の学習で非常に緩い上界となる。

そこで本論文は、プライバシー損失確率変数

[
c(o)=
\log
\frac{\Pr[M(D)=o]}
{\Pr[M(D')=o]}
]

の対数モーメント

[
\alpha_M(\lambda)
=================

\log
\mathbb{E}
\left[
\exp(\lambda c(o))
\right]
]

を管理する。対数モーメントは適応的な複数ステップでも加法的に合成できる。

[
\alpha_M(\lambda)
\leq
\sum_t\alpha_{M_t}(\lambda)
]

累積後は、次式により ((\varepsilon,\delta)) へ変換する。

[
\delta
======

\min_\lambda
\exp\bigl(\alpha_M(\lambda)-\lambda\varepsilon\bigr)
]

これにより、サブサンプリングされたガウス機構の性質を直接利用し、一般的な強合成定理よりも大幅に小さいプライバシー損失を算出できる。理論上も、強合成定理に存在する追加的な (\sqrt{\log(T/\delta)}) 程度の要因を削減している。

### 3.3 実装上の工夫

TensorFlow上に、次の二つの構成要素を実装している。

* **Sanitizer**：事例ごとの勾配計算、勾配クリッピング、ガウスノイズ付加を行う。
* **Privacy Accountant**：学習の各ステップで対数モーメントを累積し、使用済みの ((\varepsilon,\delta)) を計算する。

通常のTensorFlowはバッチ平均勾配を計算するため、本手法に必要な事例単位の勾配を得る `per_example_gradient` 演算も実装している。また、MNISTでは差分プライバシー付きPCAを用いて入力次元を削減している。

## 4. 実験方法

### Moments Accountantの評価

以下の条件で、従来の強合成定理とMoments Accountantを比較した。

[
q=0.01,\qquad \sigma=4,\qquad \delta=10^{-5}
]

ここで (q=L/N) は各ステップにおけるサンプリング率である。学習エポック数を増やした際の累積 (\varepsilon) を比較している。

### MNIST

データは訓練60,000枚、テスト10,000枚の28×28グレースケール画像である。

モデル構成は以下である。

* 差分プライバシー付き60次元PCA
* 1,000ユニットのReLU隠れ層
* 10クラスのSoftmax出力
* lot size：600
* 各層の勾配クリッピング閾値：4
* 初期学習率：0.1
* 10エポックで0.052まで線形に減衰

ニューラルネットワーク用ノイズ (\sigma) とPCA用ノイズ (\sigma_p) について、((2,4))、((4,7))、((8,16)) の3条件を比較した。

### CIFAR-10

CIFAR-10は訓練50,000枚、テスト10,000枚の32×32 RGB画像からなる。

使用したモデルは、5×5畳み込みを用いる2層の畳み込み層と、384ユニットの全結合層2層からなる。ただし、畳み込み層を訓練データから完全にプライベート学習するのではなく、公開データとみなしたCIFAR-100で事前学習し、主に上位の全結合層とSoftmax層を差分プライバシー付きで再学習している。

主な設定は、ノイズ倍率 (\sigma=6)、クリッピング閾値3、lot size 2,000または4,000である。

## 5. 結果

### プライバシー会計

Moments Accountantは、強合成定理よりも非常に小さい (\varepsilon) を算出した。

| エポック数 |               強合成定理 | Moments Accountant |
| ----: | ------------------: | -----------------: |
|   100 |  (\varepsilon=9.34) | (\varepsilon=1.26) |
|   400 | (\varepsilon=24.22) | (\varepsilon=2.55) |

いずれも (\delta=10^{-5}) である。特に400エポックでは、同じアルゴリズムに対して従来法が ((24.22,10^{-5}))-DPしか示せないのに対し、提案法は ((2.55,10^{-5}))-DPを示している。

### MNIST

| プライバシー保証           | テスト精度 |
| ------------------ | ----: |
| ((0.5,10^{-5}))-DP |   90% |
| ((2,10^{-5}))-DP   |   95% |
| ((8,10^{-5}))-DP   |   97% |
| 非プライベート基準モデル       | 98.3% |

(\varepsilon=8) の条件では、非プライベートモデルとの差は約1.3ポイントに抑えられた。また、DP-SGDでは訓練精度とテスト精度の差が小さく、ノイズ付加と勾配制限が正則化として働いていることが観察された。

パラメータ分析では、PCAによって精度が約2ポイント改善し、入力次元を784から60へ削減することで学習時間がおよそ10分の1になった。lot sizeとノイズ倍率は精度への影響が特に大きく、MNISTでは経験的に (\sqrt{N}) 程度のlot sizeが良好だった。

### CIFAR-10

(\delta=10^{-5}) の条件で、以下の結果が得られた。

| (\varepsilon) | テスト精度 |
| ------------: | ----: |
|             2 |   67% |
|             4 |   70% |
|             8 |   73% |

公開データによる事前学習を利用した非プライベート基準モデルは約80%であり、(\varepsilon=8) のプライベートモデルとの差は約7ポイントだった。MNISTよりも複雑なタスクでは、プライバシーと精度のトレードオフが大きくなっている。

## 6. 考察

本論文の最大の意義は、単にSGDへノイズを加えたことではなく、**非凸な深層学習を反復実行した場合のプライバシー損失を、実用的な水準まで厳密化したこと**にある。DP-SGDの構造自体は比較的単純だが、Moments Accountantがなければ長期間の学習に対する (\varepsilon) が過大になり、実用性を主張できなかった。

また、勾配クリッピングには二つの相反する効果がある。(C) を小さくすると感度とノイズ量は減るが、真の勾配方向が歪む。(C) を大きくすると勾配は保持されるが、ノイズの標準偏差 (\sigma C) も大きくなる。したがって、精度はノイズ倍率だけでは決まらず、クリッピング、lot size、学習率、訓練ステップ数を一体として調整する必要がある。

一方、次の制約がある。

第一に、評価対象はMNISTとCIFAR-10という比較的小規模な画像分類に限られる。特にCIFAR-10では、畳み込み層をCIFAR-100で事前学習しており、モデル全体を機微データからDP-SGDで一から学習した結果ではない。公開かつ関連性の高いデータが利用できない状況では、同程度の精度が得られるとは限らない。

第二に、理論解析では各事例を独立確率 (q) で選ぶポアソンサンプリングを仮定する一方、実装ではデータをランダムに並べ替えて分割する方式を使用している。このサンプリング方式の差は、厳密なプライバシー保証を実装へ移す際に注意を要する。

第三に、保証の単位は原則として一つの訓練レコードである。一人の利用者が多数のレコードを提供する場合、利用者単位の保証はgroup privacyによって弱くなる。したがって、実環境では「一レコード」と「一利用者」のどちらを保護対象とするのかを明示する必要がある。

第四に、(\varepsilon=8) は一桁のプライバシー予算ではあるが、あらゆる用途で十分に強いとは限らない。医療や位置情報など高機密分野では、精度だけでなく、隣接関係、ユーザー当たりのレコード数、許容する攻撃モデルを踏まえて予算を決める必要がある。

---

## 7. まとめ

本論文は、以下の三点を確立した。

1. 事例単位の勾配クリッピングとガウスノイズ付加によるDP-SGD
2. 反復学習のプライバシー損失を高精度に追跡するMoments Accountant
3. TensorFlow上での実装と、MNIST・CIFAR-10による実用可能性の検証

MNISTでは ((8,10^{-5}))-DPで97%、CIFAR-10では同条件で73%を達成した。特にMoments Accountantによって、従来の合成定理と比較してプライバシー損失の上界を大幅に改善した点が、理論面と実用面の双方で重要である。

---

## 8. 総評

### 長所

理論、アルゴリズム、実装、実験が一つの論文内で統合されている。特に、非凸目的関数を持つ多層ニューラルネットワークについて、モデル全体に対する明示的な差分プライバシー保証を示した点は新規性と実用的価値が高い。

Moments Accountantは単なる解析上の改善ではなく、同じ学習処理に対する (\varepsilon) を、400エポック時の24.22から2.55まで縮小している。この改善が、DP深層学習を「形式的には可能」な段階から「一定の精度で実行可能」な段階へ進めている。

### 短所

CIFAR-10実験は公開データによる事前学習への依存が大きく、完全なend-to-endのプライベート深層学習とはいえない。また、実験対象が限定的であり、大規模データ、自然言語、利用者単位プライバシーなどへの一般化は未検証である。サンプリング方式やデータ依存のハイパーパラメータ調整についても、理論上の保証と実装上の運用をより厳密に対応付ける余地がある。

### 査読結果

**判定：採択（Strong Accept相当）**

理由は、明確な問題設定、強い理論的新規性、再利用可能なアルゴリズム、実装可能性、実験的裏付けを備えているためである。実験規模や公開事前学習への依存は弱点だが、主要な貢献を損なうものではなく、後続研究の基盤として十分な価値がある。なお、本論文は実際にセキュリティ分野の主要国際会議であるACM CCS 2016に採択・掲載されている。([Google Research][1])
