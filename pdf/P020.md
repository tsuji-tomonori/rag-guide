## 1. はじめに

**MTEB: Massive Text Embedding Benchmark**は、テキスト埋め込みモデルを多様な用途で統一的に評価するためのベンチマークを提案した論文である。著者はNiklas Muennighoff、Nouamane Tazi、Loïc Magne、Nils Reimers。58データセット、8タスク、112言語を収録し、33モデルを比較している。([arXiv][1])

本論文の中心的な主張は、**STSで高性能なモデルが検索やクラスタリングでも高性能とは限らず、あらゆる用途を支配する万能な埋め込みモデルは存在しない**という点である。

## 2. 背景

テキスト埋め込みは、文書検索、クラスタリング、重複検出、意味類似度計算、分類器の特徴量などに利用される。しかし従来の研究では、STSや分類など少数のタスクだけでモデルを評価する場合が多く、検索・再ランキング・クラスタリングへの転用可能性が十分に検証されていなかった。

また、研究ごとに前処理、分類器、距離関数、ハイパーパラメータが異なるため、性能差が埋め込みモデル自体によるものか、評価パイプラインによるものかが判別しにくかった。SentEvalは検索やクラスタリングを含まず、BEIRは主に情報検索に限定されるため、複数用途を横断する統一基盤が必要だった。

## 3. 提案手法

提案されたMTEBは、新しい埋め込み学習法ではなく、**既存モデルを同一条件で比較する評価フレームワーク**である。設計原則は次の4点である。

* 多様性：複数タスク、言語、文章長を収録する
* 簡潔性：テキスト列を固定次元ベクトルへ変換できれば評価可能
* 拡張性：データセットや評価タスクを追加可能
* 再現性：ソフトウェア、データセット、結果をバージョン管理する

モデル側に要求される基本インターフェースは、文字列のリストを同一次元のベクトルへ変換することだけであり、論文では10行未満のコードでモデルを接続できるとしている。([ar5iv][2])

8種類のタスクと主要指標は以下の通りである。

| タスク                 | 評価内容                     | 主要指標              |
| ------------------- | ------------------------ | ----------------- |
| Bitext Mining       | 異言語間で翻訳対応文を探索            | F1                |
| Classification      | 埋め込み上でロジスティック回帰を学習       | Accuracy          |
| Clustering          | Mini-batch k-meansで文書を分類 | V-measure         |
| Pair Classification | 文ペアが重複・言い換えか判定           | Average Precision |
| Reranking           | 候補文書を関連度順に並べ直す           | MAP               |
| Retrieval           | コーパス全体から関連文書を検索          | nDCG@10           |
| STS                 | 文ペアの意味的類似度を推定            | Spearman相関        |
| Summarization       | 自動要約と人手要約の近さを評価          | Spearman相関        |

さらに、入力の長さと関係をSentence-to-Sentence、Paragraph-to-Paragraph、Sentence-to-Paragraphに分類し、短文だけでなく論文要旨や文書検索も扱っている。

## 4. 実験方法

実験では、単語埋め込み、BERT系エンコーダ、T5系エンコーダ、GPT系デコーダ、多言語モデル、API経由の埋め込みモデルなど、合計33モデルを評価した。代表例としてGloVe、BERT、SimCSE、Contriever、LaBSE、MiniLM、MPNet、SGPT、GTR、ST5、OpenAI Adaが含まれる。自己教師ありモデルと教師ありファインチューニング済みモデルを分けて分析している。

基本的には各データセットのテスト分割を使用し、MSMARCOだけはBEIRの慣例に従って開発分割を使用した。最終的には30超のモデルについて約5,000回の実験を実施している。([ar5iv][2])

英語の総合比較では、Bitext Miningを除いた7タスク、56データセットを評価している。速度評価はSTS15を用い、NVIDIA A100 80GB、CUDA 11.6環境で実施された。また、速度だけでなく埋め込み1件当たりの保存容量も比較している。

## 5. 結果

英語サブセットにおける主要結果は以下の通りである。数値は各タスクの主要指標を100倍した平均値である。

| 評価対象    | 最良モデル              |   スコア |
| ------- | ------------------ | ----: |
| 総合平均    | ST5-XXL            | 59.51 |
| 分類      | ST5-XXL            | 73.42 |
| クラスタリング | ST5-XXL            | 43.71 |
| ペア分類    | GTR-XL             | 86.13 |
| 再ランキング  | MPNet              | 59.36 |
| 検索      | SGPT-5.8B-msmarco  | 50.25 |
| STS     | ST5-XXL            | 82.63 |
| 要約評価    | multilingual MPNet | 31.57 |

総合平均ではST5-XXLが首位だったが、全タスクで首位になったモデルはなかった。検索ではMSMARCO向けに学習されたSGPTが強く、再ランキングではMPNet、ペア分類ではGTRが優位だった。特にクラスタリングでは、ST5-XXLの43.71に対して、約50分の1の規模とされるMPNetが43.69でほぼ同等だった。

教師ありファインチューニング済みモデルは、自己教師ありモデルより全般的に高性能だった。また、モデル規模と性能には強い相関が見られたが、大規模モデルほど推論速度と保存容量の負担が増加した。速度重視ならGloVe、速度と性能の両立ならMiniLMまたはMPNet、最高性能重視ならST5-XXL、GTR-XXL、SGPT-5.8Bが候補と整理されている。

多言語評価では、翻訳文探索はLaBSEが広範な言語で強く、多言語分類とSTSではmultilingual MPNetが総合的に良好だった。SGPT-BLOOMはBLOOMの事前学習データに多く含まれた中国語、フランス語、ポルトガル語などで強かった一方、未学習言語では性能が低下した。

## 6. 考察

本論文の最も重要な発見は、**埋め込みの品質が単一の尺度では表せない**ことである。STSやペア分類は対称的な文ペアを扱うが、検索では短いクエリと長い文書という非対称な入力を扱う。そのため、NLIやSTS向けに学習したモデルは類似度計算には強くても検索には弱く、MSMARCO向けモデルは逆の傾向を示した。

MTEBの総合スコアは有用だが、無条件にモデル選定へ用いるべきではない。56データセットの単純平均であるため、15データセットを持つ検索や12データセットを持つ分類の影響が大きく、1データセットしかない要約評価の影響は小さい。著者自身もこのタスク不均衡を限界として認めている。実運用では総合順位より、対象用途に近いタスクのスコア、推論速度、ベクトル次元、利用言語を優先すべきである。

評価上の注意点もある。クラスタリングでは正解ラベル数をk-meansのクラスタ数として与えているため、完全な未知クラスタ発見ではなく、**クラスタ数が既知という条件下での埋め込み分離性能**を測っている。また分類では埋め込み上にロジスティック回帰を追加するため、純粋な距離構造だけでなく線形分離可能性を評価している。

より重大なのは学習データ汚染である。著者らはSciDocsRRの一部がMPNetやMiniLMの学習データに含まれた可能性を指摘しながら、十分な数のデータセットを平均すれば影響は小さいとして、重複を補正していない。この仮定はランキングの信頼性に影響し得るため、将来のベンチマークでは学習データの申告や汚染耐性評価が必要である。

そのほか、数百語を超える長文が不足していること、多言語の検索・クラスタリングがないこと、コードや画像を含む評価がないことも原版MTEBの限界である。

## 7. まとめ

MTEBは、従来ばらばらに評価されていたテキスト埋め込みモデルを、8タスク、58データセット、112言語にわたって統一評価したベンチマークである。

実験から得られた結論は明確である。

> 「最高の埋め込みモデル」を一つ選ぶことはできず、検索、類似度、分類、多言語処理など、目的に応じてモデルを選ぶ必要がある。

ベンチマーク実装、評価結果、リーダーボードが公開されているため、モデル研究だけでなく、実システムでの埋め込みモデル選定にも大きな価値を持つ。

## 8. 総評

**論文としての評価：高い。**

新しいモデルを提案する論文ではないものの、評価方法の断片化という実務的・研究的に重要な問題を解決している。評価範囲、再現性、拡張性、公開リーダーボードという点で貢献が大きく、「STSの順位だけでモデルを選んではならない」ことを大規模実験で示した点も説得力がある。

一方、総合スコアのタスク不均衡、既知クラスタ数を使うクラスタリング設定、学習データ汚染、長文・多言語検索の不足には注意が必要である。したがって、MTEBはモデル候補を絞るための有力な基準だが、最終的には対象ドメインのデータによる追加評価が必要である。
