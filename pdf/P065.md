# 1. はじめに

**論文名:** *From Local to Global: A GraphRAG Approach to Query-Focused Summarization*
**主題:** 大規模な文書集合全体を俯瞰する質問に対し、知識グラフと階層的要約を使って回答するGraphRAGの提案。

通常のRAGは、「特定の人物が何を述べたか」のように、答えが少数の文書断片に局在する質問には強い一方、「文書集合全体の主要テーマは何か」「全体を通じてどのような傾向があるか」といった**大域的なセンスメイキング質問**には弱い。本論文は、この問題を単純な検索ではなく、文書集合全体に対する**クエリ指向要約**として捉え直している。

中心的なアイデアは、文書からエンティティ知識グラフを構築し、グラフをコミュニティに分割してあらかじめ要約しておくことである。質問時には各コミュニティ要約から部分回答を作り、それらを統合して最終回答を生成する。

# 2. 背景

一般的なベクトルRAGでは、質問と意味的に近いテキストチャンクを埋め込み空間から検索し、上位のチャンクだけをLLMへ渡す。この方法では、検索された局所的な情報が、文書集合全体を代表している保証がない。

例えば「ニュース記事全体から見た公衆衛生上の主要課題」という質問では、個別記事を数件検索するだけでは、話題の頻度、分布、相互関係、多様な観点を十分に把握できない。そのため、必要なのは局所検索ではなく、コーパス全体を対象とした要約である。

既存研究にも、階層的なテキスト要約や知識グラフを利用するRAGは存在する。しかし本研究の特徴は、グラフを単なる検索・経路探索のために使うのではなく、グラフが持つ**モジュール性**を利用し、Leiden法によって階層的なコミュニティへ分割する点にある。各コミュニティを意味的なまとまりとして要約することで、コーパス全体を漏れなく分割統治する。

# 3. 提案手法

GraphRAGは、インデックス構築時と質問応答時の二段階から構成される。

## インデックス構築

まず文書をチャンクへ分割する。各チャンクについてLLMが、人物・組織・場所などのエンティティ、その説明、エンティティ間の関係を抽出する。必要に応じて、日時や出来事などの検証可能な主張も抽出できる。

同一名称のエンティティや関係を集約し、エンティティをノード、関係をエッジとする知識グラフを構築する。関係が複数回抽出された場合、その回数をエッジの重みとして利用する。なお、論文中の実装ではエンティティ同定に基本的に完全文字列一致を使っている。

次にLeiden法を再帰的に適用し、グラフを階層的なコミュニティに分割する。最下層ではノード、エッジ、関連情報を直接要約し、上位層では下位コミュニティの要約を統合する。これにより、細粒度な話題からコーパス全体の概観まで、複数解像度の要約が得られる。

## 質問応答

質問時には、指定された階層のコミュニティ要約を次のmap-reduce方式で処理する。

1. コミュニティ要約をシャッフルして複数の入力単位に分ける。
2. 各入力単位から質問への部分回答を並列生成する。
3. 各部分回答に0～100の有用度スコアを付け、0の回答を除外する。
4. 有用度の高い回答からコンテキストへ追加し、最終的な大域回答を生成する。

つまり、直接「関連文書を検索する」のではなく、**コーパスの各領域が質問に対して何を示すかを並列に判断し、最後に統合する**方式である。

# 4. 実験方法

実験には次の2データセットを使用している。

* 技術系ポッドキャストの文字起こし：約100万トークン、1,669チャンク
* ニュース記事集合：約170万トークン、3,197チャンク

いずれも600トークン単位で分割し、隣接チャンク間に100トークンの重複を設けている。ポッドキャスト側では8,564ノード・20,691エッジ、ニュース側では15,754ノード・19,520エッジのグラフが生成された。

評価質問は、LLMに想定利用者を5種類、各利用者のタスクを5種類、各組み合わせの質問を5件生成させ、各データセットについて計125問を作成した。質問は個別事実の検索ではなく、コーパス全体の理解を必要とするよう設計されている。

比較条件は以下の6種類である。

* **C0～C3:** GraphRAG。C0が最上位、C3が最下位のコミュニティ要約
* **TS:** グラフを作らず、元テキストに同じmap-reduce要約を直接適用
* **SS:** ベクトル類似度でチャンクを検索する通常のRAG

回答生成時のコンテキストは全条件で8,000トークンに統一された。8k、16k、32k、64kを予備比較した結果、8kが包括性で最も良かったためである。グラフ構築にはGPT-4-turboを使用し、ポッドキャストデータのインデックス構築には約281分を要した。

評価は二段階で行われた。

**実験1**では、LLM-as-a-judgeによる一対比較を各質問につき5回実行し、包括性、多様性、利用者の判断支援度、直接性を評価した。

**実験2**では、回答から検証可能な主張を抽出するClaimifyを用い、47,075件の重複除去済み主張を取得した。主張数を包括性、主張をクラスタリングした際のクラスタ数を多様性の指標とした。

# 5. 結果

通常のベクトルRAGに対して、GraphRAGを含む大域的手法は明確に優位だった。

包括性の勝率は、ポッドキャストで72～83%、ニュースで72～80%だった。多様性の勝率は、それぞれ75～82%、62～71%であり、多くの比較で統計的に有意だった。一方、直接性では通常RAGが優れており、GraphRAGの回答は情報量が多い反面、冗長になりやすいことが示された。

GraphRAGと、グラフを使わない全文map-reduce要約TSとの差は小さい。ポッドキャストではC2がTSに対して包括性57%、多様性57%の勝率、ニュースではC3が包括性64%、多様性60%の勝率だった。ただし主張ベースの実験2では、GraphRAG各条件とTSの間に一貫した有意差は確認されなかった。

計算効率ではグラフの利点が大きい。最上位コミュニティを使うC0の質問当たり入力トークン数は、TSの約2.3～2.6%で、97%以上削減されている。最下位のC3でもTSより26～33%少ない。C0は大幅に低コストでありながら、ベクトルRAGに対して包括性約72%、多様性約62%の勝率を維持した。

主張数でも、ニュースではC0が平均34.18件、SSが25.23件、ポッドキャストでは最良のC2が32.46件、SSが26.50件だった。すべての大域的条件は包括性でSSを有意に上回った。LLM判定と主張ベース判定の一致率は、非引き分け事例に限定すると包括性78%、多様性69～70%だった。

# 6. 考察

本研究の最も重要な点は、「大域的な質問を、検索問題として解くべきではない」と示したことである。GraphRAGは、文書全体を意味的なコミュニティへ事前整理することで、局所的な類似検索では拾いにくい横断的傾向を回答へ反映できる。

一方、実験結果からは、性能向上の主要因が「グラフそのもの」なのか、「コーパス全体をmap-reduceで処理したこと」なのかは完全には切り分けられていない。TSもSSを大きく上回り、主張ベース評価ではGraphRAGとTSの有意差がほぼ見られない。したがって、グラフの明確な価値は、絶対的な回答品質よりも、**階層的な情報整理、複数解像度での探索、繰り返し質問時のトークン効率**にあると考えるのが妥当である。

評価上の課題も残る。対象は2種類、約100万トークン規模のコーパスに限られ、質問生成、回答生成、評価、主張抽出の多くをLLMに依存している。人間による利用者評価、出典との厳密な整合性、幻覚率は検証されていない。また、比較対象のSSは比較的単純なベクトルRAGであり、再ランキング、クエリ分解、反復検索、長文コンテキスト、ハイブリッド検索などの強いベースラインとの比較が必要である。論文自身も、異なる領域への一般化と生成内容の正確性評価を今後の課題として挙げている。

# 7. まとめ

本論文は、大規模文書集合に対する大域的質問を扱うため、LLMで構築した知識グラフを階層的コミュニティへ分割し、コミュニティ要約をmap-reduce方式で統合するGraphRAGを提案した。

通常のベクトルRAGと比べ、回答の包括性と多様性は大幅に向上した。特に同じコーパスへ多数の大域質問を行う用途では、事前生成した上位コミュニティ要約によって、回答品質を保ちながら質問時のトークン消費を大幅に削減できる。ただし、グラフを使わない全文要約との差は限定的であり、GraphRAGの本質的な優位性は効率性と階層構造にある。

# 8. 総評

**公開上の査読状況:** arXiv v2の原稿には「Preprint. Under review」と記載されており、原稿上は査読中である。

**模擬査読結果: Weak Accept（弱い採択）**

着眼点は明確で、通常RAGの重要な弱点を実用的なシステムとして解決している。パイプラインの再現性も高く、統計検定に加えて主張ベース評価を追加した点も評価できる。

ただし、グラフなしの大域要約との差が小さいこと、強いRAGベースラインや人間評価が不足していること、正確性・幻覚率を直接評価していないことから、GraphRAGの構成要素が性能へ与える因果的寄与は十分には確立されていない。採択相当ではあるが、最終版では強力なベースライン、グラフ構築要素のアブレーション、品質・コスト曲線、人間および出典整合性評価を追加することが望まれる。
