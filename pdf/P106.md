# P106：An Analysis of Fusion Functions for Hybrid Retrieval

**著者**：Sebastian Bruch、Siyu Gai、Amir Ingber
**掲載**：ACM Transactions on Information Systems, Vol. 42, No. 1, Article 20
**概要**：語彙検索と意味検索を統合するハイブリッド検索について、スコアの凸結合とReciprocal Rank Fusion（RRF）を理論・実験の両面から比較した論文である。([arXiv][1])

## 1. はじめに

本論文は、新しい検索モデルを提案するというより、**既存のハイブリッド検索における融合関数を分析する研究**である。

主な結論は以下である。

* 正規化した検索スコアの凸結合は、正規化方法の選択にそれほど敏感ではない。
* RRFは一般に考えられているほど「無調整」ではなく、パラメータによって性能が大きく変わる。
* 凸結合は、in-domainとout-of-domainの双方でRRFより高いランキング性能を示す。
* 凸結合の重みは、ごく少量のラベル付きクエリで調整できる。

したがって著者らは、検索器のスコアを取得できる場合、RRFよりも**正規化スコアの凸結合を優先すべき**だと主張している。([ar5iv][2])

## 2. 背景

テキスト検索には、大きく二つの方式がある。

**語彙検索**は、BM25などを用いてクエリと文書の単語一致を評価する。計算効率とゼロショット性能に優れる一方、同義語や言い換えを扱いにくいという語彙不一致問題がある。

**意味検索**は、クエリと文書を埋め込みベクトルに変換し、コサイン類似度や内積によって検索する。意味的な対応を捉えられるが、学習領域から外れたデータではBM25より性能が低下する場合がある。

そこで、両者の結果を統合するハイブリッド検索が用いられる。代表的な融合方法は次の二つである。

1. 語彙スコアと意味スコアを正規化し、重み付きで加算する凸結合
2. 各検索結果の順位だけを利用するRRF

先行研究では、凸結合はスコア尺度や正規化方法に敏感であり、順位だけを使うRRFの方がゼロショット検索に適していると報告されていた。本論文はこの見解を再検証している。([ar5iv][2])

研究課題は次の3点である。

* **RQ1**：凸結合は妥当な融合関数か。正規化方法にどの程度依存するか。
* **RQ2**：RRFは本当にパラメータに頑健で、領域間で一般化するか。
* **RQ3**：RRFが元の検索スコア間の距離を捨てることは、性能に悪影響を与えるか。

## 3. 提案手法

### 3.1 正規化スコアの凸結合

語彙検索スコアを (f_{\mathrm{Lex}})、意味検索スコアを (f_{\mathrm{Sem}}) とすると、融合スコアは次のように定義される。

[
f_{\mathrm{CC}}(q,d)
====================

\alpha \phi_{\mathrm{Sem}}(f_{\mathrm{Sem}}(q,d))
+
(1-\alpha)\phi_{\mathrm{Lex}}(f_{\mathrm{Lex}}(q,d))
]

ここで、(\alpha) は意味検索の寄与率、(\phi) は正規化関数である。

論文では、通常のmin-max正規化、理論的min-max正規化、z-score正規化、片方だけの正規化、正規化なしを比較している。

特に著者らが採用する**理論的min-max正規化**は、

[
\phi_{\mathrm{tmm}}(f_o(q,d))
=============================

\frac{f_o(q,d)-\inf f_o}
{\max_{d'} f_o(q,d')-\inf f_o}
]

と定義される。BM25の理論的最小値は0、コサイン類似度の理論的最小値は−1である。この正規化を使った凸結合を、論文では**TM2C2**と呼ぶ。([ar5iv][2])

### 3.2 正規化間の順位同値性

本論文の重要な理論的貢献は、正規化方法が変わっても、凸結合の重み (\alpha) を変換すれば同じ順位を再現できる条件を示したことである。

各正規化が単調な線形変換であり、語彙スコアと意味スコアの相対的な拡大率がクエリ間でおおむね一定なら、ある正規化における (\alpha) に対して、別の正規化でも順位同値となる (\alpha') が存在する。

つまり、min-maxとz-scoreでは最適な (\alpha) の位置は変化するが、生成可能なランキングそのものは大きく変わらない。重要なのは、正規化の細かな種類よりも、**スコアを有界で比較可能な尺度にすること**である。([ar5iv][2])

### 3.3 RRFのパラメータ化

標準的なRRFは、

[
f_{\mathrm{RRF}}(q,d)
=====================

\frac{1}{\eta+\pi_{\mathrm{Lex}}(q,d)}
+
\frac{1}{\eta+\pi_{\mathrm{Sem}}(q,d)}
]

で表される。一般には (\eta=60) が使われる。

しかし著者らは、語彙検索と意味検索に同じ (\eta) を使う必然性はないとして、次のように再定義する。

[
f_{\mathrm{RRF}}(q,d)
=====================

\frac{1}{\eta_{\mathrm{Lex}}+\pi_{\mathrm{Lex}}(q,d)}
+
\frac{1}{\eta_{\mathrm{Sem}}+\pi_{\mathrm{Sem}}(q,d)}
]

この見方では、二つの検索器を統合するRRFには二つのパラメータがある。一方、凸結合には (\alpha) 一つしかない。したがって、RRFを「ノンパラメトリック」とみなすことは適切でないと論じている。([ar5iv][2])

### 3.4 Smooth RRF

RRFはスコアの大小関係を離散的な順位に変換するため、スコアがわずかに逆転しただけでも融合値が不連続に変化する。

この問題を調べるため、著者らは順位計算の指示関数をシグモイドで近似した**Smooth RRF（SRRF）**を導入した。平滑化の強さを制御する (\beta) を小さくすると、元スコアの分布がより保存される。

この実験は、良い融合関数には、入力の小さな変化が出力の急変を引き起こさない**Lipschitz連続性**が必要だという仮説を検証するために行われている。([ar5iv][2])

## 4. 実験方法

### データセット

全9データセットを使用している。

| 分類                      | データセット                                       |
| ----------------------- | -------------------------------------------- |
| In-domain               | MS MARCO Passage、Natural Questions、Quora     |
| Out-of-domain／zero-shot | NFCorpus、HotpotQA、FEVER、SciFact、DBPedia、FiQA |

Natural Questions、MS MARCO、Quoraは、使用した意味検索モデルの学習データに含まれるためin-domainとされている。([ar5iv][2])

### 検索モデル

* **語彙検索**：PISAによるBM25
  (k_1=0.9)、(b=0.4)
* **意味検索**：all-MiniLM-L6-v2
  384次元埋め込み、コサイン類似度
* **ベクトル検索**：FAISSのIndexFlatIPによる厳密検索
* 各検索器から上位1,000件を取得し、その和集合を再ランキング

付録では、BM25、SPLADE、Tas-B、all-MiniLM-L6-v2の異なる組み合わせでも検証している。([ar5iv][2])

### 評価指標

基本的には以下を用いる。

* Recall@1000
* NDCG@1000

データ規模が小さいNFCorpusとSciFactのみ、Recall@100およびNDCG@100を使用する。

有意差検定には、クエリ単位の対応あり両側t検定を使用し、(p<0.01) を有意とする。

主比較では、TM2C2の (\alpha=0.8) と、標準RRFの (\eta=60) を比較する。さらに、RRFのパラメータ調整、SRRF、学習クエリ数を変えたサンプル効率実験を行っている。([ar5iv][2])

## 5. 結果

### 5.1 TM2C2とRRFの比較

NDCGの主結果は次の通りである。

| データセット   | TM2C2 |   RRF |      差 |
| -------- | ----: | ----: | -----: |
| MS MARCO | 0.454 | 0.425 | +0.029 |
| NQ       | 0.542 | 0.514 | +0.028 |
| Quora    | 0.901 | 0.877 | +0.024 |
| NFCorpus | 0.327 | 0.312 | +0.015 |
| HotpotQA | 0.699 | 0.675 | +0.024 |
| FEVER    | 0.744 | 0.721 | +0.023 |
| SciFact  | 0.753 | 0.730 | +0.023 |
| DBPedia  | 0.512 | 0.489 | +0.023 |
| FiQA     | 0.496 | 0.464 | +0.032 |

TM2C2は、全データセットでRRFより高いNDCGを示し、差はいずれも統計的に有意だった。

RecallではRRFがHotpotQAで0.888対0.884、DBPediaで0.567対0.564とわずかに上回ったが、ランキング品質を示すNDCGではTM2C2が一貫して優れている。([ar5iv][2])

### 5.2 RRFのパラメータ感度

RRFの (\eta_{\mathrm{Lex}}) と (\eta_{\mathrm{Sem}}) を変更すると、NDCGは大きく変動した。

In-domainでは意味検索が強いため、語彙検索側の (\eta_{\mathrm{Lex}}) を大きくして語彙検索の寄与を弱める設定が有効だった。例えば ((\eta_{\mathrm{Lex}},\eta_{\mathrm{Sem}})=(10,4)) は、MS MARCOで0.451、Quoraで0.896となり、TM2C2に近づいた。

一方、この設定をout-of-domainに適用すると、HotpotQAは0.621、FEVERは0.649まで低下した。つまり、in-domainで調整したRRFパラメータは、領域外データへ十分に一般化しない。([ar5iv][2])

### 5.3 正規化の影響

min-max、理論的min-max、z-scoreなどの有界な正規化では、性能曲線の最適位置が左右に移動するものの、到達可能な最大性能はおおむね同等だった。

一方、正規化を完全に除くと、BM25と意味検索のスコア尺度の違いによって性能が不安定になった。

したがって結論は、**正規化は必要だが、正規化方式の細かな選択は決定的ではない**というものである。([ar5iv][2])

### 5.4 Smooth RRF

SRRFは、多くのデータセットで通常のRRFを改善した。例えば標準RRF (\eta=60) を平滑化すると、MS MARCOは0.425から0.431、NFCorpusは0.312から0.323、DBPediaは0.489から0.501へ向上した。

HotpotQAでは、SRRFが0.705となり、TM2C2の0.699を上回る設定も確認された。ただし、全データセットではTM2C2の方が安定して高かった。

この結果は、順位だけに離散化せず、元スコアの距離情報をある程度保存することが有効である可能性を示している。([ar5iv][2])

### 5.5 サンプル効率

TM2C2の (\alpha) は、多くのデータセットで**訓練クエリの5%未満**を使った時点でほぼ収束した。

一方、RRFのパラメータはサンプルによる変動が大きく、十分なデータを与えても最終性能はTM2C2を下回る傾向があった。

著者らは、実用上は (\alpha\in[0.6,0.8]) が広いデータセットで有効だったと報告している。([ar5iv][2])

## 6. 考察

### 論文の意義

本研究の最も重要な点は、RRFと凸結合の比較を単なるベンチマークにとどめず、**順位同値性、スコア分布、パラメータ数、連続性**という観点から説明したことである。

特に、次の実務的知見は有用である。

* スコアを取得できるなら、有界に正規化して凸結合する。
* 少量でもラベル付きクエリがあれば (\alpha) を調整する。
* ラベルがない場合でも、0.6～0.8程度を初期値として検討できる。
* RRFの (\eta=60) は普遍的な最適値ではない。
* RRFを使う場合も、検索器ごとの寄与が同じでよいかを検討する必要がある。

一方、RRFが無価値という結論ではない。RRFは、異なるシステムからスコアを取得できない場合、スコア尺度の意味が不明な場合、実装を単純化したい場合には依然として強力なゼロショット基準である。

### 限界

第一に、主実験は二つの検索器の融合に限定されている。三つ以上の検索器や、複数クエリを統合するRAG-Fusion型の設定に同じ結論が成立するかは未検証である。

第二に、主な意味検索モデルはall-MiniLM-L6-v2であり、現在の大規模・多言語埋め込みモデルとは性能特性が異なる可能性がある。ただし、付録で複数の検索モデルの組み合わせを検証している点は評価できる。

第三に、評価の中心がNDCG@1000である。これは候補生成段階の深いランキング評価には妥当だが、検索結果の上位数件だけを利用するRAGやWeb検索では、NDCG@10、MRR、Recall@20などで結論が変わる可能性がある。

第四に、SRRFの改善だけからLipschitz連続性の効果を断定するのは難しい。平滑化は連続性だけでなく、ランキング関数そのものも変更するためである。著者らも、この実験は厳密な因果的証明ではないことを認めている。

第五に、クエリごとに最適な (\alpha) を選ぶOracleと固定 (\alpha) の間には比較的大きな差がある。例えばMS MARCOでは0.454対0.547、NQでは0.542対0.637である。この結果は、固定重みよりも**クエリ適応型の融合**に大きな改善余地があることを示している。([ar5iv][2])

## 7. まとめ

本論文は、ハイブリッド検索における凸結合とRRFを理論的・実験的に比較した。

中心的な結論は次の通りである。

* 語彙検索と意味検索のスコアは相補的であり、線形な凸結合でも高い性能を得られる。
* 正規化は必要だが、有界で単調な変換であれば、その具体的な方式は大きな問題ではない。
* RRFはパラメータに敏感で、in-domainで調整した設定がout-of-domainに一般化しにくい。
* スコアの距離情報を捨てることが、RRFの性能を制限している可能性がある。
* 理論的min-max正規化を用いたTM2C2は、RRFより高性能で、少量の訓練データでも調整できる。

実務的には、**スコアへアクセスできる環境ではTM2C2型の重み付きスコア融合を第一候補とし、スコアを利用できない場合にRRFを使う**という判断が妥当である。

## 8. 総評

### 評価

本論文は、単純だが広く使われている融合関数を丁寧に再検討し、従来の「RRFは無調整で頑健」という理解に重要な修正を加えている。理論解析、複数データセットによる検証、追加モデルでの再現、統計的有意差、サンプル効率の分析がそろっており、研究としての完成度は高い。

一方で、TM2C2自体は複雑な新規アルゴリズムではなく、貢献の中心は分析と設計指針である。また、Lipschitz連続性に関する主張は示唆的ではあるものの、完全に立証されたとは言いにくい。
