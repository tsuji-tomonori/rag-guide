**対象論文:** Kelvin Guu et al., “REALM: Retrieval-Augmented Language Model Pre-Training,” ICML 2020. ([Proceedings of Machine Learning Research][1])

## 1. はじめに

本論文は、言語モデルが世界知識をニューラルネットワークのパラメータ内に暗黙的に記憶する従来方式に対し、外部文書を検索して利用する **REALM（Retrieval-Augmented Language Model）** を提案している。

中心的な着想は、質問応答などの予測を行う前にWikipediaのような大規模コーパスから関連文書を検索し、その文書を入力とともに読み込ませることである。さらに、検索器を固定せず、Masked Language Modelingの予測性能を学習信号として、検索器と回答側のエンコーダを同時に事前学習する。著者らは、数百万件規模の文書を対象とする検索器を、教師なしの言語モデル目的から訓練できることを本研究の主要な貢献としている。([arXiv][2])

## 2. 背景

BERTやT5などの事前学習済み言語モデルは、多量のテキストから事実知識を獲得できる。しかし、その知識はモデルパラメータのどこに保存されているかが分かりにくく、新しい知識を増やすにはモデル規模そのものを大きくする必要がある。また、知識が更新された際に、該当する事実だけを差し替えることも難しい。

一方、オープンドメイン質問応答では、あらかじめ正解を含む文書が与えられない。システム自身が数百万の文書から関連文書を検索し、そこから回答を抽出する必要がある。従来手法ではTF-IDFやBM25などの疎な検索、あるいは固定された検索インデックスを用いることが多く、検索器の能力が言語モデルの事前学習と十分に連動していなかった。REALMは、この検索過程を言語モデル事前学習の内部に組み込むことを狙っている。

## 3. 提案手法

REALMは、入力 (x) に対する出力 (y) の確率を、潜在的な検索文書 (z) を介して次のようにモデル化する。

[
p(y\mid x)=\sum_{z\in Z}p(y\mid x,z)p(z\mid x)
]

ここで、(p(z\mid x)) が検索器、(p(y\mid x,z)) が検索文書を利用して予測する知識拡張エンコーダである。文書自体には正解文書ラベルが与えられず、潜在変数として周辺化される。

**検索器**では、入力と文書をそれぞれBERT系エンコーダでベクトル化し、内積

[
f(x,z)=\operatorname{Embed}*{input}(x)^\top
\operatorname{Embed}*{doc}(z)
]

によって関連度を計算する。全文書に対するsoftmaxが検索分布 (p(z\mid x)) となる。

**知識拡張エンコーダ**では、入力 (x) と検索文書 (z) を連結して別のTransformerに入力する。事前学習時にはマスクされた語を予測し、質問応答へのファインチューニング時には、検索文書中の開始位置と終了位置を推定して回答スパンを抽出する。したがって、この実験での回答は検索文書中に連続した文字列として存在することを前提としている。

全文書について周辺化することは計算不可能なため、上位 (k) 文書だけをMIPS（Maximum Inner Product Search）で検索して近似する。ただし、文書エンコーダを学習すると事前計算済みの文書ベクトルが古くなる。この問題に対し、学習処理と並行して全文書を再エンコードし、数百ステップごとにMIPSインデックスを非同期更新する。検索後の上位文書については、最新パラメータで検索確率と勾配を再計算する。

学習を安定させるため、さらに以下の工夫を導入している。

* 人名・地名・組織名・日付など、外部知識を必要としやすい箇所をマスクする「salient span masking」
* 外部検索が不要な例のための空文書
* 元文書の非マスク版をそのまま取得する自明な検索の禁止
* 検索器をInverse Cloze Taskで、回答エンコーダをBERT-baseで初期化するウォームスタート

これらは単なる補助的な工夫ではなく、後述のアブレーションでは性能に大きく影響している。

## 4. 実験方法

評価には3種類のオープンドメイン質問応答データセットを用いている。

| データセット                |      学習／テスト件数 | 概要                 |
| --------------------- | ------------: | ------------------ |
| NaturalQuestions-Open | 約79,000／4,000 | 実際のGoogle検索クエリ     |
| WebQuestions          |  約3,000／2,000 | Google Suggestから収集 |
| CuratedTREC           |  約1,000／1,000 | 検索サービスへの実ユーザ質問     |

評価指標は、予測文字列がいずれかの参照回答と一致するかを測るExact Matchである。

知識コーパス (Z) には2018年12月20日時点の英語Wikipediaを使用し、各文書を最大288 BERT wordpiecesに分割して、約1,300万件の検索候補を構築した。事前学習コーパス (X) には、Wikipediaを使う設定とCC-Newsを使う設定の2種類がある。

事前学習は20万ステップ、バッチサイズ512、学習率 (3\times10^{-5}) で行われた。主学習に64基のGoogle Cloud TPU、文書埋め込みとインデックス構築に16基のTPUを使用している。事前学習時は空文書を含む上位8文書を周辺化し、質問応答の推論時は上位5文書を利用する。

比較対象には、BERTベースライン、DrQA、HardEM、GraphRetriever、PathRetriever、密検索を用いるORQA、およびモデルパラメータ内に知識を保存して直接回答を生成するT5が含まれる。特にORQAとはファインチューニング条件が共通であり、REALM事前学習の効果を比較しやすい設計になっている。

## 5. 結果

主要なExact Match結果は以下の通りである。

| モデル                 |       NQ | WebQuestions | CuratedTREC | パラメータ数 |
| ------------------- | -------: | -----------: | ----------: | -----: |
| T5-11B              |     34.5 |         37.4 |           ― |  約113億 |
| ORQA                |     33.3 |         36.4 |        30.1 |  約3.3億 |
| REALM：Wikipedia事前学習 |     39.2 |         40.2 |    **46.8** |  約3.3億 |
| REALM：CC-News事前学習   | **40.4** |     **40.7** |        42.9 |  約3.3億 |

REALMはNaturalQuestionsで従来最高のT5-11Bを5.9ポイント、WebQuestionsで3.3ポイント上回った。CuratedTRECでは、ORQAを16.7ポイント上回っている。T5-11Bと比較すると、モデルパラメータ数は約30分の1である。

ORQAとの直接比較では、REALMはNaturalQuestionsで最大7.1ポイント、WebQuestionsで4.3ポイント、CuratedTRECで16.7ポイント改善している。両者は同じ密検索・回答抽出系の構成を持つため、この差は検索器を含むREALM事前学習の有効性を強く示している。

NaturalQuestions開発セットでのアブレーションでは、完全なREALMがExact Match 38.2、事前ファインチューニング前のRecall@5が38.5であった。ORQAはそれぞれ31.3、13.9である。REALMで学習した検索器だけを残した場合は37.4、エンコーダだけを残した場合は35.3であり、検索器とエンコーダの両方が改善に寄与している。

また、salient span maskingをランダムトークンマスクに置き換えると38.2から32.3へ、ランダムスパンマスクでは35.3へ低下した。MIPSインデックスの更新頻度を30分の1にすると28.7まで低下しており、知識を必要とする学習例の選択と、最新の検索インデックスを維持することが重要である。

定性的な例では、「3 is a ___ prime」の正解“Fermat”に対する確率が、BERTでは (1.1\times10^{-14}) だったのに対し、関連文書を検索したREALMでは周辺確率0.129まで上昇した。検索文書が予測に実際に利用されていることを示す例となっている。

## 6. 考察

本研究の最大の意義は、検索器を質問応答データだけで学習するのではなく、通常の教師なしテキストから事前学習した点にある。検索文書を条件にしたとき正解語の予測確率が高くなるなら、その文書の検索スコアを上げるという学習原理は明快であり、潜在文書検索とMasked Language Modelingを自然に結び付けている。

検索結果として実際の文書が提示されるため、純粋なパラメトリック言語モデルよりも回答根拠を追跡しやすい。また、知識コーパスを新しいWikipediaに交換することで、一部の事実予測を更新できることも示されている。ただし、エンコーダ自身も知識をパラメータ内に記憶するため、コーパスを更新しても古い予測が残る場合がある。したがって、外部知識化によって知識更新問題が完全に解決されるわけではない。

一方で、以下の制約がある。

第一に、事前学習には64基のTPUと、インデックス構築用の16基のTPUを必要とし、計算資源と検索インフラの負担が大きい。モデルパラメータ数ではT5-11Bより小さいものの、約1,300万文書の格納領域、検索インデックス、再構築コスト、検索レイテンシはパラメータ数の比較には含まれない。そのため、「30倍小さい」という主張はモデル本体の規模に限定して解釈すべきである。

第二に、性能がマスク方式、初期化、インデックス更新頻度に強く依存している。REALMの基本目的だけで自動的に高品質な検索器が得られるというより、適切な帰納バイアスと大規模な非同期システムが必要である。

第三に、実験対象は英語の短答式・抽出型質問応答に限定され、回答が文書中の連続スパンとして存在することを仮定している。複数文書にまたがる推論、回答生成、長文回答、検索文書に明示されていない推論型の問題については、この実験だけでは有効性を判断できない。

## 7. まとめ

REALMは、言語モデルの知識をモデルパラメータだけに閉じ込めず、大規模な外部文書コーパスから必要な情報を検索する方式を、事前学習・ファインチューニング・推論の全段階に導入した研究である。

特に重要なのは、正解文書の教師ラベルを使わず、マスク語予測を改善した文書を有用な検索結果として学習する点である。非同期MIPS更新やsalient span maskingなどの技術的工夫により、約1,300万件の候補を扱いながら、当時のオープンドメイン質問応答で大幅な性能向上を達成した。([arXiv][2])

## 8. 総評

**査読評価：Strong Accept相当**

新規性は高く、検索拡張型言語モデルを単なる質問応答パイプラインではなく、言語モデルの事前学習方式として定式化した点が特に優れている。潜在変数モデルとしての定式化、密検索、MIPSによる大規模化、非同期インデックス更新まで一貫して設計されており、主要結果だけでなく検索器・エンコーダ・マスク方式・更新頻度に関するアブレーションも説得力がある。

弱点は、膨大な事前学習資源、複雑な検索インフラ、抽出型質問応答に限定された評価、モデル本体と外部メモリを合わせた総コストの不明瞭さである。しかし、これらを考慮しても、技術的新規性と実験結果の強さが上回るため、採択が妥当である。
