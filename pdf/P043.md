# 1. はじめに

**論文名:** *Enabling Large Language Models to Generate Text with Citations*
**著者:** Tianyu Gao, Howard Yen, Jiatong Yu, Danqi Chen

本論文は、大規模言語モデルが回答を生成する際に、各主張を裏付ける文書を引用させ、その性能を自動評価するベンチマーク **ALCE（Automatic LLMs’ Citation Evaluation）** を提案する。中心的な問題意識は、LLMの回答が流暢でも、事実誤認やハルシネーションを含み、利用者が根拠を検証しにくいことである。

ALCEは、単に「引用らしい番号が付いているか」ではなく、回答の正しさ、引用による主張の支持、不要な引用の少なさを分離して評価する。論文は2023年5月に投稿され、改訂版は同年10月に公開され、EMNLP 2023に採択されている。([arXiv][1])

# 2. 背景

従来の検索拡張型生成や生成検索システムには、二つの問題があった。

第一に、検索文書をLLMへ入力しても、モデルがその文書に忠実な回答を生成する保証はない。検索結果とは無関係な知識を混ぜたり、検索文書にない情報を補ったりする可能性がある。

第二に、既存の引用付き回答システムは商用検索エンジンや非公開モデルに依存することが多く、評価も人手中心であった。このため、研究者が同一条件で手法を再現・比較することが難しかった。

ALCEでは、質問、固定された検索コーパス、検索器、生成モデル、引用評価を一体化し、再現可能なエンドツーエンド評価を行う。対象データセットは次の3種類である。

| データセット  | 質問の性質              | 検索コーパス                 | 求められる回答        |
| ------- | ------------------ | ---------------------- | -------------- |
| ASQA    | 複数の解釈を持つ曖昧な事実質問    | Wikipedia、約2,100万パッセージ | 複数の解釈を網羅した長文回答 |
| QAMPARI | 回答が複数存在する列挙型質問     | Wikipedia、約2,100万パッセージ | 正解エンティティのリスト   |
| ELI5    | Why／How／What型の説明質問 | Sphere、約8.99億パッセージ     | 複数文書を統合した説明文   |

各データセットの開発集合から1,000問を抽出する。文書は原則として100語単位のパッセージに分割され、各回答文は最大3件まで引用できる。ASQAとQAMPARIには2018年12月時点のWikipediaスナップショットが使われている。

# 3. 提案手法

## ALCEのタスク定義

質問を (q)、検索コーパスを (D) とする。システムは、複数の文または主張 (s_1,\ldots,s_n) からなる回答を生成し、各主張 (s_i) に対して、それを支持するパッセージ集合 (C_i) を引用する。

ALCEは回答を次の3側面から評価する。

### 流暢性

ASQAとELI5では、生成文と人間の回答との分布的な近さを測る **MAUVE** を用いる。ただし著者らも、MAUVEは文章長や文体に敏感であるため、主として極端に不自然な回答を検出するための健全性確認として扱っている。

### 正解性

データセットごとに異なる指標を採用する。

* **ASQA:** 正解となる短い回答が生成文に含まれる割合であるExact Match Recall
* **QAMPARI:** 正解エンティティのPrecisionとRecall。5個以上の正解を出した場合はRecallを100%とするRecall-5も使用
* **ELI5:** 正解文から`text-davinci-003`で3個の短いサブクレームを生成し、NLIモデルTRUEを用いて、モデル回答が各クレームを含意するかを測るClaim Recall

ELI5でROUGEを主指標にしなかった理由は、検索結果をそのままコピーするだけでも比較的高いROUGEが得られる一方、回答の重要な観点を網羅しているとは限らないためである。

### 引用品質

引用品質は、**引用再現率**と**引用適合率**に分けられる。

引用再現率では、ある回答文に付与されたすべての引用パッセージを連結し、その内容が回答文全体を含意しているかをNLIモデルTRUEで判定する。引用が存在し、かつ引用集合が文全体を支持するとき、その文のスコアを1とする。

引用適合率では、各引用について、その引用を除外しても残りの引用だけで回答文が支持され、かつ除外した引用単独では回答文を支持できない場合、その引用を「無関係」と判定する。したがって、複数の根拠を引用すること自体は罰せず、明らかに不要な引用を検出する設計である。

## 生成方法

ALCEでは、以下の構成を比較する。

**VANILLA**は、検索上位 (k) 件のパッセージをそのままプロンプトに入れ、回答と引用を生成させる最も単純な方法である。

**SUMM／SNIPPET**は、検索文書を要約、または関連部分のみの抜粋に圧縮する。4Kコンテキストでは全文パッセージを約5件しか入れられないため、圧縮によって10件程度を入力することを目的とする。

**INTERACT**では、モデルが必要な文書の全文を確認する`Check`、回答文を生成する`Output`、処理を終了する`End`という操作を交互に実行する。

**INLINESEARCH**では、モデルが回答中に検索クエリを生成し、検索上位100件の中から追加文書を取得する。

**CLOSEDBOOK**では検索文書を与えず、モデル内部の知識だけで回答する。

さらに、4個の候補回答を生成し、引用再現率が最も高い回答を選ぶ **RERANK** と、検索なしで回答した後に各文へ類似パッセージを付ける **POSTCITE** を検証している。

# 4. 実験方法

主要実験では、4Kコンテキストの`gpt-3.5-turbo-0301`を使用する。比較対象として、16K版ChatGPT、8Kコンテキストの`gpt-4-0613`、LLaMA、Vicuna、LLaMA-2-Chatなどを評価する。

Wikipedia検索ではGTRまたはDPR、Sphere検索ではBM25を使用し、最初に上位100パッセージを取得する。そのうち、通常は上位5件、圧縮手法では10件、GPT-4の長文脈実験では最大20件をモデルに入力する。VANILLAでは原則として2個のin-context demonstrationも与える。

多くの実験は異なる乱数シードで3回行い、平均値と標準偏差を報告している。ただし、4候補を生成するRERANK、ChatGPT-16K、GPT-4については計算費用のため1回のみである。

人手評価では、ASQAとELI5から各100問を抽出し、ChatGPT VANILLA、ChatGPT RERANK、Vicuna-13B VANILLAを比較する。評価項目は、回答の有用性、引用集合が回答文全体を支持するか、各引用が完全・部分的・非支持のどれに該当するか、の3項目である。

# 5. 結果

代表的な結果は以下の通りである。数値は百分率で、正解性指標はデータセットごとに定義が異なる。

| データセット・手法                   | 主な正解性指標 | 引用再現率 | 引用適合率 |
| --------------------------- | ------: | ----: | ----: |
| ASQA: ChatGPT VANILLA       |    40.4 |  73.6 |  72.5 |
| ASQA: ChatGPT + RERANK      |    40.2 |  84.8 |  81.6 |
| ASQA: GPT-4、20パッセージ         |    44.4 |  73.0 |  76.5 |
| QAMPARI: ChatGPT VANILLA    |    20.8 |  20.5 |  20.9 |
| QAMPARI: ChatGPT CLOSEDBOOK |    32.9 |  10.0 |  10.0 |
| ELI5: ChatGPT VANILLA       |    12.0 |  51.1 |  50.0 |
| ELI5: ChatGPT + RERANK      |    11.4 |  69.3 |  67.8 |
| ELI5: GPT-4、20パッセージ         |    18.3 |  48.5 |  53.4 |



主要な観察結果は次の通りである。

**単純なVANILLAが強い。** 複雑な検索・対話機構を導入しても、検索上位文書をそのまま入力するVANILLAを安定して上回ることはできなかった。

**要約や抜粋は正解性を改善する場合があるが、引用品質を下げる。** ASQAではSUMMにより正解性が40.4から43.3へ上昇した一方、引用再現率は73.6から68.9、適合率は72.5から61.8へ低下した。圧縮時に、回答生成には役立つが正確な出典対応に必要な情報が失われたと考えられる。

**RERANKは引用品質に有効である。** ASQAでは引用再現率が73.6から84.8へ、ELI5では51.1から69.3へ上昇した。ただし、正解性はほとんど改善していない。RERANKは「正しい回答を作る」より、「生成済み回答の中から引用指標上よいものを選ぶ」手法である。

**検索なしの回答は正解でも引用できない。** CLOSEDBOOKはQAMPARIとELI5でVANILLAより高い正解性を得たが、POSTCITEによる引用品質は大幅に低かった。モデル内部知識から生成された正しい文章と、検索コーパス中の表現が一致せず、事後的に根拠を割り当てにくいことが原因として示されている。

**コンテキスト長だけでは不十分である。** ChatGPT-16Kでは入力パッセージを増やしても改善しなかった一方、GPT-4では5件から20件へ増やすことで正解性が改善した。重要なのは長いコンテキストを持つことだけでなく、複数文書を選別・統合する能力である。

**検索性能と生成性能の間に大きな差がある。** 検索上位に正解根拠が含まれていても、モデルの正解性は検索Recallを大きく下回った。検索器の改善だけでなく、取得した複数文書を利用する生成側がボトルネックになっている。

人手評価では、自動評価と人間評価によるモデル順位がほぼ一致した。Cohenのκ係数は、引用再現率で0.698、引用適合率で0.525であり、人手ラベルに対する自動評価の正解率はそれぞれ85.1%と77.6%だった。

# 6. 考察

本論文の重要な示唆は、**RAGに検索文書を追加しただけでは、根拠付き回答は実現しない**という点にある。システムには、関連文書の検索、必要情報の選択、複数文書の統合、個々の主張と出典の対応付けという別々の能力が必要である。検索Recallが上昇しても生成性能が比例して上がらない結果は、この分解を明確に支持している。

一方、ALCEの引用評価が測るのは主として、**回答が引用文書に含意されているか**である。引用元そのものが誤っている場合でも、回答と引用が整合していれば高得点になり得る。したがって、「引用に忠実であること」と「現実世界で真実であること」は区別する必要がある。ALCEは前者を大きく前進させたが、情報源の信頼性、最新性、複数情報源間の矛盾までは評価していない。

また、ELI5の正解性と引用品質の双方で同じNLIモデルTRUEを使用するため、NLIモデル固有の誤りが複数の指標へ相関して入り込む可能性がある。ELI5のサブクレームも単一の参照回答から3件だけ自動生成されるため、参照回答とは異なるが妥当な説明を過小評価する可能性がある。著者らの検証ではサブクレームの93.3%が適切と判断され、NLI判定精度は80.0%だったが、完全な評価器ではない。

引用適合率にも粒度の問題がある。ひとつの文に複数の原子的主張が含まれる場合、ある引用が文の一部だけを支持していても、自動評価では「無関係」と誤判定され得る。著者らも、NLIモデルが「部分的支持」を認識できず、適合率の偽陽性が多いことを認めている。

RERANKにも注意が必要である。候補選択に用いた自動引用再現率と、最終評価に用いる指標が同じなので、評価指標への過適合が起こり得る。人手評価でも改善が確認されている点は評価できるが、人手評価は2データセット、各100問、3モデルに限定され、QAMPARIでは実施されていない。

そのほか、MAUVEの文章長依存性、Exact Matchによる言い換えの見落とし、数学・コード・マルチホップ推論を含まない点、モデルの重みを更新する学習手法をほぼ扱っていない点が限界となる。これらは論文中でも明示されている。

# 7. まとめ

本論文は、引用付きLLM回答を、流暢性、正解性、引用再現率、引用適合率の複数軸で評価するALCEを提案した。

実験から、単純な検索文書入力が複雑な対話型検索より強いこと、文書圧縮には正解性と引用品質のトレードオフがあること、事後的な引用付与は不十分であること、長いコンテキストを有効利用する能力が重要であることが示された。

最大の貢献は、引用付き生成を主観的なデモンストレーションから、比較可能な研究課題へ変えた点にある。現在のRAGシステムを評価する際にも、回答の正しさと、回答が提示された根拠によって本当に支持されているかを分けて考えるという設計思想は重要である。

# 8. 総評

本論文は、問題設定の重要性、ベンチマークの再現性、評価軸の明確さ、幅広いモデル・検索・プロンプト手法の比較、人手評価による自動指標の検証という点で高く評価できる。特に、正解性と引用品質を分離したことは、検索拡張生成の評価に対する本質的な貢献である。

一方で、NLI評価器への強い依存、部分的支持の扱い、情報源自体の信頼性を評価しない点、RERANKにおける選択指標と評価指標の重複、人手評価規模の限定性は改善の余地がある。これらは結論を無効にするほどではないが、ALCEスコアをそのまま「事実性」と解釈すべきではない。

**模擬査読判定: 採択（Accept、8/10）**

新規性と研究上の有用性が高く、主要な主張は実験と人手評価によって十分に支持されている。評価器の限界を踏まえても、今後の引用付き生成・RAG研究の共通基盤として価値が高い。
