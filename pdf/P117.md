# P117：Term-weighting approaches in automatic text retrieval

## 1. はじめに

本論文は、**Gerard Salton と Christopher Buckley**による語重み付けの比較研究である。1988年に *Information Processing & Management*, 24巻5号, pp.513–523に掲載された。

論文の中心的な主張は、複雑な句・シソーラス・知識表現を導入する前に、**単語単位の索引語へ適切な重みを付けるだけで非常に強い検索性能が得られる**というものである。特定の新しいアルゴリズムを一つ提案するというより、TF、IDF、ベクトル長正規化を体系的に組み合わせ、情報検索手法が最低限上回るべきベースラインを明確化した研究である。([SciSpace][1])

## 2. 背景

ベクトル空間型の情報検索では、文書 (D) とクエリ (Q) を語の重みベクトルとして表現し、その内積やコサイン類似度によって関連度を計算する。

初期の検索方式では、語が存在すれば1、存在しなければ0という二値表現が用いられていた。しかし、すべての語を同等に扱うと、一般的な語と内容を強く特徴づける語を区別できない。そのため、各語の重要度を実数値で表現する必要がある。

当時は、単語共起、複合語・句、シソーラス、辞書、知識ベースなどを利用した複雑な文書表現も研究されていた。ところが著者らは、これらには次の問題があると整理している。

* 厳しい条件で複合表現を抽出すると、抽出数が少なく単語表現との差が小さい。
* 条件を緩くすると、有用な表現と同時に大量のノイズが生成される。
* 語の依存関係や句の効果が、コレクションや文脈によって安定しない。
* シソーラスや知識ベースの構築コストが高い。

そこで本論文は、**単語表現そのものを高度化する語重み付け**に焦点を当てる。([SciSpace][1])

## 3. 提案手法

### 3.1 基本的な考え方

語の重みを、次の3要素に分解する。

| 要素                   | 役割                |
| -------------------- | ----------------- |
| Term Frequency       | 文書・クエリ内での出現頻度     |
| Collection Frequency | コレクション全体における語の希少性 |
| Normalization        | 文書長・ベクトル長の影響の補正   |

著者らは、文書側とクエリ側の重み付けをそれぞれ3文字で記述し、`文書重み.クエリ重み`という形式で表現する。これは一般にSMART表記と呼ばれる。

### 3.2 各成分

**第1文字：語頻度成分**

* `b`：二値重み。語があれば1
* `t`：生の語頻度 (tf)
* `n`：拡張正規化語頻度

[
n(tf)=0.5+0.5\frac{tf}{\max tf}
]

`n`は、語が一度しか出現しない場合でも重みが過度に小さくならないよう、0.5から1.0の範囲に正規化する。

**第2文字：コレクション頻度成分**

* `x`：コレクション頻度を使用しない
* `f`：通常の逆文書頻度

[
idf(t)=\log\frac{N}{n_t}
]

* `p`：確率的逆文書頻度

[
p(t)=\log\frac{N-n_t}{n_t}
]

ここで (N) は全文書数、(n_t) は語 (t) を含む文書数である。

**第3文字：正規化成分**

* `x`：正規化なし
* `c`：コサイン正規化

[
w'_{i}=\frac{w_i}{\sqrt{\sum_j w_j^2}}
]

### 3.3 最良候補となる重み付け

論文が代表的な最良方式として扱うのが、次の組合せである。

[
\texttt{tfc.nfx}
]

文書側の `tfc` は、生のTFにIDFを掛け、ベクトル長で正規化する。クエリ側の `nfx` は、拡張TFにIDFを掛けるが、ベクトル長正規化は行わない。

クエリ正規化を省略できる理由は、一つのクエリについて全候補文書を順位付けする場合、クエリの正規化係数はすべての類似度に共通する定数となり、順位を変えないためである。([SciSpace][1])

## 4. 実験方法

### 4.1 データセット

6種類の情報検索コレクションを使用している。

| コレクション |    文書数 | クエリ数 |
| ------ | -----: | ---: |
| CACM   |  3,204 |   64 |
| CISI   |  1,460 |  112 |
| CRAN   |  1,398 |  225 |
| INSPEC | 12,684 |   84 |
| MED    |  1,033 |   30 |
| NPL    | 11,429 |  100 |

CACM、CISI、CRAN、INSPEC、MEDは通常の自然言語文書または抄録に近い。一方、NPLは元の自然言語テキストではなく、既に索引語ベクトルへ変換された状態で提供されていた。

NPLでは、クエリの平均長が7.16語、標準偏差が2.36と短く均質であり、すべてのクエリ語の出現頻度が1だった。このため、ほかの5コレクションとは性質が大きく異なる。([SciSpace][1])

### 4.2 比較条件

TF成分、コレクション頻度成分、正規化成分を組み合わせ、合計1,800通りの設定を実験した。このうち検索順位が異なる実質的な設定は287通りだった。

代表的な比較方式には、以下が含まれる。

* `tfc.nfx`：完全重み付け方式
* `txc.nfx`：文書側ではIDFを使わない方式
* `tfx.tfx`：正規化なしの古典的TF-IDF
* `nxx.bpx`：最良の確率的方式
* `bfx.bfx`：TFを二値化したIDF方式
* `bxx.bpx`：二値独立確率モデル
* `txc.txx`：TF中心の初期SMART方式
* `bxx.bxx`：一致語数だけを数える方式

### 4.3 評価指標

再現率が0.25、0.50、0.75となる3点で適合率を求め、その平均を各クエリの性能とした。さらに全クエリについて平均している。

加えて、各方式が287方式中で何位だったかを評価する。順位は小さいほど高性能である。([SciSpace][1])

## 5. 結果

### 5.1 通常の自然言語コレクション

NPLを除く5コレクションでは、次の傾向が得られた。

| 方式        | 5コレクションでの平均順位 |
| --------- | ------------: |
| `tfc.nfx` |      **11.2** |
| `txc.nfx` |      **16.4** |
| `tfx.tfx` |          84.4 |
| `nxx.bpx` |          86.2 |
| `bxx.bxx` |           260 |

最良だったのは、文書にTF-IDFとコサイン正規化を適用し、クエリに拡張TFとIDFを適用する` t fc.nfx`である。文書側でIDFを使わない`txc.nfx`も比較的近い性能を示した。

一方、単純なTF-IDFであっても、文書長正規化とクエリ側の拡張重みを用いない` tfx.tfx`は大きく性能が低下した。したがって、この研究の結論は単に「TF-IDFが良い」というものではなく、**TF、IDF、文書長正規化、クエリ重み付けをどの側に適用するかが重要**というものである。

二値ベクトル間の一致語数だけを計算する`bxx.bxx`は、代表方式の中でほぼ最悪だった。([SciSpace][1])

### 5.2 NPLコレクション

NPLでは結果が逆転し、確率的方式`nxx.bpx`が287方式中2位、平均適合率0.2752となった。二値文書表現を使う`bxx.bpx`も8位、平均適合率0.2596だった。

これは確率モデルが一般に優れていたためではない。NPLでは文書とクエリが短く、クエリ語はすべて1回しか出現しないため、TFや文書長正規化が本来の機能を果たさない。著者らは、この例外的結果を通常の自然言語検索へ一般化すべきではないと述べている。([SciSpace][1])

## 6. 考察

### 6.1 TF・IDF・正規化の役割

本論文では、各成分の役割が明確に分離されている。

* TFは、文書内で繰り返される中心的な語を強調し、主として再現率を改善する。
* IDFは、多くの文書に出現する一般語を抑え、識別力の高い語を強調する。
* 文書長正規化は、長い文書が偶然多数のクエリ語に一致する問題を抑える。

特に重要なのは、文書とクエリを同じ方式で重み付けする必要がないという点である。文書は長さの差が大きいため正規化が重要だが、クエリ正規化は順位に影響しない。

### 6.2 コレクション特性への適応

著者らは、万能な単一設定ではなく、入力特性に応じた設定を推奨している。

短いクエリでは各語が重要なので`n`、長いクエリでは語頻度による識別が必要なので`t`が適する。専門語が中心の文書では`n`、語彙が多様な文書では`t`、短い統制語彙ベクトルでは`b`が適する。

文書長のばらつきが大きければ`c`による正規化を使用し、短く長さが均質なら正規化を省略できる。([SciSpace][1])

### 6.3 研究上の限界

現代の評価基準から見ると、以下の限界がある。

第一に、1,800通りの設定を同じコレクション上で探索し、その中の最良方式を報告している。独立した検証集合や交差検証がないため、設定選択による過適合の可能性が残る。

第二に、統計的有意差検定やクエリごとの信頼区間が示されていない。平均値や平均順位の差が、偶然ではないことを直接確認できない。

第三に、評価は再現率3点での適合率に限定されている。現在一般的なMAP、MRR、nDCG、上位 (k) 件のPrecisionなどは使用されていない。

第四に、複合語、句、シソーラス、知識ベースより単語重み付けが優れるという主張は、主として過去研究の総括に基づいている。本論文内で最新の複合表現方式と全面的な再比較を行ったわけではない。

それでも、NPLの異常な結果を単なる失敗として除外せず、文書長や語頻度分布から理由を説明している点は、実験分析として優れている。

## 7. まとめ

本論文が示した中心的な結論は、次のとおりである。

1. 単語単位の表現でも、TF、IDF、長さ正規化を適切に設計すれば強力な検索性能が得られる。
2. 通常の自然言語文書では、文書側の` t fc`または`nfc`、クエリ側の`nfx`、`tfx`、`bfx`が有力なベースラインとなる。
3. 文書とクエリの長さ、語頻度、語彙の統制度によって最適な重みは変化する。
4. 正規化なしの単純TF-IDFを使うだけでは不十分であり、特に文書長正規化とクエリ側の重み設計が重要である。
5. より複雑な検索方式は、まずこれらの強い単語重み付けベースラインを上回る必要がある。([SciSpace][1])

## 8. 総評

本論文の最大の貢献は、TF-IDFを単一の公式として扱うのではなく、**局所頻度、コレクション頻度、長さ正規化という独立した設計要素へ分解し、文書とクエリで別々に検証したこと**である。6種類のコレクションと多数の組合せを用いた比較は当時として包括的であり、結果を実装可能な推奨設定まで落とし込んでいる。

一方、現代的な査読では、統計的検定、独立テストセット、前処理条件の詳細、複合表現方式との直接比較が要求される。特に、多数の設定を探索した同一データ上で「最良方式」を選択している点は、追加検証が必要である。
