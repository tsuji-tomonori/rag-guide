# 論文情報

**題名:** *Reciprocal Rank Fusion outperforms Condorcet and individual Rank Learning Methods*
**著者:** Gordon V. Cormack、Charles L. A. Clarke、Stefan Büttcher
**発表:** SIGIR 2009、pp. 758–759
**概要:** 複数の検索システムが出力した順位を、極めて単純な式で統合する **Reciprocal Rank Fusion（RRF）** を提案した2ページのポスター論文である。([Webis][1])

## 1. はじめに

本論文は、複数の情報検索システムやランキング学習モデルが生成した順位リストを統合し、単独のシステムよりも精度の高い順位を得る手法としてRRFを提案する。

RRFは教師なしの順位統合法であり、学習データや各検索システムのスコア正規化を必要としない。著者らは当初RRFを単純なベースラインとして考案したが、実験ではCondorcet Fuseや個別のランキング学習手法を一貫して上回る結果が得られたとしている。([G. V. Cormack][2])

## 2. 背景

ランキング学習では、文書とクエリの特徴量および関連度ラベルを用いて順位関数を学習する。しかし、十分な関連度ラベルの準備にはコストがかかる。

これに対して順位統合では、複数の検索システムが出力した順位を組み合わせる。代表的な比較対象として、論文では次の手法を扱っている。

**Condorcet Fuse**は、文書の各ペアについて、どちらを上位に置いたシステムが多いかを多数決で決定する。一方、**CombMNZ**は、文書を返したシステム数と各システムの検索スコアの合計を掛け合わせる。

Condorcet Fuseは順位だけを利用できるが、多数の弱い選好が少数の強い選好を打ち消す可能性がある。CombMNZは各システムの検索スコアを使用するため、システム間でスコア尺度が異なる場合に不安定になり得る。([G. V. Cormack][2])

## 3. 提案手法

文書集合を (D)、入力される順位リスト集合を (R)、順位リスト (r) における文書 (d) の順位を (r(d)) とすると、RRFスコアは次式で定義される。

[
\mathrm{RRFscore}(d)
====================

\sum_{r\in R}
\frac{1}{k+r(d)}
]

実験では定数として

[
k=60
]

を使用する。

処理は単純である。

1. 各順位リストについて、文書の順位から (1/(k+r(d))) を計算する。
2. 同じ文書に対する値を全順位リストについて加算する。
3. 合計スコアの降順に文書を並べる。

上位文書ほど大きな値が加算されるが、下位文書の寄与も完全にはゼロにならない。定数 (k) は、ある一つのシステムだけが文書を極端に高く評価した場合、その影響が過度に大きくなることを抑制する。

また、RRFは検索システムが出力する生のスコアを利用せず、順位だけを使う。このため、BM25、言語モデル、ランキング学習モデルなど、互いにスコア尺度が異なるランキングも直接統合できる。([G. V. Cormack][2])

## 4. 実験方法

論文では三段階の実験を行っている。

| 実験          | データ・入力                               | 主な目的                |
| ----------- | ------------------------------------ | ------------------- |
| パイロット実験     | Wumpus Searchの30設定を4種類のTRECコレクションに適用 | (k) の感度確認           |
| TREC提出結果の統合 | TREC Robust、TREC 3、TREC 5、TREC 9     | 実際の異種検索システムの統合性能を検証 |
| LETOR 3実験   | 583,850文書–クエリ対、複数のランキング学習結果          | RRFをメタランキング学習器として評価 |

比較対象は、各実験における最良の個別システム、Condorcet Fuse、CombMNZである。LETOR 3ではListNet、AdaRank-MAP、RankSVM、RankBoost、および著者らのLogistic Gradient Descent法も比較対象に含まれる。

主要評価指標は平均適合率の平均であるMAPである。著者らは、P@k、R-precision、NDCGでも同様の傾向が得られたと述べている。([G. V. Cormack][2])

## 5. 結果

### 5.1 定数 (k) の感度

最初のパイロット実験では、(k=0) から (500) まで変化させている。

* (k=0): MAP 0.2072
* (k=60): MAP 0.2145
* (k=80): MAP 0.2147
* (k=500): MAP 0.2098

最大値は (k=80) の0.2147だが、(k=60) の0.2145との差は極めて小さい。したがって、著者らは (k=60) を固定値として採用した。比較対象では、最良の個別システムが0.2016、Condorcetが0.2074、CombMNZが0.2039であり、RRFが最も高かった。([G. V. Cormack][2])

### 5.2 TREC実験

| コレクション      |        RRF |       最良の個別結果 | Condorcet |    CombMNZ |
| ----------- | ---------: | ------------: | --------: | ---------: |
| TREC Robust | **0.3686** |        0.3586 |    0.3652 |     0.3575 |
| TREC 3      |     0.4350 |        0.4226 |    0.4256 | **0.4381** |
| TREC 5      | **0.3394** |        0.3165 |    0.3213 |     0.3237 |
| TREC 9      | **0.2830** | 0.3519／0.2801 |    0.2750 |     0.2671 |

TREC 9の0.3519は人間を介したシステムの結果であり、自動システムとしての最良値は括弧内の0.2801である。したがって、RRFは自動システムの最良結果をわずかに上回るが、人手を利用したシステムには大きく及ばない。

RRFは全コレクションでCondorcetを上回り、CombMNZに対しては4件中3件で上回った。パイロット実験を含む7件の比較では、RRFはCondorcetに7勝0敗、CombMNZに6勝1敗だった。著者らの符号検定では、それぞれおおむね (p=0.008) と (p=0.04) と報告されている。([G. V. Cormack][2])

### 5.3 LETOR 3実験

| 手法                        |        MAP |
| ------------------------- | ---------: |
| RRF                       |     0.6051 |
| Condorcet                 |     0.5917 |
| CombMNZ                   | **0.6107** |
| ListNet                   |     0.5846 |
| Logistic Gradient Descent |     0.5837 |
| AdaRank-MAP               |     0.5778 |
| RankSVM                   |     0.5737 |
| RankBoost                 |     0.5622 |

RRFは、最良の個別ランキングであるListNetを絶対値で0.0205、相対的には約3.5～4％上回った。すべての個別ランキング手法に対する差は (p<0.003) とされている。

Condorcetに対してもRRFが0.0134高く、(p=0.004) だった。一方、CombMNZはRRFより0.0056高かったが、この差は (p=0.2) で統計的に有意ではなかった。したがって、LETOR 3において「RRFがすべての統合法を上回った」とまではいえないが、個別のランキング学習器より安定して高い性能を示した。([G. V. Cormack][2])

## 6. 考察

### RRFが有効である理由

RRFの強みは、複数のランキングに共通して上位に現れる文書を高く評価しつつ、少数のシステムだけが強く支持する文書も救い上げられる点にある。

Condorcetでは、ある文書を非常に高く評価する少数のシステムが存在しても、多数のシステムによる小さな順位差でその文書が負ける場合がある。RRFでは上位順位の寄与が比較的大きいため、異なる検索モデルが持つ多様性を活用しやすいというのが著者らの説明である。([G. V. Cormack][2])

### 実用上の長所

RRFには次の利点がある。

* 学習処理が不要である。
* システム間のスコア正規化が不要である。
* 実装が非常に単純である。
* 各順位リストを一つずつ処理して加算でき、全順位を同時に保持する必要がない。
* (k) の値に対する性能感度が比較的小さい。

特に、各検索システムが全く異なる意味のスコアを返す場合に、スコアではなく順位だけで統合できる点は重要である。([G. V. Cormack][2])

### 限界

第一に、「教師なし」という表現には注意が必要である。RRFの適用時には学習データを使わないが、推奨値 (k=60) は関連度判定済みのパイロットデータ上のMAPを見て選ばれている。完全に理論的・無調整で決められた値ではない。

第二に、論文は2ページと短く、再現に必要な情報が十分ではない。TRECで使用した具体的なrun ID、CombMNZのスコア処理やカットオフ、順位外文書の扱い、信頼区間の計算単位などが詳細には記述されていない。

第三に、比較対象がCondorcetとCombMNZを中心としており、他の順位統合法との網羅的な比較は行われていない。また、入力ランキングの数、品質、相関、多様性が性能に与える影響も分析されていない。

第四に、論文タイトルの「outperforms」はやや強い表現である。CombMNZはTREC 3でRRFを上回り、LETOR 3でも数値上はRRFより高い。TREC 9では人手を介した個別システムがRRFを大幅に上回っている。したがって、適切な結論は「多くの条件で非常に強く、個別システムやCondorcetを安定して上回った」である。

第五に、7実験に対する符号検定は直感的だが、標本数が小さく、クエリ単位の性能差や効果量を十分に活用していない。統計的評価については、より詳細なクエリ単位検定が望まれる。

## 7. まとめ

本論文は、次式だけで表される単純な順位統合法を提案した。

[
\mathrm{RRFscore}(d)
====================

\sum_{r\in R}
\frac{1}{60+r(d)}
]

RRFは検索スコアの尺度に依存せず、複数のランキングにおける順位だけを使って文書を統合する。TRECおよびLETOR 3の実験では、Condorcetや多くの個別ランキング手法を上回り、複雑な学習ベース手法の強力なベースラインとなり得ることを示した。

最大の貢献は、新しい複雑なモデルを導入したことではなく、「極めて単純な順位ベースのアンサンブルが、複雑なランキング学習器より強い場合がある」ことを明確な実験結果によって示した点にある。([G. V. Cormack][2])

## 8. 総評

### 査読結果

**判定：Weak Accept／ショート・ポスター論文として採録相当**

実際に本論文はSIGIR 2009のポスター論文として掲載されているが、当時の個別査読スコアや査読コメントは公開情報からは確認できない。([Webis][1])

採録を支持する最大の理由は、提案手法が非常に単純であるにもかかわらず、複数のTRECコレクションとLETOR 3で一貫して強い結果を示していることである。手法の新規性は数式自体の複雑さではなく、順位の逆数和という設計が異種検索システムの統合に有効であることを実証した点にある。

一方、フルペーパーとして評価する場合は、実験設定、統計検定、再現手順、比較手法、入力ランキングの多様性分析が不足している。また、「常に上回る」と受け取れる主張は、CombMNZやTREC 9の結果を考慮して弱めるべきである。

総合すると、**理論的には単純だが、実用性と後続研究への影響が非常に大きい、優れたショート論文**である。特に、複雑な新規モデルを提案する研究に対して、まずRRFのような単純で強いアンサンブルを比較対象に置くべきだという重要な教訓を与えている。
