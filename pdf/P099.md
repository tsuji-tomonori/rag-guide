# P099：PubLayNet: Largest Dataset Ever for Document Layout Analysis

## 1. はじめに

本論文は、文書画像中の**本文・見出し・箇条書き・表・図**を検出する文書レイアウト解析のために、大規模データセット **PubLayNet** を構築した研究である。著者は Xu Zhong、Jianbin Tang、Antonio Jimeno Yepesで、ICDAR 2019に採択された。

最大の貢献は、新しい認識モデルそのものではなく、同一論文の**PDF表現とXML表現を照合してアノテーションを自動生成する仕組み**にある。1,162,856件のPubMed Central論文を処理し、最終的に36万ページ超、約357万レイアウト要素を含む、発表当時として最大規模の公開データセットを作成した。([arXiv][1])

---

## 2. 背景

PDFは人間が読むには適しているが、文書の論理構造を明示的には保持しないため、機械処理では「どこが本文か」「どこが表か」といったレイアウト解析が必要になる。

従来の文書レイアウトデータセットは、人手でアノテーションされることが多く、規模は数百ページ程度にとどまっていた。そのため、Faster R-CNNやMask R-CNNを使う場合も、ImageNetやCOCOのような一般画像データで事前学習したモデルを転移する必要があった。しかし、自然画像と文書画像では対象物の形状やテクスチャが大きく異なるため、事前学習データとして必ずしも最適ではない。

一方、PubMed Central Open Accessでは、同じ科学論文がPDFと構造化XMLの両方で提供されている。XMLには、タイトル、段落、図、表、リストなどの意味的構造が記録されているため、著者らはこれをPDF上の座標へ対応付けることで、大量の教師データを自動生成した。([ar5iv][2])

---

## 3. 提案手法

### 対象カテゴリ

PubLayNetでは、XML中の細かな要素を次の5種類へ集約している。

| カテゴリ   | 主な内容                   |
| ------ | ---------------------- |
| Text   | 本文、要旨、著者情報、キャプション、脚注など |
| Title  | 論文タイトル、独立した節・小節タイトル    |
| List   | 箇条書き、番号付きリスト           |
| Table  | 表の本体                   |
| Figure | 図の本体                   |

インラインの節タイトルはTextとして扱い、複数のサブ図からなる図は全体を一つのFigureとして扱う。したがって、ラベルは視覚的に識別しやすい一方、文書の細かな論理構造は省略されている。

### 自動アノテーション処理

処理は概ね次の流れで行われる。

1. **XMLの前処理**
   XML要素を、読み順が明確な要素、読み順が不明確な要素、図、表、リストに分類する。

2. **PDFの解析**
   PDFMinerを利用して、PDFからテキストボックス、テキスト行、画像、直線・矩形などの幾何要素と座標を抽出する。

3. **文字列の照合**
   PDFとXMLでは文字コード、改行、数式表現などが一致しないため、Unicode正規化を行い、レーベンシュタイン距離を用いた曖昧文字列照合でXML要素とPDF上のテキスト行を対応付ける。

4. **図・表領域の推定**
   キャプションと周囲の本文領域から候補領域を作り、その内部にある画像・文字・幾何図形を囲む最小矩形を図または表の領域とする。表については原則として「表本体はキャプションの下に存在する」と仮定している。

5. **ポリゴン生成**
   Text、Title、Listについては、複数のテキスト行の左右端をたどって階段状のポリゴンを生成する。FigureとTableについては矩形領域をセグメンテーションとして再利用する。

6. **品質管理**
   ページ内のPDF要素のうち、アノテーションされた面積の割合を品質指標とする。通常ページでは99%以上、形式のばらつきが大きいタイトルページでは90%以上を採用条件とした。

公式データは、バウンディングボックスとポリゴンセグメンテーションを含み、MS COCOの物体検出形式に準じたJSONで公開された。([GitHub][3])

## 4. 実験方法

データは、未見の論文テンプレートへの一般化を評価するため、**ページ単位ではなくジャーナル単位**で訓練・開発・テストへ分割された。開発・テストセットについては、自動生成後に人手で重大な誤りを除去し、中程度の誤りを修正している。

| 分割  |    ページ数 |  レイアウト要素数 |
| --- | ------: | --------: |
| 訓練  | 340,391 | 3,311,660 |
| 開発  |  11,858 |   127,815 |
| テスト |  11,983 |   131,775 |
| 合計  | 364,232 | 3,571,250 |

実験は三つ行われた。

### 実験1：PubLayNet上のレイアウト検出

Faster R-CNNとMask R-CNNをDetectronで学習した。バックボーンはImageNetで事前学習したResNeXt-101-64×4dで、180,000イテレーション、8 GPU、実効バッチサイズ8で学習している。評価指標はCOCO方式のバウンディングボックスmAP、すなわちIoU 0.50～0.95の平均である。

### 実験2：表検出への転移学習

PubLayNetの表ページで事前学習したモデルを、ICDAR 2013 Table Competitionの訓練ページ170枚でファインチューニングし、正式テストセット238枚で評価した。

### 実験3：異なる文書領域への転移学習

科学論文とはレイアウトが異なる、米国の医療保険に関するSummary Plan Description文書20件、2,131ページを人手でアノテーションした。PubLayNet、COCO、ImageNetによる初期化を、5分割の文書単位交差検証で比較した。

---

## 5. 結果

### PubLayNet上の検出性能

テストセットにおける主なmAPは以下のとおりである。

| カテゴリ   | Faster R-CNN | Mask R-CNN |
| ------ | -----------: | ---------: |
| Text   |        0.913 |      0.917 |
| Title  |        0.812 |      0.828 |
| List   |        0.885 |      0.887 |
| Table  |        0.943 |      0.947 |
| Figure |        0.945 |      0.955 |
| マクロ平均  |        0.900 |  **0.907** |

両モデルともマクロmAPが0.9を超え、Mask R-CNNがわずかに優れていた。表と図は形状が規則的で視覚的特徴が明確なため高精度である。一方、Titleは領域が小さく、本文との境界も曖昧なため最も低い結果となった。

### ICDAR 2013の表検出

PubLayNetで事前学習したFaster R-CNNは、Precision 0.972、Recall 0.964、F1 0.968を達成した。これは当時の最良結果と同等であり、既存手法が約1,600サンプルを使用していたのに対し、本実験では170ページのファインチューニングで同水準に到達した。

### 異分野文書への転移

SPD文書におけるマクロmAPは次のようになった。

| 初期化              | Faster R-CNN | Mask R-CNN |
| ---------------- | -----------: | ---------: |
| PubLayNet・ゼロショット |        0.470 |      0.465 |
| ImageNetから微調整    |        0.584 |      0.602 |
| COCOから微調整        |        0.611 |      0.620 |
| PubLayNetから微調整   |    **0.641** |  **0.663** |

ゼロショット性能は低く、科学論文と保険文書のドメイン差は大きい。しかし、ファインチューニング後はPubLayNet事前学習が総合的に最良だった。例外として、Faster R-CNNの表検出ではCOCO初期化が0.560、PubLayNet初期化が0.541であり、科学論文と保険文書の表形式の違いが転移を妨げたと考察されている。

## 6. 考察

本論文の本質的な価値は、モデルの新規性ではなく、**構造化文書と描画済み文書の冗長性を利用して教師データを大量生成した点**にある。さらに、ジャーナル単位のデータ分割によって、同一テンプレートが訓練・テストへ混入する問題を軽減したことは、評価設計として優れている。

一方、以下の限界がある。

第一に、収録文書は生物医学系の科学論文に偏っている。6,500以上のジャーナルを含むため科学論文内の多様性はあるが、帳票、新聞、契約書、スライド、請求書などへの一般化は保証されない。

第二に、品質指標は「アノテーションで覆われた面積」を測るものであり、クラスラベルや境界が正しいことを直接検証していない。誤ったクラスで領域全体が覆われている場合でも、品質値は高くなり得る。

第三に、図・表の抽出にはキャプション位置などのヒューリスティクスが使われているため、横置きの表、上部以外にキャプションがある図、複雑な複合図などでは系統的な誤差が発生し得る。

第四に、ポリゴンアノテーションを生成してMask R-CNNを学習しているにもかかわらず、主要評価はバウンディングボックスmAPのみであり、マスクAPによるセグメンテーション品質の検証がない。

第五に、異分野転移実験は20文書と小規模であり、アノテータ間一致度、信頼区間、複数シードによる分散も報告されていない。したがって、転移学習の有効性は示されているものの、効果量の安定性には検証の余地がある。

また、PubLayNetは要素間の関係、たとえば「この見出しに属する段落」「この図に対応するキャプション」といった論理構造を含まない。著者自身も、XMLに含まれる関係情報を将来利用する必要性を認めている。

---

## 7. まとめ

PubLayNetは、PDFとXMLを自動照合することで、36万ページ超の文書レイアウトデータを低コストで構築した研究である。Faster R-CNNおよびMask R-CNNで高い文書レイアウト検出性能を示しただけでなく、表検出や異分野文書への転移学習においても、一般画像データによる事前学習より有効な初期値を提供した。

現在の視点ではカテゴリの粗さやドメイン偏りが目立つが、大規模な文書画像事前学習という方向性を明確に示し、後続の文書AI研究に利用可能な基盤を提供した点が重要である。

---

## 8. 総評

**査読判定：Strong Accept（採択）**

データセット論文として、規模、公開性、実験的裏付け、後続研究への有用性が非常に高い。特に、単に大量データを公開しただけでなく、ジャーナル単位分割、厳格な自動品質フィルタ、人手による開発・テストセット修正、複数の転移学習実験まで実施している点を高く評価できる。

弱点は、科学論文へのドメイン偏り、自動アノテーションの正確性を直接測る人的評価の不足、クラス体系の粗さ、セグメンテーション評価の欠如である。しかし、これらは研究の中心的貢献を覆すものではなく、ICDAR 2019時点ではコミュニティへのインパクトが極めて大きい。

実際に本論文はICDAR 2019に採択され、公式リポジトリではBest Paper Award受賞が報告されている。([GitHub][3])
