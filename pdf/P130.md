# 1. はじめに

本論文は、Andrei Z. Broderが1997年に発表した、**大規模文書集合における重複・近似重複検出**の基礎的研究である。文書同士の関係を、ほぼ同一であることを表す「類似度（resemblance）」と、一方が他方に含まれることを表す「包含度（containment）」として定式化し、小さな文書スケッチだけから推定する方法を提案している。現在MinHash系手法として理解されている考え方の原型に当たる。論文は *Compression and Complexity of SEQUENCES 1997* の会議録に掲載された。([IEEE Xplore][1])

# 2. 背景

Webの急速な拡大により、完全に同一、あるいは一部分だけが異なる文書が大量に発生した。検索エンジンでは、これらを検出して検索結果の重複を除去したり、複製ページをクラスタリングしたりする必要がある。

しかし、Hamming距離やLevenshtein距離のような通常の文字列距離には、次の問題がある。

* 文字列の挿入や削除によって、その後の位置がずれると距離が大きくなる
* 書式や広告部分の差にも敏感である
* 全文書対を比較すると、文書数に対して二乗規模の計算が必要になる
* 比較時に全文書を保持・走査する必要がある

そこで本論文は、各文書を独立に処理して小さなスケッチを生成し、以後はスケッチだけで文書間の関係を推定する問題設定を採用した。

# 3. 提案手法

## 文書のシングル化

まず文書を正規化し、単語・文字・行などのトークン列に変換する。句読点、大文字・小文字、書式命令など、比較上無視したい情報は正規化段階で除去する。

連続する (w) 個のトークン列を **(w)-shingle** と呼び、文書 (D) から得られる集合を (S(D,w)) とする。

シングルの扱いには二つの選択肢がある。

* **Option A**：同じシングルの第1出現、第2出現などを区別し、出現頻度を保存する
* **Option B**：重複したシングルを一つにまとめ、集合として扱う

Option Aは情報量が多い一方で処理が重く、Option Bは頻度情報を失うものの実装効率が高い。論文ではOption Bの方が実用上高速であると述べている。

## 類似度と包含度

文書 (A,B) の類似度を次のように定義する。

[
r_w(A,B)=
\frac{|S(A,w)\cap S(B,w)|}
{|S(A,w)\cup S(B,w)|}
]

これはシングル集合に対するJaccard類似度である。

一方、文書 (A) が文書 (B) にどの程度含まれているかを、

[
c_w(A,B)=
\frac{|S(A,w)\cap S(B,w)|}
{|S(A,w)|}
]

と定義する。類似度は対称だが、包含度は一般に

[
c_w(A,B)\neq c_w(B,A)
]

である。また、

[
d_w(A,B)=1-r_w(A,B)
]

は距離関数、すなわち三角不等式を満たすため、クラスタリングに利用できる。ただし、この性質の証明は本論文の予備版では省略されている。

## ランダムサンプリングによる推定

全シングル集合を保存する代わりに、シングルにランダムな順序を与え、値が小さいものだけを保存する。

文書 (D) に対して、ランダム置換後の最小 (s) 個を

[
M(D)=\operatorname{MIN}_s(\pi(S(D,w)))
]

として保存する。文書 (A,B) の和集合における最小 (s) 個のうち、両方の文書に存在する要素の割合を求めると、類似度 (r_w(A,B)) の**不偏推定量**になる。

この方式の重要な点は、文書長にかかわらずスケッチを一定サイズにできることである。

包含度については、ハッシュ値が (0 \bmod m) となるシングルを保存する可変サイズサンプル (L(D)) を用い、

[
\hat c(A,B)=\frac{|L(A)\cap L(B)|}{|L(A)|}
]

として推定する。ただし、短い文書が非常に長い文書に含まれるかを調べる場合、短い文書側のサンプルが少なくなり、推定分散が大きくなる。

## Rabinフィンガープリント

数十バイトのシングルそのものを保存する代わりに、各シングルを短いビット列へ写像するため、Rabinフィンガープリントを使用する。

Rabinフィンガープリントは多項式演算に基づき、衝突確率を理論的に評価できる。また、隣接するシングルは一つのトークンだけが入れ替わるため、ローリングハッシュとして更新でき、各シングルを最初から計算する場合の (O(wn)) から、実質的に文書長 (n) に比例する処理へ高速化できる。

# 4. 実験方法

本論文には、現代的な意味で独立した詳細な実験セクションはない。理論解析に加え、AltaVista検索エンジンが収集したWeb文書に対する大規模クラスタリング事例が簡潔に報告されている。

使用されたデータと条件は以下である。

* 文書数：3,000万件以上
* 総データ量：150GB以上
* クラスタ形成の類似度閾値：50%
* 典型的な文書のシングル集合：約1,000要素
* 1シングルの長さ：約400ビット
* Rabinフィンガープリント長：40ビット

また、サンプル数と推定精度について、超幾何分布を二項分布で近似して解析している。論文では、実用上は100サンプル程度が妥当で、200サンプルあれば十分大きいと評価している。

一方で、実験で使用した正確なシングル幅 (w)、スケッチサイズ (s)、前処理条件、クラスタの正解ラベル、比較手法などは十分に記載されていない。詳細は別論文に委ねられている。

# 5. 結果

3,000万件以上の文書を50%以上の類似度でクラスタリングした結果、次の規模のクラスタが得られた。

| 分類                 | クラスタ数 |    文書数 |
| ------------------ | ----: | -----: |
| 全クラスタ              |  360万 | 1,230万 |
| 完全に同一な文書だけのクラスタ    |  210万 |   530万 |
| 完全重複と近似重複が混在するクラスタ |  150万 |   700万 |

この結果は、大規模Web集合に完全重複だけでなく、大量の近似重複文書が存在することを示している。

理論・実装面では、次の結果が示されている。

* 類似度と包含度をスケッチから不偏推定できる
* 類似度用スケッチは文書長に依存しない固定サイズにできる
* 100～200サンプルの場合、スケッチはおおむね300～800バイトになる
* 2文書の比較時間はスケッチサイズを (s) として (O(s))
* 特定の疎性仮定の下では、(r) 文書の貪欲クラスタリングを (O(rs)) で実装できる

論文の確率計算では、真の類似度が50%未満の文書対を90%以上と誤推定する確率は0.1%未満とされている。ただし、これは提示された分布近似に基づく理論値であり、実データ上の精度評価とは区別する必要がある。

# 6. 考察

本論文の最大の貢献は、曖昧だった「ほぼ同じ文書」という概念を、シングル集合の交差問題として単純かつ計算可能な形に落とし込んだ点にある。

特に、文書ごとに独立してスケッチを作成できるため、新しい文書の追加時に既存文書を再処理する必要がない。全文を保持せずに比較できることから、分散処理、検索エンジン、Webクローリング、重複除去に適している。また、定義・確率的推定・Rabinフィンガープリント・データ構造までを一つの処理系として接続した点も優れている。

一方、以下の制約がある。

**意味的類似性は扱えない。**
同じ内容を別の表現で書き換えた文書は、シングルが一致しないため類似度が低くなる。本手法が測るのは意味ではなく、局所的な文字列・トークン列の一致である。

**シングル幅への依存が大きい。**
(w) が小さいと語順変更や偶然の一致に弱く、(w) が大きいと一語の変更が複数シングルへ波及する。適切な (w) は文書種別や目的によって異なる。

**前処理が結果を左右する。**
HTMLタグ、句読点、数字、ナビゲーション、広告などをどこまで除去するかによって類似度が大きく変わる。

**包含度推定は文書長の差に弱い。**
非常に短い文書と長い文書を比較すると、短い文書側のサンプル不足により推定が不安定になる。

**実験的検証が不十分である。**
精度・再現率、誤検出例、ベースライン比較、パラメータ感度、実行時間、メモリ使用量が体系的に報告されていない。また、貪欲クラスタリングは処理順序や代表スケッチの選び方に依存するが、その影響も検証されていない。

# 7. まとめ

本論文は、文書をシングル集合へ変換し、集合のJaccard類似度と非対称な包含率によって文書関係を定義した。さらに、ランダム置換後の小さなサンプルだけを保存することで、全文を再度参照せずに類似度を推定できることを示した。

提案の本質は、

[
\text{文書比較}
\rightarrow
\text{シングル集合の比較}
\rightarrow
\text{ランダムな小規模スケッチの比較}
]

という変換にある。大規模データに対して、精度と記憶容量のトレードオフを制御可能にした点が、本論文の中心的成果である。

# 8. 総評

**査読判定：Weak Accept（採択）**

新規性、理論的簡潔性、実装可能性、実際のWeb規模への適用という観点から、採択に値する研究である。特に、固定長スケッチによってJaccard類似度を推定する発想は汎用性が高く、文書以外の集合データにも適用できる。

ただし、査読上は次の修正を要求したい。

* 距離関数の三角不等式に関する完全な証明
* 実験で使用した (w)、(s)、正規化処理の明記
* 正解ラベルを用いた適合率・再現率評価
* 編集距離や既存コピー検出法との比較
* クラスタリングの順序依存性と誤結合の分析
* 包含度推定が不安定になる条件の定量評価
