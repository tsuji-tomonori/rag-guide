# P149：Fairness and Abstraction in Sociotechnical Systems

**著者**：Andrew D. Selbst、danah boyd、Sorelle A. Friedler、Suresh Venkatasubramanian、Janet Vertesi
**掲載**：FAT* 2019, pp. 59–68
**主題**：公平性を機械学習モデル単体の性質として扱うのではなく、人間・組織・制度・法律を含む社会技術システム全体の性質として捉えるべきだと論じた概念論文である。([ACMデジタルライブラリ][1])

## 1. はじめに

公平性を考慮した機械学習、すなわちFair-MLでは、予測精度と同様に公平性を数理的な指標として定義し、モデルの入力・出力や誤り率を調整する研究が進められてきた。

本論文の中心的な批判は、こうした研究が機械学習モデルをブラックボックスとして切り出し、その外側にある社会的・制度的文脈を抽象化しすぎている点にある。精度や再現率はモデルの性質として評価できるが、公平性、正義、適正手続は、雇用制度や刑事司法制度を含むシステム全体について評価すべき性質である。

したがって、技術的なサブシステムだけを見て「公平なアルゴリズム」と呼ぶことは、著者らの表現では**抽象化の誤り**に当たる。論文はこの誤りを五つの「罠」として整理し、Science and Technology Studies、以下STSの知見を用いて回避策を提示する。([Sorelle][2])

## 2. 背景

コンピュータ科学における抽象化は、複雑な対象を入力・出力・内部処理に分け、再利用可能なモジュールとして扱うための重要な方法である。しかし、社会的価値を扱う場合、どこまでをシステムの内側とみなすかによって、評価結果が大きく変わる。

本論文では、分析対象の境界を次のように段階的に捉えている。

| フレーム        | 含まれる対象                     | 問題点                        |
| ----------- | -------------------------- | -------------------------- |
| アルゴリズム・フレーム | モデル、入力、出力                  | 社会的文脈をほぼ考慮しない              |
| データ・フレーム    | モデル、データ表現、ラベル、属性           | データ上の偏りは扱うが、制度や人間の判断は外部に残る |
| 社会技術フレーム    | モデル、データ、人間、組織、法律、権力関係、運用過程 | 公平性をシステム全体の結果として評価できる      |

Fair-MLは従来の機械学習より広くデータ生成過程を扱うものの、多くの場合はデータ・フレームにとどまっているというのが著者らの診断である。([Sorelle][2])

論文で繰り返し用いられる事例は、刑事司法における公判前リスク評価である。モデルが予測するのは「被告人が出廷しない可能性」などだが、社会的に重要な最終結果は、裁判官が釈放、保釈、勾留のどれを選ぶかである。裁判官がモデルの勧告を無視したり、特定の事案だけ変更したりすれば、モデル単体で保証された公平性は、最終判断には引き継がれない。([Sorelle][2])

## 3. 提案手法

本論文が提案するのは新しい学習アルゴリズムではなく、Fair-MLの研究・設計・導入を評価するための**五つの抽象化の罠と、その回避プロセス**である。

| 罠                             | 内容                                              | STSに基づく対応策                                                          |
| ----------------------------- | ----------------------------------------------- | ------------------------------------------------------------------- |
| **Framing Trap：フレーミングの罠**     | 公平性が実現されるシステム全体ではなく、モデルやデータだけを評価対象にする           | 「異種混成的エンジニアリング」により、人間、組織、規制、インセンティブも抽象化境界の内側に置く。([Sorelle][2])      |
| **Portability Trap：可搬性の罠**    | ある社会的文脈向けの公平性手法を、別の組織・地域・用途にもそのまま移植できると考える      | 技術に埋め込まれた利用者の「スクリプト」、前提条件、想定用途、適用限界を明記し、導入先ごとに再検討する。([Sorelle][2])  |
| **Formalism Trap：形式主義の罠**     | 公平性を単一の数式や制約条件に還元し、手続性、文脈依存性、政治的争点性を失う          | SCOTの「解釈の柔軟性」「関連社会集団」「閉鎖」の概念を用い、誰の公平性を形式化しているのかを明示する。([Sorelle][2]) |
| **Ripple Effect Trap：波及効果の罠** | 技術を既存制度へ導入すると、人間の行動、組織内の権力、評価対象そのものが変化することを無視する | 「強化の政治」や「反応性」を予測し、試験導入、継続監視、シナリオ分析を行う。([Sorelle][2])                |
| **Solutionism Trap：技術解決主義の罠** | 問題には技術的解決策が必要であり、アルゴリズムを作ること自体が前提だと考える          | 非技術的政策や現行の人間中心プロセスと比較し、「作らない」「導入しない」という選択肢を残す。([Sorelle][2])        |

著者らは、実際の設計時には五つの罠を論文中とは逆の順序で検討することを勧めている。

**技術を使うべきか → 導入によって社会がどう変わるか → 公平性を十分に表現できるか → 現地の文脈に適合するか → 人間と制度を含む適切なシステム境界になっているか**

どこかの段階で条件を満たせない場合、開発を中止するか、その限界を明示すべきだとされる。([Sorelle][2])

## 4. 実験方法

本論文は概念・理論研究であり、通常の意味での機械学習実験は行っていない。

データセット、学習モデル、ベースライン、評価指標、統計検定は使用されていない。代わりに、次の方法で議論を構築している。

1. Fair-ML、法学、社会学、STS、HCIにまたがる先行研究の統合
2. 公平性の数理的形式化に対する概念分析
3. 公判前リスク評価を中心とする事例分析
4. 五つの失敗類型とSTS理論との対応付け
5. リスク評価システムの設計過程を想定した適用例の提示

したがって、本論文でいう「方法」は実験プロトコルではなく、学際的文献に基づく批判的分析と設計フレームワークの構築である。([Sorelle][2])

## 5. 結果

本論文の成果は、数値的な性能向上ではなく、Fair-MLの問題設定を再構成した点にある。

第一に、公平性はモデルの局所的な性質ではなく、**人間を含む意思決定過程のエンドツーエンドな性質**であると整理された。裁判官や採用担当者の判断をモデル外部に置いたままでは、社会的結果に関する公平性保証は成立しない。

第二に、「公平な分類器」や「公平性モジュール」を別の用途に移植できるという考えが否定された。公平性手法は、その用途、地域、法律、組織、利用者の行動を前提としており、文脈から切り離すと意味が変わる。

第三に、数理的公平性指標の選択自体が規範的・政治的判断であることが示された。偽陽性率と偽陰性率のどちらを重視するかは、数学だけでは決められず、その判断によって不利益を受ける人々を含めた社会的検討が必要である。

第四に、技術は中立的に既存制度へ追加されるのではなく、制度を変化させる介入であるとされた。導入後には、権限の再配分、指標への迎合、入力情報の操作、目的の変質などが起こり得る。

最終的に、Fair-ML研究者が提示すべきものは、単一の「公平な解」よりも、技術利用の是非を含めて検討する設計・監視・異議申立てのプロセスであると結論付けられている。([Sorelle][2])

## 6. 考察

本論文の最大の意義は、従来のFair-MLが暗黙に置いてきた「公平性をモデルの制約条件として実装できる」という前提を問い直した点にある。特定の公平性指標を満たすことは重要であっても、それだけで制度全体が公平になるわけではない。

特に重要なのは、社会的要素を単なる外部環境や事後的な説明変数ではなく、システム仕様の一部として扱う点である。裁判官、被告人、弁護士、行政機関、企業、規制機関、影響を受ける集団が、モデルと同じシステム内の構成要素になる。

一方、以下の限界がある。

**実証的検証がない。** 五つの罠は説得的だが、この分類によって実際に害が減るか、公平性が改善するかは比較実験やフィールド調査では確認されていない。

**十分なシステム境界の基準が曖昧である。** 著者らは少なくとも一つの技術要素と一つの社会要素を含めることを示唆するが、どの関係者や制度まで含めれば十分なのかを決定する手続は具体化されていない。

**利害対立の解決方法が不足している。** 多様な関連社会集団を参加させても、公平性に関する価値観が一致するとは限らない。誰が最終決定権を持つか、参加の正当性をどう確保するかは別途設計する必要がある。

**波及効果の予測は本質的に困難である。** 試験導入や監視は有効だが、政治的変化、戦略的行動、制度的権力の変化を事前に完全にモデル化することはできない。

**事例の偏りがある。** 議論の多くが米国の刑事司法と雇用制度に依存しているため、異なる法制度や文化圏への適用には追加検証が必要である。

ただし、これらの限界は中心命題を否定するものではない。むしろ、社会的複雑性を完全にモデル化できないという事実を認めたうえで、限界を文書化し、導入後も再評価を続けるべきだという論文の主張を補強している。([Sorelle][2])

## 7. まとめ

本論文は、「どの公平性指標を採用するか」という問題より前に、次の問いを検討すべきだと主張する。

**何をシステムとして扱うのか、誰の価値を公平性として採用するのか、技術導入によって人々の行動や制度がどう変わるのか、そしてそもそも技術を導入すべきなのか。**

公平性はアルゴリズムに付与できる固定的なラベルではない。公平な結果を目指すには、モデル、データ、人間、組織、法制度、権力関係、異議申立ての仕組みを含む社会技術システム全体を設計・評価しなければならない。

本論文のメッセージは、**「公平なアルゴリズムを作る」ことから、「公平性を継続的に検討できる制度とプロセスを作る」ことへの転換**と要約できる。([Sorelle][2])

### 査読結果

**独自査読判定：採択、Strong Accept相当**

公平性研究の問題を、個々の指標やアルゴリズムではなく、Fair-MLの研究方法そのものの問題として整理した点が非常に優れている。五つの罠は概念的に明快であり、異分野間で議論するための共通語彙としても有用である。特に、「技術を作らないことも正当な設計結果である」という指摘は、技術解決主義に陥りやすい研究分野への重要な介入となっている。

弱点は、提案フレームワークの実証評価と運用手順が不足していることである。採択後の発展課題として、異なる制度・地域における比較事例、ステークホルダー参加の具体的手続、導入後評価の指標を提示することが望ましい。
