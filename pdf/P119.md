# P119 論文要約

**論文:** *Unsupervised Domain Adaptation for Sparse Retrieval by Filling Vocabulary and Word Frequency Gaps*
**著者:** Hiroki Iida, Naoaki Okazaki
**採録:** AACL-IJCNLP 2022 Long Paper, pp. 752–765 ([ACL Anthology][1])

## 1. はじめに

本論文は、SPLADEに代表されるニューラル疎検索モデルを、教師ラベルのない専門分野へ適応させる問題を扱う。

SPLADEは、事前学習済み言語モデルを用いてクエリと文書を語彙次元の疎ベクトルへ変換し、入力に明示されていない関連語も展開できる。一方、学習元のMS MARCOにほとんど出現しない専門用語については完全一致を軽視しやすく、対象分野との語彙・単語頻度の差が大きいほど性能が低下する。

そこで著者らは、対象分野のラベルなしコーパスだけを利用し、語彙ギャップと単語頻度ギャップを同時に補う **CAI（Combination of AdaLM and IDF）** を提案する。中心的な発想は、言語モデル側で専門語彙を学習させることと、検索時に希少語の重みをIDFで補正することを組み合わせる点にある。([ACL Anthology][2])

## 2. 背景

BM25などの語彙検索は、クエリ語と文書語の完全一致に強く、専門用語や固有名詞を適切に評価できる。しかし、同義語や言い換えによる語彙不一致には弱い。

SPLADEは、Masked Language Modelの出力を用いて各テキストを語彙サイズの疎ベクトルに変換する。元のテキストに存在しない語にも非ゼロ重みを与えられるため、ニューラルなクエリ・文書展開が可能であり、転置インデックスによる高速検索も維持できる。([ACL Anthology][2])

ただし、SPLADEには以下の問題がある。

* MS MARCOで低頻度だった語の完全一致重みが小さくなりやすい。
* 医療・科学分野では、一般分野と語彙集合や単語頻度分布が大きく異なる。
* 対象分野ごとに大量の検索教師データを作ることは高コストである。

既存の教師なしドメイン適応手法GPLは、対象文書から疑似クエリを生成して検索モデルを再学習する。しかし著者らは、疑似クエリ学習だけでは、疎検索における希少語の完全一致問題を十分に改善できないと指摘する。([ACL Anthology][2])

## 3. 提案手法

CAIは、次の三段階から構成される。

### 3.1 AdaLMによる語彙拡張

まず、対象分野のラベルなしコーパスからWordPieceトークナイザを作り、BERTの既存語彙に専門語を追加する。

語彙サイズを3,000語ずつ増やし、対象コーパスで頻度の高い新語を順番に採用する。数字や記号だけから構成されるトークンは除外する。新語の埋め込みは、その語を元のトークナイザで分割したサブワード埋め込みの平均によって初期化される。最終語彙数は、医療分野で71,694語、科学分野で62,783語となった。([ACL Anthology][2])

語彙拡張には二つの狙いがある。

第一に、「phytate」のような専門語を単一または少数の意味のあるトークンとして扱えるようにする。第二に、SPLADEの疎ベクトル次元そのものを拡張し、専門語に対応する明示的な検索次元を追加する。

### 3.2 対象分野での継続事前学習

語彙拡張後の言語モデルを、対象分野コーパス上でMasked Language Modelingにより継続事前学習する。入力トークンの15%をマスクし、元のトークンを予測させる。

この処理により、SPLADEが専門分野に適した語を文書・クエリ展開先として選びやすくなる。その後、このドメイン適応済みモデルを初期値として、MS MARCOの教師データでSPLADEを学習する。([ACL Anthology][2])

### 3.3 IDFによる文書ベクトル補正

SPLADEで生成した文書の疎ベクトルを、対象検索コーパスから計算したIDFで重み付けする。

[
w_t^{\mathrm{IDF}}=
\begin{cases}
\log \frac{N}{N_t} & N_t>0\
1 & N_t=0
\end{cases}
]

ここで、(N) は対象コーパスの全文書数、(N_t) はトークン (t) を含む文書数である。文書ベクトル (s_D) は、

[
\hat{s}_D=w^{\mathrm{IDF}}\odot s_D
]

と補正される。IDF補正は文書側だけに適用される。これにより、対象分野で文書頻度の低い専門語の完全一致が強調される。([ACL Anthology][2])

### 3.4 BM25とのハイブリッド

最終的なハイブリッド方式では、SPLADEとBM25のスコアを単純加算する。

[
S_{\mathrm{Hybrid}}(Q,D)
=S_{\mathrm{BM25}}(Q,D)+S_{\mathrm{SPLADE}}(Q,D)
]

ニューラルな意味的展開と、BM25の語彙的一致を相補的に利用する構成である。([ACL Anthology][2])

## 4. 実験方法

教師あり学習のソースドメインにはMS MARCOを使用し、対象ドメインの検索ラベルはドメイン適応に用いないゼロショット設定とした。

| 項目       | 内容                                                                |
| -------- | ----------------------------------------------------------------- |
| 医療データセット | BioASQ/B-ASK、NFCorpus、TREC-COVID                                  |
| 科学データセット | SCIDOCS、SciFact                                                   |
| 評価指標     | nDCG@10                                                           |
| 医療適応コーパス | PubMed抄録、約17 GB                                                   |
| 科学適応コーパス | S2ORC抄録、約7.3 GB                                                   |
| 主な比較対象   | BM25、docT5query、Dense Retrieval、Cross Encoder、LaPraDoR、GPL、SPLADE |
| 初期言語モデル  | BERT-base-uncased                                                 |

対象の5データセットは、BEIR内でMS MARCOとの重み付きJaccard類似度が特に低く、語彙・頻度分布の差が大きいデータセットとして選ばれている。([ACL Anthology][2])

SPLADEはMargin-MSEとFLOPS正則化で学習された。継続事前学習は8基のNVIDIA A100 40 GBを用いて1エポック、検索モデルの学習は1基のA100で実施された。GPLでは各文書から3件の疑似クエリを生成し、計算量を抑えるため対象コーパスを最大100万文書に制限している。([ACL Anthology][2])

## 5. 結果

主要な平均nDCG@10は次のとおりである。

| モデル                   | 平均nDCG@10 |
| --------------------- | --------: |
| SPLADE                |     0.462 |
| **SPLADE＋CAI**        | **0.497** |
| Cross Encoder         |     0.497 |
| SPLADE-Doc            |     0.435 |
| **SPLADE-Doc＋CAI**    | **0.480** |
| LaPraDoR              |     0.504 |
| **Hybrid-SPLADE＋CAI** | **0.514** |

SPLADE＋CAIは、元のSPLADEを5データセットすべてで上回った。平均では0.462から0.497へ約7.6%の相対改善であり、Cross Encoderと同値になった。SPLADE-Docでは0.435から0.480へ約10.3%改善しており、クエリ側を単純なBag-of-Words表現にした場合にも効果が大きい。BM25と組み合わせたHybrid-SPLADE＋CAIは平均0.514で、比較モデル中の最高値を記録した。([ACL Anthology][2])

### GPLとの比較

| 基盤モデル           | Original |   GPL |       CAI |
| --------------- | -------: | ----: | --------: |
| Dense Retrieval |    0.422 | 0.457 | **0.459** |
| SPLADE          |    0.462 | 0.477 | **0.497** |
| SPLADE-Doc      |    0.435 | 0.432 | **0.480** |

Dense RetrievalではCAIとGPLの差は小さいが、SPLADEではCAIがGPLを0.020上回った。SPLADE-DocではGPLが元モデルより悪化したのに対し、CAIは0.048改善している。したがって、CAIは特に疎表現と語彙的一致を利用する検索方式に適している。([ACL Anthology][2])

### アブレーション

AdaLM部分の平均nDCG@10は、元のSPLADEが0.462、継続事前学習のみが0.476、語彙拡張のみが0.462、語彙拡張と継続事前学習を組み合わせたAdaLMが0.491だった。

語彙を増やすだけでは改善せず、対象分野での継続事前学習と組み合わせることで初めて明確な効果が得られた。これは、単に疎ベクトルの次元数を増やすことよりも、追加語彙を意味的に学習することが重要であることを示す。([ACL Anthology][2])

### 希少語分析

NFCorpus上のRSJ重み分析では、CAIは「検索上重要かつIDFが高い語」の推定重みを理想値に近づけた。一方、GPLはこの種の希少語に対する重み付け問題を悪化させる場合があった。([ACL Anthology][2])

事例分析では、“Phytates for the Treatment of Cancer”というクエリに対し、CAI版は希少語“phytate”を含む正解文書を1位に検索した。通常のSPLADEは“phytate”を含まない不正解文書を1位とした。“phytate”はMS MARCO内の文書頻度で下位2%に属する語であり、CAIが学習元で希少な語の完全一致を回復した例となっている。([ACL Anthology][2])

## 6. 考察

本手法の最大の長所は、ドメインシフトを抽象的な表現分布の差としてだけでなく、**語彙集合の差**と**単語頻度の差**に分解し、それぞれに対応する処理を設計した点にある。

AdaLMは、専門語を疎ベクトル上の独立した検索次元として導入し、継続事前学習によってその語の意味や共起関係を学習する。IDF補正は、ニューラルモデルが過小評価しやすい希少語の完全一致を検索時に復元する。この二つは、意味的展開と語彙的一致という異なる役割を持つため、相補性が高い。

また、改善幅がDense RetrievalよりSPLADEおよびSPLADE-Docで大きかったことは、提案手法の目的と実験結果が整合している。単に総合性能が上がっただけでなく、希少語の重み分析と検索事例によって改善機構を説明している点も評価できる。

一方、次の制約がある。

**評価範囲の限定:** 実験対象は英語の医療・科学分野に限定され、さらにMS MARCOとの語彙差が特に大きい5データセットが選択されている。極端なドメインシフトに対する有効性は示されたが、法律、金融、ニュース、非英語環境や、語彙差が中程度の領域に一般化できるかは不明である。([ACL Anthology][2])

**計算資源:** 教師ラベルは不要だが、17 GBまたは7.3 GBのドメインコーパスと、8基のA100による継続事前学習を必要とする。「教師なし」ではあっても、必ずしも低コストな適応手法ではない。([ACL Anthology][2])

**アブレーションの不足:** AdaLM内部については詳細な分解がある一方、IDFのみを適用した条件が主要アブレーションに含まれていない。このため、CAI全体の改善のうち、AdaLMとIDFがそれぞれどの程度寄与したのかを完全には分離できない。また、BM25との加算係数に関する感度分析も示されていない。([ACL Anthology][2])

**効率評価:** NFCorpusでは文書ベクトルの平均非ゼロ要素数が291.7であり、疎性を維持していると報告されている。しかし、語彙拡張に伴うインデックス容量、検索レイテンシ、インデックス構築時間の体系的な比較はない。実運用上の効率性を主張するには、複数データセットでの速度・メモリ評価が望まれる。([ACL Anthology][2])

さらに、RSJ分析では、式(10)の「理想重み－モデル推定重み」という定義と、その後の正負の解釈に符号上の不整合が見られる。分析の定性的結論を直ちに否定するものではないが、指標の定義と図の読み方は明確化すべきである。([ACL Anthology][2])

## 7. まとめ

本論文は、疎検索モデルの教師なしドメイン適応において、専門語彙の不足と希少語の過小評価を主要因として捉えた。

提案するCAIは、

1. 対象分野の語彙を言語モデルへ追加する
2. 対象分野コーパスで継続事前学習する
3. 文書の疎ベクトルをIDFで補正する

という比較的単純で解釈可能な構成を採用している。

実験では、SPLADEおよびSPLADE-Docを一貫して改善し、GPLより高い平均性能を示した。さらにBM25とのハイブリッドにより、対象5データセットの平均で最高性能を達成した。特に、学習元で低頻度だった重要な専門語の完全一致を回復できることが、本手法の本質的な貢献である。([ACL Anthology][2])

## 8. 総評

**独立査読結果: 採択（Weak Accept）**

語彙・頻度ギャップという明確な問題設定に対して、語彙拡張、継続事前学習、IDF補正を組み合わせた設計は合理的である。5データセットすべてでの改善、GPLとの比較、アブレーション、RSJ分析、具体例がそろっており、性能向上の理由にも一定の説明力がある。

特に、既存技術を単に組み合わせるだけでなく、それらを疎検索特有の「低頻度語の完全一致」という問題に結び付けた点に実用的な新規性がある。

一方、計算資源の大きさ、対象分野の限定、IDF単独条件や統計的有意差検定の不足、検索速度・インデックス容量の評価不足は改善すべきである。新規アルゴリズムとしての革新性は中程度だが、問題設定、実証結果、分析の総合的な完成度は採択水準に達している。実際、本論文はAACL-IJCNLP 2022のLong Paperとして採録されている。([ACL Anthology][1])
