以下は、**Findings of the Association for Computational Linguistics: EMNLP 2020 掲載版**に基づく改訂版です。出版版では著者がRodrigo Nogueira、Zhiying Jiang、Ronak Pradeep、Jimmy Linの4名となり、Core17・Core18でのゼロショット評価、チェックポイント分析、ロジット正規化、ターゲットトークン実験などが追加・拡張されています。([ACL Anthology][1])

# 1. はじめに

本論文 **“Document Ranking with a Pretrained Sequence-to-Sequence Model”** は、情報検索における文書再ランキングを、従来のBERT型二値分類ではなく、T5を用いた**sequence-to-sequence問題**として定式化した研究です。

クエリと候補文書をT5に入力し、関連文書なら `true`、非関連文書なら `false` というターゲットトークンを生成するように学習します。推論時には実際に文章を生成するのではなく、最初のデコードステップにおける `true` と `false` のロジットから関連度を計算します。

出版版における主要な主張は次の3点です。

1. MS MARCO Passage Rankingで、T5によるランキングがBERT-largeを上回る。
2. MS MARCOだけで学習したモデルが、Robust04、Core17、Core18にゼロショット転移できる。
3. 少量データ環境ではT5がBERTより大幅にデータ効率に優れ、その要因の一部は出力ターゲットトークンに埋め込まれた事前学習知識で説明できる可能性がある。([ACL Anthology][2])

# 2. 背景

大規模文書集合に対してTransformerを直接適用するのは計算コストが高いため、ニューラル検索では一般に二段階以上の検索パイプラインを用います。

第一段階ではBM25などの語彙的検索手法によって候補文書を高速に取得し、第二段階ではBERTなどの高精度モデルを用いて候補文書だけを再ランキングします。本論文もBM25またはBM25+RM3が取得した上位1,000文書を対象とします。([ACL Anthology][2])

従来のBERT型rerankerでは、クエリと文書をエンコーダに入力し、`[CLS]` の表現をランダム初期化された全結合層に与えて関連・非関連を分類します。この方式では、BERT本体は事前学習されていますが、最終分類層はタスク固有データから新たに学習しなければなりません。

一方、T5ではエンコーダ、デコーダ、語彙出力層を含むネットワーク全体が事前学習されています。著者らは、関連性を `true`／`false` のような自然言語トークンに対応付ければ、最終判断部分でも事前学習済みの意味・語彙知識を利用できるため、特に訓練例が少ない場合に有利になると考えました。([ACL Anthology][2])

# 3. 提案手法

## 3.1 入出力形式

T5への入力は次のテンプレートです。

```text
Query: [Q] Document: [D] Relevant:
```

`[Q]` にクエリ、`[D]` に候補文書を挿入します。教師出力は以下です。

* 関連文書：`true`
* 非関連文書：`false`

`true` と `false` は、T5のSentencePieceトークナイザにおいて、それぞれ単一トークンとして表現されます。これにより、複数サブワードのロジットを統合する処理を回避しています。([ACL Anthology][2])

## 3.2 関連度スコア

推論時には、最初のデコードステップで得られる `true` と `false` のロジットだけを取り出します。文書 (d) のクエリ (q) に対する関連度は、

[
s(q,d)
======

\frac{\exp(z_{\mathrm{true}})}
{\exp(z_{\mathrm{true}})+\exp(z_{\mathrm{false}})}
]

と計算されます。これは、

[
s(q,d)=\sigma(z_{\mathrm{true}}-z_{\mathrm{false}})
]

とも表せます。

各候補文書を独立に評価し、このスコアの降順に並べ替えます。したがって、形式上は生成モデルを利用していますが、ランキング処理自体は**2つのターゲットトークンを利用したpointwise二値分類**に近い構造です。([ACL Anthology][2])

## 3.3 長文書への適用

Robust04、Core17、Core18には512トークンを超える長文書が含まれます。そこで文書を、

* 10文単位
* ストライド5文

のスライディングウィンドウでパッセージに分割します。それぞれのパッセージを独立に評価し、最大の関連確率を文書全体のスコアとします。

[
s(q,d)=\max_{p\in d}s(q,p)
]

これはBERT–MaxPと同種の集約方法です。BM25スコアとの補間も検討されましたが、性能向上がなく追加パラメータも必要になるため採用されていません。([ACL Anthology][2])

# 4. 実験方法

## 4.1 データセット

| データセット           | 規模と用途                                    |
| ---------------- | ---------------------------------------- |
| MS MARCO Passage | 約880万パッセージ、約53万のクエリ・関連文書ペア。学習および同一ドメイン評価 |
| Robust04         | 249トピック、約52.8万文書                         |
| Core17           | 50トピック、約186万件のNew York Times記事           |
| Core18           | 50トピック、約60万件のWashington Post記事           |

Robust04、Core17、Core18の関連性ラベルは学習やチェックポイント選択には使用せず、テストにだけ使用します。そのため、これら3コレクションでの評価はゼロショット転移です。([ACL Anthology][2])

ただし、クエリの入力形式はモデルによって異なります。

* BM25・BM25+RM3：短いtopic title
* T5：文章形式のtopic description

著者らは、topic descriptionの方がMS MARCOの自然言語質問に近いためT5への入力として採用しています。([ACL Anthology][2])

## 4.2 学習設定

| モデル      | パラメータ数 |   学習時間 |
| -------- | -----: | -----: |
| T5-base  |   220M |  約12時間 |
| T5-large |   770M |  約48時間 |
| T5-3B    |     3B | 約160時間 |

共通設定は以下です。

* 学習率：(10^{-3})
* 学習回数：10万ステップ、約10エポック
* バッチサイズ：128
* 正例と負例を同数にしたクラス均衡バッチ
* 最大入力長：512トークン
* 出力長：ターゲットトークンとEOSの2トークン
* 単一Google TPU v3-8を使用
* 最終チェックポイントを評価に使用

T5-11Bは計算コストのため実験されていません。推論ではgreedy decodingを使用しますが、最初のデコードステップのロジットしか利用しないため、beam searchを用いてもランキング結果は変わりません。([ACL Anthology][2])

## 4.3 比較手法と評価指標

比較対象は以下です。

* BM25
* BM25+RM3
* BM25+BERT-large
* Birch
* BERT–MaxP
* PARADE

MS MARCOでは公式指標のMRR@10を使用します。Robust04、Core17、Core18ではAP、nDCG@20、Jdg@20を使用します。Jdg@20は上位20件のうち、人手で関連性が判定済みの文書の割合です。

## 4.4 少量データ実験

BERT-baseとT5-baseを、以下の正例数で比較します。

* 1,000件
* 2,500件
* 10,000件
* 全約530,000件

各条件では正例と同数の負例を使用します。少量データ条件は異なるサンプルで5回反復し、平均値と95%信頼区間を報告します。BERTは学習率 (10^{-6})、T5は (10^{-3})、バッチサイズ32で3エポック学習されます。([ACL Anthology][2])

## 4.5 追加分析

出版版では、初稿に加えて以下が詳しく検証されています。

* MS MARCOでの学習エポック数とゼロショット転移性能の関係
* `true` ロジットをランキング間で比較可能にする正規化方法
* `true`／`false` を別のターゲットトークンに置換した場合の性能
* ターゲットトークンの意味、極性、事前学習頻度の影響

# 5. 結果

## 5.1 MS MARCO

| モデル             | パラメータ数 | Dev MRR@10 | Test MRR@10 |
| --------------- | -----: | ---------: | ----------: |
| BM25            |      ― |      0.184 |       0.186 |
| BM25+BERT-large |   340M |      0.372 |       0.365 |
| BM25+T5-base    |   220M |      0.381 |           ― |
| BM25+T5-large   |   770M |      0.393 |           ― |
| BM25+T5-3B      |     3B |  **0.398** |   **0.388** |

T5-baseは、BERT-largeよりパラメータ数が少ないにもかかわらず、開発セットで0.381対0.372と上回りました。T5-3Bは0.398で最良となり、T5-largeとの差は対応のあるt検定で (p<0.01) と報告されています。

出版版のMS MARCO結果は、初稿で報告されていた数値より全体的に高くなっています。また、テストセットではT5-3Bが0.388を記録し、BERT-largeの0.365を上回りました。([ACL Anthology][2])

## 5.2 ゼロショット転移

以下は主要結果です。T5モデルはMS MARCOだけで学習されています。

| モデル            | Robust04 AP / nDCG@20 | Core17 AP / nDCG@20 | Core18 AP / nDCG@20 |
| -------------- | --------------------: | ------------------: | ------------------: |
| Birch          |       0.3697 / 0.5325 |     0.3323 / 0.5092 |     0.3522 / 0.4953 |
| BM25           |       0.2531 / 0.4240 |     0.2087 / 0.3877 |     0.2495 / 0.4100 |
| BM25+T5-3B     |       0.3876 / 0.6091 | 0.3193 / **0.5629** | 0.3749 / **0.5493** |
| BM25+RM3       |       0.2903 / 0.4407 |     0.2823 / 0.4467 |     0.3135 / 0.4604 |
| BM25+RM3+T5-3B |   **0.4062 / 0.6122** | **0.3564** / 0.5612 | **0.3998** / 0.5492 |

Robust04では、当時の強い手法であるBERT–MaxPのnDCG@20が0.5453、PARADEが0.5713であるのに対し、BM25+RM3+T5-3Bは0.6122を達成しています。

T5-3Bは、各コレクションにおいてT5-base、T5-large、および対応するBM25系ベースラインより有意に優れており、Bonferroni補正後の対応のあるt検定で (p<0.05) と報告されています。ただし、この検定結果はBirch、BERT–MaxP、PARADEとの直接比較に対して報告されたものではありません。([ACL Anthology][2])

また、T5による上位文書はBM25よりJdg@20が低い傾向があります。特にCore18では、BM25のJdg@20が0.9620であるのに対し、BM25+T5-3Bは0.8600です。未判定文書は通常の評価では非関連として扱われるため、著者らはT5の値を真の性能に対する下限と解釈しています。ただし、未判定文書が実際に関連しているとは限らないため、比較には一定の不確実性が残ります。([ACL Anthology][2])

## 5.3 少量データでの性能

正例1,000件と負例1,000件だけを使用した場合、T5-baseはMRR@10で約0.254となり、BM25の0.184を約0.07上回りました。一方、BERT-baseはBM25を下回っています。

正例10,000件と負例10,000件の場合、T5-baseは約0.294となり、BERT-baseを約0.09上回りました。10,000正例は全正例の約2%ですが、T5はBM25に対する完全訓練時の改善幅の約60%を、この少量データだけで獲得しています。([ACL Anthology][2])

したがって、出版版でもT5の利点が最も明確なのは、モデル規模そのものよりも**少量教師データに対するデータ効率**です。

## 5.4 チェックポイント選択

MS MARCOでは、学習を長く続けるほど概ね性能が向上します。ただし、T5-baseには後半で軽い過学習が見られます。

一方、MS MARCOで学習したモデルをRobust04に適用すると、すべてのモデルサイズで、学習エポック数が増えるほどゼロショット性能が悪化する傾向が確認されました。つまり、MS MARCOへの適合を進めすぎると、他ドメインへの汎化能力が低下します。

より早いチェックポイントを使えばRobust04の性能を改善できますが、適切なチェックポイントを選ぶためにRobust04の検証データを使うと、厳密なゼロショットではなくなります。この結果は、同一ドメイン性能と転移性能にトレードオフがあることを示しています。([ACL Anthology][2])

## 5.5 ロジット正規化

| スコア計算方法                     |    MRR@10 |
| --------------------------- | --------: |
| `true` の生ロジットのみ             |     0.026 |
| 全語彙に対するsoftmax              |     0.379 |
| `true`／`false`だけに対するsoftmax | **0.381** |

`true` の生ロジットだけを使うと性能はほぼゼロになります。これは、異なる入力例から得られた単一トークンの生ロジットを直接比較できないことを意味します。

全語彙softmaxと二トークンsoftmaxの差は小さいため、本論文の二トークン正規化が唯一の有効な方法というわけではありません。ただし、二トークンだけを正規化する方法がわずかに高性能で、計算も単純なため標準方式として採用されています。

## 5.6 ターゲットトークンの影響

| 条件              | 関連／非関連         |             1K正例 |            10K正例 |   全530K正例 |
| --------------- | -------------- | ---------------: | ---------------: | --------: |
| Baseline        | true / false   | **0.254 ±0.014** |     0.294 ±0.002 |     0.374 |
| Alternate       | yes / no       |     0.218 ±0.040 | **0.301 ±0.004** | **0.378** |
| Reverse         | false / true   |     0.243 ±0.025 |     0.282 ±0.006 |     0.374 |
| Antonyms        | hot / cold     |     0.240 ±0.021 |     0.246 ±0.005 |     0.375 |
| Related Words   | apple / orange |     0.206 ±0.026 |     0.260 ±0.003 |     0.376 |
| Unrelated Words | hot / orange   |     0.194 ±0.018 |     0.242 ±0.005 |     0.377 |
| Subwords        | `_ab` / `_de`  |     0.179 ±0.014 |     0.228 ±0.005 |     0.377 |

全データを用いると差は0.004以内となり、任意のラベル対応を学習できます。これに対し、少量データではターゲットトークンの選択が大きく影響します。特に、意味を持たないサブワード条件は1K正例でBM25を下回ります。

ただし、結果は「意味的に自然なラベルほど必ず高性能」という単純な順序にはなっていません。10K正例では `yes`／`no` が `true`／`false` を上回り、`apple`／`orange` が `hot`／`cold` より良い結果です。著者らも、意味だけでなく事前学習コーパスにおける出現頻度や生成経験が影響している可能性を指摘しています。([ACL Anthology][2])

# 6. 考察

## 6.1 評価できる点

第一に、提案手法が単純です。新しいランキング専用ネットワークを設計せず、既存のT5に入力テンプレートとターゲットトークンを与えるだけで強力なrerankerを構築しています。

第二に、出版版では評価範囲が大きく拡張されています。MS MARCOだけでなく、性質と年代の異なるRobust04、Core17、Core18でゼロショット転移を確認しており、特定データセットだけに適合した結果ではないことを示しています。

第三に、単なる性能比較にとどまらず、以下の分析を行っています。

* 学習データ量
* モデルサイズ
* 学習エポック数
* ロジット正規化
* ターゲットトークンの意味と極性

特に、生ロジットがランキングに使えないこと、過度なファインチューニングがゼロショット転移を悪化させること、少量データではターゲットトークン選択が重要になることは、後続のprompt-based rankingや生成型ランキングにもつながる重要な知見です。

## 6.2 「生成型ランキング」という表現

本手法はT5のencoder–decoder構造を使用しますが、通常の意味で文や系列を生成しているわけではありません。利用するのは最初のデコードステップの `true`／`false` ロジットだけです。

数学的には、

[
\sigma(z_{\mathrm{true}}-z_{\mathrm{false}})
]

による二値分類と等価です。したがって、本手法の本質は「文章生成」よりも、**事前学習済み語彙トークンを分類ラベル、すなわちverbalizerとして利用すること**にあります。

## 6.3 原因分析の限界

著者らは、T5のデータ効率が高い理由を、事前学習済みのデコーダと出力語彙層を利用できるためと説明しています。しかし、実験では次の要因が同時に変化しています。

* encoder-onlyとencoder–decoderの違い
* BERTとT5の事前学習目的
* 事前学習データ
* モデルサイズ
* トークナイザ
* ランダム分類層と語彙出力層

そのため、データ効率向上の原因が事前学習済み出力層だけにあるとは断定できません。

より直接的な検証には、同一のT5バックボーンを用いて、

1. `true`／`false` による語彙出力
2. ランダム初期化した分類層
3. ランダム初期化した語彙トークン
4. 凍結したデコーダと学習可能な分類層

を比較する必要があります。

## 6.4 ターゲットトークン分析の限界

ターゲットトークン実験は興味深い一方、各カテゴリーについて1組の単語対しか試していません。例えば、`apple`／`orange` の結果から一般的な「意味的関連語」の効果を結論づけることは困難です。

また、性能には以下が交絡します。

* 単語頻度
* 事前学習時の出力頻度
* 出力バイアス
* 埋め込み空間上の距離
* 品詞
* トークンが出現する典型的文脈
* ラベルとして使われた事前経験

著者らも、より多くの単語対を調べなければ強い結論は出せないと認めています。([ACL Anthology][2])

## 6.5 ゼロショット評価の注意点

ゼロショット性能は強力ですが、いくつか注意点があります。

第一に、BM25には短いtitle、T5には情報量の多いdescriptionが入力されており、入力情報が完全には統制されていません。

第二に、T5は従来の検索システムと異なる文書を上位に出すため、Jdg@20が低下しています。不完全な関連性判定の下では、T5が過小評価されている可能性もありますが、逆に未判定文書を関連とみなす保証もありません。

第三に、T5の事前学習データに対象コーパスの文書が含まれていた可能性は排除されていません。論文自身もこの可能性を注記しています。([ACL Anthology][2])

## 6.6 計算効率

T5-3BはMS MARCOで高性能ですが、単一TPU v3-8で約160時間の学習を必要とします。さらに、クエリごとに上位1,000文書をcross-encoder形式で個別評価し、長文書については複数パッセージを処理します。

論文では以下が報告されていません。

* クエリ当たりの推論時間
* 文書当たりのレイテンシ
* スループット
* 消費電力
* BERTとの計算量を揃えた比較
* 蒸留モデルとの性能・速度比較

したがって、ランキング精度の優位性は明確ですが、オンライン検索システムへの導入可能性までは十分に検証されていません。

# 7. まとめ

本論文は、文書ランキングを、

> クエリと候補文書から、関連性を表すターゲットトークンを予測する問題

として定式化しました。

MS MARCOではT5-base、large、3BのすべてがBERT-largeを上回り、T5-3Bは開発セットでMRR@10 0.398、テストセットで0.388を記録しました。また、MS MARCOだけで学習したモデルが、Robust04、Core17、Core18で強いゼロショット性能を示しています。

特に重要なのは少量データ環境での性能です。T5は正例1,000件程度でもBM25を上回り、同量のデータで学習したBERTを大幅に上回ります。

出版版の追加分析からは、次のことが分かります。

* 異なる文書間で生ロジットを直接比較してはいけない。
* 十分なデータがあればターゲットトークンの違いはほぼ吸収できる。
* 少量データではターゲットトークンの意味や事前学習経験が重要になる。
* MS MARCOへの過剰なファインチューニングは、他ドメインへの転移性能を損なう。

本研究の本質的な貢献は、長い文章を生成することではなく、**事前学習済みの語彙出力経路をランキング用の分類ヘッドとして再利用したこと**にあります。
