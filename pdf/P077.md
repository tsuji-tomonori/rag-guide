# P077：A Survey on Retrieval-Augmented Text Generation for Large Language Models

**著者**：Yizheng Huang、Jimmy X. Huang
**対象本文**：arXiv v2、2024年8月23日公開。のちに *ACM Computing Surveys*, Vol. 58, No. 12, Article 300, pp. 1–38として2026年5月15日に掲載された。([arXiv][1])

## 1. はじめに

本論文は、大規模言語モデルに外部情報検索を組み合わせるRetrieval-Augmented Generation、すなわちRAGについて、**情報検索の観点から技術体系を整理したサーベイ論文**である。

LLMは学習後に知識を更新しにくく、専門領域への対応、最新情報の利用、事実性の確保に課題がある。RAGは、質問に関連する情報を外部コーパスから検索し、その情報を生成時のコンテキストとして与えることで、モデルの再学習を行わずに知識を補完する。

本論文の中心的貢献は、RAGを単純な「検索して生成する」処理ではなく、次の4段階から構成されるシステムとして整理した点にある。

1. Pre-Retrieval
2. Retrieval
3. Post-Retrieval
4. Generation

さらに、RAGの評価方法、代表的研究の構成要素、検索器と生成器の関係、今後の研究課題を包括的に整理している。([arXiv][2])

## 2. 背景

論文は、LLMが抱える主要な問題として以下を挙げている。

第一に、一般的なデータを中心に学習しているため、医療や法律などの専門領域では十分な性能を発揮できない場合がある。第二に、学習完了後に生じた出来事や更新情報をモデル内部に取り込めない。第三に、もっともらしいが事実ではない文章を生成する、いわゆるハルシネーションが発生する。

基本的なRAGは、次の3工程で構成される。

**Indexing**では、文書をチャンク化し、転置インデックスや埋め込みベクトルとして保存する。
**Retrieval**では、ユーザークエリに関連する文書を検索・順位付けする。
**Generation**では、クエリと検索文書をLLMに入力し、回答を生成する。

しかし近年のRAGは、この直線的な構成から、生成途中で再検索したり、検索結果を評価してクエリを書き換えたりする反復的・適応的システムへ発展している。本論文は、こうした複雑化したRAGを共通の処理段階に分解する必要性を指摘している。

## 3. 提案手法

本論文は新しいRAGアルゴリズムを提案するのではなく、既存研究を整理するための**統一的な分類体系**を提案している。

### Pre-Retrieval

検索を実行する前の準備段階であり、以下の3要素からなる。

**Indexing**では、転置インデックスに加えて、埋め込みを用いる近似最近傍探索を扱う。ベクトルインデックスは、グラフ型、Product Quantization、Locality-Sensitive Hashingに分類される。

**Query Manipulation**では、検索性能を向上させるためにクエリを変換する。具体的には、関連語を追加するQuery Expansion、質問を別表現に変換するQuery Reformulation、LLMへの指示によって検索用クエリを生成するPrompt-based Rewritingが含まれる。

**Data Modification**では、検索対象データを変更・拡張する。LLM自身に関連知識を生成させるInternal Data Augmentationと、Wikipediaや知識ベースなどから情報を追加するExternal Data Enrichmentに分類される。([arXiv][3])

### Retrieval

検索と初期順位付けを行う段階である。論文はRAGの検索戦略を5種類に分類している。

* Basic Retrieval：1回だけ検索する直線的な方式
* Iterative Retrieval：生成結果を使って検索を反復する方式
* Recursive Retrieval：複雑な質問をサブクエリに分解し、階層的に検索する方式
* Conditional Retrieval：条件判定に応じて検索方法や処理経路を変更する方式
* Adaptive Retrieval：モデルの信頼度や生成内容に応じて、検索の有無やタイミングを動的に決定する方式

この分類により、単発検索とマルチホップ検索だけでなく、FLARE、Self-RAG、CRAGなどの動的なRAGも共通の枠組みで説明できる。([arXiv][3])

### Post-Retrieval

最初に取得した文書集合を生成に適した形へ改善する段階である。

**Re-Ranking**では、検索済み文書をより高精度なモデルで再評価する。教師なし再ランキング、教師あり再ランキング、合成データによる再ランキング学習に分類される。

**Filtering**では、無関係、低品質、重複、矛盾を含む文書やトークンを削除する。Self-RAGの自己評価、RECOMPのコンテキスト圧縮、FiD-TFのトークンフィルタリングなどが含まれる。

再ランキングが文書の順序を改善する処理であるのに対し、フィルタリングは不要な文書そのものを除外する処理である。([arXiv][3])

### Generation

検索結果を最終回答に反映する段階であり、EnhancingとCustomizationに分けられる。

**Enhancing**には、クエリと文書を直接連結する方法、複数文書や複数生成結果を統合するアンサンブル方式、生成結果を評価して再検索・再生成するフィードバック方式が含まれる。

**Customization**は、ユーザー属性、目的、文体、状況などに応じて回答を個人化する処理である。論文では、他の構成要素と比較してCustomizationを中心に扱う研究が少ないと指摘している。([arXiv][3])

## 4. 実験方法

本論文自体は、新しいRAGモデルを実装して統一条件下で比較する実験論文ではない。実験に相当する部分は、既存研究の分類と、既存ベンチマーク結果の再整理である。

まず、代表的なRAG研究について、内部・外部検索源、マルチホップの有無、学習の有無、4段階のどこを改善しているかを表形式で比較している。また、各研究で使用されたBM25、DPR、Contrieverなどの検索器と、T5、BART、LLaMA、GPT系モデルなどの生成器も整理している。([arXiv][3])

定量的な分析には主として次の既存研究の結果が用いられている。

**MIRAGE**では、GPT-3.5を生成器とし、PubMedおよびWikipediaを検索対象として、BM25、Contriever、SPECTER、MedCPT、複数検索器を統合するReciprocal Rank Fusionを比較している。評価対象はMMLU-Med、MedQA-US、MedMCQA、PubMedQA、BioASQ-Y/Nである。

**eRAG**では、Natural Questionsにおいて、BM25、RetroMAE、SPLADEv3とDeBERTa-v3再ランキングを用い、Recall@5などの検索性能と下流のRAG性能との相関を分析している。

**BERGEN**では、検索を使わないClosed-book条件と、正解文書を与えるOracle条件を比較し、モデルサイズと検索による性能向上量の関係を調べている。([arXiv][3])

評価指標は検索側と生成側に分けられる。検索側にはPrecision、MAP、MRR、NDCG、Context Relevance、Rejection Rate、Error Detection Rateがあり、生成側にはEM、F1、Accuracy、BLEU、ROUGE-L、Answer Relevance、Faithfulnessなどがある。RAGAS、ARES、RECALL、RGB、MIRAGE、eRAG、BERGENなどの評価フレームワークも比較されている。

ただし、公開版では、文献を収集したデータベース、検索式、検索期間、採択・除外基準、研究品質の評価方法といった、システマティックレビューとしての選定手順は明示されていない。

## 5. 結果

既存研究の分類から、次の傾向が示されている。

多くのRAG研究は外部データを利用し、単発検索よりも反復的なマルチホップ検索を採用している。また、データ自体を変更する研究よりも、クエリ書き換えや検索・ランキングを改善する研究が多い。生成段階ではEnhancingが多く、Customizationは相対的に研究例が少ない。([arXiv][3])

検索器と生成器の比較では、高性能な生成モデルが多数存在する一方、検索器には依然としてBM25が広く使用されている。生成器ではT5系が多く、調査対象期間ではGPT-4などの大規模な生成器が必ずしも主流ではなかった。([arXiv][3])

MIRAGEの平均正解率では、検索なしが60.69%であったのに対し、PubMedに対するRRF-2は70.26%となり、9.57ポイント向上した。一方、Wikipediaに対するRRF-2は60.14%であり、検索なしをわずかに下回っている。この結果は、単に検索を追加すればよいのではなく、**コーパス、検索器の学習データ、対象タスクの整合性が重要**であることを示している。

また、PubMedではRRF-2が70.26%、RRF-4が69.71%、WikipediaではRRF-2が60.14%、RRF-4が59.14%であり、検索器を増やしても必ずしも性能は向上しない。

eRAGの結果では、Recall@5で測定した検索性能が高くなるほど、RAGの回答性能も向上した。特にSPLADEv3にDeBERTa-v3再ランキングを組み合わせた構成が高い性能を示しており、検索結果の再ランキングが下流性能に大きく影響することが示されている。

BERGENでは、検索による改善量とLLMのパラメータ数との間に単純な比例関係は見られなかった。小規模モデルの方が検索から大きな恩恵を受ける場合があり、検索付きLLaMA2-7BがClosed-bookのLLaMA2-70Bを上回る例も報告されている。

## 6. 考察

本論文から得られる重要な示唆は、RAG性能を改善する際、生成モデルの巨大化だけに注目すべきではないという点である。

実用上は、次の順序で検討することが合理的である。

1. タスクに適した検索コーパスを選択する
2. コーパスに適合した検索器を使用する
3. 再ランキングとフィルタリングを導入する
4. 不十分な検索結果では回答を拒否できるようにする
5. その上で生成モデルやプロンプトを改善する

特にeRAGの考え方は、検索品質を「人間から見た文書の関連性」だけでなく、**その文書が生成器の正答にどれだけ寄与するか**で評価すべきことを示している。

一方、本論文にはいくつかの限界がある。

第一に、文献検索・選定手順が明確でないため、収録研究の網羅性や選択バイアスを検証しにくい。第二に、Iterative、Conditional、Adaptive Retrievalの境界には重複があり、一つの手法が複数カテゴリに該当し得る。第三に、MIRAGE、eRAG、BERGENなど、異なるモデル、データセット、プロンプト、評価指標で得られた結果をまとめているため、「検索品質が生成器より常に重要」と一般化する際には注意が必要である。

また、論文はLLMのハルシネーションを最新情報へのアクセス不足と強く関連付けているが、実際のハルシネーションには、不適切な推論、曖昧な質問、コンテキスト間の矛盾、生成時の過度な補完なども関係する。そのため、RAGはハルシネーションを軽減する手段ではあるが、単独で解決するものではない。

論文が示す将来課題は、検索ノイズ、検索失敗時の回答拒否、複数文書の統合、検索遅延、再ランキングの計算コスト、モジュール間最適化、マルチモーダル情報の対応付けと一貫した生成である。

## 7. まとめ

本論文は、RAGを「Indexing → Retrieval → Generation」という単純な構成から、**Pre-Retrieval → Retrieval → Post-Retrieval → Generation**という4段階の統一フレームワークへ拡張して整理した。

最大の知見は、RAGの性能が生成器の規模だけでなく、検索コーパスとの適合性、検索器の品質、再ランキング、フィルタリング、反復検索の設計によって大きく左右されることである。

新規アルゴリズムや統一的な再実験を提示する論文ではないが、RAGシステムの各構成要素を理解し、既存手法を比較し、研究・開発上の改善箇所を特定するための基礎資料として有用である。([arXiv][3])

## 8. 総評

### 査読結果

**公開上確認できる最終結果：採択・掲載**

本論文は2024年2月にACL Rolling Reviewへ匿名投稿されているが、OpenReviewの公開ページには査読コメント、スコア、最終判定が掲載されておらず、公開返信数も0件である。したがって、ACL ARRにおける具体的な査読結果は確認できない。([OpenReview][4])

一方、最終的には2026年5月に *ACM Computing Surveys* に掲載されているため、公開情報から確認できる最終的な出版判定は**採択**である。([ACM Digital Library][5])

### 内容に基づく模擬査読判定

**判定：Weak Accept／採択寄り**

評価できる点は、RAGを情報検索中心の4段階へ整理した分類体系が明快であり、検索器、生成器、評価指標、将来課題まで一つの枠組みで把握できることである。特に、検索品質と生成性能の関係、ドメイン適合性、再ランキングの重要性を既存結果から具体的に示した点は有用である。

主な懸念は、文献選定プロトコルが不明確であること、分類間の境界が一部曖昧であること、異質な既存実験から比較的強い一般論を導いていることである。検索式、収録基準、研究品質評価、統一条件での再実験を追加できれば、サーベイとしての再現性と証拠の強度はさらに高まる。

**総合評価：7.5／10。RAGの全体像を学ぶ入門・整理資料として高い価値を持つ一方、厳密なシステマティックレビューや定量的メタ分析として読むべき論文ではない。**
