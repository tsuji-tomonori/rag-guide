# 論文情報

**Language Models as Knowledge Bases?**
Fabio Petroniほか、EMNLP-IJCNLP 2019。事前学習済み言語モデルが、追加学習なしで事実知識や常識知識を想起できるかを検証した論文である。([arXiv][1])

## 1. はじめに

本論文の中心的な問いは、**「言語モデルのパラメータを、暗黙的な知識ベースとして利用できるか」**である。

通常の知識ベースは、たとえば

[
(\text{Dante},\ \text{born-in},\ \text{Florence})
]

のような構造化された三つ組で事実を保持する。一方、言語モデルに

> Dante was born in [MASK].

と入力し、`Florence`を予測できるなら、その事実がモデルのパラメータに記憶されていると考えられる。

著者らはこの考えを検証するため、事前学習済みモデルをファインチューニングせずに評価するベンチマーク **LAMA（LAnguage Model Analysis）** を提案した。主な結論は、BERT、とくにBERT-largeが、一定範囲の事実知識・常識知識をパラメータ内に保持しているというものである。

## 2. 背景

従来型の知識ベースをテキストから構築するには、固有表現抽出、共参照解析、エンティティリンキング、関係抽出などを組み合わせたパイプラインが必要になる。各処理には教師データや関係スキーマが必要であり、前段の誤りが後段へ伝播する問題もある。

これに対して言語モデルは、大規模テキストから自己教師あり学習によって訓練されるため、明示的な知識スキーマを事前に設計せず、自然言語の形で多様な関係を問い合わせられる可能性がある。

論文では次のモデルを比較している。

* fairseq-fconv
* Transformer-XL large
* ELMo original
* ELMo 5.5B
* BERT-base
* BERT-large

とくにBERTは、左右両方の文脈からマスク位置を予測するMasked Language Modelingで訓練されているため、穴埋め形式の知識問い合わせと学習目的がよく整合している。

## 3. 提案手法

### LAMAプローブ

LAMAでは、知識を主語・関係・目的語の三つ組

[
(s,r,o)
]

として用意し、関係ごとの自然言語テンプレートを使って穴埋め問題へ変換する。

たとえば、

[
(\text{Dante},\text{place-of-birth},\text{Florence})
]

を

> Dante was born in [MASK].

に変換し、正解目的語`Florence`がモデルの出力候補中で何位になるかを測る。

使用する知識源は次の4種類である。

| 知識源        | 内容                             |            規模 |
| ---------- | ------------------------------ | ------------: |
| Google-RE  | Wikipediaから人手抽出された出生地・生年月日・死亡地 |   5,527事実、3関係 |
| T-REx      | WikidataとWikipediaを対応付けた関係知識   | 34,039事実、41関係 |
| ConceptNet | 常識的な概念間関係                      | 11,458事実、16関係 |
| SQuAD      | 文脈なしでも回答可能な質問を穴埋め文へ変換          |          305問 |

Google-REとT-RExでは関係ごとにテンプレートを人手で作成し、SQuADについても質問を手作業で平叙文型の穴埋め問題へ変換する。ConceptNetでは、主語と目的語を含む既存文の目的語部分をマスクする。

### 評価上の制約

モデル間の比較を成立させるため、以下の制約が置かれている。

* 正解は**単一トークン**に限定する。
* 三つ組のうち、原則として目的語だけを予測する。
* 全モデルの語彙の共通部分である約21,000語だけを候補とする。
* 複数の正解目的語がある場合、評価対象以外の正解を候補から除く。
* 指標にはPrecision@kを使用する。

単一トークンに限定した理由は、複数トークン生成ではビーム幅、長さ正規化、候補スコアリングなどの追加要因が入り、純粋な知識保持能力を測りにくくなるためである。

## 4. 実験方法

各モデルは追加学習を行わず、公開済みの事前学習済み重みをそのまま使用する。BERTでは目的語位置を`[MASK]`に置き換え、マスク位置の出力分布から候補語を順位付けする。単方向モデルでは、目的語直前までの文脈から次トークンを予測する。ELMoでは順方向・逆方向モデルの確率を平均する。

比較対象は次の通りである。

**Freq**
関係ごとに、評価データ中で目的語として頻出する語を優先する。関係固有の単純な頻度バイアスでどこまで解けるかを見る診断的ベースラインである。

**REn／REo**
既存の関係抽出モデルを使い、Wikipedia文から知識グラフを構築する。REnは文字列一致による単純なエンティティリンキング、REoは正しいエンティティ対応を与えるオラクル版である。

**DrQA**
SQuADに対する教師ありオープンドメイン質問応答システム。Wikipedia検索と読解モデルを組み合わせる。ただし、言語モデル側との比較のため、回答を単一トークンに制限している。

主指標は関係ごとに平均したP@1であり、補助的にP@10やP@100も調べている。

## 5. 結果

主要なP@1の結果は以下の通りである。

| データセット     | BERT-large |          主な比較対象 |
| ---------- | ---------: | --------------: |
| Google-RE  |   **10.5** |      REオラクル 7.6 |
| T-REx      |       32.3 | REオラクル **33.8** |
| ConceptNet |   **19.2** |    頻度ベースライン 4.8 |
| SQuAD      |       17.4 |   DrQA **37.5** |

### 事実知識

Google-REでは、BERT-largeがP@1で10.5を記録し、オラクルのエンティティリンキングを与えられた関係抽出モデルの7.6を上回った。とくに出生地と死亡地では強い一方、生年月日はほぼ予測できていない。

より大規模なT-RExでは、BERT-largeの32.3は関係抽出オラクルの33.8に近い。ただし、関係の種類によって大きな差がある。

* 1対1関係：BERT-large 74.5
* 多対1関係：34.2
* 多対多関係：24.3

「国の首都」のように答えがほぼ一意に決まる関係は強いが、「外交関係を持つ国」「所属チーム」のように複数の正解を持つ関係では弱い。

T-RExでは、BERTの正解が上位10件に入る割合は約60%、上位100件では約80%だった。したがって、正解が必ず最上位に来るわけではないが、内部表現には相当量の候補知識が含まれている。

### 常識知識

ConceptNetではBERT-largeが19.2で、他の言語モデルを大きく上回った。たとえば「Ravens can [MASK]」に対して`fly`、「Birds have [MASK]」に対して`wings`や`feathers`など、正解と完全一致しない場合でも意味的に妥当な語を上位に出す例が確認された。

### 質問応答

SQuADではP@1でDrQAの37.5に対しBERT-largeは17.4であり、最上位回答の精度には明確な差がある。一方、P@10ではBERT-largeが57.1、DrQAが63.5となり、検索機構もタスク固有の教師あり学習も持たないモデルとしては比較的小さな差に収まった。

### 信頼度とプロンプト依存性

BERT-largeでは、最上位予測の確率が高いほど正解率も高く、目的語が訓練コーパスに多く出現するほど性能が高い傾向があった。一方で、同じ事実でも問い合わせ文の表現を変えると順位が変動する。BERTとELMo 5.5Bは他モデルより変動が小さいが、これらのモデルはWikipediaを広く学習しており、評価文または類似文を訓練時に見た可能性がある。

## 6. 考察

本論文から支持されるのは、**BERTのパラメータには、外部検索なしで呼び出せる一定量の関係知識が存在する**という主張である。しかし、「正しく回答できる」ことをそのまま「事実を理解している」と解釈することはできない。

著者自身も、BERTが関係を理解しているのではなく、主語と目的語の共起や表層的な関連性を学習しただけである可能性を指摘している。さらに、BERTの学習データと評価用事実の双方がWikipediaに由来するため、一般化能力と訓練文の記憶を明確に分離できていない。

重要な限界は次の通りである。

**テンプレート依存性**
知識を取り出せるかどうかが質問表現に左右される。著者らは結果を「モデルが持つ知識の下限」と解釈しているが、特定テンプレートが訓練文と類似している場合には、逆に性能を過大評価する可能性もある。

**単一トークン制約**
人名、地名、組織名など現実の知識の多くは複数トークンである。この制約により、評価される事実集合が大きく偏る。

**モデル間の統制不足**
各モデルでアーキテクチャ、パラメータ数、学習コーパス、コーパス規模が異なる。そのため、BERTの優位性が双方向性、モデルサイズ、学習データ量のどれによるものか切り分けられない。

**知識ベースとしての機能不足**
通常の知識ベースには、出典、更新可能性、一貫性、否定、時間情報、複数正解、問い合わせ結果の検証可能性が求められる。LAMAが示しているのは「事実の想起能力」であり、言語モデルがそのまま完全な知識ベースになることではない。

また、本論文はシステムを「unsupervised」と表現しているが、より正確には**タスク固有のファインチューニングを行わないzero-shot評価**である。関係テンプレートは人手で作成され、評価データ自体も人手または既存知識ベースから構築されている。

## 7. まとめ

本論文は、事前学習済み言語モデルを穴埋め形式で問い合わせるLAMAプローブを提案し、BERTが事実知識と常識知識をパラメータ内に保持していることを示した。

BERT-largeは、限定された条件では、オラクル情報を与えられた関係抽出ベースラインに近い、またはそれを上回る性能を示した。一方で、性能は関係の種類、質問表現、訓練データ中の出現頻度に強く依存し、複数トークン回答や多対多関係には弱い。

したがって、本論文の最も妥当な結論は、**「言語モデルは知識ベースそのものではないが、事実を保持するパラメトリックメモリとして利用できる」**というものである。

## 8. 総評

### 評価

本論文の最大の貢献は、「言語モデルが知識を持っているか」という抽象的な問いを、再現可能な穴埋め評価として定式化した点にある。LAMAは単純で理解しやすく、複数のモデル・知識源・ベースラインを同じ枠組みで比較している。言語モデルのパラメータを知識記憶装置として分析する研究課題を明確にした点で、研究上の意義は大きい。

一方、論文タイトルが示唆する「言語モデルが知識ベースになり得る」という主張に対し、実験が検証している範囲は限定的である。Wikipedia由来データとの重複、手作業テンプレート、単一トークン制約、モデル間の学習条件の違いを考えると、結果は「知識の一般化」よりも「事前学習コーパスからの想起」を示している可能性が高い。

### 査読結果

**実際の採否：採択。**
本論文はEMNLP-IJCNLP 2019の本会議論文として採択・掲載されている。([arXiv][1])

**模擬査読判定：弱採択（Weak Accept）。**

* 新規性：4/5
* 技術的妥当性：3/5
* 実験の充実度：3.5/5
* 記述の明瞭性：4/5
* 研究上の重要性：4.5/5

採択を支持する理由は、問いの重要性、LAMAという明快な評価枠組み、BERTのパラメトリック知識を示した実証結果である。ただし、結論は「言語モデルは知識ベースである」ではなく、**「言語モデルは限定的かつ不安定な形で事実知識を記憶・想起できる」**まで狭めて解釈すべきである。
