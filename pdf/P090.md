# P090：DocLayNet: A Large Human-Annotated Dataset for Document-Layout Analysis

## 1. はじめに

本論文は、PDF文書のレイアウト解析に用いる大規模データセット **DocLayNet** を提案した研究である。新しいニューラルネットワーク構造ではなく、学習・評価基盤となるデータセットの構築が主な貢献である。

DocLayNetは、IBM Researchが作成した **80,863ページ、11クラス、1,107,470個のレイアウト要素**からなる人手アノテーション済みデータセットである。論文は2022年のACM SIGKDD国際会議で発表された。([arXiv][1])

---

## 2. 背景

文書レイアウト解析では、PDFページ内の領域を本文、見出し、表、図、数式などに分割する。これはOCR結果を単に文字列として取得するだけでなく、文書を構造化データへ変換するために必要となる。

従来の大規模データセットとして、約30万ページのPubLayNetと約50万ページのDocBankが存在した。しかし、これらはPubMedやarXivのXML・LaTeXソースを利用して自動生成されており、データの大部分が定型的な科学論文である。そのため、科学論文上では高い性能を示しても、企業の年次報告書、マニュアル、法令、特許、政府文書のような自由度の高いレイアウトには汎化しにくい。([arXiv][2])

自動生成アノテーションには低コストという利点がある一方、構造化された元データが存在する文書しか利用できない。また、組版上の「見えない表」を実際の表として扱うなど、人間が知覚するレイアウトとプログラム上の構造が一致しない場合がある。DocLayNetは、この問題を人手アノテーションによって解決しようとしている。([ar5iv][3])

---

## 3. 提案手法

### データセットの構成

DocLayNetは、次の6種類の文書ドメインから構成される。

| 文書カテゴリ | ページ比率 |
| ------ | ----: |
| 財務報告書  |   32% |
| マニュアル  |   21% |
| 科学論文   |   17% |
| 法律・規則  |   16% |
| 特許     |    8% |
| 政府入札文書 |    6% |

特に財務報告書とマニュアルを多く含めることで、企業固有のデザインや複雑な自由レイアウトを増やしている。

アノテーション対象は以下の11クラスである。

**Caption、Footnote、Formula、List-item、Page-footer、Page-header、Picture、Section-header、Table、Text、Title**

クラスは、出現頻度、視覚的な識別可能性、単一ページだけで判断できること、ページ全体を十分に被覆できることを基準として選ばれた。AuthorやAffiliationのように、文字内容を読まなければ区別できない意味的クラスは採用されていない。([arXiv][2])

### 人手アノテーション

アノテーション工程は4段階で実施された。

第1段階では、利用可能な公開PDFを収集・解析し、表や図を含むページを意図的に多く抽出した。第2段階では、11クラスを決定し、100ページを超える詳細なアノテーションガイドラインを作成した。

第3段階では、974ページの正解アノテーションを用いて40人を訓練した。難易度の異なる2段階の試験を通過できなかった8人を除き、32人が本番作業へ進んだ。訓練期間は約12週間である。

第4段階では、32人が約3か月かけて約8万ページをアノテーションした。アノテータ同士は互いの結果を閲覧できず、テキスト領域ではPDFの文字セルに矩形を自動スナップさせることで、境界の精度と作業効率を高めた。1ページ当たりの作業時間は、おおむね20～60秒であった。([arXiv][2])

### 重複アノテーション

80,863ページのうち、7,059ページには2件、1,591ページには3件の独立したアノテーションが付与され、合計91,104件のアノテーションインスタンスが作成された。

この重複アノテーションにより、単なるラベル品質検査だけでなく、アノテータ間一致度を「人間が到達可能な性能の参考値」として測定できる。([ar5iv][3])

### データ分割と形式

学習・検証・テスト集合は、それぞれ69,375、6,489、4,999ページである。分割はページ単位ではなく、**文書単位**で行われる。同じ文書のページが学習集合とテスト集合に混在しないため、文書固有のテンプレートを記憶することによるデータリークを防止できる。([GitHub][4])

データは拡張COCO形式で提供され、1,025×1,025ピクセルのPNG、矩形アノテーション、元PDF、PDF内の文字列と座標を記録したJSONが含まれる。([arXiv][2])

## 4. 実験方法

評価指標には、COCOで一般的な **mAP@0.5–0.95** を使用する。IoU閾値を0.50から0.95まで0.05刻みで変化させ、その平均適合率を計算する。

ベースラインとして以下の物体検出モデルを使用した。

* Mask R-CNN：ResNet-50、ResNet-101
* Faster R-CNN：ResNet-101
* YOLOv5x6

すべてCOCO 2017の事前学習重みから初期化され、1,025×1,025ピクセルのRGB画像で学習・評価された。重複アノテーションページについては、学習時には1件だけを使用している。([arXiv][2])

加えて、次の実験を実施した。

1. 学習データ量を段階的に増加させる学習曲線実験
2. 11クラスを6、5、4クラスへ統合・削除するラベルアブレーション
3. 文書単位分割とページ単位ランダム分割の比較
4. PubLayNet、DocBank、DocLayNet間のクロスデータセット評価

---

## 5. 結果

### ベースライン性能

全クラス平均のmAPは次の通りである。

| 評価対象              | mAP@0.5–0.95 |
| ----------------- | -----------: |
| 人間同士の一致度          |        82～83 |
| Mask R-CNN R50    |         72.4 |
| Mask R-CNN R101   |         73.5 |
| Faster R-CNN R101 |         73.4 |
| YOLOv5x6          |     **76.8** |

YOLOv5x6が最も高い性能を示したが、全体では人間同士の一致度より約5～6ポイント低い。論文ではクラスごとの違いも含め、おおむね6～10%の差が残ると説明している。

YOLOv5x6は、Textで88.1、Tableで86.3、Pictureで77.1を達成し、これらのクラスではアノテータ間一致度の範囲を上回った。一方、Formulaは66.2、Page-footerは61.1であり、細かい領域や視覚的に曖昧な要素は依然として難しい。ここで人間の値は「正解率」ではなくアノテータ間の一致度なので、この結果を単純に超人的性能と解釈することはできない。([ar5iv][3])

### データセットサイズ

Mask R-CNN R50の性能は、学習データを増やすにつれて急速に向上するが、全データの80%付近から飽和した。100%データで5回学習した際の変動幅は約1%であり、80%と100%の結果は誤差範囲内だった。

したがって、同種のページを追加するよりも、新しい文書カテゴリ、レイアウトスタイル、データ拡張、アノテーション整合性の改善が有効だと著者らは結論づけている。([ar5iv][3])

### データ分割の影響

11クラス設定では、文書単位分割のmAPが72であるのに対し、ページ単位ランダム分割では84となった。5クラス設定でも78から87へ上昇した。

この約10ポイントの差は、同じ文書の類似ページが学習・テスト集合に混入すると、モデルの汎化能力を大幅に過大評価することを示している。([arXiv][2])

### クロスデータセット評価

各データセットで学習したモデルの全体スコアは次のようになった。

| 学習データ     | PubLayNetで評価 | DocBankで評価 | DocLayNetで評価 |
| --------- | -----------: | ---------: | -----------: |
| PubLayNet |           93 |         34 |           30 |
| DocBank   |           48 |         68 |           27 |
| DocLayNet |           59 |         47 |           78 |

PubLayNetとDocBankのモデルは、自身のテスト集合では高性能だが、別データセットでは大きく低下した。DocLayNetモデルにもドメイン差はあるものの、低下幅が比較的小さく、多様な未知レイアウトに対する頑健性が高い。

## 6. 考察

本研究の最も重要な成果は、単にページ数を増やしたことではなく、**レイアウトの多様性、人手による自然な構造解釈、文書単位での厳密な評価**を一つのデータセットにまとめた点である。特に、ページ単位分割が約10ポイントもの性能過大評価を引き起こすことを定量的に示した点は、文書AI全般の評価設計に対する重要な警告である。

また、重複アノテーションによって、教師ラベルには本質的な曖昧さが存在することを数値化した点も評価できる。レイアウト解析では、見出しと本文、図と複数の部分図、タイトルとセクション見出しなどの境界が必ずしも一意ではない。モデル性能を単一の正解ラベルだけで評価するのではなく、人間同士の不一致を考慮する考え方は妥当である。

一方で、以下の限界がある。

第一に、文書の約95%が英語であり、多言語文書に対する代表性は限定的である。さらに、スキャン文書、手書き文書、領収書、請求書、自然画像内の文字などは対象外であり、「汎用」という主張は主としてデジタル生成された技術文書の範囲に限られる。([arXiv][2])

第二に、アノテーションは非重複の軸平行矩形に制限されている。このため、入れ子構造、重複領域、回転文字、不規則な形状を正確には表現できない。また、読み順、見出し階層、図とキャプションの対応、表のセル構造などは直接アノテーションされていない。

第三に、データセット間比較では各データセットのラベル定義が異なるため、ラベルの統合や削除が必要となる。例えば、DocLayNetでは各リスト項目を別々に扱うが、PubLayNetではリスト全体を一つの領域として扱う。このため、クロスデータセット実験はDocLayNetの優位性を示唆するものの、完全に同一条件の比較ではない。([ar5iv][3])

第四に、ベースラインは主に従来型の画像ベース物体検出器である。テキスト情報を利用するマルチモーダルモデルやTransformer系レイアウト解析器は詳細に評価されておらず、データセットの可能性をすべて検証した実験ではない。

---

## 7. まとめ

DocLayNetは、科学論文に偏っていた既存レイアウトデータセットの問題を解決するため、6種類の文書ドメインから80,863ページを収集し、11クラスを人手でアノテーションしたデータセットである。

主な知見は次の通りである。

* YOLOv5x6の全体mAPは76.8で、人間同士の一致度82～83には届かない。
* 学習性能はデータ量80%付近で飽和する。
* ページ単位のランダム分割は、mAPを約10ポイント過大評価する。
* DocLayNetで学習したモデルは、PubLayNetやDocBankで学習したモデルより未知データセットへの性能低下が小さい。

したがって、本研究の価値はモデルの性能更新ではなく、実環境に近い文書レイアウト解析を可能にする、再利用性の高い評価・学習基盤を提供した点にある。

## 8. 総評

**査読結果：採択。**
本論文は、査読付き国際会議 **The 28th ACM SIGKDD Conference on Knowledge Discovery and Data Mining（KDD ’22）** に採択され、会議録に収録されている。公開されている論文情報から確認できる査読結果は採択の事実までであり、個別の査読スコアや査読コメントは記載されていない。([arXiv][1])

総合的には、データセット論文として非常に完成度が高い。人手アノテーションの規模だけでなく、アノテータの訓練、詳細なガイドライン、重複アノテーション、文書単位分割、データ量・クラス数・分割方法のアブレーションまで含め、データセットの信頼性を多面的に検証している。

特に、文書単位で分割しなければ評価値が大幅に膨らむことを実証した点は、DocLayNetそのものを超えて重要な貢献である。一方、多言語性、スキャン文書、読み順や階層構造、複雑な幾何形状などは今後の拡張課題である。

**総合評価：強い採択相当。**
新規アルゴリズムの提案ではないが、文書AI研究のベンチマーク設計とモデル汎化性能の改善に長期的な影響を与える、実用性と再現性の高い研究である。
