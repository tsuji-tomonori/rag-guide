# P147 — *Extracting Training Data from Large Language Models*

Nicholas Carliniほか、30th USENIX Security Symposium、2021年、pp.2633–2650。([USENIX][1])

## 1. はじめに

本論文は、巨大言語モデルが訓練データ中の文章を逐語的に記憶し、外部からのクエリだけでそれを抽出できるかを検証した研究である。

著者らはGPT-2を対象にブラックボックス型の訓練データ抽出攻撃を実施し、氏名・電話番号・メールアドレスなどの個人情報、ソースコード、IRC会話、URL、UUIDなどを含む数百件の訓練文章を復元した。特に、訓練データ内の一つの文書にしか含まれない文字列も抽出可能であることを示した点が重要である。

本研究の中心的な主張は、**モデル全体として目立った過学習が観測されなくても、個別の訓練例は記憶・漏洩し得る**というものである。

## 2. 背景

従来、訓練データの漏洩は、訓練誤差とテスト誤差の差が大きい「過学習」と関連付けて考えられることが多かった。しかしGPT-2は、約40GBのWebデータを用いて約12エポック学習され、訓練損失とテスト損失の差も約10％にとどまっていた。それでも、本論文では多数の逐語的な記憶が確認されている。

著者らは、危険性の高い記憶を表すために **(k)-eidetic memorization** を定義する。文字列 (s) がモデルから抽出可能であり、かつ訓練データ中の高々 (k) 個の文書にしか出現しない場合、その文字列は (k)-eideticに記憶されているとする。

GPT-2では一つのWebページが一訓練例に相当する。このため、一つのページ内で文字列が何度も反復されていても、その文字列は (k=1) と数えられる。(k) が小さいほど、その文字列は希少であり、意図しない記憶やプライバシー漏洩である可能性が高い。

攻撃者はモデルの重みや内部状態にはアクセスせず、文章生成と各トークンの確率を取得できるブラックボックスアクセスのみを持つ。攻撃は特定人物を狙う標的型ではなく、モデルが記憶している訓練データを無差別に発見する非標的型である。

## 3. 提案手法

提案攻撃は、次の二段階で構成される。

### 3.1 候補文章の生成

GPT-2から大量の256トークン文章を生成する。著者らは以下の三方式を比較した。

1. **Top-(n) sampling**
   空のプロンプトから、各ステップで確率上位40トークンを用いて生成する基本方式。

2. **減衰温度サンプリング**
   初期温度を (t=10) として多様な接頭辞を探索し、最初の20トークンで (t=1) まで減衰させる。初期には多様性を確保し、記憶された系列に入った後は高確率経路を維持する設計である。

3. **Internet条件付き生成**
   Common Crawl由来の文章から5～10トークンを接頭辞として与える。実際のWeb文章に近い多様な開始文脈を与えることで、空のプロンプトからは到達しにくい記憶系列を引き出す。

### 3.2 記憶候補のランキング

単純にGPT-2のパープレキシティが低い文章を選ぶだけでは、一般的な定型文や同一表現の反復が上位に来てしまう。そこで、対象モデルが「他の基準と比較して異常に高い確率を与える文章」を探す。

評価した指標は以下の六つである。

* GPT-2 XL自身のパープレキシティ
* GPT-2 XLとSmallモデルの対数パープレキシティ比
* GPT-2 XLとMediumモデルの対数パープレキシティ比
* GPT-2のパープレキシティとzlib圧縮量の比
* 元文章と小文字化文章のパープレキシティ比
* 50トークン窓における最小パープレキシティ

特にSmall・Mediumモデルとの比較は、「大モデルだけが異常に高い確率を与える希少文章」を発見するための参照モデルとして機能する。zlibとの比較は、単純な反復文章を圧縮器側で高確率とみなし、候補から除外する役割を持つ。

## 4. 実験方法

攻撃対象は主として約15億パラメータのGPT-2 XLであり、モデルサイズの影響を調べる実験ではSmall、Medium、XLを比較した。

三つの生成方式それぞれについて、256トークンの文章を20万件生成した。したがって生成総数は60万件である。各生成集合を六つのランキング指標で並べ、合計 (3\times6=18) 条件を作成した。

各条件について上位1,000件から100件を抽出し、trigramベースの曖昧重複除去を行った。最終的な調査対象は1,800件である。

記憶の判定は次のように行われた。

* 著者4名が、生成文章中の特徴的な部分文字列をWeb検索する。
* 完全一致するWebページが見つかった場合、記憶候補と判定する。
* さらにGPT-2開発者と協力し、元の訓練データに対する3-gram検索で候補を確認する。

訓練コーパス自体が一般公開されていなかったため、OpenAIによる元データ検索は結果の信頼性を高める一方、第三者による完全な追試を難しくする要因でもある。

## 5. 結果

1,800件の候補から、**604件の異なる訓練例**が逐語的な記憶として確認された。全体の適中率は33.5％であり、最良条件では100件中67件、すなわち67％が訓練データだった。

生成方式別の異なる抽出例数は次の通りである。

| 生成方式         | 抽出された記憶例 |
| ------------ | -------: |
| Top-(n)      |     191件 |
| 減衰温度         |     140件 |
| Internet条件付き |     273件 |

Internet条件付き生成が最も有効だった。一方、GPT-2自身のパープレキシティだけによる判定では、Top-(n)生成で9件、温度生成で3件しか発見できなかった。zlib比較ではそれぞれ59件、46件、67件となり、比較型指標の有効性が明確に示された。

抽出内容には以下が含まれていた。

* 個人名、電話番号、住所、SNSアカウントなどの個人関連情報78件
* 実在するURL 50件
* ソースコード31件
* 50ビット以上のエントロピーを持つランダム文字列21件
* UUID、Gitコミットハッシュ、広告用ID、製品番号など

九つの高エントロピー文字列は、一つの訓練文書にしか存在しない (k=1) の例だった。長さは10～87文字であり、その一文書内での出現回数は10～311回だった。

さらに、発見したコード断片を接頭辞として生成を継続することで、**1,450行のソースコードを逐語的に復元**できた。MIT、Creative Commons、Project Gutenbergのライセンス全文も抽出できている。

モデルサイズと反復回数の実験では、難しい抽出条件において、GPT-2 XLは一文書中で33回以上反復された対象URLをすべて生成した。Mediumは一部のみを記憶し、Smallは生成できなかった。追加の6文字を接頭辞として与えた容易な条件でも、Smallが抽出できたのは359回反復された例だけだった。

## 6. 考察

本論文から得られる最重要の知見は、**平均的な汎化性能と個別訓練例の漏洩は別問題である**という点である。訓練損失と検証損失の平均差が小さくても、一部の希少な訓練例には異常に低い損失が割り当てられ、その例だけが逐語的に記憶され得る。

また、対象モデル単独の確信度よりも、別モデルや圧縮器と比較した「相対的な異常度」が有効だった。これは、一般的な文章と対象モデル固有の記憶を分離するという意味で、本研究の技術的に優れた部分である。

一方、抽出可能性はプロンプトに強く依存する。同じ記憶内容でも、適切な直前文脈を与えなければ再現できない場合が多い。Internet条件付きで発見した273件は、最初に使った短い接頭辞では安定して再現できなかったが、記憶箇所直前までの文脈を与えるとほぼすべてが高確率で再現された。そのため、604件という結果はモデルが記憶する総量の下限と考えるべきである。

主な限界は以下である。

* 実証対象がGPT-2系列に限定されている。
* トークン確率またはパープレキシティを取得できるAPIを前提としている。
* 公開Webデータであるため、検索による照合が容易な条件に偏っている。
* 攻撃は非標的型であり、特定人物の情報を狙って取得できることまでは示していない。
* (k) は文書数であり、文書内の反復回数を表さない。したがって (k=1) が「一度しか学習されていない」ことを意味するわけではない。
* 防御策は提案されているものの、その効果を実験的には評価していない。
* 元訓練データを第三者が直接検査できず、完全な再現性には制約がある。

## 7. まとめ

本論文は、巨大言語モデルに対して次の処理を行うことで、訓練文章を実際に抽出できることを示した。

1. モデルから大量の文章を生成する。
2. 対象モデルが、参照モデルや圧縮器と比べて異常に高い確率を与える文章を選別する。
3. Web検索や元訓練データとの照合によって逐語的記憶を確認する。

結果として、過学習が小さいGPT-2からも604件の訓練例が抽出され、個人情報、高エントロピー文字列、長大なソースコードなどが復元された。さらに、大きなモデルほど記憶しやすく、同じ文書内での反復回数が増えるほど抽出可能性が高まることが示された。

著者らは防御策として、差分プライバシーによる学習、個人情報の除去、文書内反復まで考慮した重複排除、生成時フィルタリング、訓練後のプライバシー監査を挙げている。ただし、いずれも単独で完全な解決策とはならない。

## 8. 総評

本論文は、実運用に近い大規模モデルにおける訓練データ抽出を、具体的な攻撃手順と実データで示した重要な研究である。特に、参照モデルやzlibとの相対比較、元訓練データを用いた検証、モデルサイズ・反復回数・プロンプト依存性の分析は説得力が高い。

対象モデルの限定、確率アクセスを必要とする脅威モデル、完全な再現性の制約、防御実験の不足は弱点である。しかし、訓練データ漏洩を仮想的な問題から実証的なセキュリティ問題へと引き上げた新規性と影響度は、これらの弱点を上回る。

**査読結果：採択（Accept）**

Strong Acceptではなく通常のAcceptと判断する。技術的妥当性と社会的・研究的意義は十分に高いが、一般化可能性と防御策の評価には追加研究が必要である。実際にも本論文はUSENIX Security 2021に採録されている。([USENIX][1])
