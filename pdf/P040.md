# 論文情報

**題名:** *Passage Re-ranking with BERT*
**著者:** Rodrigo Nogueira, Kyunghyun Cho
**初版:** 2019年1月、最終改訂版 v5 は2020年4月。([arXiv][1])

## 1. はじめに

本論文は、検索クエリに対して候補パッセージを並べ替える「パッセージ再ランキング」にBERTを適用した研究である。

BM25などの高速な検索手法で取得した候補を、BERTを用いてクエリとの意味的関連性に基づき再評価する。新しいTransformer構造を提案するのではなく、事前学習済みBERTを検索ランキングへ転用するだけの簡潔な方法によって、MS MARCOとTREC-CARの両方で当時の最高性能を達成した点が中心的な貢献である。MS MARCOでは、従来最高性能に対してMRR@10を相対約27%改善した。

## 2. 背景

一般的なオープンドメイン質問応答・検索システムは、次のような多段構成を取る。

1. BM25などで、数百から1,000件程度の候補文書を高速に取得する。
2. 高コストなモデルで各候補を精密に再ランキングする。
3. 上位10～50件を回答抽出・回答生成モデルへ渡す。

本論文が扱うのは、このうち第2段階の再ランキングである。検索対象全体にBERTを適用するのではなく、BM25で絞り込まれた候補だけをBERTで処理する。

当時すでにDRMM、KNRM、Co-PACRR、DUETなどのニューラルランキングモデルが提案されていたが、大規模なランキング用データセットが不足しており、古典的な検索手法に対して一貫した優位性を示すことは難しかった。

著者らは、次の二つがそろったことで状況が変化したと主張する。

* 実ユーザーのクエリと関連パッセージを含む大規模なMS MARCOデータセット
* 大規模コーパスから言語知識を獲得した事前学習済みBERT

すなわち、「大規模な教師データ」と「事前学習モデル」の組み合わせを、読解だけでなく情報検索にも適用した研究である。([ar5iv][2])

## 3. 提案手法

### 入力形式

クエリをBERTのSentence A、候補パッセージをSentence Bとして入力する。概念的には次の形式になる。

```text
[CLS] クエリ [SEP] 候補パッセージ [SEP]
```

クエリは最大64トークンに切り詰め、クエリ、パッセージ、特殊トークンを合わせた全長を最大512トークンとする。

### 関連度の計算

BERTの最終層における `[CLS]` ベクトルを単層ニューラルネットワークへ入力し、その候補パッセージがクエリに関連する確率 (s_j) を求める。

各パッセージはほかの候補とは独立に評価され、最終的に関連確率の降順で並べ替えられる。現在の用語では、クエリとパッセージを一つのTransformerへ同時入力する**クロスエンコーダ型再ランカー**に相当する。

### 学習目的

関連パッセージ集合を (J_{\mathrm{pos}})、非関連パッセージ集合を (J_{\mathrm{neg}}) とすると、損失は二値クロスエントロピーである。

[
L =
-\sum_{j\in J_{\mathrm{pos}}}\log s_j
-\sum_{j\in J_{\mathrm{neg}}}\log(1-s_j)
]

負例には、BM25で取得した上位候補中の非関連パッセージを使用する。したがって、完全に無関係なランダム文書ではなく、語彙的にはある程度クエリに近い「難しい負例」を学習できる。

一方、損失関数は候補間の順位関係を直接最適化するpairwiseまたはlistwise損失ではなく、候補ごとの関連・非関連を判定するpointwise学習である。

## 4. 実験方法

### MS MARCO

MS MARCOの学習データは、クエリ、関連パッセージ、非関連パッセージからなる約4億組のデータを含む。

開発セットは約6,900クエリ、評価セットは約6,800クエリで、各クエリについてBM25が取得した上位1,000パッセージが与えられる。平均すると各クエリに関連パッセージは約1件だが、BM25の候補集合内に正解が存在しないクエリもある。

主な学習設定は以下の通りである。

| 項目           |               設定 |
| ------------ | ---------------: |
| モデル          |       BERT Large |
| 最大系列長        |              512 |
| バッチサイズ       |              128 |
| 学習ステップ       |          100,000 |
| 処理したペア数      |           1,280万 |
| 学習時間         |            約30時間 |
| 計算資源         |         TPU v3-8 |
| 初期学習率        | (3\times10^{-6}) |
| Warmup       |       10,000ステップ |
| Weight decay |             0.01 |
| Dropout      |              0.1 |

実際に学習した1,280万ペアは全学習データの2%未満である。さらに3日間、合計約5,000万ペアまで学習しても、開発セットの改善は確認されなかった。

評価指標には、最初の関連パッセージが何位に出現したかを評価する **MRR@10** を用いる。

### TREC-CAR

TREC-CARでは、Wikipediaの記事タイトルと節タイトルを連結したものをクエリとし、その節に含まれる段落を関連パッセージとする。

* 学習セット：約230万クエリ
* 検証セット：約58万クエリ
* テストセット：約2,254クエリ
* 評価指標：MAP

TREC-CARのテスト文書はWikipedia由来であるため、通常のBERTは事前学習段階でテスト文書を見ている可能性がある。著者らはこのデータリークを避けるため、TREC-CARの学習側に属するWikipedia部分だけを使ってBERTを事前学習し直している。この対応は、実験設計上の重要な長所である。

ファインチューニング用の候補はAnseriniのBM25で取得した上位10件から構成され、全体で約3,000万ペアとなる。そのうち1,280万ペアを使って学習している。

## 5. 結果

主要結果は以下の通りである。論文中の「36.5」などの表記は、通常の小数表記では0.365に相当する。

| 手法               | MS MARCO Dev MRR@10 | MS MARCO Eval MRR@10 | TREC-CAR MAP |
| ---------------- | ------------------: | -------------------: | -----------: |
| BM25（Lucene、未調整） |               0.167 |                0.165 |        0.123 |
| KNRM             |               0.218 |                0.198 |            — |
| Conv-KNRM        |               0.290 |                0.271 |            — |
| IRNet            |               0.278 |                0.281 |            — |
| BERT Base        |               0.347 |                    — |        0.310 |
| **BERT Large**   |           **0.365** |            **0.358** |    **0.335** |

TREC-CARでは、調整済みAnserini BM25が0.153、従来ニューラル手法Co-PACRRが0.148であったため、BERT Largeの0.335は非常に大きな改善である。

MS MARCO評価セットでは、従来最高のIRNetが0.281、BERT Largeが0.358であり、絶対値で0.077、相対値で約27.4%改善している。これは論文の中心的な成果である。

### 学習データ量と性能

学習データ量を変化させた実験では、10万クエリ・パッセージペアだけで、BERT Largeは従来最高手法IRNetをMRR@10で1.4ポイント上回った。

性能は1,000件から100万件程度までは急速に改善するが、1,000万件前後からほぼ飽和している。これは、BERTの事前学習によって、下流タスクの教師データを比較的少量しか使用しなくても高い性能を得られることを示している。

## 6. 考察

### 研究上の意義

最大の意義は、事前学習済み言語モデルが情報検索の関連度判定にも極めて有効であることを、単純かつ再現可能な構成で示した点にある。

従来のランキングモデルでは、クエリ語と文書語の完全一致、n-gram一致、局所的な類似度などを明示的にモデル化することが多かった。これに対してBERT再ランカーは、クエリとパッセージを同時に自己注意へ入力することで、文脈を考慮した細粒度の相互作用を学習する。

特に、複雑な検索専用アーキテクチャを設計せず、事前学習済みモデルを二値分類としてファインチューニングするだけで大幅な改善を得たことは、その後のニューラル検索研究に強い影響を与える設計原則となった。

### 計算コスト

最大の問題は推論コストである。BERT Largeはクエリと候補パッセージの組ごとにTransformer全体を実行するため、1クエリにつき1,000候補を再評価すると1,000回のBERT推論が必要になる。

論文は検索精度を詳細に報告している一方、次の実用指標を報告していない。

* 1クエリ当たりのレイテンシ
* 1秒当たりの処理クエリ数
* GPUまたはTPU使用量
* 候補数と精度・速度のトレードオフ
* BERT BaseとLargeの計算効率比較

したがって、品質改善は明確であるが、オンライン検索サービスでそのまま利用可能かどうかは判断できない。

### 実験比較の問題

MS MARCOで比較対象となっているIRNetは、当時のリーダーボード上の未公開手法である。モデル構造や学習条件が論文内で説明されていないため、厳密な比較可能性には限界がある。

また、次の分析も不足している。

* 複数ランダムシードによる平均値・分散
* 統計的有意差検定
* 負例選択方法のアブレーション
* 最大系列長やクエリ切り詰め長の影響
* BERTの事前学習とモデル規模の寄与分解
* 成功例・失敗例の定性的分析

BERT BaseとLargeの比較、学習データ量の比較は行われているが、方法のどの構成要素が改善を生んだのかを分離するには不十分である。

### データセット上の制約

MS MARCOは関連判定が疎であり、実際には関連していても正解としてラベル付けされていないパッセージが負例として扱われる可能性がある。また、BM25の上位1,000件内に正解がなければ、再ランカーは正解を上位に移動できない。したがって、最終性能は第一段検索の再現率に制約される。

TREC-CARについても、著者らが用いた自動判定はすべての候補を評価できる利点がある一方、人間による関連性判断とは異なるバイアスを持つ可能性がある。論文では、手動判定が各参加システムの上位5件程度に限られるため、自動判定を採用している。

## 7. まとめ

本論文は、BM25が取得した候補パッセージを、クエリとともにBERTへ入力して関連度を推定する再ランキング手法を提案した。

手法自体は、BERTの `[CLS]` 表現を使った二値分類という非常に単純なものである。しかし、MS MARCOではMRR@10が0.358、TREC-CARではMAPが0.335となり、当時の既存手法を大幅に上回った。さらに、比較的少量のファインチューニングデータでも高性能になることを示した。

本研究の本質的なメッセージは、検索専用の複雑な構造を設計するよりも、強力な事前学習モデルにクエリと文書を共同入力してファインチューニングする方が、高品質な関連度判定を実現できる場合がある、という点にある。

| ------ | --------------------------------------- |
| 新規性    | モデル構造の新規性は低いが、BERTの検索再ランキングへの適用は当時として重要 |
