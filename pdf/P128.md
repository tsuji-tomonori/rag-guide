# P128：Measuring Attribution in Natural Language Generation Models

**書誌情報**：Hannah Rashkinほか、*Computational Linguistics*, 49(4), pp.777–840, 2023。
**主題**：生成文が「指定された情報源によって裏付けられているか」を測る、タスク横断的な評価枠組み **AIS（Attributable to Identified Sources）** の提案。([ACL Anthology][1])

## 1. はじめに

自然言語生成モデルは流暢な文章を生成できる一方、入力文書にない情報を生成する「ハルシネーション」が問題になる。本論文は、生成文の世界知識としての正しさを直接判定するのではなく、**生成された主張が、システムによって明示された情報源から正当に導けるか**を評価する。

著者らの主要な貢献は、AISの形式的定義、実務的な二段階アノテーション手順、会話型質問応答・要約・表から文章への生成という異なるタスクでの検証、詳細なアノテーション指針と評価データの公開である。

## 2. 背景

従来研究では、生成文の問題を「hallucination」「faithfulness」「factual consistency」「groundedness」など、タスクごとに異なる名称と基準で評価していた。そのため、会話生成、要約、data-to-textを横断して利用できる統一的かつ再現可能な評価基準が不足していた。

AISが特に重視するのは、**attributionとfactualityの分離**である。

* **Attribution**：その主張を、指定された情報源が支持しているか。
* **Factuality**：その主張が、現実世界において真であるか。

したがって、誤った情報源を正確に要約した出力はAISである可能性があるが、事実として正しいとは限らない。逆に、現実には正しい主張でも、指定された情報源に書かれていなければAISではない。このためAISは、事実性評価全体の「必要な一部分」であり、情報源の信頼性評価とは別に用いる必要がある。

また、会話中の「彼」「それ」「25歳」といった表現は、単独では意味が確定しない。そこで本論文は、文脈から補完された明示的な命題を表す語用論上の概念 **explicature** を導入している。

## 3. 提案手法

### AISの形式的定義

生成文を (s)、言語的文脈と時点の組を (c)、指定された情報源を (P) とする。生成文がAISであるためには、次の条件を満たす必要がある。

1. システムが生成文とともに情報源 (P) を特定している。
2. 文脈 (c) のもとで生成文 (s) の意味が解釈可能である。
3. 文脈から復元されたexplicature (E(c,s)) が、独立して解釈できる命題である。
4. 一般的な聞き手が、十分な確信をもって「情報源 (P) によれば、(E(c,s)) である」と判断できる。

つまりAISは、単純な文字列一致ではなく、照応、省略、時制、常識的推論を含めて、生成文が情報源を正確に表現しているかを判定する枠組みである。

### 二段階アノテーション

形式定義を人手評価に落とし込むため、評価を次の二段階に分ける。

**第1段階：Interpretability**

情報源を隠した状態で、生成文と必要な会話履歴だけを提示し、「生成文が伝えている情報を明確に理解できるか」を二値判定する。情報源を先に見せると、評価者が曖昧な生成文を情報源から補完して、過度に寛容になるためである。

**第2段階：Attribution**

第1段階で解釈可能とされた場合に限り、情報源を提示し、「生成文に含まれるすべての情報が、情報源によって完全に支持されているか」を判定する。一部だけ支持されている場合や、細部が変更されている場合も非AISとなる。

さらに、表示崩れ、欠損、文字化け、専門知識なしには理解できない情報源などは **Flag** として評価対象から除外する。各例は5人が評価し、多数決を最終判定とする。AIS値は、基本的に「解釈可能とされた例のうち、完全に情報源帰属可能だった割合」である。

## 4. 実験方法

3種類のNLGタスク、4データセットでAISを検証した。

| タスク     | データセット              | 比較対象                                            |
| ------- | ------------------- | ----------------------------------------------- |
| 会話型質問応答 | QReCC               | T5の事前学習／fine-tuning、証拠文書あり／なし、Small／Base        |
| 知識接地型会話 | Wizard of Wikipedia | WoW Baseline、Dodeca、T5、faithfulness制御付きT5       |
| 要約      | CNN/DailyMail       | 抽出型MatchSum、ハイブリッド型Pointer-Generator、抽象型BigBird |
| 表から文章生成 | ToTTo               | ByT5-Base／XL、T5-Small／Base／XL                   |

QReCC、Wizard of Wikipedia、CNN/DailyMailでは主に200例を抽出し、ToTToでは200件のモデル出力と200件の参照文を評価した。会話型質問応答では、検索性能の影響を除くため、正解を含むoracle取得文書・文を情報源として使用している。

評価はインド・ハイデラバードを拠点とする9名の有給フルタイム評価者が担当した。評価者は言語学の専門家ではないが、パイロット評価、判定理由の記述、著者によるフィードバック、難例の裁定、内部監査を通じて訓練された。

評価品質は、F1、ペア一致率、Krippendorffのαで測定した。さらにQReCCとCNN/DailyMailについては各50例を著者2名が評価し、クラウド評価者との一致度も調べた。またCNN/DailyMailの200例を用いて、解釈可能性判定の削除、5段階尺度、文単位評価というアブレーションも行った。

## 5. 結果

主要結果は以下の通りである。Intは解釈可能率、AISは解釈可能例に対する帰属可能率を表す。

| データセット              | システム              |   Int |  AIS |
| ------------------- | ----------------- | ----: | ---: |
| QReCC               | T5-FT・証拠あり・Small  |  99.0 | 87.9 |
| QReCC               | T5-FT・証拠なし・Small  |  57.8 | 25.2 |
| QReCC               | 参照回答              |  99.0 | 87.8 |
| Wizard of Wikipedia | 制御付きT5            |  99.5 | 92.4 |
| Wizard of Wikipedia | WoW Baseline      |  84.4 | 19.8 |
| Wizard of Wikipedia | 参照回答              | 100.0 | 15.6 |
| CNN/DailyMail       | MatchSum・抽出型      |  90.0 | 99.4 |
| CNN/DailyMail       | Pointer-Generator |  90.0 | 97.8 |
| CNN/DailyMail       | BigBird・抽象型       |  90.0 | 87.2 |
| CNN/DailyMail       | 参照要約              |  86.0 | 54.1 |
| ToTTo               | T5-Small          |  86.5 | 88.6 |
| ToTTo               | 参照文               |  83.9 | 91.0 |

QReCCでは、証拠文書を入力しないモデルのAISが約22～25%まで低下した一方、fine-tuning済みかつ証拠ありのモデルは約87～88%を達成した。これはAISが、モデルが本当に情報源を利用しているかを識別できることを示す。

Wizard of Wikipediaでは、faithfulnessを高める制御を導入したT5が92.4%となり、通常のT5の39.8%、ベースラインの19.8%を大きく上回った。したがって、AISはモデル設計による帰属性の改善を明確に捉えている。

CNN/DailyMailでは、抽出型、コピー機構を持つハイブリッド型、抽象型の順にAISが低下した。これは情報源を直接コピーする方式ほど、非帰属情報を追加しにくいためである。ただし、AISが高いことは、要約の情報量、簡潔性、読みやすさまで優れていることを意味しない。

ToTToでは各モデルのAISは85.1～88.6%に集中し、モデル規模を大きくしても一貫した改善は見られなかった。参照文は91.0%で、他のデータセットとは異なりモデル出力を上回った。

### アノテーション品質

AIS判定の評価者間F1は0.92～0.95、Krippendorffのαは0.69～0.79であり、タスクをまたいで中程度から高い一致が得られた。解釈可能性判定は、特に長い要約を扱うCNN/DailyMailで低く、αは0.46だった。

一方、専門家との一致は評価者間一致より低い。CNN/DailyMailの解釈可能性では専門家とのαが−0.04だったが、主要指標であるAISでは0.61だった。評価者は専門家より曖昧な表現を許容する傾向があり、固有名詞の同一性や省略表現で判定が分かれた。

### 参照データの問題

重要な発見は、**gold referenceであってもAISとは限らない**ことである。Wizard of Wikipediaの参照回答は15.6%、CNN/DailyMailの参照要約は54.1%しか完全なAISと判定されなかった。

Wizard of Wikipediaでは、データ作成者が個人的経験や感想を加えていた。CNN/DailyMailでは、要約の元となった記事ハイライトが、本文にない見出し、画像、読者の背景知識などを利用していた。この結果は、参照文を無条件に正解とみなす既存評価の危険性を示している。

### アブレーション

元の二値・出力全体評価は平均203.7秒で最も高速だった。文単位評価は258.1秒を要し、より細粒度になる代わりに約1分の追加コストが発生した。5段階尺度は評価者から好まれ、専門家との一致がわずかに改善したものの、統計的に有意ではなかった。どの設計でもモデル順位は変わらなかった。

## 6. 考察

本論文の最大の強みは、曖昧だった「情報源への忠実性」を、言語学的に明確な定義と再現可能な評価工程に落とし込んだ点である。explicatureを導入したことで、会話履歴を必要とする応答にも適用できる。また、証拠なしモデルのスコア低下や、faithfulness制御付きモデルの改善を検出しており、構成概念妥当性も比較的高い。

一方、AISには明確な限界がある。第一に、AISは情報源との整合性だけを測り、情報源自体の正確性、信頼性、偏りを評価しない。第二に、実験ではoracle情報源を使っているため、実運用で重要となる検索失敗や不適切な出典選択を対象としていない。

第三に、出力全体を二値判定するため、わずか一箇所の誤りで全体が非AISとなり、誤りの位置や種類を示せない。反対に、情報源から短く自明な文をコピーすれば高得点を得られるため、関連性、完全性、有用性、情報量などの指標と併用する必要がある。

第四に、判定は評価者の常識、固有名詞に対する知識、どの程度の推論を許すかに依存する。専門家との一致が必ずしも高くないことから、AISは完全に客観的な尺度ではなく、評価方針と許容する推論範囲の明文化が不可欠である。

さらに、質問、命令、雑談と情報提示が混在する応答は形式定義の対象外である。英語のみ、一つの評価者集団、各条件約200例という検証範囲も、長文・複数情報源・専門領域への一般化には十分ではない。著者ら自身も、AISを事実確認全体の必要条件にすぎないと位置付け、将来的には自動評価と人手評価の組み合わせが必要だと論じている。

## 7. まとめ

本論文は、生成文が明示された情報源によって完全に支持されているかを測るAISを提案した。AISは新しい生成モデルそのものではなく、**生成結果の根拠性を測るための評価規約**である。

二段階アノテーションにより、解釈不能な生成文と、解釈可能だが情報源に支持されない生成文を分離できる。実験では、証拠文書の利用やfaithfulness制御による改善を捉え、複数のNLGタスクで安定した評価者間一致を示した。

特に、既存データセットの参照回答にも多くの非AIS情報が含まれることを明らかにした点は重要である。ただし、AIS単独では真偽、情報源の品質、回答の関連性や完全性を保証できず、これらを測る補完的評価と併用すべきである。
