# P126：Learning Filter-Aware Distance Metrics for Nearest Neighbor Search with Multiple Filters

## 1. はじめに

本論文は、ベクトル類似度だけでなく、複数のメタデータ条件をすべて満たすデータを検索する **MultiFilterANN** を扱う。中心的なアイデアは、ベクトル距離とフィルタ不一致度を組み合わせる際のペナルティを人手で決めるのではなく、学習クエリから自動的に推定し、その距離関数を検索時だけでなくグラフインデックス構築時にも利用することである。

論文はICML 2025の第1回Vector Databases Workshop（VecDB 2025）のProceedings掲載論文である。([OpenReview][1])

## 2. 背景

通常の近似最近傍探索では、クエリベクトル (q) に近いベクトルを検索する。一方、実際の検索システムでは、「特定の言語」「特定のカテゴリ」「特定の年代」のような離散的フィルタが併用される。

AND型の複数フィルタでは、クエリのラベル集合 (S_q) が候補ベクトルのラベル集合 (S_v) にすべて含まれる、すなわち

[
S_q\subseteq S_v
]

を満たす候補だけを返す必要がある。

従来方式には、まず通常のANN検索を実行してから条件不一致の候補を除去するポストフィルタ方式と、探索中に条件不一致ノードを完全に排除する方式がある。前者は大量の候補を余分に取得する必要があり、後者はグラフ上の探索経路を遮断して局所解に陥りやすい。先行するMultiFilterANNでは不一致度に固定ペナルティを加える方法が導入されたが、適切なペナルティはベクトル距離の分布やラベルの疎密によって異なるという問題がある。([arXiv][2])

## 3. 提案手法

### フィルタ認識距離

クエリ (q) と候補 (v) のラベル一致率を

[
m(q,v)=\frac{|S_q\cap S_v|}{|S_q|}
]

と定義する。すべてのフィルタを満たす場合は (m(q,v)=1) となる。

提案スコアは

[
D(q,v)=d(q,v)+w_m\left(1-m(q,v)\right)
]

である。ここで (d(q,v)) はコサイン距離などの通常のベクトル距離、(w_m\geq0) はフィルタ不一致に対するペナルティ重みである。(w_m) が大きいほどフィルタ一致が重視され、小さいほどベクトル距離が重視される。([OpenReview][1])

### 学習用ペアの生成

各訓練クエリについて全件走査を行い、フィルタを完全に満たす候補 (v_i) を正例とする。その正例よりベクトル距離は近いが、フィルタを完全には満たさない候補 (v_j) を負例として組み合わせる。

つまり、通常のベクトル距離だけなら負例 (v_j) が上位になる状況で、フィルタを満たす正例 (v_i) が提案距離では上位になるように学習する。([OpenReview][1])

### 線形計画問題

重み (w_m) とスラック変数 (s_{q,i,j}) を、次の線形計画問題で求める。

[
\min_{w_m,s}
\left(
w_m+
\alpha\frac{1}{|\mathcal{T}|}
\sum_{(q,i,j)\in\mathcal{T}}s_{q,i,j}
\right)
]

制約は

[
d(q,v_i)+w_m(1-m(q,v_i))+\epsilon
\leq
d(q,v_j)+w_m(1-m(q,v_j))+s_{q,i,j}
]

である。実験ではマージンを (\epsilon=0.01) としている。スラックを許すことで、すべての順位制約を完全に満たせない場合にも解を得られる。実装にはPuLPが用いられている。([OpenReview][1])

直感的には、フィルタ不一致候補が持つ「ベクトル距離上の有利さ」を打ち消すのに必要な最小限のペナルティを、複数の訓練クエリから推定している。

### インデックス構築への適用

検索時だけでなく、データ点間にも

[
D(v_1,v_2)
==========

d(v_1,v_2)+
w_m\left(1-
\frac{|S_{v_1}\cap S_{v_2}|}{|S_{v_1}|}
\right)
]

を適用する。これにより、ベクトル空間上で近いだけでなく、類似したラベルを持つデータ点同士がグラフ上で接続されやすくなる。

FilteredDiskANNのGreedy Searchを変更したWeightedGreedySearchを用いてグラフを構築し、検索時にも同じ学習済みスコアで探索候補を優先する。さらに、条件を満たすデータが10万件未満と推定される場合は、転置インデックスで対象を絞って厳密検索を行い、それ以上ならグラフ検索を行うクエリプランナを使用する。([OpenReview][1])

## 4. 実験方法

実験には次の2種類のデータセットが用いられている。

**YFCC1M**は100万件の画像データで、192次元のCLIP埋め込みを使用する。クエリはテキスト記述の埋め込みであり、カメラ機種、撮影年、場所などをフィルタラベルとする。クエリ条件は単一ラベルまたは2ラベルのAND条件である。訓練クエリは11,865件、評価クエリは5,718件である。

**Wikipedia-35M**は約3,500万件のWikipediaパッセージからなり、各データは768次元の埋め込みを持つ。ラベルはYAKEによるキーワード抽出、Universal Sentence Encoderによる意味的ラベル対応、Wikipediaカテゴリから生成される。訓練クエリは308件、評価クエリは464件である。([OpenReview][1])

比較手法は、学習距離をインデックス構築と検索の両方で使う提案手法、通常距離で構築したインデックスに固定ペナルティ検索を適用する方式、通常のANN検索後にフィルタリングする方式の3種類である。

評価指標はRecall@10と、検索中に行われた平均ベクトル距離計算回数である。後者をハードウェア非依存の検索コスト、すなわちレイテンシの代理指標としている。([arXiv][2])

## 5. 結果

学習された重みは、Wikipedia-35Mで

[
w_m=0.204148
]

YFCC1Mで

[
w_m=0.017787
]

となった。値が大きく異なることから、固定値ではなくデータセットごとにペナルティを調整する必要性が示唆される。([OpenReview][1])

Wikipedia-35Mでは、提案手法のRecall–距離計算回数曲線が固定ペナルティ方式とポストフィルタ方式をほぼ一貫して上回っている。YFCC1Mでも、特に少ない距離計算回数で高いRecallを達成している。一方、計算量を十分に増やした領域では固定ペナルティ方式との差は縮小している。論文全体としては、固定ペナルティ方式に対して精度が5～10ポイント向上したと報告している。

フィルタ認識距離で構築したグラフを通常の無条件ANN検索に利用した場合も、標準グラフとほぼ同等のRecallが維持されている。したがって、フィルタ検索用にグラフを変更しても、無条件検索の品質を大きく損なわないことが示されている。

ただし、クエリプランナによってグラフ検索に回されたYFCC評価クエリは5,718件中1,727件、約30%に限られる。残りの選択性が高いクエリは厳密検索側に送られている。Wikipediaでは464件すべてがグラフ検索に回されている。([OpenReview][1])

## 6. 考察

本手法の長所は、非常に単純で解釈可能なモデルでありながら、フィルタ分布に応じて検索グラフ自体を変化させられる点にある。単に検索時の候補順位を変えるだけでなく、類似ラベルを持つデータ間の経路をインデックスに埋め込むため、フィルタを満たす近傍へ到達しやすくなる。

一方、以下の点には注意が必要である。

第一に、学習されるのはデータセット全体で1個のスカラー (w_m) である。したがって、ラベルごとの重要性、頻度、相関、クエリごとの選択性を直接学習しているわけではない。論文が問題意識として挙げる「クエリによって異なるフィルタ重要度」に対しては、依然として表現力が限定的である。

第二に、インデックス構築時のラベル一致度は

[
|S_{v_1}\cap S_{v_2}|/|S_{v_1}|
]

であり、一般に (v_1) と (v_2) を入れ替えると値が変わる。このため、提案された (D(v_1,v_2)) は通常の数学的意味での距離尺度ではなく、非対称な探索スコアと呼ぶ方が正確である。DiskANN系アルゴリズムに非対称スコアを導入した際の理論的性質は論文では十分に解析されていない。([OpenReview][1])

第三に、実験では「学習済み重み」と「フィルタ認識インデックス構築」を同時に変更している。一方のベースラインは標準インデックスと固定ペナルティの組合せであるため、性能向上が重み学習によるものか、インデックス構築によるものかを切り分けられない。「標準インデックス＋学習済み重み」や「固定重み＋フィルタ認識インデックス」といったアブレーションが必要である。([arXiv][2])

第四に、関連研究ではCAPS、ACORN、IVF2、Filtered-DiskANNなどを挙げているが、実験比較は固定ペナルティとポストフィルタの2方式に限られる。また、実測レイテンシ、インデックス構築時間、メモリ使用量、インデックスサイズ、複数試行の分散も報告されていない。システム研究として一般性を主張するには、評価がまだ限定的である。([OpenReview][1])

最後に、学習には訓練クエリごとの全件走査と正解近傍が必要であり、3,500万件規模では事前計算コストが大きい。論文自身も、将来的にはクエリや正解近傍を必要とせず、データセット統計だけから距離関数を学ぶことを課題としている。([arXiv][2])

## 7. まとめ

本論文は、複数フィルタ付きANN検索における固定ペナルティを、ランキング制約に基づく線形計画で学習する方式へ置き換えた。さらに、その重みを検索時だけでなくグラフインデックス構築時にも用いることで、フィルタを満たす候補へ到達しやすいグラフを生成する。

手法は単純で実装しやすく、Wikipedia-35Mのような大規模データでも有効性が示されている。特に、少ない距離計算回数で高いRecallを得られる点は実用上重要である。一方で、学習対象が単一の重みであること、比較手法とアブレーションが不足していることから、現時点では一般的なフィルタ付きANN問題を包括的に解決したというより、固定ペナルティをデータ駆動で調整する有望な第一段階と評価するのが適切である。
