# 1. はじめに

本論文 **“ReAct: Synergizing Reasoning and Acting in Language Models”** は、大規模言語モデルに「言語による推論」と「外部環境に対する行動」を交互に生成させる **ReAct** を提案した研究である。著者は Shunyu Yao らで、ICLR 2023に採択されている。([arXiv][1])

ReActの中心的な発想は、LLMに回答を直接生成させるのではなく、次のループを実行させることである。

> **Thought（推論）→ Action（行動）→ Observation（観測）→ Thought → …**

例えば質問応答では、LLMが「まず人物Aを検索し、得られた情報から人物Bを調べる必要がある」と推論し、Wikipedia検索を実行し、その結果を読んで次の検索や最終回答を決める。従来のChain-of-Thoughtがモデル内部だけで推論を完結させるのに対し、ReActは外部情報や環境から得た観測を推論過程にフィードバックする。

---

# 2. 背景

本研究以前、LLMの利用方法は主に次の二つに分かれていた。

**推論中心の手法**では、Chain-of-Thought（CoT）のように中間的な推論文を生成させる。複雑な問題を段階的に解ける一方、推論はモデル内部の知識だけに依存するため、誤った事実を生成すると、その誤りが後続の推論へ伝播しやすい。

**行動中心の手法**では、検索、Web操作、ロボット操作などのアクションをLLMに直接生成させる。しかし、Act-only方式では、高水準の目標分解、進捗管理、例外処理、探索方針の修正が明示的に行われない。このため、同じ失敗行動を繰り返したり、現在の状態を見失ったりする。([ar5iv][2])

論文が取り組む問いは、次のように整理できる。

> 推論と行動を別々に扱うのではなく、相互に補完させれば、LLMはより正確かつ柔軟に問題を解けるのではないか。

ReActでは、推論が行動計画を導き、行動によって得られた観測が次の推論を修正する。著者らはこれを **reason to act** と **act to reason** の相乗効果として位置づけている。([ar5iv][2])

# 3. 提案手法

## 3.1 基本構造

通常のエージェントは、観測や履歴を入力として環境内のアクションを選択する。ReActでは、そのアクション空間に自然言語による **Thought** を追加する。

Thoughtは環境そのものを変化させない内部的な言語行動であり、Observationも返されない。一方、Actionは検索や移動などを実行し、環境からObservationを受け取る。([ar5iv][2])

Thoughtには、主に以下の役割がある。

* 問題や目標をサブゴールへ分解する
* 現在までに判明した情報を要約する
* サブゴールの完了状況を管理する
* 次に調査すべき対象を決める
* 検索失敗時にクエリを変更する
* 例外や想定外の観測に応じて計画を修正する
* 最終回答を複数の観測から統合する

重要なのは、特殊なニューラルネットワークを新たに設計しているわけではない点である。基本的には、Thought・Action・Observationを含む少数の実演例をプロンプトとして提示し、既存のLLMに同形式の軌跡を生成させる。

## 3.2 タスクに応じた推論密度

ReActでは、すべてのタスクで毎回Thoughtを生成するわけではない。

質問応答や事実検証では、比較的短い推論過程の各段階で、

> Thought → Action → Observation

を密に繰り返す。

一方、ALFWorldのように数十回の行動を必要とするタスクでは、各行動の前に推論すると冗長になる。そのため、目標分解、探索方針の変更、サブゴール完了など、重要な場面だけでThoughtを生成する **疎な推論** を用いる。([ar5iv][2])

## 3.3 内部知識と外部知識の統合

著者らは、ReAct単体に加えてCoTのSelf-Consistencyとの組み合わせも試している。

* **ReAct → CoT-SC**：ReActが規定ステップ内に回答できない場合、CoT-SCへ切り替える
* **CoT-SC → ReAct**：複数のCoT出力で多数派が十分に形成されない場合、外部検索を行うReActへ切り替える

これは、CoTの柔軟な推論能力とReActの外部情報による事実性を組み合わせる方式である。

# 4. 実験方法

## 4.1 使用モデル

主要実験では、重みを更新しない **PaLM-540B** をfew-shot promptingで使用している。一部の追加実験ではGPT-3 `text-davinci-002`、ファインチューニング実験ではPaLM-8BおよびPaLM-62Bを使用した。([ar5iv][2])

## 4.2 知識集約型タスク

### HotpotQA

複数のWikipedia記事を横断して推論するmulti-hop質問応答である。評価指標は完全一致率、Exact Matchである。

### FEVER

与えられた主張について、Wikipediaの情報に基づき、支持、反証、情報不足を分類する事実検証タスクである。評価指標は分類精度である。

両タスクとも、モデルには質問または主張のみを与え、根拠文書は直接与えない。ReActは次のWikipedia APIを利用する。

* `search[entity]`
* `lookup[string]`
* `finish[answer]`

HotpotQAでは6例、FEVERでは3例の手作業によるReAct軌跡をプロンプトに使用した。比較対象はStandard、CoT、21サンプルによるCoT-SC、Act-onlyである。([ar5iv][2])

## 4.3 対話的意思決定タスク

### ALFWorld

家庭環境内で物品を探し、加熱、冷却、洗浄、移動などを行うテキストゲームである。134件の未観測ゲームで評価した。

各タスクタイプについて3本の軌跡を作成し、そのうち2本を用いる順列から6種類のプロンプトを構成している。比較対象には、Thoughtを除去したAct-onlyと、模倣学習エージェントBUTLERが含まれる。

### WebShop

実在の商品情報をもとに構築された模擬オンラインショッピング環境である。ユーザーの価格、色、サイズ、用途などの条件を満たす商品を検索し、オプションを選択して購入する。

500件のテスト指示を用い、希望属性をどの程度満たしたかを示すScoreと、すべての条件を満たした割合であるSuccess Rateを評価した。ReActはone-shot promptingで、模倣学習および模倣学習＋強化学習と比較された。([ar5iv][2])

## 4.4 ファインチューニング

ReActが生成し、最終回答が正解だった3,000件の軌跡を使って、PaLM-8Bおよび62Bをファインチューニングした。入力からThought、Action、Observationを含む軌跡全体を生成するよう学習させている。([ar5iv][2])

# 5. 結果

## 5.1 HotpotQA・FEVER

| 手法             | HotpotQA EM | FEVER Accuracy |
| -------------- | ----------: | -------------: |
| Standard       |        28.7 |           57.1 |
| CoT            |        29.4 |           56.3 |
| CoT-SC         |        33.4 |           60.4 |
| Act-only       |        25.7 |           58.9 |
| ReAct          |        27.4 |           60.9 |
| CoT-SC → ReAct |        34.2 |       **64.6** |
| ReAct → CoT-SC |    **35.1** |           62.0 |
| 教師ありSoTA       |        67.5 |           89.5 |

数値は論文Table 1による。

ReAct単体はAct-onlyを両タスクで上回ったが、HotpotQAではCoTより低い。したがって、「ReAct単体がすべての推論タスクでCoTを上回った」という結果ではない。

一方、FEVERではReActがCoTを上回っている。事実のわずかな違いによってラベルが変化するタスクでは、内部知識だけで推論するより、外部情報を検索する効果が大きいと考えられる。

最も高い結果はReActとCoT-SCを切り替えるハイブリッド方式であり、HotpotQAで35.1、FEVERで64.6を達成した。ただし、教師ありのタスク特化手法とは依然として大きな差がある。

## 5.2 幻覚と失敗分析

著者らは、HotpotQAについてReActとCoTから正解例・不正解例を各50件ずつ抽出し、合計200軌跡を人手で分析した。

正解例のうち、推論過程と事実の両方が正しかった割合は、ReActが94%、CoTが86%であった。正解ラベルを偶然出したものの、途中に誤った事実や推論を含むFalse Positiveは、ReActが6%、CoTが14%だった。

不正解例では、CoTの失敗の56%が幻覚に分類されたのに対し、ReActでは幻覚による失敗は観察されなかった。ただし、ReActには推論ループや行動選択ミスがあり、失敗の47%が推論エラー、23%が不適切な検索結果に起因していた。([ar5iv][2])

## 5.3 ALFWorld

| 手法                 | 全タスク成功率 |
| ------------------ | ------: |
| Act-only：best of 6 |     45% |
| ReAct：6設定の平均       |     57% |
| ReAct：best of 6    | **71%** |
| ReAct-IM：best of 6 |     53% |
| BUTLER：best of 8   |     37% |

ReActの最良設定は71%で、BUTLERの37%を34ポイント上回った。また、ReActの6プロンプト平均57%もAct-onlyの最良値45%を上回っている。

Act-onlyでは、物品探索の途中で目標を見失う、既に確認した場所を再探索する、複数のサブゴールを適切に管理できないといった問題が見られた。ReActでは、Thoughtが作業記憶と高水準の計画として機能している。

## 5.4 WebShop

| 手法        |    Score | Success Rate |
| --------- | -------: | -----------: |
| Act-only  |     62.3 |        30.1% |
| ReAct     | **66.6** |    **40.0%** |
| 模倣学習      |     59.9 |        29.1% |
| 模倣学習＋強化学習 |     62.4 |        28.7% |
| 人間        |     82.1 |        59.6% |

ReActはSuccess RateでAct-onlyを9.9ポイント上回り、従来手法に対して約10ポイントの改善を示した。ただし、人間の59.6%には大きく及ばない。

# 6. 考察

## 6.1 強み

最大の強みは、**極めて単純なプロンプト形式で、推論、検索、計画、環境操作を統一したこと**である。特定タスク専用の複雑なエージェント設計を導入せず、Thought・Action・Observationという共通形式を、質問応答、事実検証、テキストゲーム、Web操作へ適用している。

また、Act-onlyとの比較が、同じ実演軌跡からThought部分だけを除去する形で設計されている。したがって、少なくとも両者の比較では「推論文を追加すること」の効果が比較的明確である。

さらに、外部情報を観測として取り込むことで、CoTのもっともらしい誤情報を抑制した点は重要である。単に最終精度を上げるだけでなく、どの検索結果に基づいて回答したかを追跡できるため、誤りの診断もしやすい。

## 6.2 限界

第一に、ReAct単体の質問応答性能は必ずしも高くない。HotpotQAではCoTを下回り、教師ありSoTAとの差も大きい。ReActは「常に高精度な検索拡張推論」ではなく、検索が成功するかどうかに強く依存する。

第二に、論文で使われたWikipedia APIは単純で、ページ名や検索語が適切でなければ情報を取得できない。検索結果エラーがReActの失敗の23%を占めており、外部情報による事実性と、検索失敗による脆弱性が表裏一体になっている。

第三に、ALFWorldでは最良プロンプトの71%が強調されているが、6プロンプト平均は57%である。平均値でも比較手法より高いものの、プロンプト選択による変動は無視できない。実用上は、最良設定を事前に知ることができないため、平均性能や分散を重視すべきである。

第四に、「推論軌跡が読める」ことと「その軌跡がモデル内部の真の判断過程に忠実である」ことは異なる。本論文が実証しているのは主として可読性、診断可能性、外部情報との対応関係であり、Thoughtが内部計算の忠実な説明であることまでは保証していない。

第五に、主要モデルのPaLM-540Bは当時一般公開されておらず、完全な再現性に制約がある。著者らもこの問題を認め、プロンプトの公開やGPT-3による追加実験で補っている。([ar5iv][2])

最後に、実験環境はWikipedia、ALFWorld、模擬WebShopに限定されている。実世界のAPIや物理環境で行動させる場合、誤操作、個人情報取得、不適切な外部アクセスなどの安全性が問題になる。論文でも、外部環境と接続されたLLMが有害な行動を取る可能性を明示している。([ar5iv][2])

# 7. まとめ

ReActは、LLMに推論だけ、または行動だけをさせるのではなく、

> **推論によって行動を選び、行動結果によって推論を更新する**

という閉ループを導入した手法である。

知識タスクでは、外部検索によってCoTの幻覚を抑制し、特にCoT-SCとのハイブリッドで最良結果を得た。意思決定タスクでは、目標分解、状態追跡、探索方針の修正にThoughtが有効であり、ALFWorldとWebShopでAct-onlyや学習ベースの比較手法を上回った。

一方、検索失敗、推論ループ、プロンプト依存性、実行コスト、実世界での安全性は未解決である。ReActは完成された汎用エージェントというより、現在のLLMエージェントにおける「推論とツール利用を交互に行う」という基本設計を確立した研究と評価できる。

# 8. 総評

**実際の査読結果：ICLR 2023採択、Notable Top 5%。** OpenReview上でもICLR 2023の上位5%論文として記録されている。([OpenReview][3])

**本稿に対する査読判定：Strong Accept。**

提案自体は単純だが、推論と行動を一つの軌跡として統合し、質問応答からWeb操作まで複数領域で有効性を示した点に高い新規性と一般性がある。特に、外部観測によるグラウンディング、推論を用いた長期行動の管理、失敗過程の診断可能性という三点は、その後のLLMエージェント研究の基本構成につながる重要な貢献である。

ただし、知識タスクでは教師ありSoTAに遠く、ReAct単体がCoTを常に上回るわけでもない。したがって評価すべき中心的貢献は、絶対性能よりも、**Reasoning–Action–Observationループを汎用的なLLM利用形式として提示し、その有効性を体系的に実証したこと**にある。
