# 1. はじめに

**論文名:** *Unsupervised Dense Information Retrieval with Contrastive Learning*
**提案モデル:** Contriever / mContriever
**著者:** Gautier Izacardほか
**掲載:** Transactions on Machine Learning Research（TMLR）、2022年

本論文は、検索クエリと関連文書の教師データを使わずに、実用的な密検索器を学習できるかを検証した研究である。中心的な提案は、同じ文書から切り出した二つのテキスト断片を正例とする対照学習により、検索向けの文書表現を獲得する **Contriever** である。

主要な結論は、Contrieverが教師なし設定のRecall@100でBEIRの15データセット中11データセットにおいてBM25を上回り、さらに少数例学習、MS MARCOによる教師あり学習、多言語・言語横断検索の初期値としても有効だというものである。([arXiv][1])

# 2. 背景

従来のBM25やTF-IDFは、クエリと文書に同一または類似した単語が現れることを利用する。そのため教師データを必要とせず、未知ドメインにも比較的安定して適用できる一方、言い換えや同義表現を扱いにくい「語彙ギャップ」がある。

一方、DPRなどの密検索器は、クエリと文書をニューラルネットワークでベクトル化するため意味的な一致を捉えられる。しかし、高性能化には大量のクエリ・関連文書ペアが必要であり、MS MARCOなどで学習したモデルを未知ドメインへそのまま適用すると、BM25に負けることが少なくなかった。特に、英語以外では大規模な検索教師データ自体が乏しい。([arXiv][2])

先行する教師なし事前学習としては、文書中の一文をクエリ、その周辺を文書とみなすInverse Cloze Task、略してICTがあった。しかし、教師あり学習の初期値としては有効でも、教師なし検索器としてはBM25との差が残っていた。そこで本論文は、画像分野で成功していた対照学習と大量の負例を扱うMoCoを、文書検索へ本格的に導入している。([arXiv][2])

# 3. 提案手法

## 3.1 検索モデル

Contrieverは、クエリと文書を独立に符号化するbi-encoderである。同一のBERT-baseエンコーダをクエリ側と文書側で共有し、最終層のトークン表現を平均して一つのベクトルを得る。関連度は次の内積で計算する。

[
s(q,d)=f_\theta(q)^\top f_\theta(d)
]

クエリと文書を別々に符号化するため、文書ベクトルを事前に索引化でき、大規模文書集合に対して近似最近傍探索を利用できる。著者らは、クエリ用と文書用に別モデルを置くより、同一エンコーダを共有する方がゼロショット・少数例設定で頑健だったと報告している。([arXiv][2])

## 3.2 正例の生成

検索ラベルの代わりに、「同じ元文書から得られた二つの断片は意味的に関連する」と仮定する。具体的には、256トークンの文書断片から二つのspanを独立にランダム抽出し、これを正例ペアとする。

各spanの長さは元文書の5～50％から選び、さらに各トークンを10％の確率で削除する。独立クロップでは二つのspanが部分的に重なることもあり、意味的一致だけでなく語彙的一致も学習できる。ICTと異なり、クエリ側と文書側が同じクロップ分布から生成される点も重要である。([arXiv][2])

## 3.3 負例とMoCo

同一文書由来のペアを正例、異なる文書由来の断片を負例とし、正例の内積を負例より高くする対照目的で学習する。

大量の負例を保持するためにMoCoを利用する。クエリエンコーダは通常の勾配降下で更新し、キーエンコーダはクエリエンコーダの指数移動平均で更新する。過去のミニバッチから得たキー表現をキューに保存することで、巨大なバッチを直接計算せず、多数の負例を利用できる。([arXiv][2])

最終設定では、キューサイズ131,072、モメンタム0.9995、温度0.05を使用する。WikipediaとCCNetから半数ずつデータを取り、BERT-baseを初期値として、バッチサイズ2,048、50万ステップ学習する。したがって「教師なし」とは検索関連度ラベルを使わないという意味であり、モデルを完全にランダム初期化しているわけではない。([arXiv][2])

## 3.4 多言語版

多言語版のmContrieverはmBERTを初期値とし、CCNetの29言語を各言語同確率でサンプリングして学習する。英語のMS MARCOだけで追加学習しても、他言語の検索性能が改善するかを検証している。([arXiv][2])

# 4. 実験方法

実験は大きく四つの設定からなる。

**完全教師なし検索**では、NaturalQuestions、TriviaQAおよびBEIRを使用する。BEIRでは上位10件の順位品質を測るnDCG@10と、上位100件に関連文書が含まれる割合を測るRecall@100を評価する。著者らは、検索結果を後段のQAモデルや再ランキングモデルへ渡す用途を重視し、特にRecall@100を主要指標としている。([arXiv][2])

**教師あり・少数例設定**では、ContrieverをMS MARCOで追加学習するほか、SciFact、NFCorpus、FiQAの小規模なドメイン内データでfine-tuningする。MS MARCOでは、最初にランダム負例で学習し、そのモデルでhard negativeを取得して再学習する二段階方式を採用する。([arXiv][2])

**多言語設定**では、同じ言語の文書を検索するMr. TyDiと、26言語のクエリから英語Wikipediaを検索するよう構成したMKQAを評価する。後者は、文字体系が異なるアラビア語や日本語から英語文書を検索する言語横断検索である。([arXiv][2])

比較対象にはBM25、ICT、REALM、SimCSEのほか、教師ありのDPR、ANCE、TAS-B、GenQ、ColBERT、SPLADE v2、cross-encoderによる再ランキングが含まれる。

# 5. 結果

## 5.1 完全教師なし検索

NaturalQuestionsのRecall@100は、BM25の78.3に対してContrieverは82.1であり、3.8ポイント上回った。TriviaQAでは両者とも83.2で同等だった。また、ICTのNaturalQuestions 66.8、TriviaQA 73.6を大きく上回っている。([arXiv][2])

BEIRでは、ContrieverがRecall@100でBM25を15データセット中11データセット上回った。ただし、単純平均ではContrieverが60.1、BM25が63.6であり、Contrieverの方が低い。これはTREC-COVIDで17.2対49.8、Tóuche-2020で22.5対53.8と大きく負けているためである。つまり、「勝ったデータセット数は多いが、少数の大敗によって平均値では負ける」という結果である。([arXiv][2])

上位順位を重視するnDCG@10では、平均がContriever 36.0、BM25 41.7であり、BM25が15データセット中12データセットで最高だった。したがって、教師なしContrieverの強みは上位100件の候補集合を広く回収する能力であり、最上位を正確に並べる能力ではまだBM25に及ばない。([arXiv][2])

## 5.2 MS MARCOでの追加学習

MS MARCOでfine-tuningしたContrieverは、BEIRの平均Recall@100を従来の65.0から67.1へ改善した。nDCG@10は平均46.6で、比較された密bi-encoderの中では強い性能を示す。さらに外部cross-encoderで再ランキングすると平均50.2となり、9データセットで最高値を得ている。([arXiv][2])

ただし、50.2という結果はContriever単体ではなく、既存のMS MARCO向けMiniLM cross-encoderを組み合わせた結果である。そのため、検索器単体の改善と再ランキング器による改善は分けて解釈する必要がある。([arXiv][2])

## 5.3 少数例学習

小規模なドメイン内データで学習した場合、ContrieverはBERT初期化を一貫して上回った。

* SciFact: 84.0対75.2
* NFCorpus: 33.6対29.9
* FiQA: 36.4対26.1

さらにMS MARCOを経由すると、それぞれ84.8、35.8、38.1まで向上する。特にFiQAでは、教師なしContrieverを直接少数例学習した36.4が、MS MARCOで事前に教師あり学習したBERTの30.9を上回っている。([arXiv][2])

## 5.4 多言語・言語横断検索

Mr. TyDiの平均Recall@100は、教師なしmContrieverが77.2で、BM25の74.3を上回った。一方、MRR@100は25.0対33.3でBM25に負けており、英語実験と同様に「広く候補を回収できるが、最上位順位は弱い」という傾向が見られる。

英語のMS MARCOだけでfine-tuningすると、平均Recall@100は87.0、MRR@100は38.4へ改善した。Mr. TyDiの教師データまで使用した場合は、それぞれ93.6、65.2に達する。([arXiv][2])

MKQAの言語横断検索では、MS MARCOで学習したmContrieverの平均Recall@100が65.6となり、翻訳データなどを用いるCORAの59.8を上回った。アラビア語クエリから英語文書を検索するRecall@100も53.3であり、単純な単語一致では困難な異文字体系間検索を実現している。([arXiv][2])

## 5.5 アブレーション

最も明確な結果は正例生成方法である。教師なしnDCG@10の平均はICTが25.9、ランダムクロップが32.2、クロップ＋トークン削除が33.8だった。提案の中心である単純なクロップが、従来の検索事前学習タスクを大きく上回った。([arXiv][2])

一方、負例数を同じにした場合、MoCoは平均30.1、in-batch negativesは31.9であり、MoCo自体が常に高性能なわけではない。MoCoの利点は、巨大なバッチを使わずキューを拡大できるスケーラビリティにある。負例数を2,048から131,072へ増やすと、特に教師なし設定の平均性能が改善した。([arXiv][2])

# 6. 考察

本論文の重要な点は、新しい複雑な検索アーキテクチャを提案したことではなく、**適切な正例生成、大量の負例、多様な生テキスト**を組み合わせれば、対照学習だけでも検索に適した表現を獲得できると実証したことである。

独立クロップがICTより有効だった理由として、クエリ側と文書側が同一の分布になること、部分的な語彙重複が検索に必要な完全一致信号を残すこと、同じ文書の異なる部分から主題レベルの意味的一致も学べることが考えられる。著者らも、対称なクロップ分布がMoCoの学習を安定させた可能性を指摘している。([arXiv][2])

実用上は、Contriever単独で検索結果を最終表示するより、RAG、オープンドメインQA、fact checkingなどの**第一段候補生成器**として使い、その後にcross-encoderで再ランキングする構成に適している。Recall@100は強いが、nDCG@10やMRRではBM25に劣るという結果と整合する。

主な限界は以下である。

第一に、教師なしBEIRの主張は評価指標に敏感である。11/15データセットでBM25を上回る一方、平均Recall@100とnDCG@10ではBM25に負けているため、「BM25を全面的に置き換えた」と解釈すべきではない。

第二に、計算量が大きい。BERT-baseを50万ステップ、バッチサイズ2,048、最大13万件以上の負例で学習するため、ラベル作成コストは削減できても、計算資源まで低コストとはいえない。索引サイズ、検索遅延、電力消費についてBM25との詳細比較もない。

第三に、複数seedの分散や統計的有意差が表に示されていない。特に数ポイント以下の改善については、再実験による安定性の確認が望ましい。

第四に、TREC-COVIDのような時間的に新しい領域や、Tóuche-2020のような長文検索では大きく性能が低下する。著者らも、事前学習データがCOVID-19以前であることと、長文が密検索器に適していない可能性を挙げている。([arXiv][2])

第五に、多言語化では言語数を増やせば常に良くなるわけではない。Mr. TyDiの11言語だけで事前学習したモデルは、29言語モデルより教師なし時およびMS MARCO追加学習後の性能が高く、「curse of multilinguality」が確認されている。([arXiv][2])

# 7. まとめ

Contrieverは、同一文書からのランダムクロップを正例とし、MoCoで大量の負例を利用することで、検索ラベルなしに密検索器を学習する手法である。

教師なしの上位候補回収、少数例への適応、教師あり検索の初期値、多言語・言語横断検索では大きな効果を示した。一方、最上位順位の精度、長文・時間的ドメインシフト、計算コストではBM25や疎検索との相補性が残っている。

この研究の本質的な貢献は、「検索向けの自己教師あり事前学習は複雑な疑似クエリ生成を必ずしも必要とせず、単純な文書クロップでも十分に強力である」と示した点にある。

# 8. 総評

**長所:** 提案が単純で再利用しやすく、教師なし、少数例、教師あり、多言語、言語横断という広い条件で一貫して検証されている。アブレーションも充実しており、特にICTとランダムクロップの比較は明確である。コードと事前学習モデルも公開されている。([OpenReview][3])

**弱点:** 手法上の新規性は主として既存のMoCoとテキストクロップの組み合わせであり、アーキテクチャ上の革新性は限定的である。また、教師なし性能の優位性がRecall@100に偏っていること、計算効率・統計的有意差の分析が不足していること、最高nDCGの一部が外部再ランキング器に依存することには注意が必要である。

**私見による査読判定:** **採録（Accept）**
新規性は中程度だが、実験的貢献、技術的妥当性、実用的影響が十分に大きく、主張も主要な表によって裏づけられている。ただし、採録時には「BM25を上回る」という表現をRecall@100に限定し、nDCG@10との違いをより明確にする軽微な修正を求める。

**実際の査読結果:** 本論文はOpenReview上で **Accepted by TMLR** とされ、2022年8月29日にTMLR論文として公開されている。([OpenReview][3])
