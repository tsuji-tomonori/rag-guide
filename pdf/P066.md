# RAFT: Adapting Language Model to Domain Specific RAG

## 1. はじめに

本論文は、専門領域の文書群に対してRAGを利用する際、LLMをどのように追加学習すれば、検索文書をより正確に利用できるかを扱っている。

提案手法 **RAFT（Retrieval-Augmented Fine-Tuning）** は、質問と正解文書だけでなく、回答に不要な「撹乱文書」を同時に与えてファインチューニングする。これにより、モデルに単なる領域知識の暗記ではなく、複数文書から必要な根拠を選び、引用し、推論して回答する能力を学習させる。([arXiv][1])

論文が対象とするのは、任意の未知領域に対応する一般的なRAGではなく、訓練時点で対象となる文書集合を利用できる **in-domainのopen-book設定** である。

## 2. 背景

専門領域へLLMを適応させる代表的な方法には、次の二つがある。

通常のRAGは、推論時に関連文書を取得してプロンプトへ追加する。しかし、基盤モデルは一般目的で学習されており、検索された複数文書のうち、どれが回答に必要かを十分に判別できるとは限らない。検索結果に無関係な文書が混ざると、モデルがその内容に影響される問題もある。

一方、Domain-Specific Fine-Tuning（DSF）は、領域内の質問・回答ペアを学習するため、回答形式や領域知識を獲得できる。しかし、訓練時に文書を読ませていないモデルへ、推論時だけRAG文書を与えても、文書から適切な根拠を抽出する能力が保証されない。実際、本論文の実験では、DSFへRAGを追加すると、一部データセットで性能が低下している。

著者らはこれを試験にたとえている。

* DSF：教科書を暗記して「閉巻試験」を受ける
* 通常のRAG：勉強せず、当日だけ教科書を持ち込む
* RAFT：教科書を使った「開巻試験」の練習を事前に行う

RAFTの目的は、検索器そのものを改善することではなく、**検索結果を読む生成モデルをRAG向けに訓練すること**である。

## 3. 提案手法

質問を (Q)、回答の根拠を含む正解文書を (D^*)、無関係な撹乱文書を (D_i)、教師回答を (A^*) とする。

RAFTでは、訓練データを次の二種類に分ける。

[
P%:\quad Q+D^*+D_1+\cdots+D_k \rightarrow A^*
]

[
(1-P)%:\quad Q+D_1+\cdots+D_k \rightarrow A^*
]

前者では、モデルが撹乱文書を無視し、正解文書から情報を抽出することを学ぶ。後者では正解文書を与えないため、モデルはファインチューニング中に獲得したパラメトリックな知識から回答する必要がある。著者らは、両者を混ぜることで「文書を読む能力」と「領域知識の記憶」を両立させようとしている。推論時には通常のRAGと同様、質問と検索器が返したtop-k文書を入力する。

教師回答 (A^*) には、最終回答だけでなく、以下を含める。

1. 正解文書からの直接引用
2. 引用を用いたChain-of-Thought形式の説明
3. 最終回答

Gorilla APIBench以外の推論付き回答はGPT-4-1106を用いて生成している。これにより、モデルに「どの文を根拠として、どのように答えへ到達するか」を明示的に学習させる。

したがって、RAFTの本質はモデル構造の変更ではなく、**RAGの実運用状態を模擬した教師データ設計**にある。

## 4. 実験方法

基盤モデルにはLLaMA2-7B系列を使用し、RAFTでファインチューニングしたモデルを以下と比較している。

* LLaMA2-7B-chat
* LLaMA2-7B-chat + RAG
* Domain-Specific Fine-Tuning（DSF）
* DSF + RAG
* GPT-3.5 + RAG

評価対象は、医療文書QAのPubMed QA、複数文書推論を要求するHotpotQA、およびAPIドキュメントから正しいAPI呼び出しを生成するGorilla APIBenchのHuggingFace、Torch Hub、TensorFlow Hubである。Natural QuestionsとTriviaQAは、主に正解文書の混合比率や撹乱文書数の分析に使用されている。

さらに、次のアブレーションを行っている。

* CoT付き回答と最終回答のみの比較
* 訓練データに正解文書を含める割合 (P) の変更
* 訓練時の撹乱文書数の変更
* 推論時のtop-k文書数を変えた場合の頑健性評価

ただし、本文では最適化手法、学習率、エポック数、データ件数、検索器・チャンキング方法、評価指標の厳密な定義、乱数シードなどが十分に記述されておらず、実験再現性には改善の余地がある。

## 5. 結果

主要な結果は次の通りである。数値は論文のTable 1による。

| データセット         |  RAFT | 比較上重要な結果                          |
| -------------- | ----: | --------------------------------- |
| PubMed         | 73.30 | DSF+RAG 71.60、GPT-3.5+RAG 71.60   |
| HotpotQA       | 35.28 | DSF+RAG 4.41、ただしGPT-3.5+RAGは41.50 |
| HuggingFace    | 74.00 | DSF 61.06、DSF+RAG 42.59           |
| Torch Hub      | 84.95 | DSF 84.94で、ほぼ同等                   |
| TensorFlow Hub | 86.86 | DSF 86.56で、小幅な改善                  |

HotpotQAとHuggingFaceでは特に大きな改善が見られる。HotpotQAではDSF+RAGとの差が30.87ポイント、HuggingFaceでは31.41ポイントであり、検索文書を含めた状態で訓練する効果が強く表れている。

一方、「すべてで大幅に改善した」と単純化することはできない。HotpotQAではGPT-3.5+RAGを下回り、Torch HubとTensorFlow HubではDSFとの差がそれぞれ0.01、0.30ポイントにとどまる。

CoTのアブレーションでは、PubMed、HotpotQA、HuggingFace、TensorFlow Hubの4データセットで改善した。特にHotpotQAは25.62から35.28、HuggingFaceは59.07から74.00へ向上した。ただしTorch Hubでは、CoTなしの86.56に対してCoTありが84.95となり、1.61ポイント低下している。したがってCoTの効果は大きいものの、一様ではない。

正解文書を含む割合 (P) は、Natural Questionsで約40%、TriviaQAで約60%、HotpotQAでほぼ100%が最良となった。常に正解文書を与えるのではなく、一部の訓練例から正解文書を除くことが有効な場合があるが、その最適値はデータセット依存である。

また、正解文書だけで訓練したモデルより、撹乱文書を含めて訓練したモデルの方が、推論時の文書数が変動しても性能を維持しやすかった。Natural Questionsでは正解文書と3件の撹乱文書、HotpotQAでは正解文書と1件の撹乱文書による訓練が良好な結果を示している。

## 6. 考察

本研究の最大の長所は、RAGの訓練時と推論時の条件差を明確に捉えた点である。通常のSFTでは文書を読まずに回答を学習するが、推論時には検索ノイズを含む文書が入力される。RAFTはこの不一致を、正解文書と撹乱文書を混ぜた訓練によって直接解消している。

また、モデル構造や検索器を変更せず、訓練データの作り方だけで適用できるため、実装上の価値が高い。引用付き教師回答も、単なる正答率だけでなく、根拠に基づく回答を促す設計として妥当である。

一方、以下の制約がある。

第一に、対象範囲が狭い。本論文は、訓練時と評価時に同じ領域、さらには基本的に同じ文書集合を利用する設定を扱っている。未知文書への転移、文書集合の頻繁な更新、領域外質問への一般化は主要な評価対象ではない。企業内の固定文書群には適しているが、日々変化するニュースやWeb検索には再学習コストが問題となる。

第二に、CoTの効果と教師モデルの効果が分離されていない。CoT付きデータはGPT-4-1106で生成されているため、性能向上が「推論過程を学んだ結果」なのか、「強力な教師による高品質かつ長い回答を学んだ結果」なのかは判別しにくい。

第三に、比較対象が限定的である。検索認識型ファインチューニングや長文QAの関連研究は挙げられているが、それらとの直接比較がない。また、誤差範囲、複数シード、統計的有意差も報告されておらず、「significant」という表現を統計的意味で解釈することはできない。

第四に、正解文書を除いた状態でも正答を出す訓練は、モデルの暗記を促す。その結果、文書更新後にも古い知識を回答したり、入力文書とパラメトリック知識が矛盾した場合に、どちらを優先するかが不明確になる可能性がある。

## 7. まとめ

RAFTは、専門領域RAGの生成モデルを、実際の検索環境に近い条件でファインチューニングする手法である。

その構成要素は、正解文書、撹乱文書、正解文書を含まない一部の訓練例、引用付きCoT回答である。これにより、モデルは領域知識を覚えるだけでなく、検索結果から必要な根拠を選別して回答する能力を獲得する。

手法自体は単純だが、HotpotQAやAPIドキュメントQAで大きな性能改善を示しており、「RAGを推論時だけ追加するのではなく、RAG環境そのものを訓練する」という設計原則を明確に示した論文である。

## 8. 総評

**査読結果：COLM 2024採択。** 公開査読では、9点のStrong Accept、7点のAccept、6点のMarginal Acceptという評価が確認でき、最終的にCOLM 2024の会議論文として採択・公開されている。([OpenReview][2])

公開査読では、実用的な有効性、分かりやすい問題設定、複数データセットとアブレーションによる検証が評価された一方、「Retrieval-Augmented」と「Retrieval-Aware」、「golden」と「oracle」など、重要用語の不統一も指摘されている。([OpenReview][2])

**本要約としての査読判断はAcceptである。** 技術的新規性は、全く新しいモデルや学習アルゴリズムの提案というより、既存要素を適切に組み合わせたデータ設計にある。そのため理論的・方法論的な新規性は中程度だが、RAGシステムの訓練と運用の不一致を明確に定式化し、簡単な方法で実用的な改善を示した貢献は大きい。

ただし、再現性に必要な実験詳細、強力な同時代手法との比較、統計的評価、動的文書集合や領域外一般化の検証が不足している。採択には値するが、結論の一般化可能範囲は「固定された専門文書群に対するRAG」へ限定して解釈すべき論文である。
