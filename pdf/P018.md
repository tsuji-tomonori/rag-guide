# 1. はじめに

**論文:** *LayoutLM: Pre-training of Text and Layout for Document Image Understanding*
**著者:** Yiheng Xuほか
**掲載:** KDD 2020

本論文は、帳票・請求書・レシートなどの文書画像を理解する際に、OCRで得られる文字列だけでなく、**文字がページ上のどこに配置されているか**を事前学習へ取り込むモデル「LayoutLM」を提案した研究である。

中心的な発想は、BERTに各単語の二次元座標を追加するという比較的単純なものである。しかし、この拡張によって帳票理解、レシート情報抽出、文書分類の3タスクで当時の最高性能を更新した。論文はKDD 2020に採録されている。([arXiv][1])

# 2. 背景

従来の文書理解手法には、大きく二つの問題があった。

第一に、BERTなどの言語モデルはOCR結果を一次元のトークン列として処理するため、文書内の空間構造を失う。例えば「Passport ID」という項目名の値は、その右側または下側に配置されることが多いが、通常の言語モデルはこの関係を直接表現できない。

第二に、画像ベースのモデルは罫線、フォント、文書全体の外観などを扱える一方で、単語の意味や文脈を十分に利用しにくい。また、従来のマルチモーダル手法はタスクごとに設計され、大量のラベル付きデータを必要とする場合が多かった。

そこで本論文は、大規模な未ラベル文書画像を用いて、**テキストとレイアウトを統一的に事前学習する**という問題設定を導入した。([arXiv][2])

# 3. 提案手法

## 3.1 入力表現

LayoutLMはBERTをバックボーンとし、各OCRトークンについて以下の情報を入力する。

* 単語埋め込み
* 通常の一次元位置埋め込みとセグメント埋め込み
* 二次元位置埋め込み
* 任意の画像埋め込み

各単語のバウンディングボックスを

[
(x_0,y_0,x_1,y_1)
]

で表し、左上座標と右下座標をそれぞれ埋め込みに変換する。ページサイズの違いを吸収するため、座標は0から1000の範囲へ正規化される。概念的には、BERTの入力に

[
E_{x_0}+E_{y_0}+E_{x_1}+E_{y_1}
]

を加える構成である。これにより、自己注意機構が単語間の意味関係だけでなく、上下左右の配置関係も学習できる。([arXiv][2])

## 3.2 画像特徴

OCRで得られた各単語領域をFaster R-CNNに入力し、単語単位の画像特徴を生成する。また、`[CLS]`トークンにはページ全体の画像特徴を対応させる。画像側のバックボーンはVisual Genomeで事前学習されたResNet-101である。([arXiv][2])

ただし、LayoutLM v1では画像特徴をテキスト・レイアウトと完全に共同事前学習しているわけではない。中心となる事前学習対象はテキストと二次元配置であり、Faster R-CNNの画像特徴は主に下流タスクで結合される。

## 3.3 事前学習目的

### Masked Visual-Language Model：MVLM

BERTのMLMと同様に入力単語の一部をマスクするが、対応する二次元座標は残す。モデルは周囲の文章だけでなく、マスクされた単語の位置を利用して元の単語を予測する。

例えば、項目名の右側にあるマスク語であれば、その位置関係から「項目値である可能性」を学習できる。

### Multi-label Document Classification：MDC

IIT-CDIPに付与された文書タグを使い、文書全体の複数カテゴリを予測する補助目的である。これは高品質な文書レベル表現を得ることを狙っている。

なお、MVLMは自己教師あり学習だが、MDCは既存メタデータを使うため、厳密には弱教師あり学習に近い。([arXiv][2])

## 3.4 下流タスク

帳票理解とレシート理解では、各トークンにBIOES形式の系列ラベルを付与する。文書画像分類では、`[CLS]`表現とページ全体の画像特徴を使って文書カテゴリを予測する。([arXiv][2])

# 4. 実験方法

## 4.1 事前学習データ

IIT-CDIP Test Collection 1.0を使用している。

* 約600万文書
* 約1100万ページ
* 手紙、フォーム、請求書、広告、履歴書、科学論文など多様な文書種類

元データには単語座標がないため、Tesseractで全文書を再度OCRし、認識文字とバウンディングボックスをhOCR形式で保存している。([arXiv][2])

## 4.2 モデル構成

| モデル            | Transformer層 |  隠れ次元 | Attention head | パラメータ数 |
| -------------- | -----------: | ----: | -------------: | -----: |
| LayoutLM BASE  |           12 |   768 |             12 |  約113M |
| LayoutLM LARGE |           24 | 1,024 |             16 |  約343M |

BERTまたはRoBERTaの重みで初期化し、新規の二次元位置埋め込みはランダム初期化する。8台のNVIDIA Tesla V100 32GBを使用し、1100万ページを1エポック学習するのにBASEで約80時間、LARGEで約170時間を要した。([arXiv][2])

## 4.3 評価データ

**FUNSD:** 199枚の帳票画像を用いる。149枚が学習、50枚がテストで、単語レベルF1を評価する。ただし、本論文が扱うのは意味ラベル付与のみであり、質問と回答を結ぶsemantic linkingは評価対象外である。

**SROIE:** 学習用626枚、テスト用347枚のレシートを用いる。会社名、日付、住所、合計金額の4項目を抽出し、完全一致ベースのF1を評価する。OCR誤りの影響を除くため、実験では正解OCR情報を利用している。

**RVL-CDIP:** 40万枚、16カテゴリの文書画像分類データである。32万枚を学習、4万枚を検証、4万枚をテストに使用し、分類精度を評価する。([arXiv][2])

# 5. 結果

主要結果は以下のとおりである。差分は絶対ポイントで示す。([arXiv][2])

| タスク          |   従来最高 |   LayoutLM |    改善 |
| ------------ | -----: | ---------: | ----: |
| FUNSD帳票理解・F1 |  70.72 |  **79.27** | +8.55 |
| SROIE情報抽出・F1 |  94.02 |  **95.24** | +1.22 |
| RVL-CDIP分類精度 | 93.07% | **94.42%** | +1.35 |

FUNSDでは、テキストとレイアウトのみのBASEモデルでもF1 78.66を達成し、画像特徴を追加すると79.27まで向上した。少数の教師データしかないFUNSDで大幅な改善が得られた点は、レイアウト事前学習の有効性を強く示している。

画像特徴の効果はタスクによって異なる。FUNSDでは+0.61ポイントだが、RVL-CDIPではテキスト＋レイアウトの91.78%から、画像を含む94.42%へ+2.64ポイント改善している。したがって、文書カテゴリ分類ではページ全体の外観が特に重要である。

事前学習データを50万、100万、200万、1100万ページと増やすにつれてFUNSD性能が上昇した。また、BERTよりRoBERTaで初期化した方がBASEで約2.1ポイント、LARGEで約1.3ポイント高かった。

一方、MDCの効果は一貫していない。100万ページ・6エポックではFUNSDのF1が72.99から73.72へ上昇したが、1100万ページ・1エポックではMVLM単独の76.36に対し、MVLM＋MDCは74.75であり、むしろ低下している。

# 6. 考察

本研究の最大の貢献は、複雑な専用ネットワークを設計するのではなく、**BERTへ二次元座標を追加するだけで文書理解性能を大幅に改善できる**ことを示した点にある。特に、教師データが少ないFUNSDでの改善は説得力が高い。

一方で、以下の制約がある。

**画像の共同事前学習が不十分である。**
画像特徴はFaster R-CNNから取得して下流処理で結合される。したがって、画像・文字・配置を単一の目的関数で最初から共同学習する現在的な意味でのマルチモーダル事前学習には達していない。

**完全な帳票理解ではない。**
FUNSDには質問と回答の関係を推定するsemantic linkingが含まれるが、本論文はトークンの意味ラベル付与だけを評価している。「質問」「回答」を識別できても、どの質問とどの回答が対応するかは解いていない。

**OCRへの依存が強い。**
実運用ではOCRの文字誤認識や座標誤差が後段へ伝播する。特にSROIEでは正解OCRを利用しているため、スキャン画像から最終的な情報を抽出するエンドツーエンド性能とは一致しない。

**比較条件が完全には統制されていない。**
モデルサイズ、初期化モデル、事前学習データ量、エポック数が設定間で異なる。また、複数シードの平均・分散や統計的有意差が報告されていないため、小さな改善については確実性を判断しにくい。

**事前学習データと評価データの重複リスクがある。**
LayoutLMはIIT-CDIP全体で事前学習している。一方、RVL-CDIPはIIT-CDIPから作られたラベル付き部分集合であり、FUNSDもRVL-CDIPの部分集合である。論文には下流タスクのテスト画像を事前学習集合から除外したとの記載がないため、正解ラベルは見ていなくても、同一テスト文書の文字・配置を事前学習時に見た可能性がある。これは資料から導かれる評価汚染の懸念であり、実際に除外処理が行われていたかは本文から確認できない。([arXiv][2])

# 7. まとめ

LayoutLMは、テキストの意味だけでなく単語の二次元配置をTransformerへ直接入力し、大規模文書画像から事前学習する手法である。

特に重要なのは、絶対的な性能値よりも、**文書を単なる文字列ではなく、意味を持つ二次元構造として扱うべきである**ことを明確に示した点である。画像の共同事前学習やエンドツーエンドOCRなどには未完成な部分が残るものの、後続の文書AI研究の基本設計を確立した研究と評価できる。

# 8. 総評

**実際の掲載結果:** KDD 2020採録済み。([arXiv][1])

**独立査読判定:** **Accept／採択、7.5点相当**

新規性は5/5、技術的妥当性は4/5、実験の厳密性は3/5、再現性は3.5/5、研究上の影響度は5/5と評価する。

二次元位置埋め込みという単純で再利用可能な設計により、複数の文書理解タスクで明確な改善を示した点は採択に十分値する。一方、画像を完全には共同事前学習していない点、semantic linkingを評価していない点、OCR条件の非現実性、事前学習データと評価データの重複可能性は重要な弱点である。それでも、文書理解向け基盤モデルという研究方向を確立した新規性と波及効果が、これらの弱点を上回る。
