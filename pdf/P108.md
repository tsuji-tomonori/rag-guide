# P108 — ColBERTv2

**論文名:** *ColBERTv2: Effective and Efficient Retrieval via Lightweight Late Interaction*
**著者:** Keshav Santhanam, Omar Khattab, Jon Saad-Falcon, Christopher Potts, Matei Zaharia
**掲載先:** NAACL 2022 Main Conference, pp. 3715–3734 ([ACL Anthology][1])

## 1. はじめに

ColBERTv2は、ニューラル情報検索における「検索精度」と「インデックス容量」の両立を目指した検索モデルである。

従来のColBERTは、クエリと文書をトークン単位の複数ベクトルとして表現し、それらを検索時に照合する**Late Interaction**を採用する。単一ベクトル型のDense Retrieverよりも細粒度の対応関係を捉えやすい一方、文書中の多数のトークンベクトルを保存するため、インデックス容量が非常に大きくなる。

ColBERTv2は、この問題に対して次の二つを組み合わせる。

1. Cross-Encoderを教師とする**Denoised Supervision**
2. セントロイドと量子化残差による**Residual Compression**

これにより、従来のColBERTより検索精度を向上させながら、インデックス容量を約6～10分の1に削減した。([ACL Anthology][1])

## 2. 背景

ニューラル検索モデルは、大きく次の三種類に分けられる。

**Cross-Encoder**はクエリと文書を同時にTransformerへ入力するため高精度だが、検索対象の全文書に対して推論する必要があり、大規模検索には不向きである。

**Single-Vector Retriever**はクエリと文書をそれぞれ一つのベクトルに圧縮し、内積などで類似度を計算する。高速かつ省容量だが、複雑なクエリ―文書間関係を一回の内積に押し込める必要がある。

これに対して**Late Interaction**は、クエリと文書を独立に符号化しながら、それぞれをトークン単位の複数ベクトルとして保持する。検索時にトークン間の類似度を計算するため、Cross-Encoderに近い細粒度の照合能力と、事前インデックス可能性を両立できる。

ColBERTの関連度スコアは、次式で表される。

[
S(q,d)=\sum_{i=1}^{N}\max_{j=1}^{M} Q_iD_j^\top
]

ここで、(Q_i) はクエリの第 (i) トークン、(D_j) は文書の第 (j) トークンの埋め込みである。各クエリトークンについて、最も類似する文書トークンを選び、その最大類似度を合計する。この演算が**MaxSim**である。

ただしColBERTでは、文書中のほぼすべてのトークンについて埋め込みを保存する。Web規模のコーパスでは数億から数十億個のベクトルが必要となり、Single-Vector Retrieverより一桁程度大きなインデックスを必要とする。

また、従来のColBERTは比較的単純な正例・負例の三つ組で学習されていた。一方、SPLADEv2やRocketQAv2などでは、Cross-Encoderからの蒸留やHard Negative Miningによって精度が大幅に向上していた。このため、「Late Interaction自体が優れているのか、それともColBERTの学習方法が不十分なのか」が明確ではなかった。

## 3. 提案手法

### 3.1 Late Interactionモデル

基本アーキテクチャはColBERTを踏襲する。クエリと文書をBERTで独立に符号化し、各トークンの埋め込みを低次元ベクトルへ射影する。

実験ではクエリ側と文書側で共有された`bert-base-uncased`を使用し、各トークンを128次元のベクトルとして表現する。文書は事前に符号化・保存され、検索時にはクエリだけを符号化してMaxSimを計算する。

### 3.2 Denoised Supervision

ColBERTv2では、教師Cross-Encoderを用いて学習データ中のノイズを軽減する。

まず、初期ColBERTモデルで各学習クエリに対する上位文書を検索する。各クエリについて上位500文書を候補とし、それらを22MパラメータのMiniLM Cross-Encoderで再評価する。

その後、一つのクエリ、正例または教師が高く評価した文書、複数の低評価文書からなる**64-way tuple**を作成する。Cross-Encoderが出力する文書間のスコア分布を、KLダイバージェンスによってColBERTv2へ蒸留する。さらに、同一バッチ内の他クエリに対応する文書も負例として利用する。

MS MARCOには約80万件の学習クエリがあるが、人手ラベル付きなのは約50万件である。ColBERTv2では全80万件を利用し、ラベル付き正例がない場合はCross-Encoderの最上位文書を正例として扱う。

また、一度蒸留学習を行ったモデルで候補文書を再検索し、負例集合を更新してもう一度学習する。この反復によって、より難しいHard Negativeを学習に取り込む。

### 3.3 Residual Compression

ColBERTv2の中心的な効率化技術がResidual Compressionである。

著者らは、ColBERTのトークン埋め込みが意味ごとに局所的なクラスタを形成すると仮定した。例えば、写真に関するトークンや竜巻に関するトークンは、それぞれ近い領域に集まる。この性質を利用し、各トークンベクトル (v) を次のように表現する。

[
v \approx C_t+\tilde{r}
]

* (C_t)：最も近いクラスタのセントロイド
* (t)：セントロイドのID
* (r=v-C_t)：残差
* (\tilde{r})：量子化された残差

128次元ベクトルの場合、セントロイドIDを4バイトで保存し、残差の各次元を1ビットまたは2ビットに量子化する。したがって、1ベクトル当たりの容量は次のようになる。

| 表現                       | 1ベクトル当たりの容量 |
| ------------------------ | ----------: |
| 通常のColBERT、FP16          |      256バイト |
| ColBERTv2、1-bit residual |       20バイト |
| ColBERTv2、2-bit residual |       36バイト |

モデル構造や学習方法を圧縮向けに変更する必要がなく、既存のColBERT型モデルに後処理として適用できる点も特徴である。

### 3.4 インデックス構築と検索

インデックス構築は三段階で行う。

まず、文書埋め込みの一部をサンプリングし、k-meansによってセントロイドを求める。次に、全文書を符号化し、各トークン埋め込みを「セントロイドID＋量子化残差」として保存する。最後に、各セントロイドに所属するトークン埋め込みをまとめた転置リストを構築する。

検索時には、各クエリトークンに近い複数のセントロイドを探索し、その転置リストから候補となる文書トークンを取得する。取得したトークンだけを使って近似MaxSimを計算し、文書候補を絞り込む。

その後、上位候補文書の全トークン表現を読み込み、通常のMaxSimスコアを再計算して最終順位を決定する。すなわち、安価な近似検索による候補生成と、詳細なLate Interactionによる再順位付けを一つのインデックスで実現している。

### 3.5 LoTTEベンチマーク

本論文では、長尾領域におけるゼロショット検索性能を評価するため、**LoTTE: Long-Tail Topic-stratified Evaluation**も提案している。

LoTTEは、Writing、Recreation、Science、Technology、Lifestyleの5領域と、それらを統合したPooled設定からなる。各設定について、次の2種類のクエリを用意する。

* **Search Query:** GooAQから取得した実際の検索エンジン形式の質問
* **Forum Query:** StackExchange上の投稿タイトル

合計12個のテストセットがあり、各セットは約500～2,000クエリ、10万～200万文書で構成される。開発セットとテストセットではサブトピックと文書集合を分離し、未知領域への転移性能を測定する。

## 4. 実験方法

### 学習設定

主な学習条件は以下の通りである。

| 項目         | 設定                             |
| ---------- | ------------------------------ |
| 学習データ      | MS MARCO Passage Ranking       |
| Encoder    | `bert-base-uncased`、クエリ・文書間で共有 |
| パラメータ数     | 約110M                          |
| トークンベクトル次元 | 128                            |
| 学習率        | (10^{-5})                      |
| バッチサイズ     | 32                             |
| Warm-up    | 20,000ステップ                     |
| 学習ステップ     | 400,000                        |
| 蒸留候補       | 各クエリの検索上位500件                  |
| 蒸留単位       | 64-way tuple                   |
| 標準圧縮設定     | 各次元2ビット                        |

学習には主として4基の80GB A100 GPUを使用し、64-wayモデルの400,000ステップ学習には約5日、MS MARCOのインデックス構築には約2時間を要したと報告されている。

### 評価データ

評価は以下の三領域で行われた。

| 評価領域                | データセット                   | 主指標                          |
| ------------------- | ------------------------ | ---------------------------- |
| In-domain           | MS MARCO公式Dev、Local Eval | MRR@10、Recall@50、Recall@1000 |
| Out-of-domain       | BEIRの13タスク               | nDCG@10                      |
| Open-domain QA      | NQ、TriviaQA、SQuAD        | Success@5                    |
| Long-tail retrieval | LoTTEの12テスト              | Success@5                    |

比較対象にはBM25、DPR、ANCE、MoDIR、ColBERT、TAS-B、SPLADEv2、RocketQAv2などが含まれる。BEIR、Open-QA、LoTTEを合わせると、計28件のOut-of-domainテストとなる。

## 5. 結果

### 5.1 MS MARCO

MS MARCO公式Devセットでは、ColBERTv2が次の結果を達成した。

| モデル           |   MRR@10 | Recall@50 | Recall@1000 |
| ------------- | -------: | --------: | ----------: |
| ColBERT       |     36.0 |      82.9 |        96.8 |
| SPLADEv2      |     36.8 |         ― |        97.9 |
| RocketQAv2    |     38.8 |      86.2 |        98.1 |
| **ColBERTv2** | **39.7** |  **86.8** |    **98.4** |

独立したLocal Evalでも、ColBERTv2はMRR@10で40.8、Recall@50で86.3、Recall@1000で98.3を記録した。当時比較されたスタンドアロン検索モデルの中で、最も高いMRR@10となっている。

### 5.2 Out-of-domain検索

28件のOut-of-domainテストのうち、ColBERTv2は22件で最高性能を達成したと報告されている。特に、通常の自然言語検索質問を含むデータセットで強い傾向が見られた。

Wikipedia Open-QAでは、すべてのデータセットでColBERTおよびSPLADEv2を上回った。

| データセット            | ColBERT | SPLADEv2 | ColBERTv2 |
| ----------------- | ------: | -------: | --------: |
| Natural Questions |    65.7 |     65.6 |  **68.9** |
| TriviaQA          |    72.6 |     74.7 |  **76.7** |
| SQuAD             |    60.0 |     60.4 |  **65.0** |

一方、BEIRでは全面的に勝っているわけではない。NQ、FiQA、DBPedia、TREC-COVIDなどの検索型タスクでは高性能だが、Climate-FEVERやArguAnaなど、主張検証・文書類似性に近いタスクではSPLADEv2が上回った。これは、ColBERTv2の優位性が特に「自然な検索意図を持つクエリ」に現れることを示している。

### 5.3 LoTTE

LoTTEのPooledテストでは、Search QueryでSuccess@5が71.6、Forum Queryで63.4となった。

| Pooled設定     | RocketQAv2 | SPLADEv2 | ColBERTv2 |
| ------------ | ---------: | -------: | --------: |
| Search Query |       69.8 |     68.9 |  **71.6** |
| Forum Query  |       57.7 |     60.1 |  **63.4** |

各領域を含め、ColBERTv2はSearch QueryとForum Queryの双方で一貫して高い性能を示した。SPLADEv2に対して最大3.7ポイント、RocketQAv2に対して最大8.1ポイント改善した。

### 5.4 圧縮効率

MS MARCOのインデックス容量は次のように削減された。

| モデル・圧縮条件        | インデックス容量 |
| --------------- | -------: |
| 通常のColBERT      |  154 GiB |
| ColBERTv2、2-bit |   25 GiB |
| ColBERTv2、1-bit |   16 GiB |

2-bitでは約6倍、1-bitでは約10倍の圧縮となる。これには約4.5 GiBの転置リストも含まれている。2-bit時の25 GiBは、9百万文書について768次元の単一ベクトルをFP32で保存した場合とほぼ同等である。

圧縮による精度低下も小さい。通常のColBERTにResidual Compressionを適用した実験では、非圧縮時のMRR@10が36.2、Recall@50が82.1であったのに対し、2-bit圧縮では36.2、82.3となった。1-bitでも35.5、81.6を維持した。

検索レイテンシは設定によって約50～250ミリ秒で、多くの設定は150ミリ秒未満だった。3種類の異なる規模のデータセットで、最高精度に近い結果を約100ミリ秒で達成できた。

## 6. 考察

本論文の重要な点は、単にColBERTを圧縮したことではなく、**Late Interaction型モデルも高度な蒸留とHard Negative学習によって大幅に改善できる**ことを示した点にある。

従来は、Single-Vector Retrieverの方が容量面で有利であり、蒸留などの学習技術を適用すれば精度面でもLate Interactionに追いつけると考えられていた。ColBERTv2は、Late Interactionにも同様の学習を適用したうえで、その構造に適した圧縮を行うことで、精度と容量の双方で競争力を持たせた。

Residual Compressionが有効なのは、ColBERTのトークン埋め込み空間に強い局所構造があるためと考えられる。文書全体を一つのベクトルに圧縮するSingle-Vectorモデルとは異なり、各ベクトルが比較的限定されたトークン意味を表すため、「セントロイド＋小さな残差」で近似しやすい。この圧縮方式はLate Interactionの帰納バイアスとよく整合している。

一方、以下の限界がある。

第一に、すべての実験が英語であり、学習データもMS MARCOに限定されている。多言語検索や、より小規模な学習データで同じ効果が得られるかは検証されていない。

第二に、Cross-Encoderによる約80万クエリの再評価、複数回の候補検索、64-way蒸留を必要とするため、推論時のインデックスは軽量でも学習コストは高い。

第三に、Denoised Supervisionでは複数の要素が同時に変更されている。Cross-Encoder蒸留、Hard Negative、64-way学習、In-batch Negative、二段階の再検索を個別に除去した詳細なアブレーションは限定的であり、どの要素がどの程度性能向上に寄与したかは十分に分離されていない。

第四に、情報検索データセットには「関連しているがラベルが付いていない文書」が多く存在する。BEIR、Open-QA、LoTTEでは正解ラベルの作り方も異なるため、個々の数値を単純に比較する際には注意が必要である。

第五に、レイテンシ評価は主としてColBERTv2内部のパラメータ比較であり、他モデルを同一ハードウェア・同一最適化条件で比較したものではない。したがって、本論文から強く結論づけられるのは容量効率と検索精度であり、システム全体の速度優位性については追加評価が必要である。これらの点は著者ら自身も研究上の制約として認識している。

## 7. まとめ

ColBERTv2は、ColBERTのLate Interactionを維持しながら、Cross-Encoder蒸留によるDenoised Supervisionと、セントロイドベースのResidual Compressionを導入した検索モデルである。

主な成果は次の通りである。

* MS MARCOでMRR@10 39.7を達成
* 28件のOut-of-domain評価のうち22件で最高性能
* 長尾領域評価用データセットLoTTEを新規提案
* ColBERTのインデックスを154 GiBから16～25 GiBへ削減
* 2-bit圧縮ではほぼ精度低下なし

「細粒度なトークン照合は高精度だが容量が大きすぎる」というLate Interactionの主要な弱点を、モデル構造を大きく変更せずに解決した研究と位置付けられる。

## 8. 総評

**査読結果:** NAACL 2022 Main Conferenceに採択され、本会議プロシーディングスへ掲載された。公開された公式掲載情報から確認できるのは採択・掲載の事実までで、論文固有の査読スコアや査読コメントは確認できなかった。([ACL Anthology][1])

**本要約での評価: Strong Accept相当**

検索モデルとしての新規性、実用性、実験の広さが高い水準にある。特に、Late Interactionの意味空間とResidual Compressionを結び付けた設計は自然であり、圧縮率だけでなく、圧縮後の検索精度を詳細に示している点が強い。

また、MS MARCOだけでなく、BEIR、Wikipedia Open-QA、LoTTEを用いて領域外性能を検証しており、単一ベンチマークへの過適合ではないことを示している。LoTTEの提案も、長尾領域・実検索クエリに焦点を当てた独立した貢献である。

弱点は、蒸留学習の計算コスト、各学習要素を分離するアブレーションの不足、多言語評価の欠如、他方式との厳密なEnd-to-Endレイテンシ比較がないことである。それでも、検索精度とインデックス容量のトレードオフを大きく改善しており、Late Interaction Retrieverを実用的な大規模検索へ近づけた影響力の高い論文である。
