## 1. はじめに

**論文名:** *Product Quantization for Nearest Neighbor Search*
**著者:** Hervé Jégou, Matthijs Douze, Cordelia Schmid
**掲載誌:** IEEE Transactions on Pattern Analysis and Machine Intelligence, Vol. 33, No. 1, pp. 117–128, 2011
**DOI:** 10.1109/TPAMI.2010.57

本論文は、高次元ベクトルの近似最近傍探索を、少ないメモリと高速な距離計算で実現する**積量子化（Product Quantization: PQ）**を提案した論文である。データベースベクトルを短い符号に圧縮したままユークリッド距離を近似し、さらに転置ファイルと組み合わせることで、数十億ベクトル規模への拡張性を示している。([IEEE Xplore][1])

## 2. 背景

厳密な最近傍探索では、データベースサイズを (n)、次元数を (D) とすると、全ベクトルとの距離計算に (O(nD)) の処理が必要となる。KD-treeなどの空間分割法も、高次元では次元の呪いにより全探索に近い挙動となる。

当時の代表的な近似手法にはLSH、ランダム化KD-tree、階層的k-means、Spectral Hashingなどがあった。しかし、LSHやFLANNは索引に大きなメモリを必要とし、最終的な候補再ランキングのために元の浮動小数点ベクトルを保持しなければならない。逆に、短いバイナリ符号を使うハッシュ法は省メモリだが、ハミング距離が取り得る値が少なく、近傍を細かく順位付けしにくい。([ResearchGate][2])

通常のベクトル量子化で64ビット相当の表現力を得ようとすると、理論上 (2^{64}) 個のセントロイドが必要になる。このコードブックは学習も保存も不可能である。そこで本論文は、大きなコードブックを明示的に作らず、複数の小さなコードブックの直積として表現する。

## 3. 提案手法

### 3.1 積量子化

(D) 次元ベクトル (x) を、次元数 (D/m) の (m) 個の部分ベクトルに分割する。

[
x=(u_1(x),u_2(x),\ldots,u_m(x))
]

各部分ベクトルを、それぞれ独立した量子化器 (q_j) で量子化する。

[
q(x)=\left(q_1(u_1(x)),\ldots,q_m(u_m(x))\right)
]

各部分量子化器が (k^*) 個のセントロイドを持つ場合、直積コードブック全体では

[
k=(k^*)^m
]

個のセントロイドを表現できる。一方、実際に保存するのは (m) 個の小さなコードブックだけであり、必要な浮動小数点数は (k^*D) 個である。符号長は

[
m\log_2 k^*
]

ビットとなる。代表的な設定 (m=8,\ k^*=256) では、各部分ベクトルを8ビットで表し、全体を64ビットに圧縮できる。([ResearchGate][2])

### 3.2 SDCとADC

本論文は二種類の距離推定法を定義する。

**SDC（Symmetric Distance Computation）**では、クエリとデータベースの両方を量子化し、セントロイド間距離を表引きする。

[
\hat d(x,y)=d(q(x),q(y))
]

**ADC（Asymmetric Distance Computation）**では、データベースベクトル (y) だけを量子化し、クエリ (x) は元の浮動小数点表現のまま使用する。

[
\tilde d^2(x,y)
===============

\sum_{j=1}^{m}
\left|u_j(x)-q_j(u_j(y))\right|^2
]

クエリごとに、各部分ベクトルと全セントロイド間の距離表を作成しておけば、各データベースベクトルの距離計算は基本的に (m) 回の表引きと加算で済む。クエリ側の量子化誤差がないため、ADCはSDCより高精度である。([ResearchGate][2])

理論面では、ADCの平均二乗距離誤差が量子化器の再構成MSE以下になることを示している。一方、SDCの誤差上界は概ね (2,\mathrm{MSE}) となる。また、量子化セル内の平均歪みを加えることで距離推定のバイアスを補正する方法も導出している。ただし補正により分散が増え、最近傍の順位付け性能はむしろ低下したため、探索では補正しないADCを推奨している。([ResearchGate][2])

### 3.3 IVFADC

全データとのADC計算を避けるため、粗い量子化器 (q_c) と転置ファイルを導入する。

データベースベクトル (y) は、まず粗いセントロイド (q_c(y)) に割り当てられる。その後、残差

[
r(y)=y-q_c(y)
]

を積量子化し、対応する転置リストに「ベクトル識別子＋残差PQ符号」を格納する。

検索時にはクエリを近い (w) 個の粗いセントロイドに割り当て、対応する転置リストだけを走査する。リストが均等であれば、調べるベクトル数は概ね

[
n\frac{w}{k_0}
]

となる。ここで (k_0) は粗いコードブックのサイズである。残差は元ベクトルよりエネルギーが小さいため、同じ符号長でも直接量子化するより精密に表現できる。([ResearchGate][2])

## 4. 実験方法

主要な評価データは以下の通りである。([ResearchGate][2])

| データ        | 次元数 |      学習 |    データベース |    クエリ |
| ---------- | --: | ------: | --------: | -----: |
| SIFT       | 128 | 100,000 | 1,000,000 | 10,000 |
| Color GIST | 960 | 100,000 | 1,000,991 |    500 |

評価指標には、真の最近傍が上位 (R) 件以内に含まれるクエリの割合である **recall@R** を使用した。主な比較対象はSpectral Hashing、Hamming Embedding、FLANNである。

PQについては (m\in{1,2,4,8,16})、(k^*\in{16,64,256,1024,4096}) を変更し、符号長と検索精度の関係を調べている。IVFADCでは、粗いセントロイド数 (k_0) と走査セル数 (w) も変化させた。標準的な比較では (m=8,\ k^*=256) の64ビット符号が使われた。([ResearchGate][2])

さらに、100万画像から抽出した約20億個のSIFT記述子に対し、3万個のクエリ記述子を用いた大規模実験を行った。画像検索ではINRIA Holidaysに最大100万枚の妨害画像を加え、mAPを評価している。([ResearchGate][2])

## 5. 結果

ADCは一貫してSDCより高精度だった。例えば (m=8) では、ADCの (k^*=64) が、SDCの (k^*=256) とほぼ同等の精度を達成した。したがって、クエリを量子化しないことが、同程度の計算量で明確な精度向上につながっている。([ResearchGate][2])

約100万件のGISTデータでは、平均検索時間とrecall@100が次のようになった。

| 手法               | 設定             |      時間 | recall@100 |
| ---------------- | -------------- | ------: | ---------: |
| SDC              | 64ビット          | 16.8 ms |      0.446 |
| ADC              | 64ビット          | 17.2 ms |      0.652 |
| IVFADC           | (k_0=1024,w=8) |  8.8 ms |      0.682 |
| Spectral Hashing | 64ビット          | 22.7 ms |      0.132 |

IVFADCはADCより少ない時間で高い再現率を達成している。(w) を増やすほど精度は上がるが、走査する転置リストと計算時間も増加する。([ResearchGate][2])

Spectral Hashingと同程度のrecallを得る場合、ADCが返す必要のある候補数は約1桁少なかった。低順位領域ではIVFADCがADCをさらに上回り、Hamming Embeddingも明確に上回った。FLANNとの比較でも、広い速度・精度範囲でIVFADCが優位であり、索引構造はIVFADCが25 MB未満、FLANNが250 MB超だった。([ResearchGate][2])

部分ベクトルの作り方も重要だった。特にGISTでは、単純な連続次元分割のrecall@100が0.338だったのに対し、同じ方向ビンをまとめる構造化分割では0.652まで向上した。一方SIFTでは、空間的に近い成分をまとめる自然な並びも有効であり、最適な分割は特徴量の構造に依存する。([ResearchGate][2])

20億SIFT記述子の実験では、IVFADCの各符号に対する処理はHamming Embeddingと同じ8回程度の表引きで実装でき、浮動小数点演算を含んでも大幅には遅くならなかった。Holidays画像検索では、100万枚の妨害画像を加えた64ビット条件で、Hamming EmbeddingのmAP 0.497をIVFADCが0.517へ改善した。([ResearchGate][2])

## 6. 考察

本手法の本質は、**指数的に大きな仮想コードブックを、線形サイズの複数コードブックで表現したこと**にある。これにより、高い量子化能力、省メモリ性、表引きによる高速距離計算を同時に実現した。

また、単にベクトルを圧縮するだけではなく、クエリを非圧縮のまま比較するADC、粗い量子化と残差符号化を組み合わせるIVFADCまで一体的に設計した点が強い。特に「圧縮表現から元ベクトルを復元する」のではなく、「圧縮符号から検索に必要な距離を直接計算する」という設計が重要である。

一方、以下の制約がある。

* 対象距離は基本的にユークリッド距離であり、任意の距離尺度に対応するハッシュ法ほど一般的ではない。
* 固定された部分空間分割は、部分空間間の相関を直接モデル化しない。実験でも次元の並べ方が精度に大きく影響している。
* k-means学習は局所最適解であり、分割とコードブックを統合的に最適化していない。
* IVFADCでは全粗量子化セルに共通の残差PQを使用する。セルごとに別のPQを学習すれば精度向上が期待できるが、メモリ量が大きくなる。
* 主な評価対象はSIFTとGISTであり、現在一般的な深層学習ベースの埋め込み表現については検証されていない。
* (K>1) の近傍探索についても同様の傾向になると述べているが、詳細な結果は掲載されていない。([ResearchGate][2])

## 7. まとめ

本論文は、高次元ベクトルを複数の低次元部分空間に分割し、各部分を個別に量子化する積量子化を提案した。ADCにより、データベースのみを圧縮しながら精度の高い距離推定を可能にし、IVFADCにより検索対象を転置リストの一部へ限定した。

その結果、64ビット程度の短い符号でも高い検索精度を維持し、Spectral Hashing、Hamming Embedding、FLANNに対して、精度・速度・メモリのトレードオフで優れた結果を示した。さらに約20億ベクトルでの実験により、大規模検索への適用可能性を実証している。([ResearchGate][2])

## 8. 総評

本論文は、問題設定、数理的定式化、実装可能性、大規模実験のバランスが非常に良い。特にPQ、ADC、残差符号化、転置ファイルを段階的に組み合わせ、各要素の効果を実験で切り分けている点は高く評価できる。固定的な部分空間分割など改善余地はあるが、提案の新規性と実用性を損なう重大な欠点ではない。
