## 1. はじめに

本論文 **“Matryoshka Representation Learning”** は、単一の高次元埋め込みの「先頭部分」だけでも有用な表現になるよう訓練する **Matryoshka Representation Learning（MRL）** を提案している。

例えば2,048次元の埋め込み (z) に対して、先頭8、16、32、…、2,048次元を、それぞれ独立した埋め込みとして利用できるようにする。これにより、同じモデルと同じ埋め込みから、計算資源や要求精度に応じて表現次元を切り替えられる。論文はNeurIPS 2022のMain Conference Trackに採択されている。([arXiv][1])

---

## 2. 背景

一般的な表現学習では、モデルが出力する埋め込み次元は固定されている。ところが、実際の下流タスクでは必要な表現能力が異なる。

単純な分類や容易な入力では低次元表現で十分な一方、細粒度分類や難しい検索クエリには高次元表現が必要になる。しかし固定埋め込みでは、すべてのデータとタスクに最大次元を用いるため、保存容量、メモリ、検索計算量、分類層の計算量が過剰になる。特に検索コストは、データベースサイズ (N) と埋め込み次元 (d) に伴って増大する。

従来の対策には、次元ごとに複数モデルを学習する方法、SVDなどによる事後的な次元削減、ランダム射影、Slimmable Networkなどがある。しかし、複数モデルの学習・保存が必要になったり、データベース全体を再エンコードする必要が生じたり、低次元で精度が大きく低下したりする。MRLは、最初から情報を次元順に配置することで、この問題を解こうとする。

---

## 3. 提案手法

### 基本的な定式化

入力 (x) をエンコーダ (F) に与え、最大 (d) 次元の表現を得る。

[
z=F(x;\theta_F)\in\mathbb{R}^{d}
]

次に、利用対象とする次元集合を

[
\mathcal M={8,16,32,\ldots,d}
]

のように定める。各 (m\in\mathcal M) について、埋め込みの先頭部分

[
z_{1:m}\in\mathbb{R}^{m}
]

だけを使って元のタスクを解けるようにする。教師あり分類の場合、目的関数は概ね次の形になる。

[
\min_{\theta_F,{W^{(m)}}}
\frac{1}{N}
\sum_{i=1}^{N}
\sum_{m\in\mathcal M}
c_m,
\mathcal L
\left(
W^{(m)}F(x_i;\theta_F)_{1:m},y_i
\right)
]

論文では基本的に各損失の重みを (c_m=1) としている。最大次元だけでなく、複数の先頭部分に同時に損失を与えることで、重要な情報を前方の次元に集め、その後の次元により詳細な情報を追加する「粗から細」の表現を学習する。

### MRLとMRL-E

通常の **MRL** では、各次元 (m) に別々の分類器 (W^{(m)}) を用意する。

一方、**MRL-E（Efficient MRL）** では、最大次元用の重み行列 (W) を共有し、

[
W^{(m)}=W_{:,1:m}
]

とする。MRL-Eは分類ヘッドの追加パラメータをほぼ不要にできるが、低次元で通常のMRLより若干精度が低下することがある。ResNet50の実装では、通常のMRLによるモデル保存量の増加は約8 MBで、MRL-Eでは追加保存量がないと報告されている。

MRLは教師あり分類に限定されない。対照学習では各次元の表現を個別に正規化して対照損失を適用し、BERTのMasked Language Modelingでは入力埋め込みと出力分類器の重み共有により、自然にMRL-E型の構成になる。

## 4. 実験方法

主な実験設定は以下である。

| モダリティ | モデル       | 学習データ・評価対象                    |
| ----- | --------- | ----------------------------- |
| 画像    | ResNet50  | ImageNet-1K、ImageNet-4K       |
| 画像    | ViT-B/16  | JFT-300M                      |
| 画像・言語 | ALIGN     | 約18億の画像・テキスト対                 |
| 言語    | BERT-Base | English Wikipedia、BooksCorpus |

ResNet50では最大2,048次元に対して

[
\mathcal M={8,16,32,64,128,256,512,1024,2048}
]

を使用し、ViT-B/16とBERTでは最大768次元に対して

[
\mathcal M={12,24,48,96,192,384,768}
]

を使用している。比較対象は、次元ごとに独立して学習したFixed Featureモデル、SVD、ランダム特徴選択・射影、Slimmable Networkなどである。原則としてMRL専用の大規模なハイパーパラメータ探索は行わず、独立学習ベースラインと同じ設定を使用している。

### Adaptive Classification

最小次元から分類を開始し、最大Softmax確率が閾値を超えればそこで予測を確定する。信頼度が低ければ、8次元から16次元、32次元へと順に拡張する。

閾値はImageNet-1K検証集合のうち1万枚で決定し、残り4万枚で評価している。結果は30種類のランダム分割について平均されている。

### Adaptive Retrieval

検索では、小さい次元 (D_s) でデータベースから候補200件を取得し、候補だけを大きい次元 (D_r) で再ランキングする。

さらに、候補数を段階的に減らしながら表現次元を増やす **Funnel Retrieval** も提案している。検索精度にはmAP@10やPrecision@kを用い、計算量とHNSWによる実測時間も評価している。

## 5. 結果

### 通常の分類・表現品質

ImageNet-1KのResNet50分類では、MRLは独立学習モデルとほぼ同等以上の精度を示した。特に低次元で効果が大きい。

|    次元 | 独立学習FF |    MRL |
| ----: | -----: | -----: |
|     8 | 65.29% | 66.63% |
|    16 | 72.85% | 73.53% |
|   128 | 75.29% | 76.30% |
|   512 | 76.18% | 76.65% |
| 2,048 | 76.87% | 76.80% |

最大次元ではほぼ同等であり、低次元ではMRLが一貫して優位である。1-NN評価でも、8次元で独立学習モデルの58.93%に対してMRLは62.19%を達成した。SVDやランダム特徴選択は、とりわけ低次元で大幅に精度が低下している。

明示的に損失を与えた次元だけでなく、24次元や48次元など、その中間の次元でも精度が滑らかに向上することが確認された。このため、学習時にすべての次元を個別に最適化しなくても、推論時には細かい単位で次元を選択できる。

### Adaptive Classification

平均約36～37次元を使用する設定で、ImageNet-1K Top-1精度76.28～76.30%を達成した。これは512次元の独立学習モデルと同等の精度であり、平均表現サイズは約14分の1である。

ただし、複数の分類ヘッドを順番に評価した累積次元数で計算すると平均約62次元となり、この場合の削減率は約8.2倍である。MRL-Eでは重み共有により、この累積計算を軽減できる。

### Adaptive Retrieval

ImageNet-1Kでは、16次元で候補を抽出し、2,048次元で200件を再ランキングすることで、2,048次元の単発検索とほぼ同等の精度を保ちながら、理論FLOPsを約128分の1、同一ハードウェア上のHNSW実測時間を約14分の1に削減した。

より難しいImageNet-4Kでは、候補抽出に64次元が必要となり、削減率は理論上約32倍、実測で約6倍だった。したがって「14倍高速化」はすべてのデータセットに共通する値ではなく、データセットの難易度や検索構成に依存する。

### 他モダリティと追加評価

JFT-300MおよびALIGNでは、特に低次元領域でMRLが通常表現より大幅に高いk-NN精度を示した。BERTのMasked Language Modelingでは、MRLは通常の固定次元モデルから概ね0.5ポイント以内の精度に収まっているが、各次元でわずかに低い結果であり、NLPにおける優位性よりも「適用可能性」を示す実験と捉えるべきである。

分布外評価では通常表現と同等以上の頑健性を示し、ImageNet-Aでは約0.6ポイント改善した。長尾逐次学習のFLUIDでは、既知クラスの性能を維持しながら、新規の末尾クラスで最大約2ポイントの改善が報告されている。

## 6. 考察

本論文の最大の長所は、**単純な損失関数の変更だけで、単一の埋め込みに可変精度・可変計算量という機能を持たせたこと**である。別々の低次元モデルを多数管理する必要がなく、同じ埋め込みを先頭から切り出すだけで利用できる点は、検索データベースや埋め込みAPIとの相性がよい。

また、SVDなどの事後圧縮と異なり、低次元で利用されることを訓練時から明示している。この違いが、低次元での大きな性能差につながっている。

一方、査読上は以下の点に注意が必要である。

第一に、MRLが削減する中心的なコストは、埋め込み保存、分類ヘッド、距離計算、検索などの**下流処理コスト**である。ResNetやViTのバックボーン自体は通常どおり実行されるため、モデルの前向き計算が支配的な小規模タスクでは、全体の高速化が限定的になる可能性がある。論文自身も、Web規模では検索・利用コストが特徴抽出コストを上回るという設定を主眼としている。

第二に、Web規模のJFTおよびALIGN実験では、計算量の制約から、各次元で独立学習した強いベースラインが用意されていない。主に最大次元モデルから選択した低次元特徴との比較であるため、「Web規模でも独立低次元モデルを常に上回る」とまでは結論できない。

第三に、情報が粗から細へ配置されることは実験的に示されているが、その順序性や中間次元での補間性能を保証する理論はない。損失重み (c_m)、最小次元、次元間隔についても手動設定であり、論文自身が適応的な損失配分を将来課題として挙げている。

第四に、Adaptive Classificationの閾値やAdaptive Retrievalの候補数は、検証データや特定のハードウェア構成に依存する。分布変化がある実運用では、閾値の再較正やレイテンシを含む再評価が必要になる。

---

## 7. まとめ

MRLは、最大次元の埋め込み内部に、先頭8次元、16次元、32次元といった複数の有効な部分表現を入れ子状に学習する手法である。

その結果、単一モデルと単一埋め込みだけで、低コスト・低精度から高コスト・高精度まで連続的に切り替えられる。特に大規模ベクトル検索では、低次元による候補抽出と高次元による再ランキングを組み合わせることで、大きな実用上の高速化を実現した。

本論文の本質は単なる埋め込み圧縮ではなく、**埋め込みを固定容量の成果物から、計算予算に応じて切り替えられる多重解像度の成果物へ変えたこと**にある。

**本要約における査読判定：Accept。**

発想は簡潔だが新規性と実用性が高く、画像、画像・言語、言語という複数モダリティと、大規模分類・検索で有効性を示している。特に、低次元表現を別々に学習することなく、同一埋め込みのprefixとして利用できる設計は、ストレージ、検索、モデル運用の観点から明確な価値がある。
