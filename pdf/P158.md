# P158 RouteLLM: Learning to Route LLMs with Preference Data

## 1. はじめに

本論文は、各クエリを「高性能だが高価なLLM」と「安価だが性能の低いLLM」のどちらに送るかを、ユーザの選好データから学習する **RouteLLM** を提案する。

ルータはクエリだけを観察し、「強いモデルが弱いモデルに勝つ確率」を推定する。その確率とコスト閾値を比較して、生成前に一方のモデルだけを選択する。目標は、強いモデルの品質をできるだけ維持しながら、その呼び出し回数を削減することである。([arXiv][1])

## 2. 背景

すべてのクエリをGPT-4級モデルに送れば品質は高いが、運用コストが大きい。一方、小規模・オープンモデルだけを使うと、複雑な質問で品質が低下する。

既存のアンサンブルやカスケード方式には、複数モデルを呼び出してから回答を選択・検証するものが多く、コストだけでなくレイテンシも増加する。RouteLLMは、**回答生成前にモデルを一つだけ選ぶ**ことで、この問題を回避する。また、報酬モデルが「生成された回答」を評価するのに対し、LLMルータは「クエリだけから適切なモデルを選ぶ」という点で異なる。([OpenReview][2])

実用的なルータには、未知ドメインへの汎化、モデル更新への追従、低い推論オーバーヘッドが必要となる。本論文は特に、異なるモデルペアへのゼロショット転用を重視している。

## 3. 提案手法

クエリを (q)、強モデルを (M_s)、弱モデルを (M_w) とし、ルータは次の確率を推定する。

[
P_\theta(M_s \text{ が } M_w \text{ に勝つ}\mid q)
]

この確率が閾値 (\alpha) 以上なら強モデルへ、未満なら弱モデルへ送る。大きな (\alpha) は弱モデルを多く使うため低コストになり、小さな (\alpha) は品質重視となる。([arXiv][1])

### 学習データ

主な学習データはChatbot Arenaの約8万件の人間による比較投票である。モデル間の直接比較が極めて疎であるため、モデルをランキングに基づく10段階のtierにまとめ、上位2 tierを強モデル群、次のtierを弱モデル群として扱う。前処理後は、64モデルに関する約6.5万件の比較が使われる。([arXiv][1])

さらに、次のデータ拡張を行う。

* **Golden-label拡張 (D_{\text{gold}})**：MMLU検証セット約1,500問について、正解との一致から強弱モデルの勝敗を決める。
* **LLM-judge拡張 (D_{\text{judge}})**：Nectarのクエリに対するGPT-4とMixtralの回答を、GPT-4 judgeで比較する。約12万件を約700ドルで構築した。([arXiv][1])

### 4種類のルータ

本論文は単一アーキテクチャに限定せず、以下を比較する。

1. **Similarity-weighted ranking**：新しいクエリに類似した学習クエリを重視するBradley–Terryモデル
2. **Matrix Factorization**：クエリ埋め込みとモデル埋め込みの双線形相互作用から各モデルの品質を推定
3. **BERT classifier**：BERTのCLS表現を用いた分類器
4. **Causal LLM classifier**：Llama 3 8Bに勝敗ラベルを次トークンとして出力させる分類器

特にMatrix Factorizationは小型かつ高速で、性能と運用コストのバランスがよい。([arXiv][1])

### 評価指標

弱モデルと強モデルの性能差のうち、ルータが回復した割合を

[
PGR=\frac{r(\text{router})-r(M_w)}
{r(M_s)-r(M_w)}
]

と定義する。

さらに、強モデル呼び出し率を変化させたときのPGRの平均を **APGR**、PGRを50％または80％回復するために必要な最小強モデル呼び出し率を **CPT(50%)、CPT(80%)** とする。CPTは低いほど優れている。([arXiv][1])

## 4. 実験方法

基本実験では、強モデルに`gpt-4-1106-preview`、弱モデルにMixtral-8x7Bを使用する。比較対象は、同じ強モデル呼び出し率になるようランダムに振り分けるルータである。

評価データは以下の3種類である。

* MT Bench：160件のオープンエンド対話
* MMLU：57分野、14,042問の5-shot評価
* GSM8K：1,000問以上の算数問題に対する8-shot評価

学習データとの重複を除外したうえで評価している。また、学習時に含まれないClaude 3 Opus／Sonnet、およびLlama 3.1 70B／8Bへ、ルータを再学習せず転用する実験も行っている。([arXiv][1])

## 5. 結果

主要結果は次の通りである。数値はPGRの50％を回復する場合である。([OpenReview][2])

| ベンチマーク   | 最良CPT(50%) |   ランダム | GPT-4に対する品質 | コスト削減比 |
| -------- | ---------: | -----: | ----------: | -----: |
| MT Bench |     13.40% | 49.03% |         95% |  3.66倍 |
| MMLU     |     35.40% | 50.07% |         92% |  1.41倍 |
| GSM8K    |     33.64% | 50.00% |         87% |  1.49倍 |

MT Benchでは、(D_{\text{judge}}) で拡張したMatrix FactorizationがAPGR 0.802を達成し、ランダムルータの0.500を大きく上回った。CPT(80%)も31.31％であり、ランダムの78.08％から大幅に低下した。([arXiv][1])

一方、Chatbot Arenaだけで学習したルータは、MMLUとGSM8Kではランダム以下または同程度であった。MMLUの約1,500件の正解付きデータは全学習データの2％未満だが、追加するだけでCPT(50%)が約20％改善した。すなわち、**ルータの性能はモデル規模以上に、学習クエリと運用クエリの分布一致に依存する**。([arXiv][1])

未学習モデルへの転用でも性能を維持した。特にLlama 3.1 70B／8Bでは、SW RankingがCPT(50%) 21.18％、CPT(80%) 29.39％を達成し、再学習なしでランダムルータを大きく上回った。([arXiv][1])

ルーティング自体のコストは小さい。Matrix Factorizationは毎秒約155リクエストを処理し、100万リクエスト当たり約3.32ドルと見積もられている。最も高価なSW Rankingでも、論文の想定ではGPT-4生成コストの0.4％以下である。([arXiv][1])

## 6. 考察

本論文の最大の貢献は、LLMルーティングを単なる「難易度分類」ではなく、**人間が観測したモデル間の相対的な勝敗予測**として定式化した点にある。APGRとCPTも、品質とコストのトレードオフを単一の精度値より明確に示している。

また、巨大なCausal LLMルータが常に最良ではなく、Matrix Factorizationや類似度ベース手法が少量データで強いという結果も実務的に重要である。データ分布がずれた場合には複雑な分類器ほど不利になり、少量のインドメインデータ追加がモデルの大型化より効果的になり得る。

ただし、以下の制約がある。

第一に、実験は基本的に二値ルーティングであり、3個以上のモデルから価格・専門性・レイテンシを同時に考慮して選ぶ問題は扱っていない。

第二に、「品質を犠牲にせず2倍以上のコスト削減」という表現は限定的である。3.66倍の削減はMT Benchで得られた一方、MMLUとGSM8Kは1.41～1.49倍で、品質もGPT-4の92％、87％に低下している。

第三に、3.66倍という値は本文の計算上、常時GPT-4を使う場合との直接比較ではなく、同一PGRにおける**ランダムルータとの強モデル呼び出し率の比** (49.03/13.40) である。表の「GPT-4に対する削減」という表現とは若干の不整合がある。([arXiv][1])

第四に、(D_{\text{judge}}) はGPT-4回答をGPT-4 judgeで評価し、MT BenchもLLM judgeを使用する。このため、GPT-4固有の回答スタイルに有利な学習・評価バイアスが入り得る。

最後に、モデルペアの転用実験は有望だが、主にMT Benchで検証されている。コード、医学、多言語、長文脈など、モデルごとの専門性が交差する環境で同様に汎化するかは未確定である。著者自身も、実環境の分布差、複数モデルへの拡張、ルータ間の性能差の原因を今後の課題としている。([arXiv][3])

## 7. まとめ

RouteLLMは、クエリに対する強モデルと弱モデルの相対的な勝率を選好データから推定し、閾値によって品質とコストを制御する手法である。

主な知見は、次の3点に集約される。

* 人間の比較データから、有効なLLMルータを学習できる。
* ルータ性能には学習・運用データの分布一致が極めて重要である。
* Matrix Factorizationのような軽量手法でも、高性能かつ低オーバーヘッドなルーティングが可能である。

実運用では、公開済みルータをそのまま使うより、少量の自社クエリを用いて再較正・データ拡張することが重要となる。

## 8. 総評

**査読結果：採択相当（Weak Accept）。**

コスト削減という実務上重要な問題に対し、明確な定式化、複数のルータ設計、新しい評価指標、モデル間転用、推論オーバーヘッドまで一貫して評価している点を高く評価する。コードも公開されており、再現性と実装可能性が高い。([GitHub][4])

一方、二値ルーティングへの限定、LLM judge依存、評価分布への感度、コスト削減比の表現には注意が必要である。主張を「どの分野でも品質低下なしに2倍以上削減」と一般化することはできない。

公式の査読結果では、本論文は **ICLR 2025 Posterとして採択**されている。総合的には、基礎理論上の大きな革新というより、LLMルーティングを実用可能な学習・評価フレームワークとして確立した、影響力の高いシステム研究である。([OpenReview][5])
