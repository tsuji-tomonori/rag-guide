# P153：Membership Inference Attacks Against Machine Learning Models

## 1. はじめに

本論文は、Reza Shokri、Marco Stronati、Congzheng Song、Vitaly ShmatikovによるIEEE Symposium on Security and Privacy 2017の論文である。研究対象は、**あるデータが機械学習モデルの学習データに含まれていたかを、モデルへのブラックボックスアクセスだけから判定する「メンバーシップ推論攻撃」**である。([arXiv][1])

攻撃者には候補レコードが既知であり、そのレコードを対象モデルに入力して得られる予測確率を観測できるとする。論文の中心的なアイデアは、対象モデルが学習データと未学習データに対して示す出力の違いを、別の機械学習モデルで学習することである。

主な貢献は次の三点である。

1. ブラックボックス分類モデルに対する汎用的なメンバーシップ推論攻撃を定式化した。
2. 対象モデルの学習データを知らなくても攻撃モデルを学習できる、**シャドウモデル学習**を提案した。
3. Google Prediction API、Amazon ML、ローカルニューラルネットワークを対象として、攻撃成功率、原因、緩和策を実験的に評価した。

---

## 2. 背景

### メンバーシップ推論とは

対象モデルを (f_{\mathrm{target}})、候補レコードを ((x,y)) とする。対象モデルは、

[
f_{\mathrm{target}}(x)=\mathbf p
]

として、各クラスへの予測確率ベクトル (\mathbf p) を返す。

攻撃者の目的は、

[
(x,y)\in D_{\mathrm{target}}^{\mathrm{train}}
]

であるか、すなわち候補レコードが対象モデルの学習データに含まれていたかを判定することである。

これはモデル反転攻撃とは異なる。モデル反転攻撃が入力属性やクラスの代表的特徴を推定するのに対し、メンバーシップ推論は、**既知の特定レコードが学習に使用されたか**を推定する。医療モデルの場合、ある患者が特定疾患に関するモデルの学習データに含まれていると分かるだけでも、その患者の疾患情報を間接的に漏らす可能性がある。

### 攻撃者の能力

本論文では、攻撃者が以下の情報を持つと仮定する。

* 判定対象のレコード (x) と正解ラベル (y)
* 対象モデルへのクエリアクセス
* モデルが返す各クラスの予測確率
* 入出力の形式
* 対象モデルと同様の学習アルゴリズム、または対象モデルを構築したものと同じ機械学習サービスへのアクセス

対象モデルの内部パラメータ、ネットワーク構造、実際の学習データは知らなくてもよい。

### 攻撃が成立する直観

一般に、学習済みモデルは学習データに対して、

* 正解確率が高い
* 予測エントロピーが低い
* 誤分類の仕方が未学習データと異なる

といった挙動を示すことがある。過学習はこの差を大きくする主要因であるが、本論文は、モデルの種類や構造も情報漏洩量に影響すると指摘している。

## 3. 提案手法

### 3.1 攻撃モデル

攻撃時には、候補レコード (x) を対象モデルに入力し、予測確率ベクトル (\mathbf p) を得る。正解ラベル (y) に対応する攻撃モデル (g_y) に (\mathbf p) を入力し、

[
g_y(\mathbf p)\rightarrow {\mathrm{in},\mathrm{out}}
]

として、学習データに含まれていたかを判定する。

攻撃モデルは対象モデルのクラスごとに一つずつ構築される。これは、対象モデルの出力分布が正解クラスによって大きく異なるためである。

### 3.2 シャドウモデル

問題は、対象モデルについては、どのレコードが「in」でどのレコードが「out」かを攻撃者が知らないことである。

そこで、攻撃者は対象モデルを模倣する複数の**シャドウモデル**を構築する。各シャドウモデルについては、攻撃者自身が学習データを用意するため、メンバーシップの正解を知っている。

学習手順は次のようになる。

1. 対象モデルと似た形式のデータで複数のシャドウモデルを学習する。
2. 各シャドウモデルの学習データを入力し、その出力を「in」とラベル付けする。
3. 学習データと独立なテストデータを入力し、その出力を「out」とラベル付けする。
4. 全シャドウモデルの出力を集約する。
5. 正解クラスごとにデータを分割し、二値分類器である攻撃モデルを学習する。

対象モデルの学習データとシャドウモデルの学習データは重複しない設定で評価されている。シャドウモデル間の学習データは重複してもよい。

### 3.3 シャドウモデル用データの生成

対象データと同じ分布からデータを取得できない場合に備え、三種類の生成方法が提案されている。

**モデルベース生成**では、対象モデルへのクエリを繰り返し、特定クラスに対する予測確率が高くなるように入力特徴を変更する。山登り法によって高信頼度のレコードを探索し、シャドウモデルの学習データとして利用する。

**統計ベース生成**では、各特徴の周辺分布だけが既知であると仮定し、それぞれの分布から独立に特徴値をサンプリングする。

**ノイズ付き実データ**では、対象データと似たデータの一部特徴をランダムに変更し、分布が完全には一致しない状況を再現する。

この攻撃は単に「最大予測確率が高ければ学習データ」と判定するものではない。シャドウモデルの学習データとテストデータの双方に高信頼度レコードが含まれるため、攻撃モデルは、クラス別の確率分布全体や誤分類パターンを学習する。

## 4. 実験方法

### 使用データ

| データセット               | 内容                                               |
| -------------------- | ------------------------------------------------ |
| CIFAR-10 / CIFAR-100 | 10クラスおよび100クラスの画像分類。学習件数を変化させて評価                 |
| Purchase             | 197,324人分、600次元の購入有無特徴。2、10、20、50、100クラスにクラスタリング |
| Location             | Foursquareの位置履歴から作成した5,010人、446次元、30クラスのデータ      |
| Texas Hospital       | 67,330件の入院記録、6,170個の二値特徴、100種類の主要処置を予測           |
| MNIST                | 手書き数字10クラス                                       |
| UCI Adult            | 所得が5万ドルを超えるかを予測する二値分類                            |

Purchase、Texas Hospital、MNIST、Adultでは、対象モデルの学習にそれぞれ10,000件を使用している。CIFARでは学習件数を複数段階で変更した。

なお、Locationデータの対象モデル学習件数について、データ説明節では1,600件、実験設定節では1,200件と記述が食い違っている。

### 対象モデル

対象モデルは以下の三系統である。

* Google Prediction APIによるブラックボックスモデル
* Amazon MLによるブラックボックスモデル
* 著者らがローカルで学習したニューラルネットワーク

CIFARには畳み込み層と最大プーリング層を二組持つCNNを使用した。Purchaseには隠れ層128ユニットの全結合ネットワークを使用した。Amazon MLでは、学習パス数とL2正則化係数を変更した二つの設定を評価している。

### 攻撃モデルとシャドウモデル数

主な攻撃モデルは、隠れ層64ユニット、ReLU活性化を用いた全結合ニューラルネットワークである。別の実験では、攻撃モデル自体もGoogle Prediction APIで学習している。

シャドウモデル数は、CIFARで100、Purchaseで20、Texas Hospitalで10、Locationで60、MNISTで50、Adultで20である。

### 評価指標

評価データは、対象モデルの学習データとテストデータから同数ずつ選ばれている。そのため、メンバーと非メンバーの事前確率はそれぞれ0.5であり、ランダム推測のベースラインも0.5となる。

主な指標は以下である。

* **Precision**：メンバーと判定されたレコードのうち、実際にメンバーだった割合
* **Recall**：実際のメンバーのうち、正しくメンバーと判定された割合
* 一部の防御実験では、攻撃全体のAccuracyも報告

## 5. 結果

### 5.1 CIFARに対する攻撃

CIFAR-10では、学習データ数を2,500、5,000、10,000、15,000と増やしたとき、クラス別Precisionの中央値はそれぞれ0.78、0.74、0.72、0.71であった。

CIFAR-100では、学習データ数を4,600、10,520、19,920、29,540と増やしたとき、中央値は1.00、1.00、0.98、0.97であった。両データセットともRecallはほぼ1であり、特にCIFAR-100が極めて脆弱であった。

### 5.2 Purchaseデータに対する攻撃

100クラスのPurchaseモデルに対するクラス別Precisionの中央値は、

* Amazon MLのデフォルト設定：0.74
* Amazon MLの別設定：0.84
* Google Prediction API：0.94

であり、Recallはいずれもほぼ1であった。Googleモデルに対する全体Precisionは0.935である。

クラス数を増やした場合の全体Precisionは次のようになった。

| クラス数 | 攻撃Precision |
| ---: | ----------: |
|    2 |       0.505 |
|   10 |       0.550 |
|   20 |       0.590 |
|   50 |       0.860 |
|  100 |       0.935 |

出力クラス数が多いほど、対象モデルが各クラスを区別するために学習データの細かな特徴を保持し、メンバーシップ情報を漏らしやすくなると考察されている。

### 5.3 その他のデータセット

Googleモデルに対する攻撃Precisionは、Locationで0.678、Texas Hospitalで0.657であった。Texas Hospitalでは、半数のクラスでPrecisionが0.7を超え、20以上のクラスで0.85を超えた。

一方、Adultは0.503、MNISTは0.517であり、ほぼランダム推測と同等であった。Adultでは学習AccuracyとテストAccuracyがほぼ同じで、二値分類のため出力から得られる信号も少なかった。

### 5.4 不完全なシャドウデータ

Locationデータでは、シャドウデータの特徴をランダムに変更しても攻撃は機能した。

* 実データ：Precision 0.678
* 10%ノイズ：0.666
* 20%ノイズ：0.613

分布の推定が多少不正確でも、攻撃が一定の性能を維持することを示している。

Purchaseデータにおけるシャドウデータ生成方法の比較は次の通りである。

| シャドウデータ   | Precision | Recall |
| --------- | --------: | -----: |
| 実データ      |     0.935 |  0.994 |
| 周辺分布から生成  |     0.795 |  0.991 |
| 対象モデルから生成 |     0.896 |  0.526 |

モデルベース生成は高いPrecisionを達成したが、学習例の少ないクラスを十分生成できず、Recallが低下した。また、合成レコード1件当たり平均156回の対象モデルへのクエリが必要であった。

### 5.5 緩和策

著者らは、以下を防御策として評価した。

* 出力を上位 (k) クラスに限定
* 予測確率を丸める
* Softmaxの温度を上げて予測分布のエントロピーを増加
* L2正則化

Purchaseのローカルモデルでは、防御なしの攻撃Accuracy／Precisionが0.92／0.87だったのに対し、ラベルのみを返す場合は0.66／0.60となった。しかし、依然としてランダム推測を上回った。

強いL2正則化では攻撃Accuracy／Precisionが0.53／0.54まで低下したが、対象モデルのテストAccuracyも0.66から0.63に低下した。Texas Hospitalでも、攻撃をほぼランダムレベルに落とす強い正則化は、対象Accuracyを0.55から0.35に低下させた。

適度な正則化は、対象モデルの汎化性能を向上させながら情報漏洩を減らす場合がある。一方、出力確率を隠す、丸める、上位クラスだけ返すといった出力制限だけでは、攻撃を完全には防止できなかった。

## 6. 考察

### 攻撃が成功する理由

攻撃モデルが利用しているのは、メンバーと非メンバーに対する対象モデルの出力分布の差である。攻撃が成功するモデルでは、学習データに対して正解確率が高く、予測エントロピーが低い傾向が明確に観測された。

攻撃性能は主に、

1. 対象モデルの汎化性能
2. 学習データの代表性
3. クラス数
4. クラス当たりの学習データ数
5. モデルの構造や種類

に依存する。

過学習度が大きいほど攻撃性能が高くなる傾向は、同じ種類のモデル間では明確である。ただし、AmazonとGoogleのモデルを比較すると、過学習度だけでは漏洩量を説明できなかった。異なるモデル構造は、同程度の汎化ギャップでも異なる量の学習データ情報を保持する。

### 長所

本論文の最大の長所は、学習データのプライバシー漏洩を抽象的な可能性ではなく、**実行可能なブラックボックス攻撃として具体化したこと**である。

また、単一のデータセットやモデルだけでなく、画像、購入履歴、位置履歴、国勢調査、医療記録といった複数の領域を評価している。対象データと同一分布の実データを持たない攻撃者についても、ノイズ付きデータや合成データによる攻撃を検証しており、脅威モデルの幅が広い。

### 限界

第一に、評価ではメンバーと非メンバーを同数に設定している。実環境で対象モデルの学習データが母集団のごく一部である場合、メンバーの事前確率 (\pi) は0.5より大幅に小さくなる。実際の陽性的中率は、

[
\mathrm{PPV}
============

\frac{\mathrm{TPR}\pi}
{\mathrm{TPR}\pi+\mathrm{FPR}(1-\pi)}
]

で決まるため、同じTPRとFPRでも、低いベースレートではPrecisionが低下する。論文のPrecisionをそのまま実運用上のリスク確率と解釈することはできない。

第二に、強い結果の多くは、正解ラベルと予測確率ベクトルを取得できることを前提としている。ラベルのみの場合にも漏洩は残るが、攻撃性能は低下する。また、対象と類似した学習サービスを利用できるという仮定も、状況によっては強い。

第三に、モデルベース合成は高コストである。3万件の合成データに対して平均156クエリが必要なので、単純計算で約468万回の対象クエリになる。著者自身も、高解像度画像のような探索空間では、この生成法が機能しない可能性を認めている。

第四に、主要な比較対象はランダム推測であり、最大信頼度、正解クラス確率、予測損失、エントロピーなどに単純な閾値を設ける攻撃との体系的比較がない。シャドウモデルの必要性と追加的な効果を明確にするには、より強い単純ベースラインが望まれる。

第五に、主要図表では反復実験による分散や信頼区間がほとんど示されていない。クラウド側のモデル構造も非公開であるため、サービス間の漏洩差がどの構造的要因によるものかを因果的に分析できていない。Locationデータの学習件数の記述不一致も再現性上の問題である。

最後に、防御策の詳細評価は二つのデータセット上のローカルモデルに限定されている。差分プライバシーは理論的に有力な対策として論じられているが、本論文内では実験的に評価されていない。

---

## 7. まとめ

本論文は、機械学習モデルの予測結果から、特定レコードが学習データに含まれていたかを推定できることを示した。

中心となるシャドウモデル手法は、既知の学習・テストデータを持つ代理モデルを利用し、対象モデルのメンバーと非メンバーに対する出力差を学習するものである。対象データと同じ分布の実データがなくても、統計情報、ノイズ付きデータ、対象モデルから生成した合成データによって攻撃モデルを構築できる。

攻撃は、過学習したモデル、多クラスモデル、クラス当たりのデータが少ないモデルで特に成功した。ただし、過学習だけでは漏洩量を完全には説明できず、モデル構造も重要である。

出力制限だけでは十分な防御にならず、正則化や差分プライバシーを含む学習過程そのものの改善が必要である。

---

## 8. 総評

**新規性：非常に高い。**
ブラックボックスモデルに対するメンバーシップ推論を、シャドウモデルと攻撃分類器によって実行可能な形にした点は明確な技術的貢献である。

**技術的妥当性：高い。**
攻撃の構成は直観的かつ一般的であり、複数データセット、複数モデル、複数のシャドウデータ条件で一貫した結果が得られている。一方で、評価事前確率、単純ベースライン、統計的不確実性の扱いには改善余地がある。

**有用性・重要性：非常に高い。**
モデルの予測精度だけでなく、学習データの漏洩リスクもモデル選択や機械学習サービスの評価指標に含めるべきだという問題提起は重要である。特に医療データでの結果は、メンバーシップそのものが機微情報になり得ることを明確に示している。

**再現性：中程度。**
ローカルモデルについては比較的再現可能だが、GoogleおよびAmazonのモデル構造が非公開であり、一部の実験設定にも記述の不一致がある。

### 査読結果

**採択（Strong Accept）**

**判定理由：** ベースレートを考慮した評価、より強い単純攻撃との比較、再現性の改善は必要である。しかし、これらは中心的な結論を覆す問題ではない。新規性、実証範囲、プライバシー上の重要性が非常に高く、機械学習セキュリティ研究として十分な採択価値がある。
