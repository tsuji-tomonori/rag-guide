# Sentence-BERT: Sentence Embeddings using Siamese BERT-Networks

**著者:** Nils Reimers, Iryna Gurevych
**発表:** EMNLP-IJCNLP 2019, pp. 3982–3992 ([ACL Anthology][1])

## 1. はじめに

本論文は、BERTを検索・クラスタリングに利用しやすい**文埋め込みモデル**へ変換するSentence-BERT（SBERT）を提案している。

通常のBERTによる文ペア判定では、2文を同時に入力するクロスエンコーダを用いる。この方式は高精度だが、1万文から最も類似した文ペアを総当たりで探すには4,999万5,000回の推論が必要となり、論文の環境では約65時間かかる。SBERTでは各文を一度ずつベクトル化し、ベクトル間のコサイン類似度を計算することで、同処理を約5秒まで短縮した。

本論文の中心的な貢献は、BERTの表現能力を維持しつつ、文を独立してエンコード可能にした点にある。これにより、意味検索、重複文検出、クラスタリング、大規模検索などにBERTを現実的に適用できる。

## 2. 背景

### クロスエンコーダの問題

標準的なBERTの文ペア処理では、次のように2文を連結する。

[
[\mathrm{CLS}],\ \text{Sentence A},\ [\mathrm{SEP}],\ \text{Sentence B},\ [\mathrm{SEP}]
]

この方式では、自己注意機構を通じてSentence Aの各単語とSentence Bの各単語を直接比較できる。そのため文ペア分類や類似度推定では高精度だが、文ごとの表現を事前計算できない。

文数を (n) とすると、全ペア比較には

[
\frac{n(n-1)}{2}
]

回のBERT推論が必要になる。

### BERT出力をそのまま文ベクトルにする問題

単一文をBERTへ入力し、全トークンの平均または`[CLS]`ベクトルを文埋め込みとして使う方法も考えられる。しかし、未調整のBERT表現はコサイン類似度に適した幾何構造を持たない。

7種類のSemantic Textual Similarity（STS）データセットにおける平均Spearman相関は、単語GloVe平均が61.32であるのに対し、BERTトークン平均が54.81、`[CLS]`が29.19だった。つまり、BERTの内部表現が高性能であっても、そのまま距離空間上の文表現として利用できるとは限らない。

先行研究にはInferSentやUniversal Sentence Encoderが存在したが、本論文は事前学習済みBERTをSiamese構造でファインチューニングし、より高品質な文埋め込みを得ることを目指した。

## 3. 提案手法

### Siamese BERT

SBERTは、重みを共有する2つのBERTを用いるSiameseネットワークである。

Sentence AとSentence Bを別々にBERTへ入力し、各BERTの出力にPoolingを適用して、固定次元の文埋め込み (u) と (v) を得る。2つのBERTは同じパラメータを共有するため、実際には同一エンコーダを2回適用している。

Poolingとして以下を検証している。

* `CLS`: `[CLS]`トークンの最終表現
* `MEAN`: 全トークン表現の平均
* `MAX`: 各次元について最大値を取得

標準設定はMEAN Poolingである。

### 分類目的関数

SNLI・MultiNLIのように、文ペアへ「含意・矛盾・中立」のカテゴリが付与されている場合に用いる。

[
o=\operatorname{softmax}
\left(
W_t[u;v;|u-v|]
\right)
]

ここで ([u;v;|u-v|]) は、2つの文埋め込みとその要素ごとの差分を連結したベクトルである。損失関数にはクロスエントロピーを用いる。

差分 (|u-v|) を含めることで、意味的に近い文を埋め込み空間上でも近づけるようエンコーダを学習させる。

### 回帰目的関数

STSのように連続的な類似度スコアが付与されている場合、埋め込み間のコサイン類似度を計算する。

[
\hat{y}=\cos(u,v)
]

予測値と正解類似度との平均二乗誤差を最小化する。

### Triplet目的関数

アンカー文 (a)、類似する正例文 (p)、類似しない負例文 (n) を用いる。

[
L=
\max
\left(
\lVert s_a-s_p\rVert-
\lVert s_a-s_n\rVert+
\epsilon,\ 0
\right)
]

本論文ではユークリッド距離を用い、マージンを (\epsilon=1) としている。正例をアンカーへ近づけ、負例を一定距離以上離す学習である。

### 推論

学習後は分類器を使用しない。各文を独立にSBERTへ入力して埋め込みを保存し、コサイン類似度で比較する。

この構造は一般に**バイエンコーダ**と呼ばれる。検索対象文の埋め込みを事前計算でき、近似最近傍探索インデックスとも組み合わせられる。

## 4. 実験方法

### 基本学習

汎用SBERTは、約57万ペアのSNLIと約43万ペアのMultiNLIを結合して学習されている。設定は1エポック、バッチサイズ16、Adam、学習率 (2\times10^{-5})、全学習ステップの10%を線形ウォームアップとしている。

### 評価課題

**STS評価**では、STS 2012～2016、STS Benchmark、SICK-Relatednessの7データセットを使用し、予測コサイン類似度と正解スコアのSpearman順位相関を測定した。「教師なしSTS」と呼ばれている設定ではSTSの学習データは使用しないが、モデル自体はNLIラベルで教師あり学習されている。

**教師ありSTS Benchmark**では、訓練5,749ペア、開発1,500ペア、テスト1,379ペアを使用する。STSのみで学習する設定と、NLI学習後にSTSで追加学習する設定を比較し、ランダムシードを変えて10回実験している。

**Argument Facet Similarity**では、銃規制、同性婚、死刑に関する議論文6,000ペアを評価する。通常の10-fold交差検証に加え、2トピックで学習し、未観測の1トピックで評価するcross-topic設定も導入した。

**Wikipedia Sections**では、同一Wikipedia記事の同じ節から正例、異なる節から負例を作成する。約180万tripletで1エポック学習し、別記事からなる22万2,957件のテストtripletを評価した。

さらに、SentEvalの7分類課題、Pooling・特徴量連結方法のアブレーション、CPU・GPU上の文埋め込み生成速度を評価している。

## 5. 結果

### STS：STS用データを使わない設定

7データセット平均では以下の結果となった。

| モデル                        | 平均Spearman相関 |
| -------------------------- | -----------: |
| BERT CLS                   |        29.19 |
| BERTトークン平均                 |        54.81 |
| GloVe平均                    |        61.32 |
| InferSent                  |        65.01 |
| Universal Sentence Encoder |        71.22 |
| SBERT-NLI-large            |        76.55 |
| SRoBERTa-NLI-large         |    **76.68** |

SBERTはInferSentを約11.5ポイント、Universal Sentence Encoderを約5.3ポイント上回った。一方、SICK-RではUniversal Sentence EncoderがSBERTを上回っており、事前学習データと評価ドメインの適合性が影響していると論文では考察している。

### 教師ありSTS Benchmark

NLI学習後にSTSでファインチューニングしたlargeモデルでは、

* BERTクロスエンコーダ: **88.77**
* SBERT: **86.10**
* SRoBERTa: **86.15**

となった。

SBERTは従来の文埋め込みモデルを上回る一方、2文を直接相互注意で比較するクロスエンコーダには約2.6ポイント劣る。この結果は、SBERTが純粋に精度を維持するというより、**わずかな精度低下と大幅な高速化を交換する方式**であることを示している。

### Argument Facet Similarity

10-fold交差検証では、largeモデルのSpearman相関がBERT 76.38、SBERT 75.93であり、ほぼ同等だった。

一方、cross-topic評価ではBERTが60.34、SBERTが53.10となり、未観測トピックへの一般化では差が拡大した。個々の文を独立したベクトルへ圧縮するSBERTは、主張内容と論拠の細かな対応関係を直接比較するクロスエンコーダより不利になる。

### Wikipedia Sections

従来のBiLSTM-tripletモデルの正解率74%に対し、SBERT-largeは**80.78%**を達成した。弱教師ありtripletデータによるドメイン適応が有効であることを示している。

### SentEval

7課題平均では、InferSentが85.59、Universal Sentence Encoderが85.10、SBERT-largeが**87.69**だった。SBERTは7課題中5課題で最高性能を示し、特に感情分類系のMR、CR、SSTで改善が大きかった。

### アブレーション

NLI学習ではMEAN、MAX、CLSの差は比較的小さいが、STS回帰学習ではMAX Poolingが大きく悪化した。

| Pooling |     NLI学習 |   STS回帰学習 |
| ------- | --------: | --------: |
| MEAN    | **80.78** | **87.44** |
| MAX     |     79.07 |     69.92 |
| CLS     |     79.80 |     86.62 |

分類器への入力では ([u;v;|u-v|]) が最良であり、特に要素差 (|u-v|) が重要だった。

### 計算効率

Tesla V100上の文埋め込み生成速度は以下だった。

| モデル                        | GPU上の文数/秒 |
| -------------------------- | --------: |
| InferSent                  |     1,876 |
| Universal Sentence Encoder |     1,318 |
| SBERT-base                 |     1,378 |
| SBERT-base＋smart batching  | **2,042** |

文長の近い文を同じバッチにまとめるsmart batchingにより、不要なPadding計算を減らしている。SBERTはGPUではInferSentより約9%、Universal Sentence Encoderより約55%高速だった。一方CPUでは単純なBiLSTMであるInferSentの方が高速である。

## 6. 考察

### 評価できる点

最大の長所は、BERTの性能上の問題ではなく、**埋め込み空間の幾何構造と計算方式の問題**を明確に切り分けた点である。BERT出力を平均するだけでは距離尺度に適した表現にならないが、Siamese学習を行うことで意味類似度とベクトル距離を整合させられる。

また、分類、回帰、tripletという異なる教師信号に対応した設計は汎用性が高い。STSだけでなく、議論類似度、Wikipedia節判定、分類転移、計算速度まで評価しており、実験範囲も比較的広い。

手法自体は複雑ではないが、クロスエンコーダとバイエンコーダの精度・速度特性を実用的な形で示した点の価値は大きい。

### 注意すべき点

第一に、「教師なしSTS」は厳密には教師なし学習ではない。STSラベルを使用していないだけで、SNLI・MultiNLIという大規模な教師ありデータを利用している。

第二に、SBERTは常にクロスエンコーダと同等精度ではない。教師ありSTSやcross-topic AFSではクロスエンコーダが明確に上回る。したがって、論文要旨の「BERTの精度を維持する」という表現はやや強く、実際には**検索候補生成ではSBERT、少数候補の再ランキングではクロスエンコーダ**という二段階構成が自然である。

第三に、SBERTが削減するのは高価なTransformer推論回数である。厳密な全ペアのコサイン類似度計算自体は依然として (O(n^2)) だが、行列演算として高速に処理でき、近似最近傍探索による高速化も可能になる。

第四に、計算速度比較はPyTorch、TensorFlow、NumPyなど実装基盤が統一されておらず、モデル構造だけの厳密な比較ではない。また、検索インデックス構築時間、メモリ使用量、近似検索のrecallとlatencyは評価されていない。

第五に、実験は英語の比較的短い文が中心であり、多言語、長文、強いドメインシフト、バイアス、敵対的入力に対する評価は含まれていない。特にAFSのcross-topic結果は、学習分布から離れた場合の性能低下を示している。

## 7. まとめ

SBERTは、BERTをSiameseまたはtriplet構造でファインチューニングし、コサイン類似度で比較可能な文埋め込みを生成する手法である。

重要な知見は次の3点に集約できる。

1. 未調整のBERT平均・CLS表現は、コサイン類似度用の文埋め込みとしては弱い。
2. NLIやSTSを用いたSiamese学習により、従来の汎用文埋め込みを大きく上回る。
3. クロスエンコーダより一部精度は低下するが、大規模検索・クラスタリングを実行可能な速度に変えられる。

本論文は「最高精度の文ペア分類器」を提案したというより、BERTを**再利用可能・事前計算可能・検索可能な意味ベクトル生成器**へ変換した研究と位置づけるのが適切である。
