# 1. はじめに

本論文 **“PoisonedRAG: Knowledge Corruption Attacks to Retrieval-Augmented Generation of Large Language Models”** は、検索拡張生成（Retrieval-Augmented Generation: RAG）の知識ベースへ少数の悪意ある文書を挿入し、特定の質問に対して攻撃者が指定した誤答を出力させる **標的型知識汚染攻撃**を提案している。

中心的な結果は、数百万件の文書からなるデータベースに、対象質問ごとにわずか5件の汚染文書を追加するだけで、多くの設定において90％前後またはそれ以上の攻撃成功率を達成したことである。さらに、検索器へ問い合わせることすらできないブラックボックス設定でも攻撃が成立する。([arXiv][1])

---

# 2. 背景

RAGは、次の二段階で回答を生成する。

1. ユーザーの質問と意味的に近い文書を知識ベースから検索する。
2. 検索された上位 (k) 件の文書をコンテキストとしてLLMに与え、回答を生成する。

この構成は、LLMの内部知識だけに依存せず、更新可能な外部知識を利用できる一方、知識ベース自体が新たな攻撃面になる。攻撃者が検索対象の文書を追加・編集できる場合、その文書が検索されるように操作し、さらに生成モデルを誤答へ誘導できる可能性がある。

従来のプロンプトインジェクションは、文書内に「この答えを出力せよ」といった命令を埋め込む。一方、既存のコーパスポイズニングは、主として検索結果や下流モデルの挙動を変えることを目的としていた。本論文の特徴は、RAGを構成する **検索段階と生成段階を明示的に分離し、両方を同時に成立させる文書を設計した点**にある。

# 3. 提案手法

## 3.1 脅威モデル

攻撃者は、対象質問 (Q_i) と、その質問に対してRAGに出力させたい誤った標的回答 (R_i) を指定する。

攻撃者には、次の制約が課される。

* 元の正常な知識ベースにはアクセスできない。
* 被害側LLMにはアクセスも問い合わせもできない。
* ブラックボックス攻撃では、検索器のパラメータを知らず、問い合わせもできない。
* ホワイトボックス攻撃では、検索器のモデルとパラメータを知っている。
* 知識ベースには少数の悪意ある文書を追加できる。

目的は、対象質問に対して標的回答を生成させる割合、すなわち攻撃成功率（Attack Success Rate: ASR）を最大化することである。

## 3.2 二つの攻撃条件

攻撃成功には、次の条件が必要になる。

**検索条件**
悪意ある文書が検索結果の上位 (k) 件に含まれること。

**生成条件**
その文書をコンテキストに含めたとき、LLMが攻撃者指定の標的回答を生成すること。

PoisonedRAGでは、1件の悪意ある文書 (P) を次のように分解する。

[
P = S \oplus I
]

ここで、(S) は検索を操作する部分、(I) は生成を操作する部分、(\oplus) は両者の連結を表す。

## 3.3 生成操作部分 (I)

攻撃者は別のLLMを使い、標的となる誤答を事実であるかのように支持する、自然で説得力のある文書を生成する。

例えば、正解が「パリ」である質問に「ロンドン」と誤答させたい場合、ロンドンが正解であることを説明する架空の参考文書を作る。ただし、単に回答名を挿入するのではなく、LLMがその文書を根拠として標的回答を選びやすい文章を生成する。

生成した文書を攻撃者側のLLMに与え、実際に標的回答が生成されるかを検査し、成功するまで最大 (L) 回再生成する。デフォルト設定ではGPT-4が攻撃文書の生成に使われた。

## 3.4 検索操作部分 (S)

ブラックボックス攻撃では、非常に単純に対象質問そのものを文書の先頭に付加する。

[
S = Q
]

検索器は質問と文書の意味的類似度を評価するため、質問をそのまま含む文書は高順位で検索されやすい。この方法は検索器への問い合わせを必要としない。

ホワイトボックス攻撃では、質問埋め込みと汚染文書埋め込みの類似度が最大になるように (S) を最適化する。論文では主にHotFlipを使用し、追加実験ではTextFoolerも評価している。

# 4. 実験方法

## 4.1 データセットと検索器

評価には次の大規模QAコーパスが用いられた。

| データセット            | 知識ベースの文書数 |
| ----------------- | --------: |
| Natural Questions |    約268万件 |
| HotpotQA          |    約523万件 |
| MS-MARCO          |    約884万件 |

各データセットについて100件のクローズドエンド型質問を攻撃対象とした。誤った標的回答はGPT-4で生成した後、著者らが正解ではないことを手作業で確認している。

検索器にはContriever、MS-MARCOで学習したContriever、ANCEを使用した。標準設定はContriever、内積類似度、検索件数 (k=5) である。

## 4.2 被害側LLM

被害側の生成モデルとして、以下の8構成を評価している。

* PaLM 2
* GPT-3.5
* GPT-4
* LLaMA-2 7B、13B
* Vicuna 7B、13B、33B

標準攻撃では、対象質問1件につき5件の汚染文書を挿入する。すなわち、標準設定では汚染文書数 (N=5) と検索件数 (k=5) が等しい。

## 4.3 評価指標と比較手法

主要指標は、生成回答に標的回答文字列が含まれているかで判定するASRである。検索についてはPrecision、Recall、F1を測定し、一部の設定では著者らによる人手評価も実施した。

比較対象には、質問だけを文書として挿入するNaive攻撃、プロンプトインジェクション、既存のコーパスポイズニング、GCG、誤情報文書 (I) だけを挿入するDisinformation攻撃が含まれる。

さらに、Self-RAG、CRAG、ReActエージェント、約2,100万件のWikipedia由来チャンクを持つチャットボット、FEVER事実検証タスクにも攻撃を適用している。防御としては、質問の言い換え、パープレキシティによる異常検出、重複削除、検索文書数の増加を評価した。

# 5. 結果

## 5.1 基本的な攻撃性能

標準のPaLM 2設定におけるASRは次のとおりである。

| 攻撃設定     | Natural Questions | HotpotQA | MS-MARCO |
| -------- | ----------------: | -------: | -------: |
| ブラックボックス |               97% |      99% |      91% |
| ホワイトボックス |               97% |      94% |      90% |

8種類すべての被害側LLMを見ると、ブラックボックス攻撃のASRはNatural Questionsで88～97％、HotpotQAで93～99％、MS-MARCOで89～96％だった。ホワイトボックス攻撃はそれぞれ94～99％、91～99％、74～93％だった。検索F1も多くの条件で0.9以上となっている。

ホワイトボックス攻撃が常にブラックボックス攻撃を上回るわけではない。検索類似度を高めるための単語置換が文書の自然さや意味を損ない、生成段階での誘導性能を低下させる場合があるためである。

## 5.2 ベースラインとの比較

Natural Questionsでは、Disinformation攻撃が69％、プロンプトインジェクションが62％であるのに対し、PoisonedRAGは97％を達成した。MS-MARCOでも、既存手法の最大71％に対して90～91％だった。

ただし、HotpotQAではDisinformation攻撃が100％を記録し、PoisonedRAGのブラックボックス99％、ホワイトボックス94％を上回っている。したがって、「検索用部分と生成用部分の併用が常に最良」という結果ではない。HotpotQAでは生成操作部分だけでも十分検索されやすかったと考えられる。

## 5.3 汚染文書数の影響

検索上位5件のうち、悪意ある文書が1件だけの場合、ASRはおおむね35～54％だった。一方、5件すべてを悪意ある文書が占めると、93～100％まで上昇した。

これは、知識ベース全体に占める汚染率よりも、**特定質問に対する上位検索結果をどれだけ占有できるか**が重要であることを示す。

## 5.4 高度なRAGと実用規模の設定

Self-RAGおよびCRAGに対してもASRは70～87％であり、検索結果の自己評価や訂正機構だけでは攻撃を排除できなかった。

約2,100万件のWikipedia由来チャンクを持つチャットボットでは91～100％、ReActエージェントではブラックボックス攻撃で52～72％のASRとなった。FEVER事実検証ではブラックボックス97％、ホワイトボックス88％だった。

## 5.5 防御結果

質問を言い換えてから検索する防御でもASRは79～93％残った。完全一致の重複文書を除去する防御には、ほぼ効果がなかった。

検索件数を5件から50件へ増やすと、HotpotQAのASRは41～43％まで低下したが、攻撃を完全には防げず、LLMへ与えるコンテキスト量と計算コストも増加する。

# 6. 考察

## 長所

本論文の最も優れた点は、RAG攻撃を「検索されること」と「誤答させること」に分解したことである。この分解はRAGの処理構造と直接対応しており、単純で理解しやすいだけでなく、ブラックボックス攻撃にも自然に適用できる。

特に、質問文をそのまま検索操作部分として付加するだけで高い検索率を得た結果は重要である。検索器固有の勾配や問い合わせ結果がなくても、一般的な埋め込み検索器の性質を利用して攻撃が転移している。複数の検索器、8種類のLLM、3種類の大規模QAデータセット、高度なRAGやエージェントまで評価しており、実験範囲も比較的広い。

また、Natural Questionsの非対象質問1,000件において、汚染文書が検索された割合はブラックボックス0.3％、ホワイトボックス0.9％で、生成回答が変化した割合も0％と0.4％だった。攻撃が全体性能を大きく破壊するのではなく、特定質問に限定して潜伏できる可能性を示している。

## 限界

第一に、実用上もっとも強い仮定は、攻撃者がRAGの知識ベースへ文書を追加できることである。Wikipedia規模の実験も、Wikipedia由来データを用いて構築した知識ベースへ研究者が汚染文書を挿入したものであり、実際のWikipedia編集審査、クローラ更新、アクセス制御、出典審査などを突破した実験ではない。したがって、攻撃文書を投入できた後の危険性は強く示しているが、投入までの実運用上の難易度は十分にモデル化されていない。

第二に、標準設定では (N=k=5) であり、攻撃者の文書が検索コンテキスト全体を占有できる。論文は「数百万件中5件」という小さな全体汚染率を強調しているが、セキュリティ上重要なのは全体比率ではなく、対象質問の上位検索枠における占有率である。文書数を1件から5件に増やすとASRが大幅に上昇する結果も、この点を裏付けている。

第三に、各データセットの対象は100件のクローズドエンド型質問に限定されている。自由記述型・多面的・曖昧な質問に対して、攻撃者が特定の文章をどこまで安定して生成させられるかは未検証である。また、文字列包含によるASRは、「標的語を否定する回答」や複数候補を列挙した回答を成功と誤判定する可能性がある。人手評価も著者ら自身によるもので、独立評価者間一致度は報告されていない。

第四に、パープレキシティ検出について論文が報告したROC-AUCは0.25および0.30である。AUCが0.5未満の場合、単にスコアの向きが逆である可能性があり、判定方向を反転すれば数学的には0.75および0.70に相当する。したがって、この結果だけから「パープレキシティには検出能力がない」と結論づけるのは不十分であり、スコア方向や正常・異常ラベルの定義を再検証すべきである。

第五に、防御評価は比較的素朴である。完全一致ハッシュによる重複削除では、表現を少し変えた近似重複文書を検出できない。出典の信頼度、文書来歴、複数独立ソース間の合意、近似重複クラスタリング、投入時の隔離審査、検索順位の異常監視などを組み合わせた防御は評価されていない。

# 7. まとめ

PoisonedRAGは、RAGの知識ベースに少数の文書を追加するだけで、特定質問に対する回答を攻撃者指定の誤情報へ誘導できることを示した。

技術的な核心は、汚染文書を次の二部分へ分解した点にある。

* 検索上位に入るための検索操作部分
* LLMに誤答を生成させるための生成操作部分

とりわけ、検索器にも被害側LLMにも問い合わせないブラックボックス攻撃が高いASRを示したことは、公開投稿、ユーザー生成コンテンツ、外部Webページなどを自動収集するRAGシステムにとって重大である。

本論文から得られる実務的な教訓は、RAGの安全性をLLM側のアラインメントだけで確保することはできないという点である。文書の取り込み時点から、来歴管理、信頼できる出典の優先、近似重複検出、複数ソースによる裏付け、検索結果の多様性制約などを導入する必要がある。

# 8. 総評

## 実際の査読結果

本論文は **34th USENIX Security Symposium（USENIX Security ’25）に採択**され、論文集の3827～3844ページに掲載された。公式プログラム上の発表区分は **Short Presentation** である。したがって、実際の査読結果は「採択」である。([USENIX][2])

## 独自の査読判定

**判定：Accept（採択相当、7.5/10）**

発想は単純ながら、RAGの検索・生成構造を正確に突いた攻撃であり、ブラックボックス条件での高い成功率には十分な新規性と実用的意義がある。複数データセット、検索器、LLM、高度なRAG、エージェントを用いた評価も、セキュリティ論文として説得力がある。

一方で、標準実験が検索上位5件すべてを攻撃文書で占有し得る設定であること、実システムへの文書投入経路を評価していないこと、対象質問数と回答形式が限定的であること、ASR判定とパープレキシティ防御の評価に改善余地がある。したがって、PoisonedRAGは「直ちにあらゆる実運用RAGを破れる完成された攻撃」というより、**RAGの知識取り込みと検索結果の局所的占有が重大なセキュリティリスクになることを明確に実証した基礎的研究**として評価するのが妥当である。
