# 1. はじめに

本論文 **「OCR-free Document Understanding Transformer」** は、文書画像理解をOCRなしでエンドツーエンドに処理するモデル **Donut（Document Understanding Transformer）** を提案した研究である。請求書・領収書・名刺などの画像を直接入力し、文書分類、情報抽出、質問応答の結果を構造化テキストとして生成する。2021年にarXivへ投稿され、査読後に **ECCV 2022** の会議論文として出版された。([arXiv][1])

# 2. 背景

従来のVisual Document Understanding（VDU）は、概ね次の二段階で処理されていた。

1. OCRで文書画像から文字と座標を抽出する
2. BERT、LayoutLMなどで文字・座標・画像特徴を解析する

この構成には、OCRの推論コスト、言語や文書形式を変更するときの柔軟性不足、OCR誤りが後段へ伝播するという問題がある。特に韓国語や中国語など文字集合の大きい言語、手書き文字、低解像度画像では、OCRがシステム全体の性能上限になりやすい。

著者らの問題設定は、「文字認識」と「文書理解」を別々のモジュールで処理するのではなく、**文書画像から必要な構造化情報までを直接学習できないか**、というものである。

# 3. 提案手法

Donutは、画像エンコーダとテキストデコーダからなるTransformer型のEncoder–Decoderモデルである。

**画像エンコーダ**にはSwin Transformerを使用し、文書画像をパッチへ分割して視覚特徴系列に変換する。**テキストデコーダ**には多言語BARTを利用し、エンコーダの視覚特徴を参照しながら、自己回帰的にサブワードトークンを生成する。

タスクはプロンプトで指定する。例えば、文書分類では分類用トークン、情報抽出では解析用トークン、DocVQAでは質問文をデコーダの先頭に与える。出力は次のように、JSONへ変換可能な系列として生成される。

```text
<class>receipt</class>
```

```json
{"class": "receipt"}
```

領収書解析では、商品名、数量、単価、合計金額などの入れ子構造も同様に生成する。この仕組みによって、分類、情報抽出、質問応答をすべて「画像から構造化系列を生成する問題」として統一している。

事前学習では、画像内の全文を概ね左上から右下の読み順で生成させ、次トークン予測のクロスエントロピー損失を最小化する。著者らはこれを擬似OCRタスクと位置付けている。

学習データには、1100万枚のIIT-CDIP文書画像と、提案した合成文書生成器 **SynthDoG** による200万枚を使用した。SynthDoGでは英語、中国語、日本語、韓国語について各50万枚を生成し、Wikipediaの文章、紙のテクスチャ、背景画像、ランダムなレイアウトや画像変換を組み合わせて文書を合成する。

# 4. 実験方法

実験は3種類のタスク、計6データセットで行われた。

| タスク    | データセット                                         |
| ------ | ---------------------------------------------- |
| 文書分類   | RVL-CDIP                                       |
| 文書情報抽出 | CORD、Chinese Train Ticket、非公開の日本語名刺、非公開の韓国語領収書 |
| 文書質問応答 | DocVQA                                         |

情報抽出ではフィールド単位F1と、入れ子構造まで評価するTree Edit Distanceベースの精度を使用した。DocVQAでは編集距離に基づくANLSを使用した。

主な設定は、Swin-Bエンコーダ、BARTの先頭4層によるデコーダ、最大入力解像度2560×1920、最大出力長1536である。事前学習は64基のA100 GPU、バッチサイズ196、20万ステップで行われた。比較対象にはBERT、RoBERTa、LayoutLM、LayoutLMv2、BROS、SPADE、WYVERNなどが含まれる。

# 5. 結果

主要結果は以下の通りである。

| 評価対象                 |         Donut |                  主な比較対象 |
| -------------------- | ------------: | ----------------------: |
| RVL-CDIP分類精度         |    **95.30%** |      LayoutLMv2: 95.25% |
| RVL-CDIP推論時間         |    **752 ms** | LayoutLMv2＋OCR: 1489 ms |
| CORD F1／構造精度         | **84.1／90.9** |   LayoutLMv2: 78.9／82.4 |
| Train Ticket F1／構造精度 | **94.1／98.7** |   LayoutLMv2: 87.2／90.1 |
| 日本語名刺 F1／構造精度        | **57.8／84.4** |   LayoutLMv2: 52.2／83.0 |
| 韓国語領収書 F1／構造精度       | **78.6／88.6** |   LayoutLMv2: 72.9／78.0 |

文書分類ではLayoutLMv2をわずかに上回り、報告された推論時間は約半分だった。情報抽出では、公開・非公開の4データセットすべてで比較手法を上回り、特に入れ子構造を評価する精度で大きな改善が見られた。

一方、DocVQA全体ではDonutのANLSは **67.5** であり、一般モデルのLayoutLMv2の78.1や、専用拡張モデルLayoutLMv2-Large-QGの86.7には届いていない。ただし推論時間は782 msと短く、手書き文書部分では72.1を記録し、LayoutLMv2-Large-QGの67.3を上回った。したがって「全タスクで最高精度」という結果ではなく、**分類・情報抽出・速度面で強く、DocVQA全体精度には課題が残る**と読むのが適切である。

追加実験では、単なる画像分類やキャプション生成よりも「画像内の全文を読む」事前学習が有効だった。また、情報抽出では合成データだけでも高い性能が得られたが、DocVQAでは実文書画像による事前学習が重要だった。入力解像度を上げるほど性能は向上する一方、計算量も増加した。

# 6. 考察

本研究の最大の意義は、文書理解を「OCR結果に対する自然言語処理」から、**画像条件付き構造化テキスト生成**へ転換した点にある。OCRの認識誤り、読み順決定、座標系列化、NERや関係抽出などを単一モデル内で最適化できるため、システム構成と保守を簡素化できる。

ただし、「OCR-free」という表現には注意が必要である。推論時には外部OCRを使用しないが、1100万枚のIIT-CDIP画像の擬似ラベル作成には商用OCRが利用されている。したがって、厳密には **OCR不要の推論モデル** であり、学習データ生成まで完全にOCR非依存ではない。SynthDoGはこの依存を軽減するための重要な構成要素である。

また、以下の制約がある。

* 微小文字を読むために高解像度入力が必要で、学習・推論コストが増える
* 自己回帰生成なので、出力が長くなるほど推論時間と誤りの蓄積が増える
* 開始・終了トークンの対応が崩れると、フィールド全体が欠落扱いになる
* 左上から右下という読み順の仮定は、複雑な多段組み文書には適さない可能性がある
* 日本語名刺と韓国語領収書は非公開データであり、第三者による完全な再現が難しい
* 速度比較は使用するOCR API、通信時間、ハードウェア条件に影響される

論文自身も、大きな文書画像内の微小文字を取りこぼす事例と、解像度を上げた場合の計算量増加を報告している。

# 7. まとめ

Donutは、文書画像を直接Swin Transformerで符号化し、BARTデコーダでJSON相当の構造化系列を生成するOCR非依存の文書理解モデルである。

主要な成果は、外部OCRなしで文書分類と情報抽出において既存のOCR依存モデルと同等以上の精度を達成し、推論速度とシステム構成を改善したことである。SynthDoGによる多言語合成文書生成も、実データや言語別OCRが不足する状況への実用的な対応となっている。

一方、DocVQA全体では最高性能に届かず、高解像度処理、長い自己回帰出力、学習時の擬似OCRラベル依存などが今後の課題として残る。

# 8. 総評

**査読結果：ECCV 2022採択。**

本論文はECCV 2022の公式採択論文一覧に掲載され、Springerの査読済み会議論文としてLNCS第13688巻、498–517頁に収録されている。公開ページ上では個別の査読コメントや査読点数は確認できないが、最終的な判定は明確に **採択** である。([eccv2022.ecva.net][2])

研究としては、極端に複雑な新規モジュールを導入するのではなく、既存の画像Transformerとテキストデコーダを組み合わせ、タスクを構造化系列生成へ統一した点が優れている。特に「OCRを改善する」のではなく「推論パイプラインからOCRを除く」という問題設定は明快で、文書AIの研究方向を変えるほどの影響力を持つ提案である。

総合的には、**新規性、実用性、実験範囲、公開資源の点で高く評価できる論文**である。一方で、DocVQAの総合精度と計算資源、非公開データによる再現性、「OCR-free」という用語の範囲については、結果を解釈する際に留意すべきである。
