# 1. はじめに

本論文は、Jiawei ChenらによるRAG評価研究であり、**AAAI 2024のNLP Technical Trackに採択・掲載**されている。論文の中心的な問いは、「検索文書をLLMに与えれば性能が上がるか」ではなく、**LLMが不完全な検索結果をどの程度適切に利用できるか**である。著者らは、その能力を診断する英中二言語ベンチマーク **RGB: Retrieval-Augmented Generation Benchmark** を提案した。([AAAI Publications][1])

特に、RAGに必要な能力を次の4項目に分解している。

1. ノイズ頑健性
2. 回答拒否能力
3. 情報統合能力
4. 反事実頑健性

---

# 2. 背景

LLMは、事実ハルシネーション、知識の陳腐化、専門知識の不足といった問題を持つ。RAGは検索によって外部知識を与えることで、これらを軽減する有力な手法である。

しかし、実際の検索結果には、質問に関連しているが答えを含まない文書、推測記事、類似した別の出来事、さらには誤情報が混在する。また、正しい文書が含まれていても、LLMがそれを無視したり、誤った文書を優先したりする可能性がある。したがって、RAGの性能は検索器の精度だけでなく、**生成モデル側の証拠選択・回答拒否・複数文書推論・誤情報検出能力**にも依存する。([arXiv][2])

従来のQAベンチマークでは最終正答率を評価することが多く、失敗原因が検索なのか、文書理解なのか、情報統合なのかを判別しにくい。本論文は、この問題を能力別のテストベッドによって診断しようとしている。

---

# 3. 提案手法

## RGBの4つのテストベッド

**ノイズ頑健性**では、正解を含む文書と、質問には関連するが正解を含まないノイズ文書を混在させる。モデルはノイズに惑わされず、正解文書から回答を抽出する必要がある。

**回答拒否能力（Negative Rejection）**では、与えられる文書をすべてノイズ文書にする。モデルは内部知識や推測で回答せず、「文書内に十分な情報がない」と回答することが期待される。

**情報統合能力**では、複数のサブ質問を組み合わせ、回答に複数文書の情報が必要な質問を作る。例えば、異なる年度の受賞者を同時に尋ね、両方を正しく回答できるかを評価する。

**反事実頑健性**では、モデルが本来知っている質問に対して、回答部分を人手で改変した誤情報文書を与える。さらに、文書に誤りがある可能性をプロンプトで警告した上で、誤りを検出・訂正できるかを調べる。([arXiv][2])

## データ構築

データ構築は以下の流れで行われる。

* 当時の新しいニュース記事からChatGPTでイベント・質問・回答を生成
* 回答を人手で確認
* Google Search APIで質問ごとに10件のWebページを取得
* Webページを最大300トークンのチャンクに分割
* 英語ではall-mpnet-base-v2、中国語ではm3e-baseを用いて上位30チャンクを選択
* 正解を含むものを正例文書、それ以外を負例文書として分類

RGBには600件の基本質問に加え、情報統合用200件、反事実頑健性用200件が収録され、英語と中国語が半数ずつになるよう構成されている。([arXiv][2])

# 4. 実験方法

評価対象は、当時の代表的な英中対応LLMである。

* ChatGPT（gpt-3.5-turbo）
* ChatGLM-6B
* ChatGLM2-6B
* Vicuna-7B-v1.3
* Qwen-7B-Chat
* BELLE-7B-2M

各質問には5件の外部文書を与える。ノイズ頑健性ではノイズ率を0、0.2、0.4、0.6、0.8と変化させる。情報統合では0、0.2、0.4を用いる。すべてのモデルに原則として同一形式の指示を与えている。([arXiv][2])

評価指標は次の通りである。

* **正答率**：生成文中に正解文字列が含まれるか
* **拒否率**：指定された回答拒否文を生成したか
* **誤り検出率**：文書に事実誤りがあると指摘したか
* **誤り訂正率**：誤情報に従わず正しい答えを生成したか

回答拒否と誤り検出については、固定文との完全一致に加えて、ChatGPTを評価器とした意味的判定も実施している。反事実実験では、文書なしの正答率が70％を超えたモデルと言語条件だけを評価対象としている。([arXiv][2])

---

# 5. 結果

## ノイズ頑健性

正解文書だけを与えた場合、多くのモデルは高い正答率を示した。しかし、ノイズ率が高まるにつれて性能が低下した。

英語では、ChatGPTが96.33％から76.00％、ChatGLM2-6Bが91.33％から57.33％に低下した。中国語でも、ChatGPTは95.67％から70.67％、Vicunaは85.67％から49.67％に低下している。単純な質問では一定のノイズ耐性があるものの、5文書中4文書がノイズとなる条件では影響が顕著である。([AAAI Publications][3])

主な失敗原因として、著者らは次を挙げている。

* 質問に関係する語と回答箇所が離れている「長距離情報」
* 発表前の予測記事などに惑わされる「証拠の不確実性」
* 売上高と自動車部門売上高などを混同する「概念混同」

## 回答拒否能力

全文書がノイズである場合、意味的な拒否判定を用いても、最高拒否率は英語45.00％、中国語43.33％にとどまった。

固定された拒否文との完全一致ではさらに低く、英語の最高値はQwenの31.00％、中国語では8.67％だった。LLMは答えを裏づける情報がない場合でも、関連語を含む文書から強引に答えを推測しやすいことが分かる。([arXiv][2])

## 情報統合能力

ノイズがない場合でも、最高正答率は英語60％、中国語67％だった。ノイズ率0.4では、それぞれ43％、55％まで低下した。単純な質問に比べ、複数文書を必要とする質問は少量のノイズでも性能が悪化している。([arXiv][2])

ChatGLM2-6Bの誤り分析では、以下が確認された。

* 一方の答えを両方のサブ質問に使用する統合誤り：28％
* 一方のサブ質問を無視する誤り：28％
* 年度や対象と文書の対応を取り違える誤り：6％
* ノイズ頑健性実験と共通する誤り：38％

## 反事実頑健性

誤情報文書を与えると、文書なしでは正答できたモデルも大幅に誤誘導された。

* ChatGPT英語：89％から9％
* ChatGPT中国語：91％から17％
* Qwen中国語：77％から12％

誤情報の検出率も1～8％にすぎない。警告を明示しても、モデルは自身の内部知識より外部文書を優先しやすいことが示された。([arXiv][2])

# 6. 考察

本論文の重要な示唆は、**RAGのボトルネックは検索器だけではない**という点である。正解文書が検索結果に含まれていても、生成モデルが文書間の信頼度を比較できなければ、推測記事や類似事例に惑わされる。また、正解文書がない場合に回答を拒否する能力や、複数文書をサブ問題に対応づける能力も十分ではない。

したがって、実用的なRAGには次のような機構が必要になる。

* 回答可能性を推定するアブステンション機構
* 文書単位の関連性・信頼性評価
* 複雑な質問のサブ質問分解
* 各回答に対応する根拠文書の追跡
* 内部知識と外部文書が矛盾した場合の検証処理

一方、本研究には重要な制約もある。

第一に、実験では検索済みの5文書をモデルに与えており、主に**生成器側の文書利用能力**を評価している。実際の検索再現率、検索速度、再ランキング、チャンク分割を含むエンドツーエンドRAG全体の評価ではない。

第二に、正答率は文字列一致、拒否率は指定文との一致に依存する。そのため、能力不足と表現形式・指示追従の失敗が混在する。完全一致とChatGPT判定の間に大きな差があること自体が、この問題を示している。ChatGPTを評価器とする方法にも、評価モデル固有の判断傾向が残る。([arXiv][2])

第三に、反事実実験で最終的に評価できたのは3つのモデル・言語条件だけであり、一般化可能性は限定的である。また、反事実質問の基礎となる「既知の知識」をChatGPTで生成しているため、ChatGPTに有利または依存したデータ選択となる可能性がある。

第四に、ノイズ頑健性などではno-RAG条件との直接的な対照が示されていない。したがって、結果は「RAGがどれだけ性能を改善したか」よりも、**外部文書を与えた状態でLLMがどのように失敗するか**を診断したものとして解釈する方が適切である。

さらに、著者リポジトリでは2024年3月に、誤って分類された正例・負例文書の除去、正例文書の追加、不正確な正解の修正を行った改訂版データが公開されている。これはデータ公開と改善姿勢という点では評価できるが、論文中の数値がデータ品質にどの程度依存していたかを確認するため、改訂版での再実験結果も提示することが望ましい。([GitHub][4])

---

# 7. まとめ

本論文は、RAGに必要な能力を、ノイズへの耐性、回答拒否、複数文書統合、誤情報への耐性に分解して評価するRGBベンチマークを提案した。

実験から、LLMは単純な質問と少量のノイズには比較的強い一方で、以下の状況では著しく不安定になることが示された。

* 正解を含む文書が存在しない
* 複数文書の情報を統合する必要がある
* 関連性の高い誤情報が含まれる
* 内部知識と検索文書が矛盾する

RAGはハルシネーションを自動的に解決する仕組みではなく、検索文書の品質管理、回答可能性判定、証拠に基づく推論を組み合わせなければ、外部情報によって新たな誤りを生む可能性がある。

# 8. 総評

**公開上の査読結果：採択。**
本論文はAAAI-24 Technical Track on Natural Language Processing Iに採択され、2024年3月24日に論文集へ掲載された。([AAAI Publications][1])

**本稿に対する模擬査読判定：Weak Accept（弱い採択）。**

RAGの失敗を4能力に分解した問題設定は明快であり、英中二言語、実際の検索文書、制御されたノイズと誤情報を用いた診断的ベンチマークには高い研究価値がある。特に、「検索結果を追加すれば安全になるとは限らず、LLMが誤情報を強く信頼する」という結果は実用上重要である。

一方、固定文字列中心の評価、限定されたモデル群、反事実実験の対象数、ChatGPTによるデータ生成・自動評価、公開後のデータ修正は、個々の数値の信頼性と一般化可能性を弱めている。改訂データでの再評価、人手評価との一致率、no-RAG対照、複数検索器・最新モデルを用いた検証が追加されれば、より強い採択判定に相当する内容になる。
