# 読み物向け論文要約スライドのデザイン規定

この規定は、論文要約を「発表スライド」ではなく、単体で読める画像資料として生成するために使う。

## 基本

- 16:9、最終出力は 1920×1080 px に揃える。
- 上下左右 120px 以上の安全余白を確保する。
- 各ページは「要点が分かるタイトル」「読み物として成立する説明量」「理解を助ける図解」を持つ。
- 1ページの主要情報は 3〜5 ブロックに整理する。
- 章構成は対象 Markdown を優先し、明示的に除外された章は作らない。

## 配色

- 背景: `#F7F6F1`
- 主文字: `#2B3A4A`
- アクセント: `#5E7E96`
- 補助文字・ページ番号: `#9AA4AD`
- 補助図形: 上記色を薄くしたブルーグレー

避けるもの:

- ページごとの色テーマ変更
- 強い原色や追加アクセント
- 濃いグラデーション、強い影、写真風背景

## タイトル

- タイトルは章名ではなく、そのページの結論・要点にする。
- 全ページで同じサイズ感・太さ・色に揃える。
- 良い例: `DPRは質問と通路を同じベクトル空間で比べる`
- 避ける例: `提案手法`

標準位置:

- 章ラベル: x=120, y=126 付近、約24px、アクセント色
- タイトル: x=120, y=180〜232 付近、約52〜60px、太字、ネイビー
- 罫線: x=120 から幅 180〜270px、y=279 付近、3px、アクセント色
- 本文・図版: y=330〜990、左右 120px 内側

## 情報量

- 表紙: 要点タイトル + 論文名 + 3行要約 + 小図解
- 背景: 問題設定 + 既存手法の限界 + 例
- 手法: アーキテクチャ図 + 学習の要点
- 実験: コーパス、データセット、指標
- 結果: 主要数値、比較表、例外条件
- 考察: 成功要因、制約、実用上の示唆
- まとめ: 3つの結論 + 後続技術への接続

## レイアウト

- 2カラム、3カード、表 + KPI、フロー図を優先する。
- カードは細い淡色枠で情報単位を区切る。カード内カードは避ける。
- 図解は本文を置き換えず、理解を補助する。
- アイコンは意味を補助する場合のみ使う。
- 装飾目的だけの泡・ドット・幾何学模様は避ける。淡い線や小さなベクトル風モチーフは控えめに使える。

## ページ番号

- 全ページ必須。
- 右上、x=1800 付近で右揃え、y=84 付近。
- 書式: `03 / 07`
- 色: `#9AA4AD`

## 出力命名

- 出力先: `pdf/{id}/`
- ファイル名: `{id}_{番号}.png`
- 例: `pdf/P001/P001_01.png`
