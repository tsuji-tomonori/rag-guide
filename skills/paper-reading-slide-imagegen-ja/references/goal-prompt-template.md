# /goal 用プロンプトテンプレート

`paper-reading-slide-imagegen-ja` を使って、画面ID だけで読み物向け論文要約スライド画像を生成するための `/goal` テンプレート。

## 最小プロンプト

```text
/goal $paper-reading-slide-imagegen-ja を使用して、画面ID {ID} の論文要約スライドを image_gen で全ページ再生成する。

入力:
- 要約Markdown: pdf/{ID}.md
- 原典PDF: pdf/{ID}.pdf
- 上位デザイン規定: レイアウト・デザイン規定.md

出力:
- pdf/{ID}/{ID}_01.png から連番で保存
- 16:9、1920x1080 PNG

要件:
- 発表用ではなく、単体で読める読み物向けにする
- 各ページのタイトルは章名ではなく、そのページの要点・結論を明記する
- 全ページでタイトルのサイズ、色味、余白、ページ番号位置を統一する
- 背景 #F7F6F1、文字 #2B3A4A、アクセント #5E7E96、ページ番号 #9AA4AD を基本にする
- 1ページあたり 3〜5 個の情報ブロックと、理解を助ける図解・表・KPI・フロー図を入れる
- pdf/{ID}.md の章構成を使う
- 「総評」「査読結果」「総合評価」などのレビュー・評価章は作成しない
- 必要に応じて pdf/{ID}.pdf も確認する
- 生成後に、ファイル数、命名、1920x1080、ページ番号、総評除外、重複画像でないことを検証する
```

## 追加指定つきプロンプト

必要な指定だけ `追加指定` に足す。

```text
/goal $paper-reading-slide-imagegen-ja を使用して、画面ID {ID} の論文要約スライドを image_gen で全ページ再生成する。

追加指定:
- 読者: {例: RAGを学ぶエンジニア}
- 情報量: {例: 読み物として中密度、発表用より説明を多め}
- トーン: {例: 落ち着いた技術解説、過度な装飾なし}
- 除外: {例: 総評、査読結果、総合評価}
- 強調: {例: 実験結果の数値比較、手法の学習方法}

入力:
- 要約Markdown: pdf/{ID}.md
- 原典PDF: pdf/{ID}.pdf
- 上位デザイン規定: レイアウト・デザイン規定.md
```
