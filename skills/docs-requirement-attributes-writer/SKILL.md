# Docs Requirement Attributes Writer

Use this skill whenever writing or revising a requirement document. Requirement text alone is not enough; add attributes that make the requirement traceable, reviewable, and maintainable.

## Definition

要求の追加属性とは、要求文そのものに加えて、要求を管理するために付ける情報である。属性を付けると、要求を一意に参照し、背景、根拠、源泉、依存、衝突、受け入れ基準、優先度、安定性、変更履歴を追跡しやすくなる。

## Standard Attributes

| 属性 | 目的 |
|---|---|
| 識別子 | 要求を一意に参照し、追跡する。 |
| 説明 | 要求文だけでは足りない背景を補足する。 |
| 根拠 | なぜその要求が必要かを説明する。 |
| 源泉 | 誰または何から出た要求かを示す。 |
| 種類 | 機能要求、技術制約、サービス品質制約などを示す。 |
| 依存関係 | 他の要求との前後関係や依存を示す。 |
| 衝突 | 他の要求と矛盾する可能性を示す。 |
| 受け入れ基準 | 完了判定に使う条件を示す。 |
| 優先度 | 重要度や実施順序を示す。 |
| 安定性 | 変更される可能性を示す。 |
| 変更履歴 | いつ、誰が、なぜ変更したかを残す。 |

## How To Write

- Give every requirement a stable identifier such as `FR-001`, `NFR-001`, `TC-001`, or `SQ-001`.
- Keep rationale separate from the requirement statement.
- Write source as a person, role, document, standard, law, incident, or design decision.
- Link acceptance criteria by file path when possible.
- Mark conflicts explicitly instead of hiding unresolved tradeoffs.
- Update change history whenever the requirement meaning changes.
